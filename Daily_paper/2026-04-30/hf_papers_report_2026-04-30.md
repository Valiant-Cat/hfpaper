## 📄 Diffusion Templates: A Unified Plugin Framework for Controllable Diffusion

- **链接**: https://huggingface.co/papers/2604.24351
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉-可控图像生成（Computer Vision - Controllable Image Generation）

### 2. 一句话核心贡献
提出了一个统一的插件式框架 Diffusion Templates，将异构的可控扩散方法（如结构控制、风格调整等）抽象为标准化的缓存接口（如KV-Cache/LoRA），实现了基础模型推理与控制能力注入的彻底解耦，解决了多个控制模块难以兼容和联合调度的碎片化问题。

### 3. 使用指南
*   **输入**：基础文本提示词（Prompt），以及任意任务特定的控制条件（例如：深度图、边缘图、标量数值[如亮度/年龄]、参考图像或局部掩码等）。
*   **输出**：符合输入控制条件的高质量生成图像。
*   **硬件与性能**：仅需常规支持大参数扩散模型推理的 GPU 即可。由于模板插件在生成循环外部执行（不进入基础扩散模型的迭代去噪循环），且支持懒加载（lazy loading），因此融合多个模板能力时的计算开销极小，不会导致显存随插件数量激增。
*   **开源情况**：代码及预训练模型库（基于 FLUX.2-klein-base-4B）已在 ModelScope 完全开源。

### 4. 主要创新点
1.  **接口级能力抽象（Template Cache）**：打破了传统控制方法与特定模型架构绑定的限制，将控制信号统一转化为标准的中间表示（即 Template Cache，主要实现形式为 KV-Cache 或 LoRA），作为管线参数注入，使得异构载体能在同一抽象层下运行。
2.  **解耦的插件式计算与训练架构**：将每个控制能力包装为独立的 Template Model，并标准化了两个接口：无梯度的预处理接口（`prepare_inputs`）和计算梯度的前向接口（`forward`）。这种设计既保证了异构模型在统一管线（Template Pipeline）下的兼容性，又通过避免重复计算提升了训练和推理效率。
3.  **异构控制的高效无冲突组合**：得益于 KV-Cache 原生支持序列拼接、LoRA 支持秩维度拼接的特性，框架允许同时加载和合并任意数量、不同类型的控制插件。这一机制从底层解决了多控制途径对内部激活的竞争问题，无需复杂的手工工程介入即可实现能力的自由组合。

### 5. 实验效果
作者基于 FLUX.2-klein-base-4B 训练并发布了包含 10 种独立任务的模板模型库，在各项定性实验中表现优异：
*   **全任务覆盖与精准控制**：在结构控制、标量属性调节（亮度、色彩、锐度、年龄）、图像编辑、超分辨率、内容参考、局部重绘以及基于人类偏好的审美对齐等跨度极大的任务中，均实现了高度一致且视觉自然的控制效果。例如，在图像编辑任务中，基于该框架的插件方法比基础模型原生处理速度快约 1.8 倍。
*   **出色的多能力融合（Composability）**：实验展示了复杂的组合生成场景，如“超分+锐度增强”、“结构控制+图像编辑+色彩调节”、“局部重绘+编辑+亮度调节”等。多个模板不仅可以无缝串联生效，而且在大幅增强多条件约束能力的同时，稳定维持了原基础模型的图像生成质量。

## 📄 A Survey on LLM-based Conversational User Simulation

- **链接**: https://huggingface.co/papers/2604.24977
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 对话系统与大模型多智能体 (LLM-based Agents)，具体聚焦于**对话式用户模拟 (Conversational User Simulation)**。其下游应用广泛覆盖对话式推荐系统 (CRS)、智能教育辅导、人机交互 (HCI) 系统测试与可用性研究、以及高质量合成对话数据的生成。

### 2. 一句话核心贡献
本文首次系统性地综述了基于大语言模型的对话式用户模拟技术，提出了一套涵盖模拟粒度、交互范式与生成技术的统一分类框架（Taxonomy），并全面梳理了该领域的评估方法、核心数据集及未来面临的开放挑战。

### 3. 使用指南
*   **方法定位**：本文为综述型文献，不提供单一的代码库，而是为研究者和开发者提供构建“用户模拟器”的方法论和架构指南。
*   **输入**：目标模拟对象的属性定义（如：普通用户提示词、带有社会人口学特征的Persona、具体虚拟人物/历史人物的Role-play设定、或个体的历史交互日志），以及当前的对话上下文（Context）。
*   **输出**：符合设定用户特征、知识背景和性格意图的下一轮自然语言回复及模拟行为反馈。
*   **硬件要求**：取决于所采用的具体技术。若使用“提示工程 (Prompting)”或“检索增强 (RAG)”调用闭源API，无特殊硬件要求；若采用全量微调 (Full SFT)、参数高效微调 (如 LoRA) 或基于人类/AI反馈的强化学习 (RLHF/DPO) 来训练专属用户模拟器，则需要相应的 GPU 显存与算力支持。
*   **资源参考**：文中整理了大量主流开源数据集（如 MultiWOZ, EmpatheticDialogues, CharacterBench 等）与评估协议，可直接用于下游模型的评测与训练。

### 4. 主要创新点
1.  **提出统一的三维分类体系 (Taxonomy)**：从“模拟对象 (Who)”（涵盖一般用户、画像级、角色扮演、个体级及混合目标）、“交互范式 (How)”（人-机、人-人、机-机、多人-多机交互等）、以及“核心技术 (What)”（Prompt、RAG、微调、RL/DPO及混合栈）三个维度全面结构化了该领域的研究。
2.  **系统整理评估方法与数据集**：突破了传统指标（BLEU/ROUGE）的局限，深入分析了基于 LLM-as-a-Judge 的评估机制及人类评估的标准，并按对话类型（日常闲聊、任务导向、角色扮演、辩论协商等）对主流数据集进行了细致归类。
3.  **深度剖析应用场景与开放挑战**：不仅总结了对话模拟在解决长尾数据缺乏、推荐系统诊断和教育场景中的实际价值，还明确指出了多轮角色一致性漂移、长期记忆维持、群体多样性缺失以及AI角色产生偏见和毒性内容等亟待解决的核心痛点。

### 5. 实验效果
作为综述文献，本文未进行单一的新实验，但通过汇总分析该领域（特别是角色扮演基准测试 CharacterBench 等）的实验数据，得出了以下核心效果结论：
*   **Prompt-based 大模型**：如 GPT-4 等超大模型在基础响应恰当性和对话流畅度上表现优异，具备强大的零样本角色扮演能力，但在特定角色的内在知识（SPE）上容易产生幻觉或发生角色崩溃（RoleBreak）。
*   **微调模型 (SFT)**：尽管使用较小参数的骨干模型，经过角色专属对话数据微调的模型在**角色特定知识和一致性保持**方面，一致且显著地优于仅依赖 Prompt 的超大模型。
*   **优化算法介入 (RL/DPO)**：通过偏好对齐技术优化的模拟器（如引入记忆选择器的 DPO 或层次化多轮 RL），能显著提升模型在长周期对话中的策略连贯性、任务达成率和个性化反馈的真实感。

## 📄 ClawGym: A Scalable Framework for Building Effective Claw Agents

- **链接**: https://huggingface.co/papers/2604.26904
- **阅读来源**: HTML

### 1. 应用领域
NLP-大模型智能体 (LLM Agents) / 计算机交互与自动化 (Computer-use/Claw-style Agents) / 强化学习与微调。

### 2. 一句话核心贡献
提出并开源了 ClawGym，这是一个集“数据自动化合成与验证、智能体微调与沙盒并行强化学习训练、高可靠性基准评估”于一体的端到端框架，解决了开发能在本地工作区执行多步复杂操作的“Claw风格”智能体时面临的高质量数据匮乏与评估困难问题。

### 3. 使用指南
*   **输入**：用户的自然语言指令（如“整理特定文件夹下的CSV文件并生成报告”）以及初始的本地工作区状态（包含模拟的本地文件、目录结构和可用的计算机系统工具）。
*   **输出**：智能体执行多步工具调用后，更新的最终环境状态（如新建的文件、修改的数据记录）以及自然语言的进度反馈或总结。
*   **运行环境与硬件**：
    *   **推演与测试**：高度依赖 Docker 容器化环境（基于 OpenClaw 框架），以提供独立的虚拟文件系统和沙盒来进行黑盒化的工具调用和执行推演（Rollouts）。
    *   **训练硬件**：SFT（监督微调）和并行沙盒 RL（强化学习）阶段需要支持长上下文（如 64K tokens）的多卡 GPU 计算资源，特别是针对 Qwen3-8B 和 Qwen3-30B 等规模的大模型。
*   **开源情况**：论文明确表示发布了首个大规模 Claw 智能体合成数据集（ClawGym-SynData，含 13.5K 任务）、评估基准（ClawGym-Bench，200个实例），及相关预训练模型（ClawGym-Agents）。

### 4. 主要创新点
1. **双路径任务环境合成与混合验证 (Dual-route Synthesis & Hybrid Verification)**：
   结合“自上而下”的用户画像（Persona-driven）与“自下而上”的可用技能组合（Skill-grounded）两种策略，自动合成具备高度现实场景多样性的任务；同时动态生成配套的模拟文件资源，并设计了结合“代码确定性检查（Code-based）”与“LLM量规主观评估（Rubric-based）”的混合验证器。
2. **基于沙盒反馈的智能体微调与RL训练机制 (Black-box Rollout Training)**：
   在 OpenClaw 的真实 Docker 容器中以黑盒代理模式收集多轮交互轨迹。在 SFT 阶段设计了针对智能体数据的“损失掩码（Loss masking）”（屏蔽环境反馈的预测，专注决策学习）；进一步探索了轻量级沙盒并行强化学习，直接利用代码验证器提供的客观奖励信号进行 GRPO 策略优化。
3. **基于难度校准与协同审查的可靠基准构建 (Difficulty-Aware & Collaborative Benchmark)**：
   为解决动态执行环境评估结果易波动的问题，在构建基准（ClawGym-Bench）时首创了“模型推演难度过滤（过滤过易/过难任务）”和“前沿LLM与人类协同审查”的严格清洗流程，确保每个测试用例不仅区分度高，而且拥有至少一条绝对可行的满分解决路径。

### 5. 实验效果
该框架在自建基准 **ClawGym-Bench** 和外部公开基准 **PinchBench** 上均展现了极强的性能提升和泛化能力：
*   **小模型性能跃升**：参数较小的开源模型在经过合成数据微调后提升巨大。Qwen3-8B 在 PinchBench 上性能提升了 **38.90%**，在 ClawGym-Bench 上提升了 **43.46%**。
*   **大模型持续突破**：更大规模的模型（如 Qwen3-30B-A3B）同样获得显著增益，分别在两个基准上提升了 **54.68%** 和 **25.96%**。其中 ClawGym-30B-A3B 甚至超越了参数量大得多的 Qwen3-235B-A23B 以及部分闭源商业模型。
*   **强化学习增益**：基于沙盒并行的 RL 训练在未经过 SFT 的基座模型以及已 SFT 的模型上均能产生进一步的性能增益，证明了该端到端训练范式的有效性。

## 📄 Accelerating RL Post-Training Rollouts via System-Integrated Speculative Decoding

- **链接**: https://huggingface.co/papers/2604.26779
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理（NLP） - 大模型强化学习后训练（LLM RL Post-Training）与推理加速

### 2. 一句话核心贡献
将推测解码（Speculative Decoding）作为一种无损加速基元系统性地集成到强化学习训练框架中，在不改变模型采样分布和训练轨迹的前提下，显著打破了 RL 后训练中 Rollout（轨迹生成）环节的自回归耗时瓶颈。

### 3. 使用指南
*   **输入**：强化学习微调的 Prompt 数据集（如数学推理、工具调用任务等）。
*   **输出**：经过 RL 优化的目标大语言模型（其训练得到的模型性能与传统自回归 Rollout 训练完全一致）。
*   **系统与框架支持**：基于开源的 NeMo-RL 训练框架和 vLLM 推理后端，支持 MegatronLM 策略模型。支持同步和异步 RL 流水线。
*   **草稿模型配置**：支持通用的 EAGLE-3 外部草稿模型机制，也支持模型自带的原生多 Token 预测（MTP）头。
*   **硬件需求**：需要 GPU 分布式集群（实验基于 NVIDIA GB200 节点），底层依赖高速互联（如 NVLink）以实现策略引擎和 Rollout 引擎之间的高效权重同步。

### 4. 主要创新点
1.  **无损的 RL 训练语义集成（Verifier-Exact Semantics）**：不同于常规的静态推理加速，在动态 RL 训练循环中引入推测解码，利用拒绝采样机制严格保证生成的轨迹服从目标策略分布，确保 KL 惩罚和策略损失的计算完全无误，不影响任何 RL 优化效果。
2.  **系统级权重同步与在线草稿自适应（Online Draft Adaptation）**：设计了无梯度干扰的复用路径，在策略模型（MegatronLM）计算 GRPO 损失进行前向传播时，复用其隐藏状态和对数概率（log-prob）缓存来在线监督/对齐 EAGLE-3 草稿模型，使得草稿模型能低成本地跟随策略模型的不断更新。
3.  **与同步/异步 RL 的双向组合设计**：证明了推测解码与异步 RL（通过隐藏生成延迟）并非互斥而是互补。系统不仅支持直接降低同步 RL 的生成延迟，还能够在高并发的异步部署中，缩小关键路径上的生成时间占比，最大化 GPU 利用率。

### 5. 实验效果
*   **8B 规模真实训练（Qwen3-8B）**：在同步强化学习推理微调任务中，采用 EAGLE-3 推测解码使 Rollout 生成阶段的吞吐量提升了 **1.8 倍**，使端到端的单步训练速度提升了 **1.35 倍**。
*   **零性能损失**：在 AIME-2024 数学推理验证集上，使用推测解码的验证准确率提升曲线与传统的自回归基线完全重叠，证实了其对学习信号的无损性。
*   **超大规模仿真预测（Qwen3-235B）**：借助高保真性能模拟器预测，在结合异步 RL 且扩展至 2048 张 GPU 的前沿规模部署下，Rollout 环节加速可超 **3 倍**，预计带来高达 **2.5 倍** 的端到端训练加速。

## 📄 Large Language Models Explore by Latent Distilling

- **链接**: https://huggingface.co/papers/2604.24927
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大语言模型推理（LLM Inference） / 解码策略（Decoding Strategies） / 测试时计算扩展（Test-time Scaling）

### 2. 一句话核心贡献
提出了一种基于隐空间在线蒸馏的“探索性采样（Exploratory Sampling）”解码算法，利用大模型内部特征映射的预测误差作为新颖度信号，引导模型在推理时探索具有真实语义多样性的解空间，克服了传统采样仅产生表面词汇变化的缺陷。

### 3. 使用指南
*   **输入与输出**：输入为标准提示词（Prompt/Prefix），输出为具备高语义多样性及高质量的多条生成文本或推理轨迹集合。
*   **使用方式**：部署在模型推理（解码）阶段。通过在候选 Token 采样前计算新颖度奖励并对原始 Logits 进行重新加权来实现干预。
*   **硬件要求**：无需特殊硬件，单张标准 GPU（如 RTX 4090 或 A100）即可运行。由于蒸馏器（2层 MLP）极轻量，其显存占用小于 200MB（理论极限可优化至 50MB 左右）。
*   **性能开销**：通过异步计算流水线，训练和推理步骤与 LLM 自身的前向传播和 CPU 调度完美重叠，对整体系统吞吐量的影响极小（内部测试最坏情况开销 <5%，开源优化版本开销仅约 1.2%）。
*   **开源情况**：代码已开源（基于 vLLM V1 构建的 tLLM 框架，支持现代推理优化如 CUDA Graph 和 FlashInfer）。

### 4. 主要创新点
*   **基于内部隐变量表征的新颖度评估**：不依赖 Token 层的启发式规则，而是在测试时在线训练一个轻量级“隐式蒸馏器（Latent Distiller）”。该模块通过浅层隐藏状态预测深层隐藏状态，当遇到当前上下文中未见过的语义模式时会产生较大的预测误差，此误差被直接转化为探索新语义区域的奖励信号。
*   **利用共享在线训练实现无缝并行协同探索**：在批量并行生成（Batch Generation）场景下，所有序列共享同一个在线更新的蒸馏器。当一条序列探索了某种推理模式后，蒸馏器会迅速拟合该映射，从而自动降低其他并行序列重复同一逻辑的概率，实现了类似于强化学习中多智能体的“先到先得”隐式协同策略。
*   **极低开销的异步训练-推理流水线**：创新性地解耦了主模型生成与蒸馏器的生命周期。将蒸馏器的前向计算与 LLM 访存密集型的中间层计算并行重叠，将蒸馏器的反向传播（权重更新）与 CPU 端的采样去 token 化间隙重叠，从而在不阻塞主干生成路径的情况下实现测试时动态学习。

### 5. 实验效果
*   **数学与推理基准表现优异**：在 AIME 2024/2025（数学）、LiveCodeBench v5（代码）和 GPQA（科学问答）等高难度数据集上，该方法的 Pass@k 效率大幅超越标准的随机采样及其他启发式基线方法（如 Min-p、OverRIDE、对比解码等）。
*   **极高的采样效率**：在具备反思能力的推理模型（如 Qwen2.5、Qwen3 系列）上，仅需 Pass@8 的采样预算，就能达到甚至超越传统方法在 Pass@64 预算下的表现。

## 📄 FASH-iCNN: Making Editorial Fashion Identity Inspectable Through Multimodal CNN Probing

- **链接**: https://huggingface.co/papers/2604.26186
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉-多模态融合与时尚AI（Computer Vision - Multimodal Fusion & Fashion AI）

### 2. 一句话核心贡献
提出了 FASH-iCNN 多模态系统，通过视觉通道探测（Visual Channel Probing）技术，成功从单张服装图像中解码并提取出时尚品牌身份、历史年代和色彩传统等原本不可见的“编辑文化逻辑”。

### 3. 使用指南
*   **输入**：一张服装照片（主要输入）；可选择性提供人脸照片、设计师身份、季节、年份等作为辅助信号。
*   **处理流程**：系统首先使用 SegFormer 提取服装区域，随后通过独立的 EfficientNet-B0 骨干网络分别处理图像流（提取特征后拼接），最后输入到多层感知机（MLP）预测头中。
*   **输出**：返回该服装的时尚品牌（House Identity）、所属时代/年份（Temporal Era），以及三层级的色彩传统（基础色族、CSS命名颜色、CIELAB精确坐标）。
*   **硬件与开源情况**：基于轻量级的 EfficientNet-B0 架构，常规 GPU 即可满足训练与推理需求；文中明确描述了网络架构、损失函数（混合精度、BCE/MSE损失等）及训练策略，但未明确声明代码开源，数据集基于1991-2024年的《Vogue》秀场图像（87,547张）构建。

### 4. 主要创新点
1.  **视觉通道剥离探测（Visual Channel Probing）**：通过对比全彩、灰度、剪影和边缘图四种抽象层级的图像，揭示了“纹理和亮度”是承载时尚品牌身份的核心视觉通道（而非形状或颜色），打破了常规的视觉认知。
2.  **层级化且符合感知的色彩预测管线（Hierarchical Color Pipeline）**：构建了“Berlin-Kay基础色族分类 $\rightarrow$ CSS命名颜色分类 $\rightarrow$ 受限CIELAB坐标回归”的三阶段预测框架，有效将模型预测锚定在特定的文化传统中，大幅降低了色彩的感知误差。
3.  **自适应的上下文多模态融合**：实验证明模型对“人脸图片”等辅助输入的利用是上下文自适应的——当服装信息稀疏（如仅剩边缘图）时，辅助输入能有效补偿信息缺失；而当服装信息丰富（全彩图）时，辅助输入几乎不增加冗余信号。

### 5. 实验效果
模型在 87,547 张 Vogue 秀场图像数据集上取得了以下核心表现：
*   **品牌与年代预测**：仅使用服装图像，在 14 个时尚品牌上的 Top-1 准确率达到 **78.2%**；年代（10年跨度）预测 Top-1 准确率达 **88.6%**；具体年份预测（跨越34年）Top-1 准确率达 **58.3%**，且平均误差仅为 **2.2年**。
*   **通道消融效果**：在品牌身份预测中，移除颜色信息仅损失 **10.6** 个百分点的准确率，而移除纹理信息则会导致准确率大幅下降 **37.6** 个百分点。
*   **色彩预测优化**：相比于无约束的直接回归模型，FASH-iCNN 提出的三阶段层级色彩预测管线将色彩感知误差（CIEDE2000）降低了 **39%**。

## 📄 Unified 4D World Action Modeling from Video Priors with Asynchronous Denoising

- **链接**: https://huggingface.co/papers/2604.26694
- **阅读来源**: HTML

### 1. 应用领域
具身智能（Embodied AI）- 机器人操作 / 多模态世界模型（World Models）

### 2. 一句话核心贡献
提出统一的4D世界动作模型 X-WAM，通过轻量级深度预测分支和异步去噪采样（ANS）技术，在单一框架内高效统一了高质量的4D环境模拟（视频+3D重建）与实时的机器人动作执行。

### 3. 使用指南
*   **输入**：当前时刻的多视角 RGB 观察图像、机器人本体状态（Proprioceptive states）。
*   **输出**：未来多视角 RGB-D 视频序列（用于4D环境重建）、未来机器人状态以及可执行的低维控制动作。
*   **硬件要求**：
    *   **训练端**：算力需求极高。预训练阶段使用了 256 张 NVIDIA H20 GPU 进行 5800 小时数据的训练；微调阶段使用了 32-64 张 H20 GPU。
    *   **推理端**：需要能够运行基于 5B（50亿）参数规模的 Diffusion Transformer (DiT) 模型的 GPU 设备。借助块处理（Chunking）机制，动作推理延迟可控制在约 300ms 以支持实体机器人的实时控制（15Hz）。
*   **代码开源**：论文提及了项目主页（Project Website），具体代码、模型权重与预训练数据集需访问其主页获取。

### 4. 主要创新点
1.  **引入显式3D感知的统一4D建模**：打破了以往统一世界动作模型仅局限于 2D 像素空间的限制。通过端到端的框架，同时实现高保真视频生成、3D空间重建、高策略成功率和高效动作执行。
2.  **轻量级深度适配模块（Lightweight Depth Adaptation）**：为解决直接生成 RGB-D 导致序列长度翻倍的算力瓶颈，该模型仅复制预训练 DiT 的最后几个 Block 作为专门的深度预测分支，并采用不对称的“只读”连接机制。这既保留了预训练模型的视觉先验，又在不增加过多计算开销的情况下提取了3D几何结构。
3.  **异步去噪采样（Asynchronous Noise Sampling, ANS）**：针对高维视频（需多步去噪）与低维动作（少步即可解码）的模态差异，在推理时仅用前几步快速解码动作下发执行，剩余步数继续生成高保真视频；在训练阶段，摒弃了以往完全解耦的随机采样，而是从严格匹配推理阶段异步分布的“联合噪声分布”中进行耦合采样，极大优化了生成质量和动作解码效率。

### 5. 实验效果
*   **RoboCasa 仿真基准**：在24个厨房操作任务上取得 **79.2%** 的平均成功率，比最强基线模型（Cosmos Policy，67.1%）高出 12.1 个百分点。
*   **RoboTwin 2.0 仿真基准**：在双臂操作任务的 Clean 和 Randomized 场景下，分别达到 **89.8%** 和 **90.7%** 的高成功率，超越了先前的 Motus 等方法。
*   **4D 视觉与几何生成**：相比“两阶段视频生成+深度估计（如DreamZero + DA3）”的方法，X-WAM 的 PSNR 提升了 2.34 dB，点云的倒角距离（Chamfer Distance）从 0.0680 降低至 0.0049，实现了全面领先。
*   **真实世界部署**：在复杂的双臂协作“耳机装盒”长视距任务中验证了有效性，模型能处理不同的颜色、未见过的桌布以及干扰物，实现了流畅的端到端实时执行。

## 📄 Turning the TIDE: Cross-Architecture Distillation for Diffusion Large Language Models

- **链接**: https://huggingface.co/papers/2604.26951
- **阅读来源**: HTML

### 1. 应用领域
NLP-大语言模型微调与知识蒸馏（特别针对扩散大语言模型 Diffusion LLMs）

### 2. 一句话核心贡献
提出了首个针对扩散大语言模型（dLLMs）的跨架构知识蒸馏框架 **TIDE**，成功解决了教师与学生模型在基础架构、注意力机制以及分词器（Tokenizer）不匹配的知识转移难题。

### 3. 使用指南
*   **输入**：全掩码或部分掩码（Masked）的文本序列。
*   **输出**：并行解码、全局一致的完整文本序列（特别适用于代码生成等结构化输出）。
*   **使用流程**：
    *   若师生模型分词器相同：使用共享分词器流水线（结合 token 级别 KL 散度与双轴调度）。
    *   若师生模型分词器不同：使用跨分词器流水线（使用 Reverse CALM 算法在字节/块级别对齐）。
    *   训练过程中需冻结大参数量的教师模型，仅更新小参数量学生模型。
*   **硬件要求**：实验（含 16B 教师模型蒸馏）在单张 NVIDIA H100-80GB GPU (bfloat16 精度) 上即可完成；部署时，蒸馏后的 0.6B 学生模型峰值显存仅需 1.4GB，适合在消费级硬件上运行。
*   **开源情况**：论文中未直接提供代码仓库链接，但详细提供了基于 Qwen3、LLaDA2 等开源权重的实验设置和超参数配置。

### 4. 主要创新点
1.  **TIDAL（双轴动态调度机制）**：针对扩散模型特有的“不同时间步下教师模型可靠性剧烈波动”问题，TIDAL 联合“训练进度”和“扩散时间步”两个维度动态调节蒸馏强度，在高噪声阶段（掩码率高、教师靠猜）让学生依赖自身，在低噪声阶段（教师预测可靠）让学生全面向教师学习。
2.  **TIDE-Pool（互补掩码分割上下文丰富）**：针对高噪声级别下严重掩码导致教师模型上下文缺失的问题，将掩码位置随机划分为两个互补的子集，进行两次对称的教师模型前向传播。通过暴露部分答案作为条件（类似于演示），极大丰富了教师模型输出的空间表示信号。
3.  **Reverse CALM（反向跨分词器似然匹配）**：为克服不同分词器（Cross-Tokenizer）导致的词表不兼容障碍，提出了一种反向块级别（Chunk-level）似然匹配目标函数。它不仅避免了直接 token 映射的数学无解和前向映射的梯度爆炸问题，还实现了有界梯度更新和双端噪声过滤，有效应对了分词器未完全对齐带来的噪声。

### 5. 实验效果
*   **综合性能提升**：通过两条不同流水线将 16B MoE（LLaDA2.0）和 8B 密集型（WeDLM）教师模型蒸馏至 0.6B 学生模型，在 8 个主流基准测试中平均得分达 34.20，较非蒸馏基线（32.67）提升 1.53 分。
*   **代码生成突破**：得益于扩散模型的全局并行解码优势，蒸馏模型在结构化任务上表现卓越。HumanEval 测试中得分飙升至 48.78（最高甚至达 49.39），远超同等规模自回归（AR）基线模型的 32.30 分。
*   **极致推理效率**：蒸馏后的 0.6B 模型生成 256 tokens 仅需 1.4GB 峰值显存，显存占用较 WeDLM 教师模型降低 22 倍，推理延迟较 LLaDA2 教师模型实现 5.2 倍加速，且对比未蒸馏的基线模型，吞吐量仅有 2.6% 的微小折损。
