## 📄 Full Attention Strikes Back: Transferring Full Attention into Sparse within Hundred Training Steps

- **链接**: https://huggingface.co/papers/2605.16928
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大语言模型长上下文推理加速与模型稀疏化

### 2. 一句话核心贡献
提出了一种利用大语言模型内在注意力稀疏先验的轻量级框架，仅需数百步微调即可将标准全注意力模型无损转换为高稀疏模型，从而在长上下文推理中实现大幅加速。

### 3. 使用指南
*   **输入**：包含超长上下文（可达 512K Tokens）的文本提示（Prompt）。
*   **输出**：模型生成的文本回复，精度与原始全注意力模型几乎一致。
*   **训练与部署流程**：
    1. **离线校准**：通过少量文本数据运行一次，筛选出约 15% 负责长距离召回的“检索头”，剩余头作为仅关注近距离的“局部头”。
    2. **两阶段极轻量微调**：第一阶段冻结大模型，仅利用少量长文本训练一个 16 维的注意力投影矩阵（耗时约数百步）；第二阶段利用约 1M Tokens 进行自蒸馏端到端微调，对齐模型输出（约 600 步）。
*   **硬件要求**：训练和推理依赖 GPU 环境。为了打破显存带宽瓶颈和避免排序开销，部署时需运行作者定制的 CUDA Sparse Attention Kernel（针对 NVIDIA GPU 开发了底层原子计数器和单 warp CTA 的高效算子）。
*   **代码开源**：论文文本中暂未直接提供开源代码链接，但在复现时需具备自定义 CUDA 算子的开发/使用能力。

### 4. 主要创新点
1. **基于注意力头特化的稀疏先验 (Head Specialization)**：发现模型大部分注意力头仅需处理局部上下文，只有极少数（约15%）的“检索头”需要全量上下文。通过仅为检索头保留完整的 KV Cache，从架构层面天然去除了大量冗余计算。
2. **规避 RoPE 干扰的低维索引 (Low-dimensional Token Indexing)**：发现旋转位置编码 (RoPE) 的高频特征会干扰远距离检索。提出在施加 RoPE 之前，将查询和键投影到一个仅 16 维的极低维子空间中进行相关性评分，从而以极低的计算代价精准锁定关键 Token。
3. **动态查询感知的 Top-$p$ 稀疏机制 (Query-aware Dynamic Top-$p$ Sparsification)**：指出不同查询需要的 Token 数量差异巨大，摒弃了传统的固定数量（Top-$k$）截断策略，采用自适应的 Top-$p$（累计概率阈值）截断机制。配合无排序的高效 GPU 算子，在复杂查询下保全信息，在简单查询下最大化稀疏度。

### 5. 实验效果
*   **精度表现**：在 LongBench 和 RULER 等长上下文基准，以及 AIME24、MMLU-PRO 等长链路推理基准上，表现出几乎无损的准确率（例如在 AIME 上得分完全持平稠密基线）。在高达 512K 长度的超长上下文测试中依然能维持极高准确度，未出现其他稀疏模型的崩溃现象。
*   **加速效果**：稀疏度随上下文长度动态扩展（最高可达 97.1%），在解码阶段相较于 FlashAttention-2 实现了最高 **9.36 倍** 的速度提升，且显著降低了长上下文情况下的显存占用。

## 📄 LatentOmni: Rethinking Omni-Modal Understanding via Unified Audio-Visual Latent Reasoning

- **链接**: https://huggingface.co/papers/2605.22012
- **阅读来源**: HTML

### 1. 应用领域
**多模态大语言模型 (Multimodal LLMs) - 视听联合理解与跨模态推理**

### 2. 一句话核心贡献
提出了 **LatentOmni** 框架，通过在统一的连续潜在空间中交替生成文本逻辑与音视频潜在状态，解决了传统显式文本思维链（CoT）在压缩多模态连续信号时造成的信息丢失、时间对齐减弱和过度依赖语言先验的问题。

### 3. 使用指南
*   **输入形式**：同步的视频帧流、音频流以及文本查询指令（Prompt）。
*   **输出形式**：包含“离散自然语言”与“连续潜在状态特征”的交错推理轨迹（轨迹由特殊 Token 如 `<Unified_Latent>` 触发和 `<Stop_Latent>` 终止），最终输出纯文本格式的准确答案。
*   **实现依赖**：基于 Qwen2.5-Omni-7B 模型进行后训练（Post-training）。需要预先提取音频和视频的特征，并通过自回归语言建模头（LM Head）预测连续特征。
*   **训练要求**：训练过程整合了交叉熵损失（文本）、对比损失（时间同步）和 L2 距离损失（特征锚定）。论文单卡 batch size 为 1 并配合 12 步梯度累积进行训练。
*   **配套资源**：方法依赖于作者构建的包含 3.5 万条高质量样本的新数据集 **LatentOmni-Instruct-35K**。

### 4. 主要创新点
1.  **文本-潜在状态交错推理架构 (Interleaved Text-Latent Reasoning)**：放弃了纯文本的思维链瓶颈，引入特定控制 Token，允许模型在保留文本进行高级逻辑推理的同时，直接在连续的潜在空间中自回归地处理和回溯高密度的音视频感知证据。
2.  **全能同步位置编码 (Omni-Sync Position Embedding, OSPE)**：将多模态旋转位置编码（RoPE）扩展到统一的潜在空间，为具有物理时间对应关系的视觉帧和音频片段分配共享时间戳，有效防止了顺序生成时跨模态特征在时间上的漂移。
3.  **特征级潜在监督与专属数据集构建**：在训练中引入特征级监督（通过 InfoNCE 损失拉近音视频特征，通过 L2 损失将生成的潜在状态锚定在原始感知特征上）；并开发了一套自动化合成管线，构建了填补领域空白的音视频交错推理轨迹数据集 **LatentOmni-Instruct-35K**。

### 5. 实验效果
LatentOmni 在四大具有挑战性的全模态基准测试中均展现出**开源模型中的 SOTA（最优）性能**：
*   **OmniVideoBench**（跨模态对齐与问答）：准确率达到 35.4%，相较于其基座模型 Qwen2.5-Omni-7B 显著提升了 **6.1%**。
*   **Daily-Omni**（日常场景推理）：准确率提升 **4.5%**（对比基座）和 1.8%（对比显式纯文本 CoT）。
*   **WorldSense**（物理与时空常识）与 **LVOmniBench**（长视频多感官理解）：分别比基座模型提升了 **3.5%** 和 **3.1%**，特别是在超长视频的推理上表现出明显的优势。
*   **纯视觉泛化 (VideoMME)**：在退化为纯视觉推理的设置下，同样超越了现有的视觉潜在推理模型（如 LVR, Monet），证明了潜在推理设计的泛化鲁棒性。

## 📄 π-Bench: Evaluating Proactive Personal Assistant Agents in Long-Horizon Workflows

- **链接**: https://huggingface.co/papers/2605.14678
- **阅读来源**: HTML

### 1. 应用领域
NLP - 大语言模型智能体评估（主动式个人助手智能体、长跨度任务规划与决策）

### 2. 一句话核心贡献
提出了 $\pi$-Bench 基准测试，通过包含隐藏意图、跨会话依赖和持久化工作区的长跨度多轮工作流，首次实现了对大模型智能体“主动性”（识别未言明需求）与“任务完成度”的解耦与联合评估。

### 3. 使用指南
*   **输入**：一个自然但信息欠指定的初始用户指令（Underspecified Request），以及智能体所在的持久化项目环境（包含过往会话记录、长时记忆、工作区文件等）。
*   **输出**：智能体通过多轮交互、工具调用和信息澄清，最终生成的项目交付物（如文档、代码、系统执行结果）以及交互轨迹。
*   **硬件要求**：基准测试环境已通过 Docker 容器化封装，单次任务占用内存少于 8GB，存储少于 32GB，标准的 Linux CPU 服务器即可运行，大模型推理依赖外部 API 提供。
*   **是否开源**：基准测试的智能体脚手架基于开源的 Nanobot（MIT协议）和 AppWorld（Apache-2.0协议）构建，整体测试环境、任务配置及评估代码设计为向社区开放。

### 4. 主要创新点
*   **引入“隐藏意图”的解耦评估范式**：首次将用户未明确陈述的隐含需求、偏好或约束形式化为“隐藏意图（Hidden Intents）”，将智能体的“主动性”（Proactivity，主动发现并解决或精准提问）与“任务完成度”（Completeness，最终可验证的交付质量）解耦为两个独立的指标进行联合评估。
*   **构建跨会话依赖的长跨度持久化工作流**：构建了涵盖 5 种领域专家（研究员、营销人员、见习律师、药剂师、金融家）的 100 个多轮任务，任务被组织成包含长程交叉会话依赖（Cross-session dependencies）的情节，强制智能体必须利用早前任务的输出和偏好来推进后续任务。
*   **自动化且多维度的稳定评估系统**：设计了一个包含状态追踪的双阶段“模拟用户智能体”以推动交互进程，并在任务结束后结合 LLM 规则评分（针对语义内容）和 Python 规则验证（针对确切的工具调用状态，如发邮件、日历操作）实现了稳定且高一致性的全自动闭环评测。

### 5. 实验效果
*   **主动性仍是挑战**：在 9 个前沿大模型（文中称代为 GPT-5.4, Claude 4.6 Opus, Qwen3.6 Plus 等）上的测试表明，前沿模型的主动性得分（Proactivity）在 43.1% 到 67.0% 之间，仍有巨大提升空间；其中 GPT-5.4 (67.0%) 和 Claude 4.6 Opus (66.5%) 表现最好。
*   **主动性与任务完成度的显著差异**：实验证明模型解决任务和主动减轻用户负担是两种独立能力。例如部分模型（如 Kimi K2.5）能够获得不错的任务完成度，但这建立在极高交互轮数（依赖用户不断补充隐性需求）的基础上，其主动性得分全场最低。
*   **记忆对主动性的价值**：消融实验表明，当移除带有依赖关系的历史交互记忆时，智能体识别隐藏意图的主动性得分平均骤降 9.5 分，证明了长时记忆与过往交互对于智能体主动预判并解决后续任务的关键性作用。

## 📄 SAM 3D Animal: Promptable Animal 3D Reconstruction from Images in the Wild

- **链接**: https://huggingface.co/papers/2605.07604
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 3D重建（具体为：野生场景下的多动物三维重建与姿态估计）

### 2. 一句话核心贡献
提出了首个支持提示词（关键点或掩码）交互的单图多动物3D重建框架 **SAM 3D Animal**，并构建了大规模多动物3D数据集 **Herd3D**，有效解决了自然场景下因多动物密集重叠和频繁遮挡导致的三维重建难题。

### 3. 使用指南
*   **输入**：单张包含一只或多只动物的RGB图像，以及可选的交互提示（2D关键点或分割掩码）。
*   **输出**：图像中所有动物实体的3D网格（基于SMAL+参数化模型的形状与姿态参数）、相机参数和边界框（Bounding Boxes）。
*   **使用流程**：无需使用目标检测器对单只动物进行预先裁剪，模型可直接接受全图输入。用户在推理时可根据条件提供任意数量的关键点或掩码提示，模型会利用这些提示来消除遮挡带来的歧义（提示信息越丰富，重建精度越高；也支持完全无提示的自动重建）。
*   **硬件条件**：模型基于Transformer架构构建，论文中训练阶段使用了 4 张 RTX 4090 GPU 进行分布式训练，推理阶段需配备常规支持深度学习的GPU。

### 4. 主要创新点
1.  **引入提示驱动机制 (Promptable Design)**：首次将“提示（Prompt）”概念引入动物3D重建领域，模型能够通过接收关键点（用于骨骼对齐）和掩码（用于轮廓区分）作为特征条件，在拥挤和严重遮挡的场景下提供可靠的空间与语义引导。
2.  **基于集合预测的单阶段多实例重建 (Set-Prediction Paradigm)**：摒弃了以往“先检测裁剪，后单体重建”的繁琐流程，采用类似 DETR 的基于二分图匹配（Bipartite Matching）的集合预测范式，实现单次前向传播即可同时输出全图所有动物个体的3D结构。
3.  **构建多动物3D数据集 Herd3D**：为解决缺乏多动物遮挡场景3D标注数据的问题，定制化生成了包含超 5000 张图像、涵盖 118 种动物的大规模数据集。该数据集具有高度的物种多样性、复杂的遮挡模式，并提供了精确的实例级 SMAL+ 3D 标注。

### 5. 实验效果
模型在 Animal3D、APTv2 和跨域（OOD）数据集 Animal Kingdom 上进行了全面评估，取得了当前的 **SOTA（State-of-the-Art）** 表现：
*   **无提示模式下**：性能已经匹敌或超越现有的单动物重建方法（如 AniMer, GenZoo, 3D Fauna），在 Animal3D 上 PA-MPJPE 达到 80.7 mm。
*   **引入提示模式下**：性能得到飞跃式提升。在最困难的跨域数据集 Animal Kingdom 上，相较于最强基线模型，AP（平均精度）提升了 **54%**，mAP 提升了 **80%**；在 Animal3D 数据集上 PA-MPJPE 误差进一步降低了 5.2 mm。
*   **鲁棒性验证**：消融实验表明，即使只提供极少量随机关键点（例如1~5个），模型的重建精度也能出现阶跃性上升，在可见度极低（高遮挡）的环境下表现尤为突出。

## 📄 LoREnc: Low-Rank Encryption for Securing Foundation Models and LoRA Adapters

- **链接**: https://huggingface.co/papers/2605.13163
- **阅读来源**: HTML

### 1. 应用领域
人工智能安全与隐私保护 - 基础模型（大模型）端侧部署、参数高效微调（PEFT/LoRA）、生成式AI（覆盖计算机视觉与自然语言处理）。

### 2. 一句话核心贡献
提出了一种无需训练、无需数据的低秩加密框架（LoREnc），通过对基础模型权重进行“谱截断”并在授权的LoRA适配器中进行“补偿”，实现了基础模型与微调适配器在边缘设备上的联合安全部署，有效防止未经授权的权重复用和模型提取攻击。

### 3. 使用指南
*   **输入**：预训练的基础模型明文权重、对应下游任务的LoRA适配器明文权重。
*   **输出**：加密后的基础模型权重、伪装/加密的LoRA权重、存储在安全环境的恢复密钥。
*   **部署与硬件要求**：专为资源受限的边缘设备（On-device）设计。要求设备具备硬件级可信执行环境（TEE，如TrustZone/SGX）以安全存储恢复密钥。
*   **运行机制**：完全免训练（Training-free），无需原始训练数据集。在推理前向传播时进行动态实时解密计算，无需将完整的明文模型加载到设备内存中。
*   **开源状态**：文中未明确声明代码开源（注：论文标注此为作者加入Amazon前的工作，代码获取可能需查阅官方仓库）。

### 4. 主要创新点
1.  **免训练的谱截断与无损补偿（Spectral Truncation and Compensation）**：利用截断SVD提取并移除基础模型权重中占主导地位的低秩（核心能量）分量，使得未授权用户的输出产生严重语义崩溃；同时将这部分被移除的低秩信息通过数学拼接融合到授权的LoRA适配器中，确保授权用户能实现理论上的无损精确恢复。
2.  **基于正交重参数化的结构指纹隐蔽（Orthogonal Reparameterization）**：为了防止攻击者通过SVD的严格正交性特征识别出加密适配器，引入了等距旋转操作。该操作在参数空间内创造了无限等效分解，使保护后的权重看起来与普通高斯初始化的标准LoRA权重无异，实现了隐蔽性。
3.  **零驻留内存的动态重构机制（Zero-memory On-the-fly Reconstruction）**：解密过程（$\tilde{W} + \tilde{B}^{\prime}_{k}\tilde{A}^{\prime}_{k} + K_{\tilde{B}_{k}}K_{\tilde{A}_{k}}$）直接在推理时通过矩阵乘法与加法动态完成。由于不需要在显存/内存中还原并保存原始基础模型权重矩阵，从而满足了端侧部署的严苛内存和效率限制。

### 5. 实验效果
*   **计算机视觉（如 SD 1.5, DiT/Sana）**：未授权访问会导致生成的图像出现结构性崩溃（LPIPS/CLIP等语义指标大幅恶化，变成无意义的雪花或色块）；授权用户则能100%精确还原基线生成质量。
*   **自然语言处理（如 GPT-2, LLaMA-2在WikiText-2上）**：未授权推理会导致模型困惑度（Perplexity）呈现几何级暴增，输出无意义或循环的乱码；授权密钥介入后各项NLP评测指标与原始模型完全一致。
*   **安全性与开销**：在计算和内存开销不到 **1%** 的情况下，不仅能实现语义级的数据保护，还能有效抵御少量数据的微调攻击（Fine-tuning attacks）以及针对LoRA架构的谱反调解提取攻击（Spectral DeTuning）。

## 📄 Swift Sampling: Selecting Temporal Surprises via Taylor Series

- **链接**: https://huggingface.co/papers/2605.22678
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉与多模态大模型（多模态大语言模型，VLMs） - 长视频理解与处理

### 2. 一句话核心贡献
提出了一种名为 Swift Sampling 的免训练、极轻量级帧选择算法，通过在视频潜在特征空间中利用泰勒级数展开预测“时间突变（Temporal Surprises）”，高效筛选出最具信息量的关键帧，在几乎不增加计算开销的情况下大幅提升了长视频理解任务的性能。

### 3. 使用指南
*   **输入**：一段长视频（或均匀采样的候选帧序列）以及目标帧数预算（如限制输入给 VLM 的帧数 $K$）。
*   **输出**：包含最丰富动态信息与关键事件的 $K$ 个视频关键帧。
*   **使用方式**：作为一种即插即用的前置过滤模块使用。无需预先知道用户的文本查询（Query-agnostic），无需训练辅助网络。直接复用目标视频大模型（VLM）自身的视觉编码器（如提取第 0 层或早期层的特征）计算泰勒残差，随后挑出局部残差最大的帧即可。
*   **硬件要求/成本**：无需特殊硬件。由于完全复用已有 VLM 的早期层计算，相比常规的完整推理流程仅增加约 **0.2%** 的额外计算开销，远低于需要独立视觉编码器的传统基线方法。

### 4. 主要创新点
1.  **基于泰勒级数的视频潜在轨迹预测**：受人类大脑“预测编码”机制启发，将相邻视频帧的视觉潜在特征建模为连续可微的轨迹。通过计算特征的速度（一阶导数）、加速度（二阶导数）等，应用高阶泰勒展开来预测后续帧在潜在空间中的预期位置。
2.  **将“泰勒残差”量化为帧信息量（捕捉时间突变）**：方法不依赖外部模型打分，而是直接计算实际帧特征与泰勒预测轨迹之间的残差。残差较小代表视频处于可预测的冗余阶段（如缓慢行走）；残差极大的帧代表视频发生了突然偏离预期的事件（即“时间突变”，如画面中突然钻出一只海豹），以此作为高信息量关键帧的提取标准。
3.  **层次化局部最大值采样策略**：为避免在一段持续的高剧烈运动视频中耗尽所有帧预算，算法采用局部时间窗口内的最大值搜索。这种自适应策略能根据视频内容的复杂度自动缩放采样率，确保在长视频的全局时间跨度上捕捉到不同阶段的突变事件。

### 5. 实验效果
*   **长视频问答任务全面领先**：在三个主流长视频 QA 基准测试（Video-MME, LongVideoBench, MLVU）中，全面超越了均匀采样和此前最先进的免训练帧选择基线方法。
*   **极限条件下的显著提升**：在帧数预算严格受限且视频时长极长（超过30分钟）的场景下表现最为强悍，准确率最高可提升 **12.5%**。
*   **极高的计算性价比**：相比于基于余弦唯一性（Cosine Uniqueness）等传统基线方法，Swift Sampling 的额外计算开销降低了 **30倍**，且无需视频级别的超参数调优。
*   **下游任务的通用性**：作为即插即用的组件，在 Token 压缩（结合 UniComp）和视频字幕生成任务中，替换默认的均匀采样后，均能一致性地提升模型的最终精度。

## 📄 Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention

- **链接**: https://huggingface.co/papers/2605.22791
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 高效大语言模型 (Efficient LLMs) / 长上下文序列建模 (Long-context Sequence Modeling)

### 2. 一句话核心贡献
提出 Gated DeltaNet-2，通过将线性/循环注意力中的“擦除（Erase）”与“写入（Write）”机制解耦为独立的通道级（channel-wise）门控，解决了以往 Delta-rule 模型中单一标量门控导致的记忆干扰问题，大幅增强了固定大小内存处理长上下文的能力。

### 3. 使用指南
*   **输入/输出**：输入为序列 Token 向量（$x_t$），输出为更新后的隐状态特征。模型将无限长度的注意力缓存（KV Cache）替换为固定大小的循环状态矩阵。
*   **网络架构**：可作为标准 Transformer 块中的 Token 混合器（Token Mixer）使用。支持两种部署模式：纯循环架构（Recurrent-only），或与滑动窗口注意力（SWA）交替使用的混合架构（Hybrid）。
*   **硬件与效率**：推理时具有恒定内存（Constant Memory）和线性时间复杂度；训练时采用 Chunkwise 并行算法，能够高效映射到现代 GPU 硬件（如 Hopper/Ampere）的 Tensor Core 上。
*   **代码实现**：底层依赖定制的 Triton 融合算子。计算中需注意数值稳定性，如对数衰减（log-decay）及累计求和操作必须在 fp32 精度下进行计算，以避免累积误差。

### 4. 主要创新点
1. **读写维度的通道级解耦 (Decoupling Erase and Write)**：打破了 Gated DeltaNet 和 KDA 中用单一标量同时控制擦除和写入的限制。在 Key 轴引入“擦除门（Erase Gate）”决定移除多少旧关联，在 Value 轴引入“写入门（Write Gate）”决定注入哪些新特征，使模型能更精细地编辑压缩内存。
2. **构建一致的快速权重更新视角 (Fast-weight Update View)**：从在线优化的角度将模型形式化为对目标损失的单步梯度更新。模型先生成衰减状态，再通过门控擦除方向读取旧内容，最后向门控目标值写入残差，在数学上兼容并统一了以往的多种门控线性 RNN 模型。
3. **感知门控的高效块级 WY 算法 (Gate-aware Chunkwise Algorithm)**：将通道级衰减项吸收到非对称的 rank-1 擦除因子中，使得前向计算能维持高效的块级（Chunkwise）WY 矩阵求逆结构；同时推导了感知独立门控的反向传播机制（Gate-aware backward pass），在增加极少计算开销的情况下保留了高效的并行训练能力。

### 5. 实验效果
在 1.3B 参数量和 100B FineWeb-Edu 数据的训练规模下，Gated DeltaNet-2 展现出以下优势：
*   **综合性能最优**：在语言建模（WikiText困惑度）和零样本常识推理（PIQA, BoolQ等）的平均表现上，超越了 Mamba-2、Mamba-3 (SISO/MIMO)、Gated DeltaNet 和 KDA 等前沿循环序列模型。
*   **长上下文检索能力显著提升（核心优势）**：在 RULER 的“大海捞针 (Needle-In-A-Haystack)”基准中表现最为突出。特别是在多键检索（MK-NIAH）以及真实世界长文本检索任务（NQ, DROP等）中，凭借解耦的记忆编辑能力，有效克服了多个压缩关联特征之间的干扰，在纯循环和混合架构下均取得了最佳召回率。
*   **极高的训练吞吐量**：加入双门控后，单张 H100 上的训练吞吐量（约 36.1 Kt/s）仅比基线 KDA 产生极小的常数级衰减，依然保持了优秀的扩展性。

*   **过度自信与偏差**：模型在预测中表现出系统性的过度自信和强烈的响应偏差，其不确定性估计极不靠谱，证明当前 AI 系统远不足以作为预测科学进步的可靠工具。

*   **开源情况**：论文明确指出代码已开源。该方法设计为一个轻量级、即插即用的适配器，可无缝集成到 SAM 2 框架中。

### 4. 主要创新点
1.  **基于高阶马尔可夫模型的运动预测器 (Motion Predictor, MP)**：区别于传统的匀速运动假设（如卡尔曼滤波），该模块采用序列模型来捕捉目标的复杂非线性运动动态，并结合几何线索（形状、长宽比）来指导 SAM 2 的掩码选择。
2.  **误差检测与恢复模块 (Error Detection-Recovery Module, EDRM)**：建立目标的历史可靠状态原型 (Target Prototype)，利用提取到的高层语义特征和几何特征实时评估当前帧的跟踪质量。当检测到潜在的跟踪漂移或失败时，主动冻结原型更新并触发恢复机制，有效防止误差累积。
3.  **目标感知记忆库 (Target-Aware Memory Bank, TAMB)**：摒弃了简单的先进先出（FIFO）队列，提出了一种基于阈值的 Top-k 内存帧选择策略。该策略综合考量预测掩码质量 (IoU)、目标可见性 (防止遮挡帧污染) 以及运动评分，确保只保留最高质量的参考特征用于后续帧的注意力交互。

### 5. 实验效果
该方法在七个极具挑战性的数据集上进行了全面评估，表现优异：
*   **通用 VOT 基准 (LaSOT, TrackingNet, GOT-10k)**：以极小的延迟开销（基本与原模型持平），全面超越了现有基于 SAM 2 的 SOTA 方法（如 SAM2Long、SAMURAI 等），且展现出比传统监督学习跟踪器更强的泛化能力。
*   **复杂非线性场景 (Anti-UAV 系列数据集)**：在面临小目标、不规则非线性运动、红外跨模态等极端挑战的无人机跟踪数据集（如 Anti-UAV300 等）上，SAMOSA 的优势尤为明显，准确率 (Acc)、成功率 (Succ) 和 AUC 指标均获得了大幅度的实质性提升。

## 📄 FlowLong: Inference-time Long Video Generation via Manifold-constrained Tweedie Matching

- **链接**: https://huggingface.co/papers/2605.20910
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉-视频生成（Text-to-Video Generation）、多模态联合生成（音视频联合生成）、文本到3D场景生成（Text-to-3DGS）。

### 2. 一句话核心贡献
提出了一种名为 FlowLong 的免训练、与模型架构无关的推理期（Inference-time）框架，通过流形约束的 Tweedie 匹配和早期随机采样技术，解决了现有扩散模型在生成超出训练长度的长视频时出现的误差漂移、动作重复和一致性退化问题。

### 3. 使用指南
*   **输入**：文本提示词（支持全局提示词结合按时间段的局部提示词，以实现多镜头连贯生成）和初始随机噪声。
*   **输出**：任意长度的高质量长视频、音视频同步序列或大范围 3D 高斯溅射（3DGS）场景。
*   **使用方式**：即插即用（Plug-and-play）。完全在推理阶段运行，**无需任何额外训练或微调**。
*   **依赖与兼容性**：不依赖特定模型架构，可直接外挂于现有的流匹配/扩散视频模型（如 Wan 2.1-T2V、LTX-2）以及单步 3DGS 生成模型（如 AnySplat）。通过多窗口并行采样与内存中的重叠状态聚合算法，规避了由于自回归（KV-cache）引发的累计误差。

### 4. 主要创新点
1.  **免训练与无 KV-Cache 的并行生成架构**：摒弃了传统的自回归缓存复用策略，所有视频分段（Windows）从独立高斯噪声并行开始采样。这种设计从根本上消除了长视频生成中常见的曝光偏差（Exposure Bias）和误差漂移现象。
2.  **流形约束的 Tweedie 匹配（Manifold-constrained Tweedie Matching）**：在相邻分段的重叠区域，将各个片段预测出的“干净样本（Denoised Estimates / Clean Samples）”进行线性加权融合，而非直接在含噪状态下混合，以此作为指导损失拉回采样轨迹，强制实现平滑且符合数据流形的帧级过渡。
3.  **早期随机采样机制（Stochastic Early-Phase Sampling）**：在采样初期的“高噪声阶段”，每次完成 Tweedie 匹配修正后主动注入随机噪声，打破各独立片段基于确定性 ODE 轨迹的“惯性”，促进跨窗口的融合；随后在“低噪声阶段”平滑过渡到确定性 ODE 采样，以最大程度保留画面的细粒度视觉质量。

### 5. 实验效果
*   **长视频生成表现（30秒/60秒）**：在 VBench 基准测试（100个 MovieGen 测试提示词）中，FlowLong 在视觉质量、一致性以及尤其是**动态程度（Dynamic Degree）**上，全面超越了当前的免训练双向模型（如 RIFLEx, UltraViCo）和自回归模型，有效克服了传统方法在长视野下像素饱和、画面静止或动作重复的瓶颈。
*   **音视频联合生成**：在 LTX-2 架构上，成功生成了数倍于原生窗口长度的音视频联合序列，不仅无需微调，还保持了严格的音视频相位锁定（Phase-locked）。
*   **文本到 3D 场景生成（3DGS）**：应用于 VIST3A / AnySplat 管道时，突破了原生视频模型的视野限制。与基线相比，生成的 3D Gaussians 数量显著增加，空间覆盖度更广，且平均深度置信度得分从基线的 26.27 大幅提升至 41.52，几何结构更加致密可靠。

## 📄 ACC: Compiling Agent Trajectories for Long-Context Training

- **链接**: https://huggingface.co/papers/2605.21850
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大模型微调（大模型长上下文推理与智能体训练）

### 2. 一句话核心贡献
提出了智能体上下文编译（ACC）方法，将多轮智能体（Search、SWE、SQL）的交互轨迹自动转化为长上下文问答训练数据，有效解决了传统微调中跨轮次证据整合的监督盲区问题。

### 3. 使用指南
*   **输入**：多轮智能体执行轨迹（包含原始问题、工具调用响应、环境观察等，如网页搜索历史、代码仓库快照、数据库表）。
*   **输出**：长上下文问答对（QA pairs），即将所有收集到的真实证据、干扰项与原始问题拼接为一个极长的上下文背景，并以最终答案作为监督目标。
*   **使用方式**：无需人工标注，将生成的 QA 数据直接作为标准监督微调（SFT）的训练数据输入即可。该方法与现有的长上下文扩展架构或后训练策略正交，可无缝结合。
*   **开源情况**：完全开源，数据集（`groundhogLLM/ACC-dataset`，包含超 10,802 条轨迹，长度从 2K 到 128K 不等）和经过训练的权重（`ACC-Qwen3-30B-A3B`）均已在 HuggingFace 免费发布。

### 4. 主要创新点
1. **零人工标注的轨迹转化范式（Agent Context Compilation）**：首次系统性地将检索（Search）、软件工程（SWE）和数据库（SQL）三类真实智能体轨迹转化为长上下文 QA 数据。通过将分散在多轮交互中的信息拼装为单一长文档，自动提取具有复杂依赖关系的推理上下文，避免了传统高昂的长文档人工清洗与标注成本。
2. **打破传统 SFT 的“监督盲区”**：传统智能体 SFT 在计算 Loss 时会掩码（mask）工具返回的观测结果，仅对轮次级别的工具选择进行监督，导致模型无法学习全局证据整合。ACC 移除了中间决策步骤的监督，让最终答案的梯度直接作用于所有上下文 token，迫使模型学会跨越长距离定位和推理零散线索。
3. **基于干扰项与随机排列的难度增强**：在编译数据时，有意混入未访问的网页或未被修改的代码文件作为干扰项（Distractors），并对证据片段进行随机打乱（Permutation）。这阻断了模型对序列位置的死记硬背，迫使模型通过语义关联来进行信息检索与推理。

### 5. 实验效果
*   **长距离依赖推理实现飞跃**：基于 Qwen3-30B-A3B 使用 ACC 训练后，在 MRCR（多轮指代消解）任务上得分 68.3（**+18.1**），在 GraphWalks（长图遍历）任务上准确率达到 77.5（**+7.6**）。
*   **“以小博大”的参数效率**：经过 ACC 强化的 30B 模型，其长上下文依赖建模能力直接媲美参数量大近 8 倍的 Qwen3-235B-A22B 模型。
*   **通用能力无损**：在提升超长文本能力的同时，模型在 GPQA-Diamond、MMLU-Pro 和 AIME 等通用智力基准测试中未发生灾难性遗忘，甚至有小幅提升（如 GPQA 提升 2.49 分）。
*   **机制可解释性**：实验分析证实，ACC 训练后的模型涌现出“任务自适应的注意力重构”和“专家网络（MoE）特化”现象，证明模型真正学到了灵活处理长上下文的内在机制。

## 📄 Training Large Language Models to Predict Clinical Events

- **链接**: https://huggingface.co/papers/2605.12817
- **阅读来源**: HTML

### 1. 应用领域
**NLP-大模型微调 / 医疗人工智能**（重点聚焦于基于电子病历的临床事件概率预测与临床决策支持）。

### 2. 一句话核心贡献
提出了一种端到端的框架（Foresight Learning的临床扩展），将按时间排序的非结构化电子病历（EHR）笔记转化为包含“历史背景-自然语言问题-未来实际结果”的监督数据，从而在无需特征工程或构建特定终端分类器的情况下，微调大语言模型进行多种未来临床事件的精准预测。

### 3. 使用指南
*   **输入**：患者在特定“预测时间点”之前的所有非结构化临床记录（最多截断至 16,000 个 tokens），以及一个关于未来可能发生事件的自然语言问题（例如：“该患者在本次住院期间是否需要气管插管进行机械通气？”）。
*   **输出**：一个 0 到 1 之间的概率值（预测该事件在出院前发生的可能性），并附带基于可用临床记录生成的自然语言推理过程。
*   **硬件需求**：采用参数量为 120B 的开源大模型（gpt-oss-120b）作为基座，需要使用支持大参数量模型的 GPU 集群。由于只进行了 LoRA（Rank=32）轻量级微调，无需对全量参数进行更新，降低了训练成本。
*   **开源与数据情况**：训练数据依赖于 PhysioNet 平台上的 MIMIC-III 重症监护数据库（需申请严格的访问权限）。论文使用了通用的 LoRA 和 GRPO 强化学习/微调算法，但因医疗数据隐私协议限制，未直接公开可运行的代码库或微调后的模型权重，更多是作为该方法的系统性演示。

### 4. 主要创新点
1. **基于时间约束的自监督数据构建**：利用自然产生的时间戳，将临床笔记严格划分为“已知上下文”与“未来结果”。结合生成模型（Gemini 2.5 Flash），基于前期笔记生成预测问题，基于后期笔记解析真实标签，彻底避免了医疗预测模型中常见的“未来信息泄露（Look-ahead bias）”问题。
2. **统一的自然语言泛化预测接口**：摒弃了传统医疗预测任务中针对“死亡率”、“药物使用”等单一终点训练独立分类器的做法。将异构的临床结果统一转换为通用问答形式，单一模型即可处理药物、手术、器官支持、微生物学结果和死亡等多维度的概率预测。
3. **基于评分规则的推理微调（Foresight Learning）**：结合对数分数（Log Score）这一严格的概率预测评分规则，使用 GRPO 算法（批大小32，组大小4）来优化 LoRA 适配器。此方法不仅奖励能准确预测高概率结果的输出并惩罚过度自信的错误，同时还促使模型生成贴合真实轨迹的临床医学推理过程。

### 5. 实验效果
在 MIMIC-III 验证集抽样构建的含 500 个问题的保留测试集上，该微调模型取得了以下核心表现：
* **核心指标大幅优化**：相较于仅使用提示词工程的基础模型，微调后模型的预期校准误差（ECE）从 0.1269 骤降至 0.0398，Brier 分数从 0.1994 降至 0.1453，AUROC 从 0.6992 跃升至 0.7993，证明了其预测精度与校准度的极大提升。
* **高风险召回能力强**：Top-10% lift 指标达到 3.07，意味着在模型预测风险最高的前 10% 样本中，真实阳性事件的发生率是整体基线发生率的 3 倍，具有极高的临床落地审阅价值。
* **超越当前SOTA水平**：在相同的回顾性基准测试下，该轻量级适配器模型的整体表现不仅远超基础的大模型，甚至在点估计等指标上**略微超越了作为外部通用基准的 GPT-5**。在盲测评估中，微调模型在医学知识、数据依据等维度的推理胜率达到了 84.0%。

## 📄 Q-ARVD: Quantizing Autoregressive Video Diffusion Models

- **链接**: https://huggingface.co/papers/2605.21072
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 视频生成（特定为：自回归视频扩散模型 / Autoregressive Video Diffusion Models 的模型量化与高效推理部署）。

### 2. 一句话核心贡献
提出了首个专为自回归视频扩散模型（ARVDs）设计的量化框架 Q-ARVD，通过引入**最终质量感知的帧级权重机制**与**异常值感知的自适应双尺度量化策略**，成功解决了自回归生成中误差累积导致的敏感度不均与权重异常值异质性问题，在保持近乎无损生成质量的同时实现了显著的推理加速与显存压缩。

### 3. 使用指南
*   **输入**：全精度的预训练自回归视频扩散模型（如 Self-forcing, Causal-forcing）以及少量用于校准（Calibration）的视频文本提示/数据集。
*   **输出**：低精度量化模型（支持 W8A8、W4A8、W4A6 等多种位宽配置）及对应的量化参数（Scaling factors, 边界值等）。
*   **软硬件要求**：无需定制化专属硬件。方法基于常规 GPU（如 NVIDIA A6000）即可运行，作者提供了基于 Triton 编写的融合量化算子（包含激活量化 Kernel 和 INT8 GEMM+反量化 Kernel）。
*   **部署方式**：离线阶段完成异常通道检测与量化参数校准；在线推理阶段，根据离线切分好的正常/异常通道对激活值进行提取重排，直接调用 Triton 量化算子进行加速，额外开销极小。

### 4. 主要创新点
1. **揭示自回归视频量化的独有问题**：首次指出并实证分析了 ARVDs 量化面临的两大特有瓶颈：① **自回归误差累积导致帧级敏感度极度不平衡**（早期帧对量化误差极其敏感，呈指数级衰减）；② **权重异常值（Outliers）呈现高度异质性**（异常通道数量及分布在不同网络层和深度中差异巨大）。
2. **最终质量感知的帧权重机制 (Final-quality guided frame-weighting)**：不采用均匀校准，而是通过评估单帧量化对最终生成视频整体质量（MSE距离）的破坏程度，直接量化每一帧的敏感度。在量化校准重建的损失函数中，为敏感度高的早期帧分配更高权重，有效抑制了自回归过程中的误差传播。
3. **异常值感知的自适应双尺度量化 (Outlier-aware adaptive dual-scale quantization)**：引入改进的 Z-score (Modified Z-score) 并结合最小绝对幅度约束，实现对任意层异常通道的**自动化自适应检测**。检测后将异常通道与正常通道剥离，分别使用独立的量化器（赋予正常通道更小的缩放因子），从数学原理上降低了正常通道的舍入误差，且避免了统一处理引发的质量退化。

### 5. 实验效果
*   **视觉与精度表现**：在 VBench 扩展测试集上，针对 Self-forcing 和 Causal-forcing 模型，在 W8A8、W4A8、W4A6 三种位宽下，Q-ARVD 在核心的参考性评估指标（FVD-FP 和 LPIPS-FP）上全面大幅超越现有主流基线（MinMax, AdaRound, SmoothQuant, PTQ4DiT, SVDQuant 等）。尤其在 W4A8 和 W4A6 等极低位宽下表现出极强的鲁棒性，生成的视频在时间连贯性和语义一致性上实现了近乎无损的质量（Near-lossless visual quality）。
*   **部署与加速效果**：在 NVIDIA A6000 GPU 上通过 Triton 部署 INT8 模型，在 Batch Size 为 2 的配置下，实现了 **1.30 倍的端到端推理加速**（耗时从 18.02s 降至 13.85s）以及 **1.97 倍的模型体积缩减**（参数占用从 2.64GB 降至 1.34GB）。

计算机视觉与多模态智能体（图像生成 / AI Agent）

提出一种基于“工具编排视觉经验蒸馏”的自我进化图像生成智能体框架（GenEvolve），将传统的单步文本生图任务重构为包含事实搜索、参考图检索、技能调用和提示合成的智能体决策轨迹问题，显著提升了开放式复杂图像的生成质量与逻辑准确性。

### 3. 使用指南
*   **输入**：用户的自然语言开放式生图请求（Prompt）。
*   **输出**：经过智能体逻辑推理后生成的“提示词-参考图程序（Prompt-Reference Program）”，以及由下游图像生成模型最终渲染出的高质量图像。

2. **基于特权视角的视觉经验自我蒸馏（Visual Experience Distillation）**：不同于仅依赖图像级标量奖励（Scalar Reward）的方法，该机制通过对比同一请求下“最好”与“最差”的生成轨迹，提取结构化的“视觉经验”（如失败教训、正确检索词）。在训练中，这些经验仅作为“特权信息”输入给教师分支，以指导（蒸馏）正常上下文环境下的学生模型，提供高密度的 Token 级纠偏信号。

## 📄 Diversed Model Discovery via Structured Table Discovery

### 3. 使用指南

2. **基于信息块（Nugget-based）的动态库评估协议**：为了适应模型库的快速迭代，摒弃了固定的静态标注，提出从模型卡片中提取固定属性（如：模型、基础模型、变体、数据集、指标名、指标值）的6元组“Nugget”。通过将查询映射为对Nugget的约束，定量计算候选集的信息覆盖率和多样性。
3. **方向感知（Orientation-aware）的表格整合技术**：针对从不同模型卡片或论文中提取的表格结构混乱、甚至行列转置的问题，设计了方向感知比对算法。它能在整合前自动识别并转置未对齐的表格，最终生成一张紧凑、结构化的横向对比视图供用户决策。

### 5. 实验效果
在包含超过 **6万个 HuggingFace 模型**的数据集上，使用由学术检索数据集（LitSearch）改造而来的 **597 个模型推荐查询** 进行了全面评估。实验结果表明：
*   **覆盖率提升**：相较于传统的密集（Dense）、稀疏（Sparse）或混合（Hybrid）纯语义检索基线，引入表格结构感知的检索管道在“Nugget覆盖率”（即检索到的独立有效信息条目数）上获得了明显提升。
*   **算子性能差异**：在各种表格检索算子中，**Unionable（可联合）算子表现最强**，它能最稳定地召回大量格式对齐且信息丰富的模型评估证据。
*   **用户体验优化**：定性案例分析表明，该系统能为用户自动生成类似“Leaderboard（排行榜）”的整合对比视图，极大减少了用户阅读冗长文本的时间，提升了模型选型和权衡比较的效率。

## 📄 SpaceDG: Benchmarking Spatial Intelligence under Visual Degradation

- **链接**: https://huggingface.co/papers/2605.22536
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型（MLLMs）- 空间智能与推理、三维视觉理解、模型鲁棒性评测。

### 2. 一句话核心贡献
提出了首个针对真实世界“视觉退化（Visual Degradation）”场景的大规模空间智能数据集 **SpaceDG** 及其评测基准 **SpaceDG-Bench**，揭示了当前多模态大模型在不完美视觉输入下的脆弱性，并证明了“退化感知微调（Degradation-aware SFT）”能显著提升模型的空间推理鲁棒性。

### 3. 使用指南
*   **输入**：单视角或多视角图像（可能包含运动模糊、低光照、镜头畸变、JPEG压缩等9种视觉退化现象）以及相关的空间推理自然语言问题（如估计相机平移距离、物体相对位置、物理尺寸比较等）。
*   **输出**：关于空间属性的准确推断（格式为多选题、二分类回答或精确数值）。
*   **硬件要求**：数据集的生成依赖带有物理退化合成模块的 3D Gaussian Splatting (3DGS) 渲染管线；进行多模态大模型的监督微调（SFT）需要较高的算力支持（论文中微调实验使用了 8 张 H200 GPU）。

### 4. 主要创新点
1.  **首创物理驱动的 3D 退化数据合成引擎**：将真实世界的物理退化过程（涵盖光学、气象、光度、数字等4大类9种退化类型）直接嵌入到 3D 高斯溅射（3DGS）渲染管线中。这一方法既能合成高度逼真的受损图像，又能确保底层 3D 几何空间真实标签（Ground Truth）的绝对不变和准确。
2.  **构建系统化的空间智能任务分类框架**：设计了涵盖以相机为中心（如位姿变化）、以物体为中心（如深度、尺寸）及相机-物体关系（如跨视角方向）三大类、共计 11 种细粒度空间推理任务，有效规避了传统仅依赖二维边界框的局限性。
3.  **自动化标注与高标准的质量控制闭环**：通过 SAM3 提取 3D 实例、利用 VLM 生成视角无关的自然语言描述，并结合几何协视性（Covisibility）自动计算物理答案。此外，引入了“VLM 自动化过滤+双重专家人工审查”的机制，确保最终 SpaceDG-Bench 基准的高质量与无歧义性。

### 5. 实验效果
*   **模型普遍脆弱**：在对 25 个代表性大模型（包括 GPT-5.4、Gemini-3.1-Pro 等）的评测中，所有模型在面临视觉退化时空间推理准确率均出现一致且显著的下降（例如 Gemini-3.1-Pro 从干净图像的 63.1% 降至退化图像的 56.7%）。其中细粒度的目标级感知（如计数）受到的影响最为严重。
*   **人类表现对比**：人类在干净图像上准确率为 80.4%，但在退化条件下也会下降至 59.5%，说明该基准具有极高的内在挑战性。
*   **微调效果显著**：在 SpaceDG 上对开源模型（如 Qwen3-VL-8B-Instruct）进行退化感知微调后，其在干净图像上的准确率大幅提升（49.1% -> 73.2%），在退化输入上的表现更是从 42.1% 飙升至 66.1%（一举超越了人类的 59.5%），并且完全没有损害模型在常规通用评测基准（如 MMSI-Bench）上的性能。

## 📄 DecQ: Detail-Condensing Queries for Enhanced Reconstruction and Generation in Representation Autoencoders

- **链接**: https://huggingface.co/papers/2605.22777
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 图像生成（潜在扩散模型 Latent Diffusion Models、图像分词器/自编码器）。

### 2. 一句话核心贡献
提出了一种轻量级的细节凝聚查询（DecQ）框架，通过从冻结的视觉基础模型中间层提取细粒度信息，打破了表征自编码器中图像重建保真度与生成质量之间的权衡（Trade-off）瓶颈。

### 3. 使用指南
*   **输入**：图像（训练/重建阶段）或高斯噪声（扩散生成阶段）。
*   **输出**：重建或生成的高质量图像。
*   **使用流程**：
    1. **分词器阶段**：保持视觉基础模型（如 DINOv2）权重冻结。将其各中间层的特征通过交叉注意力模块（Condenser）输入给少量（如 8 个）可学习的 Query Tokens，以提取颜色、纹理等底层细节，并与原始 Patch Tokens 拼接后送入解码器。
    2. **扩散生成阶段**：将 Patch Tokens 与 Query Tokens 拼接为一个长序列，由扩散模型（如 DiT）联合进行去噪预测，最后共同输入解码器生成最终图像。
*   **硬件要求**：实验采用了 8 张 NVIDIA H200 GPU 进行训练。由于该模块极为轻量（计算量仅增加 3.9%，参数量增加 5.8%），常规支持标准扩散模型训练的 GPU 硬件均可适用。
*   **代码开源**：文中未明确提供开源链接，但详细阐述了网络架构配置与超参数（如基于 DINOv2-B 和 500M 参数的 ViT-XL 解码器）。

### 4. 主要创新点
1. **非破坏性的细节凝聚查询机制（Detail-Condensing Queries）**：引入少量可学习的查询 Tokens，通过单向交叉注意力机制提取冻结 VFM 中间层的底层特征。这一设计既补充了细粒度重建信息，又完全不干扰原有 VFM 预训练的强语义潜在空间。
2. **Patch 与 Query 的联合扩散生成**：在生成建模中，将携带全局语义的 Patch Tokens 与携带底层细节的 Query Tokens 进行联合去噪预测。研究发现，预测细节查询 Token 不仅能提升最终图像细节，还能反哺并促进全局 Patch Token 的生成质量。
3. **稀疏且深浅结合的特征聚合策略**：揭示了“浅层特征利于重建、深层特征利于生成”的规律。通过在特定层（如第 0、3、6、9 层）稀疏插入聚合模块，在计算开销极小的前提下，同时实现了重建与生成性能的最优平衡。

### 5. 实验效果
在核心数据集 **ImageNet 256×256** 上展现出卓越性能：
*   **高保真重建**：相比于基线 RAE（基于冻结的 DINOv2），仅使用 8 个查询 Token，就将重建的 PSNR 从 19.13 dB 大幅提升至 22.76 dB，显著改善了背景色彩、文本内容和细粒度纹理的恢复。
*   **高质量生成与加速**：生成阶段的收敛速度比基线 RAE 快 **3.3 倍**；生成质量上，无引导（without guidance）的 FID 达到 **1.41**，使用引导后 FID 进一步降至 **1.05**，优于现有先进方法。
*   **强通用性**：该架构不仅在 DINOv2 上表现优异，在更换为 SigLIP2 等其他视觉基础模型时，依然能获得一致的重建与生成提升。

## 📄 Live Music Diffusion Models: Efficient Fine-Tuning and Post-Training of Interactive Diffusion Music Generators

- **链接**: https://huggingface.co/papers/2605.22717
- **阅读来源**: HTML

### 1. 应用领域
人工智能生成内容 (AIGC) - 音乐音频生成、流媒体交互式音频处理。

### 2. 一句话核心贡献
提出了一种名为实时音乐扩散模型（LMDMs）的新架构，通过引入块级 KV 缓存和无强化学习的对抗性后训练方法，将离线音频扩散模型成功转化为可在普通消费级硬件上低延迟运行的流媒体交互式音乐生成器。

### 3. 使用指南
*   **输入**：支持多模态输入，包括全局文本提示（如“摇滚乐”及其平滑过渡）、时间对齐的音乐草图信号（如提取自实时演奏的音高、CQT特征）或伴奏音频流。
*   **输出**：连续、低延迟的流式音乐音频块（可作为数字音频工作站中的“生成式延迟”效果器）。
*   **硬件要求**：摆脱了传统大模型（如40GB VRAM 的 LMMs）对工业级算力的依赖，可在普通的消费级硬件（如搭载消费级显卡的游戏笔记本电脑）上进行本地实时推理。
*   **部署方式**：无需从头训练，可通过微调现有的开源扩散模型（如 Stable Audio Open）获得。模型支持导出为 ONNX 格式，并可直接集成到基于 C++/JUCE 开发的音频应用程序中。

### 4. 主要创新点
*   **支持 KV 缓存的注意力掩码与路由机制**：通过在初始状态注入时设计简单的路由机制以及定制的非对称注意力掩码，将干净的历史上下文与含噪的目标生成块解耦。这使得原本非因果的扩散模型能够在扩散步乃至时间跨度上使用 KV 缓存，从而达到甚至超越离线自回归模型的推理效率。
*   **ARC-Forcing 无强化学习后训练范式**：提出了一种无需强化学习（RL-free）且无需预训练奖励模型的全局对抗监督方法（Adversarial Relativistic Contrastive）。该方法直接在模型多块自回归生成的长序列上进行后训练，有效缓解了长文本生成的误差积累问题，并支持极少步数（1-8步）的高质量采样。
*   **统一流媒体场景下的多模态交互控制**：将离线扩散模型强大的可控性（包括文本提示渐变、局部音频草图引导、基于单个Stem轨道的盲生成伴奏）成功移植到近乎实时的流媒体框架中，使其成为人类乐手可以即兴互动的“生成式乐器”。

### 5. 实验效果

## 📄 Same Architecture, Different Capacity: Optimizer-Induced Spectral Scaling Laws

- **链接**: https://huggingface.co/papers/2605.21803
- **阅读来源**: HTML

### 1. 应用领域
NLP-大模型预训练、深度学习优化算法（Optimization）、大模型Scaling Laws。

### 2. 一句话核心贡献
揭示了优化器（如AdamW、Muon）的选择会从根本上改变大模型前馈网络（FFN）有效表示容量的 Scaling Laws，证明了决定模型容量的是“架构-优化器”组合，而不仅仅是单一的模型架构。

### 3. 使用指南
*   **适用场景**：大规模语言模型的预训练框架设计、优化器选型与架构探索（如评估增宽FFN时的边际收益）。
*   **输入与输出**：
    *   **输入**：训练过程中的数据（特定频段Token的FFN层前置/后置激活值），以及设定的 Rényi 阶数。
    *   **输出**：经验协方差矩阵的“软谱秩（Soft Rank，衡量弥散容量）”和“硬谱秩（Hard Rank，衡量主导特征容量）”。
*   **评估方法**：通过对比模型在不同FFN宽度下的软/硬秩增长斜率（Scaling Exponent），评估当前优化器是否真正有效地将增加的参数转化为有用的特征维度，特别是针对长尾罕见词（TAIL Tokens）。
*   **硬件要求**：度量和计算本身无需特殊硬件，论文实验在常规单机/多机GPU（NVIDIA RTX 3090 24GB）上即可完成GPT-2 160M/350M级别的扫描实验。

### 4. 主要创新点
1.  **提出优化器诱导的谱尺度定律（Optimizer-Induced Spectral Scaling Laws）**：首次发现即使架构相同，优化器也会导致截然不同的容量分配。例如，AdamW倾向于将新增的FFN宽度转化为低方差的“软容量”，而Muon/NorMuon类矩阵优化器则能有效将其转化为高方差的“硬容量”（主导表征方向）。
2.  **打破“Loss对齐等于表征对齐”的错觉**：实验证明，即使通过延长训练时间或调整学习率让AdamW达到与低秩正交优化器（Dion）相同的验证集困惑度（Loss），其内在的硬秩表示结构仍会发生崩溃。这表明优化器限制的是表征空间的几何结构，而非单纯的收敛速度。
3.  **发现架构干预与优化器几何的强耦合效应**：证明了传统的架构改进（如改变注意力头数、移除RoPE位置编码）带来的容量变化，往往被优化器的特性所主导或重塑。优化器的更新秩（Update Rank）是限制尾部（TAIL）罕见知识学习的核心瓶颈，倡导将“架构-优化器”进行联合设计（Co-design）。

### 5. 实验效果
在 GPT-2（160M 和 350M 参数）以及 FineWeb 预训练数据集上进行了广泛实验：
*   **尾部词（TAIL Tokens）表征增强**：在扩展FFN宽度时，Muon 和 NorMuon 优化器在尾部Token的硬秩容量扩展指数上远超 AdamW，大幅缩小了硬/软秩的不对称性。
*   **架构改造鲁棒性**：在极具训练挑战的 Partial-PostLN 架构设置下（例如75%的层使用PostLN），AdamW 训练直接发散，而基于正交化更新和神经元级归一化的 NorMuon 能够保持稳定收敛并达到可用的验证困惑度（PPL 18.06）。
*   **位置编码移除（NoPE）重分配**：在移除 RoPE 后，Muon/NorMuon 会显著改变模型处理位置依赖特征的深度分布，使其从偏好头部词（HEAD）逆转为偏好尾部罕见词（TAIL），从而实现容量在不同频段Token间的有效重分配。

## 📄 WorldKV: Efficient World Memory with World Retrieval and Compression

- **链接**: https://huggingface.co/papers/2605.22718
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 视频生成与交互式世界模型 (Video Generation & Interactive World Models)

### 2. 一句话核心贡献
提出了一种免训练的推理期框架 WorldKV，通过键值对缓存（KV Cache）的选择性检索与冗余压缩，在自回归视频世界模型中实现了兼顾实时生成效率与长期场景一致性的高效“世界记忆”。

### 3. 使用指南
*   **输入**：动作指令（如键盘按键、鼠标操作）或相机的轨迹位姿，以及当前生成的视觉上下文。
*   **输出**：具备长期场景记忆（多次重访同一场景时内容不发生偏移或幻觉）的连续视频流。
*   **使用方式**：纯推理期即插即用（Training-free），无需对基础模型进行微调或蒸馏，可直接包装并应用于冻结的自回归视频模型（如 Matrix-Game-2.0, LingBot-World-Fast 和 Inspatio-World）。
*   **硬件要求**：需要一定的 CPU 或 GPU 内存用于卸载（Offload）和存储被驱逐的 KV 缓存，但相比全量 KV 缓存机制，显著降低了 GPU 显存（VRAM）的峰值占用，避免了长视频生成时的 OOM（内存溢出）问题。

### 4. 主要创新点
1. **基于位姿/动作的免训练“世界检索”（World Retrieval）**：打破传统滑窗推理直接丢弃历史缓存的做法，将滑窗外被驱逐的 KV 缓存保存在 CPU/GPU 内存中。当模型重访某一场景时，根据当前相机位姿或累积动作的相似度，从内存中精准召回相关视角的历史缓存，并无缝插入当前的注意力窗口。
2. **基于 Key-Key 相似度的“世界压缩”（World Compression）**：利用连续视频帧存在大量视觉冗余的特性，将每个视频块（Chunk）的首帧设为“锚点帧”，通过计算非锚点帧与锚点帧的 Key 向量余弦相似度，裁剪掉高度相似的冗余 Token（保留新暴露区域和动态物体），使每个视频块的存储体积减半，从而在固定注意力预算下容纳 2 倍的历史记忆。
3. **揭示并验证“选择性检索优于全量注意力”现象**：发现在长序列视频生成中，全量 KV 缓存注意力不仅拖慢速度，还会因为引入过多无关信息导致“注意力稀释（Attention Dilution）”和误差累积。WorldKV 通过精准检索与压缩过滤了噪声，其场景重建的保真度不仅完胜滑窗推理，在许多场景下甚至超越了计算成本极高的全量 KV 缓存机制。

### 5. 实验效果
*   在不同参数规模的自回归视频世界模型（1.3B 的 Matrix-Game-2.0 和 14B 的 LingBot-World-Fast）上进行了长程轨迹（如一分钟级别）的广泛测试。
*   **重访保真度**：在反映场景重访一致性的定量指标（LPIPS、PSNR、SSIM、FID）上，WorldKV 表现优异。在14B模型上持平全量KV机制，在1.3B模型上全面超越全量KV机制；且在没有任何微调的情况下，与专门经过记忆模块训练的 Baseline 表现相当或更优。
*   **推理效率与显存**：将生成吞吐量维持在接近原始滑窗推理的水平（FPS 约为全量 KV 注意力的 2 倍），成功保障了实时交互的速度（如维持 ~8.5 FPS），且有效控制了显存增长曲线，解决了长时间 Rollout 必然导致的显存溢出崩溃。

## 📄 AnyMo: Geometry-Aware Setup-Agnostic Modeling of Human Motion in the Wild

- **链接**: https://huggingface.co/papers/2605.22715
- **阅读来源**: HTML

### 1. 应用领域
多模态大语言模型（MLLMs） - 穿戴式传感器人体动作理解（人体活动识别、运动-语言跨模态建模）

### 2. 一句话核心贡献
提出 AnyMo 框架，通过物理几何感知的虚拟 IMU 数据模拟和全身动作 Token 化技术，解决了穿戴设备因佩戴位置、方向等异构设置导致的泛化难题，实现了跨设备、跨数据集的开放词汇人体动作语言理解。

### 3. 使用指南
* **输入**：真实场景下采集的稀疏穿戴式 IMU 数据（多轴加速度与角速度序列），辅助以描述传感器可见佩戴位置的文本提示及具体任务指令。
* **输出**：文本序列（如具体的动作分类标签、开放式的动作过程字幕描述）或多模态对齐特征向量（用于 IMU 与文本的双向检索）。

1. **基于物理与几何感知的密集 IMU 模拟**：突破了传统局限于稀疏骨骼关节点的数据合成限制。该方法利用人体表面网格（Mesh）在全身密集生成虚拟候选 IMU 信号，结合表面法线、解剖学切线以及真实设备噪声先验，构建了极其丰富且符合物理规律的佩戴设置数据分布。
2. **设备无关的特征表示与全身 IMU Token 化**：设计了**掩码跨视图预测对比学习（MCVPCL）**目标，训练时空图卷积网络（ST-GCN）学习具有时序一致性的全身特征；随后通过乘积量化 VAE (PQ-VAE) 将多位置的稀疏 IMU 观测转化为紧凑、离散的全局全身运动 Token，有效消除了具体传感器位置和数量的依赖。
3. **结合生成与判别的多任务运动-语言指令微调**：将离散的 IMU Token 注入 LLM 词表，并通过多任务对比指令微调策略，联合优化自回归生成损失（用于字幕生成和多选题问答）与对称 IMU-文本对比损失（用于对齐动作旁白和动作标签），使大模型全面掌握了穿戴动作语义。

### 5. 实验效果
AnyMo 在三个核心任务上实现了最先进的（SOTA）泛化表现：
* **零样本人体活动识别（Zero-shot HAR）**：在 14 个完全未见过的下游数据集（涵盖经典 HAR 与真实世界场景）上，AnyMo 的平均准确率（Accuracy）、宏F1分数（Macro-F1）和 Recall@2 分别比最强基线提升了 **11.7%、11.6% 和 22.6%**。

* **动作字幕生成（Captioning）**：在零样本动作字幕生成任务中，生成的自然语言描述更为精细且保留了完整的动作级语义，其 BERT-F1 评分较基于大模型直接提示的最强基线提升了 **18.8%**。

提出了一种基于目标层面的归因框架（GoalWeaver），通过拆解和追踪对话过程中的直接贡献与间接影响，量化并揭示了 AI 模型在人机协作中对目标塑造及“微观决策”的真实贡献度。

### 3. 使用指南

*   **处理机制**：利用自动化 Pipeline（基于 GPT-4o 等 LLM 和文本嵌入模型），提取对话中的动作（Actions），将宏观目标分解为可验证的具体需求（Requirements），并追踪这些需求的演变历史。
*   **输出**：协作目标层级图、各项需求的生命周期，以及人类和 AI 各自在“目标塑造”和“目标执行”上的直接与间接贡献量化评分。
*   **硬件与部署**：无需特殊计算硬件，仅需调用大语言模型 API。研究还配套开发了一个名为 `GoalWeaver-viewer` 的基于 Web 的交互式可视化工具，用于向用户展示归因分析结果。

### 4. 主要创新点
1.  **从“产出物归因”转向“过程和目标归因”**：打破了传统方法仅依赖最终产出（如文本水印、代码最终修改）来评估 AI 贡献的局限。该框架将协作意图拆解为细粒度、可验证的“需求”，在对话回合的动态过程中量化双方的贡献比例。
2.  **捕捉并量化隐性的“间接影响”（Indirect Influence）**：不仅记录谁直接提出了需求，还追踪一方的操作（如提出澄清问题、提供草稿、暴露错误或提供选项）如何作为上下文“间接”促使另一方制定新需求，并总结出了 11 种常见的人机间接影响模式。
3.  **通过系统设计干预模型主动性（Initiative）**：证明了目标塑造行为可以被动态调控。通过不同的交互设计和 Prompt 策略（如模拟用户意图不明确，或强制 Agent 在采取行动前必须沟通），可以显著放大或抑制大模型在协作中的目标塑造行为。

### 5. 实验效果
*   **真实世界日志分析（638份真实对话）**：在宏观层面上，人类设定了绝大部分方向（占贡献的 75–89%）；但在越具体、越底层的微观要求上（尤其是编程和数据分析等技术任务中），AI 模型的贡献显著增加，甚至提出了许多人类从未考虑过的具体实现细节。
*   **控制变量模拟实验**：当模拟用户输入更加模糊（Underspecified）时，AI 生成需求的比例增加了 39 个百分点；当强制 Agent 在调用工具前先发送消息沟通（Chat-based）时，AI 的目标塑造比例几乎翻倍（达 42.9%）。
*   **用户研究**：使用 `GoalWeaver-viewer` 分析自己的对话日志后，参与者对自身执行贡献的评分（5分制）平均下调了近 2 分。用户表示该工具让他们惊讶地意识到 AI 在背后做出了大量未经用户明确指示的“微观决策”，有效纠正了用户对 AI 辅助工作的认知偏差。

## 📄 Efficient Agentic Reasoning Through Self-Regulated Simulative Planning

- **链接**: https://huggingface.co/papers/2605.22138
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理（NLP） - 智能体推理（Agentic Reasoning）、大语言模型（LLMs）、强化学习与决策规划。

### 2. 一句话核心贡献
提出了一种将智能体决策分解为“直觉执行(System I) + 模拟规划(System II) + 自我调节(System III)”的三系统架构，通过赋予模型自主决定“何时规划”与“预测未来状态”的能力，大幅降低了推理过程中的 token 消耗，同时保持甚至提升了复杂任务的成功率。

### 3. 使用指南
*   **输入**：自然语言描述的任务指令（如数学问题、科学推理、图表分析或网络搜索需求）以及环境观测状态。
*   **输出**：包含自我调节决策（是否规划）、结构化模拟规划（动作序列及预测结果）和实际执行动作（如调用工具、输出最终答案）的推理轨迹。
*   **工具依赖**：框架内集成了三种标准工具：网络搜索引擎（Web Search）、网页内容提取器（Web Browser）和无状态的 Python 代码执行沙盒（Python Sandbox）。
*   **硬件需求**：模型训练（SFT+GRPO强化学习）需要多卡 GPU 算力支持（如论文使用的 8-32 张 Hopper GPU），但推理阶段的硬件要求与同等参数规模（如 8B, 30B）的标准大语言模型一致。
*   **代码与开源**：论文详细公开了用于收集多模块推理数据（v0.1）和从现有 CoT 轨迹中重构结构化计划数据（v1.0）的详细 Prompt 系统，基于通用的开源 LLM 训练框架（如 ms-swift 和 vLLM）即可复现。

### 4. 主要创新点
1. **引入自调节机制（System III - Configurator）**：打破了传统大模型“无约束强制链式思考（CoT）”的局限。模型内置配置器，根据当前任务的不确定性、难度或紧急情况，自主决定此时是需要生成新计划、继续原有计划，还是直接执行动作，从而有效避免了过度思考带来的计算浪费。
2. **基于语言世界模型的模拟规划（System II - Simulative Planner）**：当模型决定需要规划时，利用 LLM 自身作为“世界模型”，要求其生成结构化的【状态评估 -> 提议动作 -> 预测未来状态】序列。这种机制将规划锚定在对未来的预测上，使长视野决策具备可验证的逻辑连贯性。
3. **高效的“计划重构”训练范式**：设计了 v1.0 数据标注流水线，利用强大的教师模型（Teacher LLM）对原有的交织执行轨迹进行事后标注，抽取出结构化的 System II 计划和 System III 决策节点。实验证明，在此数据上进行监督微调（SFT）并结合强化学习（RL）后，模型学会了“加深规划的视野”而不是盲目“增加规划的频率”。

### 5. 实验效果
*   **评估基准**：在涵盖数学（如 AIME-24）、科学问题（GPQA）、表格分析和 Web 信息搜索等 4 大领域的 11 个主流评测集上进行了测试。
*   **性能越级**：基于该架构训练的 30B 规模模型（*-v1.0-30B*）综合 Pass@1 准确率达到 71.3%，与规模庞大得多的顶尖模型（如 685B 的 DeepSeek-V3.2 和 1T 的 Kimi-K2.5）在同等工具配置下的表现相当。
*   **大幅降低推理成本**：与达到类似精度的同规模智能体大模型（如 MiroThinker-v1.5-30B 等）相比，该方法由于具备自调节能力，平均每条推理轨迹的 reasoning tokens 消耗**减少了 25.8% – 95.3%**，实现了极佳的精度-效率平衡。

## 📄 Unsupervised Process Reward Models

- **链接**: https://huggingface.co/papers/2605.10158
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理（NLP） - 大语言模型推理（LLM Reasoning）、强化学习对齐（RLAIF）、过程奖励模型（PRM）与测试时计算扩展（Test-Time Scaling）。

### 2. 一句话核心贡献
提出了一种完全无监督的过程奖励模型（uPRM）训练方法，无需任何人工步骤级标注或最终答案的真实性验证，通过提取大模型的下一个 Token 概率对多条轨迹联合评分并进行强化学习蒸馏，大幅降低了过程监督的扩展成本并缓解了奖励作弊（Reward Hacking）问题。

### 3. 使用指南
*   **输入**：无标注的模型推理轨迹（如数学题及多步解答），无需提供标准答案或步骤正确性标签。
*   **输出**：每个推理步骤的细粒度正确性概率/奖励分数（Dense step-wise scores）。
*   **使用方式**：可直接替代传统的监督式 PRM，用于推理阶段的测试时扩展（如 Best-of-N 采样、波束搜索验证）或作为强化学习（RL）训练过程中的密集奖励信号。
*   **计算与硬件要求**：
    *   *训练阶段*：由于使用了多轨迹联合评估机制（拼接多个轨迹），需要一定显存来处理较长的上下文（论文中使用 8 张 H200 GPU 在约 4.25 小时内完成训练）。
    *   *推理阶段*：**无额外开销**。训练好的 uPRM 独立处理单条轨迹，没有任何额外的上下文长度要求，与现有的 PRM 部署完全一致。

### 4. 主要创新点
1.  **基于下一个 Token 概率的无监督评分机制**：通过在推理步骤中交替插入正确/错误标记（`+`/`-`），直接提取底层大语言模型对这些标记的生成概率作为打分函数，从而隐式地捕捉步骤的正确性，彻底摆脱了对外部真实标签和人工标注的依赖。
2.  **利用上下文学习的多轨迹联合评估（Joint Assessment）**：为了克服 LLM 独立评估单条轨迹时不可靠的问题，uPRM 将同一批次内的多条轨迹拼接起来让大模型进行联合评分。这种设计激活了 LLM 的上下文学习能力，并通过设计启发式的惩罚项（Correction term）防止模型陷入全部预测“无错”或“全错”的退化解。
3.  **定制化的 Actor-Critic 低方差梯度估计器**：为了将这种隐式的联合评分能力蒸馏成一个独立、高效的 PRM 参数化模型，作者提出采用带熵正则化的强化学习目标，并专门设计了一个能够复用隐层状态和结合未来轨迹特权信息（Privileged Information）的 Critic 网络，实现了高效且稳定的梯度更新。

### 5. 实验效果
*   **步骤级错误检测**：在 ProcessBench 数据集上，uPRM 相比于常规的 LLM-as-a-Judge（独立评分基线）取得了最高 **15% 的绝对准确率提升**，尤其在 OlympiadBench 等高难度基准上表现最为显著。
*   **测试时扩展（TTS）效能**：在 Best-of-N 和 DVTS 采样策略下，uPRM 的表现与使用真实标签训练的监督式 PRM 相当。在使用 Llama-3.2-1B-Instruct 生成 256 个候选答案时，相比多数投票（Majority Voting）基线，准确率提升了 **最高 6.9%**。
*   **强化学习与抗奖励作弊（Reward Hacking）**：将 uPRM 作为 PURE 强化学习框架的奖励信号时，表现出显著优于传统监督式 PRM (sPRM) 的训练稳定性。sPRM 极易导致策略快速崩溃（输出空答案或无意义极短答案），而 uPRM 能够维持长期的鲁棒优化，使 Qwen2.5-Math-1.5B 在三大数学基准上的平均准确率较基于结果验证奖励（VR）的模型提升了 **4 个百分点**。

## 📄 Perception or Prejudice: Can MLLMs Go Beyond First Impressions of Personality?

- **链接**: https://huggingface.co/papers/2605.22109
- **阅读来源**: HTML

### 1. 应用领域
多模态大语言模型 (MLLM) - 视频理解与情感/社会认知计算 (Video Understanding & Social Cognition)

### 2. 一句话核心贡献
提出了首个要求多模态大模型必须基于视觉行为证据进行人格推理的基准测试 MM-OCEAN，揭示了当前 MLLM 在人格预测中普遍存在的“评分正确但缺乏事实依据（即偏见 Prejudice）”的核心缺陷。

### 3. 使用指南
*   **输入**：一段短视频（例如15秒的单人讲话视频）及其对应的语音转录文本。
*   **输出**：预测任务分为三个层级：大五人格（Big Five）维度的等级评分（T1）、基于行为证据的开放式推理文本（T2）、以及定位具体行为线索的结构化多选题解答（T3）。
*   **硬件要求**：需配备能够运行多模态大模型的 GPU（论文中评估开源模型使用了 NVIDIA H200 并基于 vLLM 进行部署），闭源模型可通过官方 API 调用；仅作 T2 裁判的纯文本 LLM 推理计算量较小。
*   **开源情况**：作者开源了 MM-OCEAN 数据集（包含 1,104 个测试视频及 5,320 个经过验证的细粒度多选题），以及智能体构建 pipeline 的提示词与输出 JSON 架构。

### 4. 主要创新点
1. **提出“有理有据的人格推理”范式 (Grounded Personality Reasoning, GPR)**：打破了传统表象人格识别（APR）仅预测数值的黑盒回归范式，强制要求模型的人格判断必须锚定在特定的、可观察的微观行为线索（如微表情、头部动作、视线变化）上。
2. **多智能体与人机协作的高质量数据构建 Pipeline**：设计了包含观察者 (Observer)、心理学家 (Psychologist)、考官 (Examiner) 和对齐者 (Aligner) 四个角色 LLM 智能体的五阶段流水线，并结合专业人工验证，生成了带有时空标签（时间戳和边界框）的精确数据集。
3. **全新的多层级诊断评估指标**：建立了一套包含“评分 (Rating)、推理 (Reasoning)、依据定位 (Grounding)”的三层评估框架，并创新性地提出了四个样本级别的失败模式指标：偏见率 (PR)、虚构率 (CR)、整合失败率 (IR) 和全面依据率 (HR)，用于精准透视模型在哪一步发生认知断裂。

### 5. 实验效果
在对 27 个具有代表性的 MLLM（13 个闭源，14 个开源）的基准测试中，得出了以下核心结论：
*   **“偏见鸿沟”普遍存在**：超过 50% 的“准确人格评分”根本没有正确检索到视频中的行为证据作为支撑。这意味着传统基准测试严重高估了模型的能力，模型多在利用表面相关性进行“盲猜”。
*   **综合推理表现极差**：全面依据率（Holistic-Grounding Rate, HR，即评分、推理、线索定位全部正确）的整体表现极低，跨度仅在 0.0% 到 15.6% 之间，业界最强模型（Gemini 3 Flash）也仅能达到 15.6%。
*   **开源与闭源的差异集中在视觉定位**：在人格评分和语言推理任务上，开源模型和闭源模型差距极小（已实现能力民主化）；但闭源顶尖模型在“细粒度时空视觉线索定位”这一维度上依然拥有压倒性的显著优势。

## 📄 TerminalWorld: Benchmarking Agents on Real-World Terminal Tasks

- **链接**: https://huggingface.co/papers/2605.22535
- **阅读来源**: HTML

### 1. 应用领域
人工智能-智能体 (AI Agents)、大语言模型应用 (LLMs)、自动化软件工程 (Automated Software Engineering)、终端/命令行环境交互 (Terminal/CLI Interaction)。

### 2. 一句话核心贡献
提出了一种可扩展的数据引擎，能够自动将真实的、自然发生的人类终端操作录屏（In-the-wild recordings）逆向工程转化为高保真、可自动评测的终端任务，并借此构建了包含1530个真实任务的评测基准 **TerminalWorld**，填补了现有基准依赖人工编写且脱离真实开发场景的空白。

### 3. 使用指南
*   **输入**：面向结果的自然语言任务指令（明确预期目标和输出路径，但不包含具体步骤）。
*   **输出**：智能体在终端中执行的一系列 Shell 命令（通过纯 CLI 交互，不含 TUI 界面应用），最终达成预期的系统状态改变。
*   **运行环境**：隔离的 Docker 容器。该基准会基于自动合成的 Dockerfile 构建专属环境以供任务回放和智能体执行。
*   **硬件要求**：运行评测框架（如 Harbor harness）及 Docker 容器仅需常规 CPU 服务器；LLM 推理通常通过 API 调用完成（无本地 GPU 强制要求）。
*   **开源情况**：数据集（包含任务指令、参考解决方案、执行环境 Dockerfile 和测试套件）已公开，为保护隐私与版权，原始录屏以超链接形式指向源平台（asciinema.org）。

### 4. 主要创新点
1.  **从真实噪音数据中蒸馏意图与参考解**：利用大语言模型（LLMs）对冗长、含有拼写错误和重试的原始真实录屏文本进行清洗，提取出面向结果的自然语言任务指令（消除泄题风险）和干净的可执行 Bash 脚本作为参考答案。
2.  **动态环境逆向工程**：不依赖静态配置，而是通过智能体分析参考脚本，自动推断依赖关系（如基础镜像、系统包），并结合 Docker 构建失败的真实报错信息进行“执行-反馈”闭环修复，成功复原出原作者的底层运行环境。
3.  **基于执行状态反馈的自动化测试生成**：通过捕获 Docker 容器执行参考答案前后的文件系统状态差异（State deltas）来生成断言测试。同时引入“三轮验证机制”（正常运行必须通过、空运行必须失败、残缺运行必须失败），彻底解决了 LLM 静态生成测试代码时极易出现的假阳性和假阴性问题。

### 5. 实验效果
*   **整体表现低迷**：当前最前沿的 LLMs 在该真实世界基准上依然挣扎。即使是表现最好的模型（Claude Opus 4.7）最高通过率也仅为 62.5%，所有评估模型的平均通过率约为 54.8%。
*   **效率悖论 (Efficiency Paradox)**：实验揭示了智能体在失败任务上消耗了不成比例的算力（占用63%的成本却只占43%的尝试次数），增加思考步数和 Token 消耗并未带来通过率的提升，表明智能体在开放且真实的终端环境中缺乏有效的规划和停止策略。
*   **打破原有评估认知**：TerminalWorld 的成绩与现有专家人工编写的基准（如 Terminal-Bench 2.0）仅表现出弱相关性（Pearson r=0.29）。例如，GPT-5.5 在旧基准上得分近 83%，但在该基准上仅为 53.5%，证明现有基准高估了模型在真实工作流中的能力。
*   **解题路径差异**：虽然任务源自人类录屏，但智能体成功解决任务时所用的命令与人类真实使用的命令重合度极低（中位数仅为 21.4%），证明智能体是通过探索寻找有效替代方案，而非单纯模仿人类。

## 📄 Minimalist Visual Inertial Odometry

- **链接**: https://huggingface.co/papers/2605.19990
- **阅读来源**: HTML

### 1. 应用领域
机器人学 - 视觉惯性里程计（Robotics - Visual-Inertial Odometry, VIO） / 资源受限平台下的移动机器人导航。

### 2. 一句话核心贡献
提出了一种极简的平面视觉惯性里程计（VIO）系统，仅使用四个带有 Gabor 光学掩膜的下视光电二极管和一个 IMU，通过联合优化的时序卷积网络（TCN）解码线速度，以极低的功耗（数十微瓦）实现了与高分辨率相机 VIO 相当的高精度运动轨迹估计。

### 3. 使用指南
*   **输入**：由4个带有定制掩膜的光电二极管采集的一维低维时间序列信号，以及 IMU（陀螺仪）提供的偏航角速度（Yaw rate）。
*   **输出**：机器人连续的二维平面运动轨迹（线速度与偏航角速度融合后的位姿估计）。
*   **硬件要求**：需要高度定制的硬件支持。包括：4个排列为 $2\times2$ 网格的 Hamamatsu 光电二极管、印在透明胶片上的正负分离 Gabor 光学掩膜（安装在二极管前方 5mm 处）、用于抑制强反射光的物理遮光罩，以及一个标准的 IMU。
*   **代码开源/部署情况**：论文未明确提供开源链接。但在部署时，系统的物理掩膜参数和 TCN 网络完全在基于物理的模拟器中进行联合训练，**无需在真实世界中收集数据或进行任何微调（Zero-shot）** 即可直接在真实机器人上部署运行。

### 4. 主要创新点
1.  **光学域物理空间滤波**：摒弃传统的高分辨率图像传感器，将 Gabor 滤波器物理化为光学掩膜。传感器在移动时，直接在光学域对地面纹理进行连续的空间交叉相关计算，将空间频率转化为时间信号，其主频率直接编码了机器人的线速度。这一设计将传感器功耗降至惊人的 $37 \mu W$，比传统相机低两个数量级。
2.  **软硬件端到端联合优化**：开发了一个端到端可微的物理模拟框架，不仅用于训练时序卷积网络（TCN）来回归速度，还能同时计算梯度并反向传播，用于寻找最优的物理 Gabor 掩膜参数（如空间频率和包络宽度），实现了传感器硬件设计与软件解码算法的联合设计。
3.  **解决方向歧义与地形适应性**：通过利用四像素阵列构造正交（Quadrature）信号（将正弦和余弦 Gabor 函数拆分为非负掩膜），成功解决了单一传感器无法判断前进/后退方向的歧义问题；同时，通过在模拟器训练中引入随机传感器高度变化（模拟现实中路面颠簸引起的视差和相位偏移），使模型隐式学会了抵抗机械振动和不平整地形的干扰。

### 5. 实验效果
系统在真实的差分驱动机器人上进行了广泛测试，涵盖了多种室内外环境和复杂地面纹理（地砖、地毯、砖块、大理石、混凝土等），总行驶距离达920米（耗时87分钟）。
*   **核心指标表现**：该极简系统的平均绝对轨迹误差（ATE）仅为 **0.34 米**，平均端点漂移率低至 **0.60%**。

## 📄 Disentangling Sampling from Training Budget in Class-Imbalanced CT Body Composition Segmentation

- **链接**: https://huggingface.co/papers/2605.20405
- **阅读来源**: HTML

### 1. 应用领域

### 2. 一句话核心贡献
将少样本学习中的“片段式采样（Episodic Sampling）”引入全监督类别不平衡的医学图像分割中，并深刻揭示了在对比评估不同采样策略时，“训练迭代预算（Training Iteration Budget）”是一个常被忽视但起决定性作用的混淆变量。

### 3. 使用指南
*   **输入**：CT 图像的 2D 切片（需经过 Hounsfield Unit 窗口化及线性归一化预处理）。
*   **输出**：像素级的多类别组织分割掩码（包含9种特定的肌肉/脂肪组织及背景）。
*   **使用方法**：该方法是一个模型无关、即插即用（plug-and-play）的数据加载/采样策略。在训练阶段，不修改网络架构和损失函数，仅用片段式采样器替代常规的随机采样器，使得每个 mini-batch 内包含固定数量的随机类别和均衡的切片（Support/Query）。
*   **硬件需求**：实验在单张 NVIDIA V100 GPU (32GB VRAM) 上完成，常规深度学习计算硬件即可支持。
*   **开源情况**：论文明确表示代码已开源。

### 4. 主要创新点
1.  **跨领域解耦与应用片段式采样**：将“片段式采样”从传统的少样本/度量学习（Metric-learning）语境中剥离出来，作为一种独立的数据批次构建（Batch Construction）策略应用于标准的全监督训练中，实现了对批次内（within-batch）罕见类和常见类的显式平衡控制。
2.  **揭示基于 Epoch 调度的评估漏洞**：研究指出，由于不同采样策略在每个 Epoch 产生的迭代次数不同（尤其在小数据集下差距极大），传统基于 Epoch 的学习率衰减和早停调度会带来不公平的“训练迭代预算”差异。论文排除了这一混淆变量，提出了“感知迭代次数（iteration-aware）”的对齐评估协议。
3.  **发现类别均衡批次的隐式正则化效应**：在严格对齐训练迭代预算后，研究发现片段式采样的巨大优势虽有收窄，但相比随机和加权采样，它能显著推迟模型过拟合（持续改进的迭代次数约多出 3 倍），证明了强制类别均衡的批次构建具有低成本的隐式正则化效果。

### 5. 实验效果
在包含 210 个扫描样本、具有极度类别不平衡的 SAROS 数据集（9个前景类别）上测试表现如下：
*   **全数据（100%）训练机制下**：三种采样策略表现相当，片段式采样平均 Dice 为 0.882，随机和加权采样均为 0.878。
*   **低数据（10%）基于 Epoch 调度下**：由于片段式采样的实际迭代次数是另外两者的 12 倍，取得了碾压性优势（平均 Dice 0.787，对比随机 0.758 和加权 0.762）。
*   **低数据（10%）对齐迭代预算下**：排除迭代次数带来的混淆后，各采样方法差距大幅缩小（片段式 0.771，随机 0.761，加权 0.759），但片段式采样在罕见类别（如 IMAT 等）上的表现依然最优，并且表现出更强的抗过拟合能力。

## 📄 SEGA: Spectral-Energy Guided Attention for Resolution Extrapolation in Diffusion Transformers

- **链接**: https://huggingface.co/papers/2605.22668
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 图像生成（Text-to-Image，扩散型 Transformer 模型的跨分辨率生成/分辨率外推）。

### 2. 一句话核心贡献
提出一种名为 SEGA 的免训练（Training-free）推理方法，通过在去噪过程中根据图像隐变量的频谱能量动态分配 RoPE 注意力缩放比例，解决了扩散型 Transformer（DiTs）在生成超出其训练分辨率的超高清图像时出现的结构崩溃和细节丢失问题。

### 3. 使用指南
*   **输入**：文本提示词（Prompt）以及目标高分辨率尺寸。
*   **输出**：高质量、结构连贯的超高分辨率图像（最高可超 3600 万像素，如 6K/8K）。
*   **部署与集成**：该方法为纯推理阶段适配（Training-free），**无需任何模型微调或重新训练**，不引入任何可学习参数。可作为“即插即用”的模块，直接集成到采用旋转位置编码（RoPE）的标准扩散模型管道中（如 Flux、Qwen 等）。
*   **硬件需求**：无需额外的训练算力，但由于超高分辨率下的 Transformer 注意力计算复杂度呈二次方增长，生成极端分辨率图像仍需大显存 GPU（论文实验基于 NVIDIA H100 GPU，全精度转为 `bfloat16` 以节省显存）。

### 4. 主要创新点
1. **内容与时间感知的动态缩放（Content-Aware Dynamic Scaling）**：打破了现有方法采用“静态、全局统一的注意力缩放”导致全局结构与局部细节难以兼顾的权衡瓶颈。SEGA 会在每个去噪步骤中，通过 FFT 实时提取当前隐状态的二维频谱图，根据图像本身的复杂度和当前的去噪进度生成动态的注意力缩放图。
2. **维度级别的 RoPE 频率映射（Dimension-wise Frequency Coupling）**：精准洞察了不同 RoPE 维度与特定空间频率（控制全局粗略结构的低频 vs. 控制局部纹理的高频）的耦合关系。SEGA 为高能量频段分配较弱的缩放以避免特征过度放大，为低能量频段分配较强的缩放以保留位置辨识度。
3. **基于频谱平坦度的自适应强度控制（Spectral Flatness Gating）**：引入“频谱平坦度（Spectral Flatness）”作为标量控制阀。在去噪早期（图像主要是纯噪声，频谱平坦），降低干预强度；随着图像结构逐渐浮现、能量在特定频段集中，系统自动增强注意力的重新分配力度。

### 5. 实验效果
*   **测试基准**：在 Aesthetic-4K 数据集以及精心构建的 Zero-Shot 复杂提示词基准上，基于 Flux 和 Qwen 架构进行了广泛评估。
*   **核心表现**：在 2K、3K、4K 及以上的四个高分辨率外推设置下，SEGA 在语义对齐（CLIP Score）和人类视觉偏好（ImageReward, PickScore 等）指标上**全面超越现有的 SOTA 免训练基线方法**（如 YaRN、DyPE、UltraImage 以及多阶段指导方法）。
*   **视觉质量**：在生成超大分辨率（如非标准比例、甚至超过原生训练分辨率数倍的 36M 像素）时，SEGA 成功消除了常见的重复伪影、模糊和语义混淆，达到了前所未有的结构连贯性和极致细节保真度。此外，在迁移到闭源/微调基线（如 SDXL、Diffusion-4K）的对比中，SEGA 依然保持最优或次优的水平，展现了极强的泛化能力。

## 📄 From Reasoning Chains to Verifiable Subproblems: Curriculum Reinforcement Learning Enables Credit Assignment for LLM Reasoning

- **链接**: https://huggingface.co/papers/2605.22074
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大模型微调 (LLM Post-training) / 强化学习 (Reinforcement Learning) / 复杂数学与逻辑推理 (Mathematical Reasoning)

### 2. 一句话核心贡献
提出了一种基于子问题课程的强化学习框架 (SCRL)，通过将高难度推理问题分解为一系列可独立验证的子问题，在无需外部奖励模型的情况下，实现了大模型推理过程中针对中间步骤的细粒度信用分配（Credit Assignment），有效克服了困难问题中因奖励稀疏导致的学习失效。

### 3. 使用指南
*   **输入**：高难度的目标问题、该问题的专家参考解答（Reasoning Chain）。
*   **输出**：经过 RL 优化后的大语言模型，具备更强的中间步骤探索能力和困难问题求解能力。
*   **工作流程**：
    1. **离线预处理**：调用外部 LLM（如 DeepSeek-V3.2），基于参考解答将原问题改写为由易到难、且每个都有确定答案的子问题序列（课程），最后一个子问题即为原问题。
    2. **结构化提示**：在训练阶段，通过 Prompt 要求模型使用特定标签（如 `<p1>...</p1>` 至 `<pK>...</pK>`）按顺序逐步生成各个子问题的推理和答案。
    3. **策略训练**：系统独立验证每个子问题的答案，使用改进的 GRPO 算法计算 Token 级别的优势值（Advantage）并更新策略模型。
*   **硬件要求**：需要具备进行大模型 RL 微调的算力，论文实验在 NVIDIA RTX 5090、A100-40GB 和 H20-96GB GPU 环境下完成。
*   **代码开源**：论文原文未直接给出代码仓库链接，但算法完全基于标准的开源框架（如 GRPO）和结构化 Prompt 构建，易于复现。

### 4. 主要创新点
1. **面向 On-policy 探索的子问题课程构建 (Verifiable Subproblem Curriculum)**：不同于以往直接提供静态提示（Hints）的方法，SCRL 将复杂问题拆解为包含原始问题在内的渐进式子问题序列，迫使模型在单次生成（Rollout）中自主探索并掌握前后依赖的中间推理结论。
2. **基于标签的子问题级归一化与细粒度信用分配 (Subproblem-level Normalization)**：利用模型输出的结构化标签，算法在同一个 Rollout 组内对*每个子问题位置*独立计算奖励的均值和方差并进行归一化，随后将优势值直接反向分配给对应答案区间的 Token，从而在没有过程监督模型（PRM）的情况下实现了细粒度的信用分配。
3. **进度感知修正与混合训练机制 (Progress-aware Correction & Mixed-group Training)**：为了防止模型“奖励破解”（即前面的子问题答错，却强行猜对后续子问题来骗取奖励），算法严格截断计算，只对从头开始连续答对的子问题序列给予奖励；同时，将包含子问题课程的 Rollout 与只包含原问题的 Rollout 混合训练，确保模型在提升中间推理能力的同时，不丧失直接解决原题的能力。

### 5. 实验效果
*   **核心数据集**：在 7 个主流数学推理基准（MATH500, AIME24, AIME25, AMC23, OlympiadBench, Omni-MATH, IMO-Bench）上进行了测试。
*   **整体表现**：在 Qwen3-4B-Base 和 Qwen3-14B-Base 上，SCRL 相比于标准的 GRPO 分别实现了 **+4.1 和 +1.9 个平均绝对百分点**的提升，全面超越了 DAPO、QuestA、NuRL 等强劲的课程学习强化学习基线。
*   **困难问题突破**：在难度最高的 AIME24, AIME25 和 IMO-Bench 三个测试集上，Qwen3-4B-Base 的 pass@1 指标平均提升达 **+3.7 分**，证明了方法在极具挑战性的任务上的探索优势。
*   **泛化能力 (OOD)**：在未包含在训练目标中的科学问答和代码生成领域（GPQA, HumanEval, LiveCodeBench），模型性能同样获得一致提升，证明 SCRL 提升了底层的多步推理通用能力，而非仅仅是对特定格式的过拟合。

## 📄 Forecasting Downstream Performance of LLMs With Proxy Metrics

- **链接**: https://huggingface.co/papers/2605.18607
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大语言模型评估、预训练数据选择与性能外推预测

### 2. 一句话核心贡献
提出利用候选模型在专家推理轨迹上的 Token 级别预测分布统计量构建“代理指标（Proxy Metrics）”，在无需自回归生成或昂贵直接评测的情况下，极其高效且准确地预测大模型在下游任务（尤其是复杂推理任务）中的真实性能。

### 3. 使用指南
*   **输入**：待评估的候选大模型、测试题目，以及该题目对应的专家推理轨迹（纯文本格式，可由人类专家或如 Llama-3.3-70B 等强模型生成）。
*   **处理过程**：无需让模型进行耗时的文本生成。直接将题目和专家轨迹输入候选模型进行**单次前向传播（Single forward pass）**，提取每个 Token 级别的预测分布。
*   **指标计算**：基于预测分布计算 10 种核心统计量（如 Top-k 准确率、熵、模型置信度等），并结合 8 种 Token 加权方案（如赋予低频词或模型高不确定性位置更高权重），得到 80 维的代理指标库。
*   **输出**：通过简单的线性模型（如 RankSVM）或幂律拟合（Power-law fits），输出候选模型在目标任务上的相对排名或外推的准确率得分。
*   **硬件与资源**：对硬件要求极低，只需标准 GPU 显存能够运行候选模型的单次推理即可。由于不涉及解码生成（Decoding），单个样本计算耗时通常不到 1 秒。使用标准的 HuggingFace Transformers 库即可实现。

### 4. 主要创新点
1.  **基于专家轨迹的免生成评估范式**：摒弃了传统的交叉熵损失（与下游能力脱节）和直接评测（稀疏且极其昂贵），巧妙地将模型对专家思维链的局部跟踪能力（Next-token 预测）作为预测最终解题能力的强信号。
2.  **细粒度 Token 级加权策略**：创新性地认识到并非所有 Token 都同等重要。通过逆词频加权（关注生僻实体/数字）和熵加权（关注模型不确定的关键推理步），精准提取出了最能反映模型真实能力的鉴别性信号。
3.  **仅依赖离散文本的强兼容性**：打破了此前 SOTA 方法（如 rBridge）需要获取专家模型完整概率分布（Logprobs）的限制。该方法仅需专家提供的离散文本 Tokens，使得闭源前沿 API 模型或人类均可作为完美的专家信号源。

### 5. 实验效果
*   **跨家族模型选择（Cross-Family Ranking）**：在涵盖 6 大系列、18 个不同架构与规模（0.6B-70B）的大模型群体中，基于代理指标的线性排序器在未见的复杂推理测试集上达到了 **0.93 的 Spearman 秩相关系数**（作为对比，通用文本交叉熵损失的相关性仅为 0.28）。
*   **预训练语料选择（Data Selection）**：在 DataDecide 测试集上，仅使用相当于目标 1B 模型算力 **万分之一（$10^{-4}$）** 的极小代理模型，即可实现超过 **85%** 的语料对排序准确率，大幅刷新了该领域的帕累托前沿。
*   **训练轨迹性能外推（Training-time Extrapolation）**：在跨越 **100倍 ($100\times$)** 算力规模的训练过程中，利用早期 Checkpoints 预测模型最终的下游任务准确率，其外推误差（RMSE）比现有的基于损失 (CE loss) 或纯算力的基线方法**降低了约 50%**。

## 📄 OmniPro: A Comprehensive Benchmark for Omni-Proactive Streaming Video Understanding

- **链接**: https://huggingface.co/papers/2605.18577
- **阅读来源**: HTML

### 1. 应用领域
多模态大语言模型 (Multimodal LLMs)、流式视频理解 (Streaming Video Understanding)、主动式AI智能体 (Proactive AI Agents)。

### 2. 一句话核心贡献
提出了首个用于全面评估全模态主动式流式视频理解的基准测试集 **OmniPro**，解决了现有基准过度依赖纯视觉信号、采用假性主动（固定轮询）以及任务覆盖面窄的问题。

### 3. 使用指南
*   **输入**：连续的视频帧流和音频流（包含语音与非语音环境音），以及用户在视频开始时提供的自然语言指令（例如：“当画面中出现白猫时告诉我它的坐标”）。
*   **输出**：模型需要自主决策**何时响应（When）**并输出**相应内容（What）**（如：在精确的时间戳输出目标状态、计数或动作指导）。
*   **评估模式**：
    *   **Probe（探针）模式**：适用于所有视觉语言模型（无需流式推理能力），在真实触发点前后查询模型，评估其静态内容理解能力。
    *   **Online（在线）模式**：要求模型具备原生流式处理能力，逐帧处理视频流并自主决定触发响应，评估真正的流式主动交互能力。
*   **硬件需求**：论文中开源模型的推理评估在 NVIDIA A800 80GB GPU 上进行。
*   **开源情况**：基准数据集已遵循 CC BY-NC 4.0 协议开源（禁止商业用途），代码与 Prompt 模板公开。

### 4. 主要创新点
1.  **全模态跨认知层级的任务分类体系**：构建了包含3个认知层级（感知、追踪、推理）、涵盖6种基础视频理解能力（警告、监控、定位、计数、旁白、预测）的9大子任务，提供了2700个高质量人类验证样本。
2.  **高度的跨模态（特别是音频）依赖性设计**：数据集中 84% 的样本依赖音频信息（语音或非语音环境音），并为每个样本提供了模态隔离标签，填补了现有基准无法区分“全模态模型”与“纯视觉模型”的空白。
3.  **首创的双模式评估协议（Dual-mode Protocol）**：彻底摒弃了传统基准中定时轮询（Polling）的伪流式评估方法。其 Online 模式强制模型在没有外部提示的情况下，于连续视听流中进行多触发点的自主决策，并对过度触发进行惩罚。

### 5. 实验效果
研究团队在 OmniPro 上评估了 11 款代表性模型（如 Gemini-3-Flash, Qwen2.5-Omni, MiniCPM-o 4.5 等），主要发现如下：
*   **整体性能与开源/闭源差距**：当前模型整体表现欠佳，闭源模型 Gemini-3-Flash 在 Probe 模式下达到最高 40.4% 的准确率，几乎是最佳开源模型（22.1%）的两倍；在复杂的推理级任务（如步骤指导）上差距尤为明显。
*   **流式主动决策极具挑战**：在真正的流式 Online 模式下，表现最好的开源模型 MiniCPM-o 4.5 的 F1 分数仅为 20.9%，特别是在需要密集生成的任务中性能骤降。
*   **长时鲁棒性严重不足**：模型性能随视频时间推移出现显著衰减，在长视频（>180秒）后期的性能平均仅保留初期（0-60秒）表现的 37%。在线流式模型在长视频后期几乎完全失效。
*   **模态利用瓶颈**：尽管音频信号能带来一致的性能提升（+2.4 到 +11.1），但各模型对音视频的融合利用能力差异巨大，且所有模型在感知“非语音声音（如环境音效）”时的表现均为最弱。

## 📄 AutoRubric-T2I: Robust Rule-Based Reward Model for Text-to-Image Alignment

- **链接**: https://huggingface.co/papers/2605.17602
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉与多模态 - 文本到图像生成（Text-to-Image Generation）、人类偏好对齐（Human Preference Alignment）、奖励模型设计（Reward Modeling）与强化学习。

### 2. 一句话核心贡献
提出了 AutoRubric-T2I 框架，通过从极少量（仅256对）图像偏好数据中自动学习和筛选稀疏且带权重的自然语言评分规则（Rubrics），构建了高可解释性、低训练成本且抗“奖励作弊（Reward Hacking）”的文本到图像对齐奖励模型。

### 3. 使用指南
*   **输入**：文本提示词（Prompt）、生成的图像、极少量的人类偏好对比数据（如256对图像偏好对）。
*   **输出**：一组精简的、带有权重的自然语言评分规则集（Rubric Set），以及针对输入图文对的最终标量奖励得分。
*   **硬件与环境依赖**：
    *   **训练/提取阶段**：无需对稠密奖励模型进行昂贵的梯度微调。提取和优化规则集仅耗时约几小时（如在 4 块 A6000 GPUs 上运行）。
    *   **推理阶段**：依赖现成的视觉语言模型（VLM，如开源的 Qwen3-VL 部署在 vLLM 上，或调用 Gemini API）作为裁判（Judge）来根据规则进行打分。由于并行评估多条规则，需要一定的显存或 API 吞吐量支持。
*   **代码开源情况**：论文实验基于开源模型（Qwen3-VL、SD-3.5-Medium）和开源强化学习流程（Flow-GRPO），正文中提供了完整的算法伪代码与详细的超参数，具备极强的可复现性（依据常规惯例，此类具备完整复现细节的框架通常会开源相关代码和抽取的规则集）。

### 4. 主要创新点
1.  **无限维空间的稀疏全局规则学习**：将规则学习形式化为无限维稀疏逻辑回归问题。区别于依赖人工编写或为每个 prompt 动态生成规则的方法，该方法在离线状态下从偏好数据中学习一套全局通用的、带有权重的自然语言评分标准。
2.  **基于 $L_1$ 正则化的规则特征筛选**：利用 VLM 裁判对候选规则集进行打分，并采用带有正向权重约束的 $L_1$ 正则化逻辑回归，自动剔除冗余或与人类偏好相关性弱的规则，保留最具判别力的 Top-N 规则，确保了奖励信号的紧凑性与高度可解释性。
3.  **基于课程学习的难例挖掘与迭代优化（Curriculum-bucketed hard-pair mining）**：在迭代过程中，根据当前规则集的预测误差边缘（Margin）将预测失败的偏好对分桶。早期关注大误差错误以发现缺失的宏观评价维度，后期关注高分误判例以挖掘细粒度规则，从而不断自适应地扩充和完善规则池。

### 5. 实验效果
*   **偏好预测准确率（奖励模型评估）**：在极低数据需求下（仅使用不到 0.01% 的标注偏好数据），AutoRubric-T2I 在 MMRB2（领域外基准）上显著提升了 VLM 的零样本判别能力，整体准确率超越了 HPSv3、UnifiedReward 等经过大规模数据微调的强大标量奖励模型，同时也击败了 AutoRule 等现有规则生成基线。
*   **下游强化学习（T2I-RL）微调表现**：作为 Flow-GRPO 的奖励信号对 SD-3.5-Medium 进行微调，在 TIIF 和 UniGenBench++ 核心数据集上，不仅整体指令遵循能力得分大幅提高（例如在 TIIF 长提示词上得分从 62.7% 提升至 67.9%），在复杂推理、空间关系、目标数量和文本渲染等细粒度属性上的改善尤为明显，成功抑制了传统标量奖励模型容易引发的“为追求高对比度/明亮度而牺牲语义细节”的作弊现象。
*   **人类主观盲测**：在 4 选 1 的人工评估中，AutoRubric-T2I 优化的模型生成图像获得了 44.8% 的最高选择率（随机基线为 25%），在统计学上显著优于基准模型和传统奖励模型优化的结果。

## 📄 TransitLM: A Large-Scale Dataset and Benchmark for Map-Free Transit Route Generation

- **链接**: https://huggingface.co/papers/2605.22355
- **阅读来源**: HTML

### 1. 应用领域
城市计算-智能交通路线规划 / 自然语言处理（NLP）- 大语言模型（LLM）时空推理与应用

### 2. 一句话核心贡献
构建了首个千万级的大规模公共交通路线规划数据集与基准（TransitLM），证明了大语言模型无需依赖外部结构化地图和传统路由引擎，仅凭海量数据训练即可直接根据起终点（OD）信息实现端到端、无地图（Map-Free）的高精度公交路线生成。

### 3. 使用指南
*   **输入**：用户的自然语言查询语句（可选）以及起终点（Origin-Destination）的 GPS 坐标或 POI 名称。
*   **输出**：结构化的 JSON 格式路线规划方案。包含详细的交通方式、线路序列、带有换乘标记的完整站点 ID 序列、预估距离/时间/票价，以及接驳（首末一公里，如步行、骑行）的具体信息。
*   **实现机制**：基于 Qwen3（0.6B-4B）等基座模型。需将所有站点 ID 注册为专用独立 Token，然后使用提供的 1390 万条连续预训练（CPT）语料让模型内化交通网络拓扑结构，最后在特定任务上进行监督微调（SFT）。
*   **硬件要求**：模型训练需使用 GPU/PPU 集群进行分布式训练（论文中采用了 DeepSpeed ZeRO-3 显存优化策略）。
*   **代码与数据开源**：论文声明该数据集（含预训练语料和微调基准数据）及 Benchmark 已/即将公开提供（论文中提及 "The dataset and benchmark are available at [链接预留]"）。

### 4. 主要创新点
1. **无地图的隐式空间接地（Map-Free Spatial Grounding）**：打破传统路线规划对结构化地图图数据库和图搜索算法（如 Dijkstra/RAPTOR）的依赖。模型通过学习海量轨迹数据，隐式掌握了公共交通的地理拓扑结构，仅凭输入的纯 GPS 坐标即可精准映射并规划出合理的登车/下车站点。
2. **专有词表扩展与两阶段训练架构**：通过将 120,845 个现实站点 ID 直接扩展进大模型词表（每个站点作为一个独立的 Token），彻底消除了通用模型基于字符组合生成的“幻觉站点”。结合大规模连续预训练（CPT）和联合监督微调（SFT），实现了单一模型在多任务（最优路线、偏好感知、多路线推荐）上的统一。
3. **填补领域数据与基准空白（TransitLM 构造）**：提取并清洗了中国四个超大城市（北京、上海、深圳、成都）的 1300 万条高价值真实路线规划日志，首次将“用户行为选择标签”、“站到站完整拓扑结构”和“全量静态网络描述”结合，提出了涵盖三个互补任务的综合评估基准体系。

### 5. 实验效果
*   **生成准确度与合理性**：微调后的 Qwen3-4B 专属模型能够生成结构高度连贯的路线，站点接地精度（Station Grounding）达 **96%**，最优路线完全匹配率（Route Exact Match）最高达 **71.0%**。路线的数值预测（预估耗时、距离、票价等）通过率超过 **92%**，平均绝对百分比误差（MAPE）低至 **2.1%** 以下。
*   **对比通用大模型（SOTA LLMs）**：即使在放宽评估标准的情况下， TransitLM 专属模型也大幅超越了最先进的通用大模型（如 GPT、Gemini、Claude，其最优匹配率均不足40.2%）。更重要的是，即使与**允许调用地图 API 工具**的智能体 LLM 相比，TransitLM 的原生端到端生成效果依然不相上下。
*   **极端环境（纯GPS无文本输入）表现**：在去除所有地名文本提示，仅保留起点与终点 GPS 坐标的消融实验中，通用大模型的路线生成能力直接崩溃（匹配率跌至 <1%），而 TransitLM 的性能几乎零衰减，有力证明了模型真正内化了深度的时空拓扑表示，而非依赖文本语义的表面记忆。

### 1. 应用领域
AI辅助形式化数学 - 定理证明代码优化与重构（涉及大语言模型智能体、Lean 4定理证明、多目标代码生成）。

### 2. 一句话核心贡献
提出了一种无需微调的检索增强代理框架 **Lean Refactor**，通过外置带有“编译成本”和“版本兼容性”等元数据的代码重构策略库，引导大语言模型实现对 Lean 证明代码在长度、编译时间和跨版本兼容性上的多目标可控优化。

### 3. 使用指南
*   **输入**：冗长的 Lean 4 证明代码，以及用户指定的优化目标（如：追求最短代码长度、最低编译时间，或适配特定的 Lean/Mathlib 版本）。
*   **输出**：经优化且能通过编译器验证的简练 Lean 4 证明代码。
*   **依赖与硬件**：该框架是“即插即用”的，无需微调模型。用户仅需调用主流大模型（如 Gemini, Claude, GPT 系列）的 API 或具备运行开源大模型的 GPU 算力，并配置本地的 Lean 4 编译器以提供实时反馈。
*   **开源情况**：研究团队开源了迄今为止最大的 Lean 重构数据集，包含超20万对长短证明代码样本以及逾9000个带有详细元数据标注的重构策略库。

### 4. 主要创新点
1.  **构建带元数据的解耦策略库**：将重构知识从 LLM 模型权重中剥离，构建了一个高度结构化的外部策略数据库。每个策略不仅包含代码转换模式，还密集标注了该策略预期的“编译时间缩减率”和“兼容的 Lean/Mathlib 版本”。
2.  **多目标条件检索与重排机制**：突破了以往单目标优化的局限。在推理阶段，系统直接根据用户的目标动态改变检索规则（例如：对兼容性目标进行版本过滤，对编译时间目标依据元数据进行重排），无需任何重新训练即可灵活控制优化方向。
3.  **大模型无关的闭环代理架构 (Agentic Workflow)**：设计了“规划器 (Planner) - 重构器 (Refactorer) - 调试器 (Debugger)”三段式智能体循环。规划器负责拆解长代码，重构器依据检索到的策略进行改写，调试器结合 Lean 编译器的报错信息进行局部修复，这套流程可无缝适配各类通用大语言模型。

### 5. 实验效果
*   **代码压缩率**：在竞赛级测试集（如 miniF2F, Putnam2025）上实现了 **>70%** 的代码长度缩减；在结构更复杂的科研级项目（如 PFR, Analysis）上实现了 **>20%** 的压缩。
*   **编译效能**：在以降低编译时间为目标时，成功削减了 **>30%** 的编译成本（基于心跳/heartbeat指标及挂钟时间测试）。
*   **综合对比与鲁棒性**：全面超越了先前的专门微调模型（ProofOptimizer）以及直接使用工具增强的 Claude Code。同时，经过特定版本策略过滤重构后的代码，在面对未来的 Lean/Mathlib 版本升级时，展现出了比原版冗长代码更强的跨版本存活率。

## 📄 DelTA: Discriminative Token Credit Assignment for Reinforcement Learning from Verifiable Rewards

- **链接**: https://huggingface.co/papers/2605.21467
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理（NLP） - 大语言模型强化学习（尤其是基于可验证奖励的强化学习 RLVR、大语言模型复杂推理能力提升）。

### 2. 一句话核心贡献
提出了一种基于鉴别器视角的 Token 级信用分配方法（DelTA），通过重新加权强化学习目标函数以放大具有正负奖励区分度的 Token 梯度，解决了响应级奖励与 Token 级更新之间的粒度不匹配问题。

### 3. 使用指南
*   **输入**：大模型生成的 Prompt 及对应响应序列（Responses），以及环境给出的响应级别可验证奖励（如：数学题最终答案的对错）。
*   **输出**：经过 RLVR 优化、推理能力增强的大模型参数。
*   **使用方式**：DelTA 可作为即插即用的模块集成到现有的序列级 RLVR 框架（如 DAPO）中。该方法通过在当前批次上执行少量交替迭代，计算出每个 Token 的固定权重系数（作为停止梯度乘数），随后代入常规的强化学习代理目标函数中进行模型更新。
*   **硬件与计算资源**：依赖常规的大模型分布式 RL 训练硬件。为解决显存和计算瓶颈，该方法仅使用最后输出层（LM-head）的隐藏状态作为轻量级梯度代理来计算系数，无需计算或存储全参 Token 梯度。计算过程带来的时间开销较小（实验表明单步训练时间仅增加约15%以内）。

### 4. 主要创新点
1.  **提出 RLVR 的局部鉴别器理论视角**：揭示了标准的序列级 RLVR 策略更新本质上是在 Token 梯度空间中构建了一个隐式的线性鉴别器。指出传统按优势加权的质心容易被高频的共享模式（如标点符号、格式化 Token）主导，从而稀释了真正能区分高低奖励响应的稀疏关键特征。
2.  **基于质心对比的判别性 Token 评分机制**：通过计算候选 Token 梯度与正、负优势响应质心之间的平方距离差，设计了一种带有熵正则化的软分配函数。该机制能够精准识别并放大更代表自身奖励阵营（而非对面阵营）的 Token 梯度方向。
3.  **高效的自归一化加权替代目标设计**：将上述算出的判别性评分映射为有界系数，并引入自归一化机制替换传统 RLVR 中的均匀 Token 平均。在不改变全局学习率尺度的前提下，动态重塑了有效更新方向，实现了更具对比度的策略梯度下降。

### 5. 实验效果
*   **核心数学推理任务**：在 7 个困难的数学基准测试（如 AIME24/25/26、HMMT25/26、Brumo 25）中，DelTA 全面超越了强基线模型（如 DAPO、GRPO、FIPO、SAPO）。在 Qwen3-8B-Base 和 Qwen3-14B-Base 上，其平均得分分别比同规模最强基线提高了 3.26 分和 2.62 分，且有效缓解了常规方法中因长度增加导致的奖励停滞问题。
*   **跨领域与跨模型泛化性**：在代码生成（HumanEval+、MBPP+）以及域外科学知识测试（GPQA、MMLU-Pro）上均展现出持续的性能提升。同时在不同的模型架构（Olmo3-7B-Base）上也验证了该方法的有效性，证明其具有广泛的通用性。

## 📄 Sensor2Sensor: Cross-Embodiment Sensor Conversion for Autonomous Driving

- **链接**: https://huggingface.co/papers/2605.22809
- **阅读来源**: HTML

### 1. 应用领域
自动驾驶 (Autonomous Driving) / 计算机视觉-跨模态生成 (Computer Vision - Cross-modal Generation)

### 2. 一句话核心贡献
提出了一种名为 Sensor2Sensor 的生成式框架，将无约束的单目行车记录仪视频（“野生”视频）转换并重渲染为高保真、多模态的自动驾驶传感器日志（包含 360 度多视角相机图像和 LiDAR 点云），解决了自动驾驶系统长尾场景验证中数据匮乏与格式不兼容的问题。

### 3. 使用指南
*   **输入**：单目行车记录仪视频帧或序列（支持任意来源的网络视频、手机录像或行车记录仪）。
*   **输出**：与目标自动驾驶系统（ADS）载体对齐的、时空同步的 8 视角环视图像和 LiDAR 点云数据。
*   **依赖与硬件**：
    *   模型参数量约为 250M，基于潜在扩散模型（Latent Diffusion）。
    *   训练阶段计算成本高昂，论文中使用了 128 块 TPU 进行多阶段训练；若要适配新的自动驾驶平台，前期需要基于目标传感器配置使用 4DGS（4D 高斯溅射）进行配对数据生成。
*   **开源情况**：论文中未明确提供开源链接，且其训练重度依赖大规模专有（Proprietary）自动驾驶日志数据集，目前尚未开源。

### 4. 主要创新点
1. **基于 4DGS 的配对数据合成管道 (Synthetic Data-Pairing Pipeline)**：针对缺乏 (行车记录仪, 多模态 AV 日志) 成对训练数据的瓶颈，利用 4D 高斯溅射（4DGS）重建现有高保真自动驾驶日志场景，并从中渲染出带有真实相机内外参分布扰动的“合成行车记录仪视角”，从而构建了完美的像素级配对训练集。
2. **多模态/多视角条件扩散架构 (Multi-modal Conditional Diffusion)**：设计了包含独立 VAE 和 U-Net 分支的条件扩散模型。一方面通过将 LiDAR 表示为范围视图自旋图像 (range-view spin images) 统一了 2D 生成范式；另一方面在 U-Net 块中引入跨传感器注意力模块 (Cross-sensor attention)，并将单目输入作为条件掩码拼接到特征中，实现了图像与 LiDAR 的几何一致性联合生成。
3. **基于 DAgger 的时序一致性视频生成 (Autoregressive Generation via DAgger)**：为解决自回归视频生成中长序列误差累积导致的“时序漂移”问题，将数据集聚合策略（DAgger）引入扩散模型的训练。通过让模型在训练阶段“看到”并修正自身在 rollout 过程中生成的降级数据，显著增强了生成视频的长期时序连贯性。

### 5. 实验效果
在作者构建的 “Fixed-Camera-to-AV” 配对验证集上，Sensor2Sensor 全面超越了现有的先进基线模型（如 X-Drive、CAT3D、VGGT 等）：
*   **图像质量**：达到了最佳的 FID（21.54）和 PSNR（22.42）。
*   **视频时序稳定性**：FVD（Fréchet Video Distance）达到 278.12，极大幅度优于所有对比基线（如 VGGT 的 2373.15），展现了卓越的帧间连贯性。
*   **点云生成**：LiDAR 生成的 Chamfer 距离（1.06）为最优，精确对齐了视觉对象。
*   **泛化与人类评估**：在直接输入完全未见过的“野生”网络事故、夜间行车等挑战性视频时，其生成的图像和 LiDAR 数据在人类主观评测中的偏好率分别高达 84.62% 和 58.46%，感知模型（如 LiDAR 车辆检测）在生成数据上的表现与真实数据高度接近，证明了其用于自动驾驶闭环仿真的潜力。

## 📄 SceneAligner: 3D-Grounded Floorplan Localization in the Wild

- **链接**: https://huggingface.co/papers/2605.22581
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉-3D场景理解与视觉定位（跨模态平面图定位）

### 2. 一句话核心贡献
提出了一种基于3D几何重建与跨模态特征匹配的方法（SceneAligner），解决了在无约束真实场景（如大规模建筑、栅格化平面图）下仅通过照片图像集合实现高精度平面图定位的难题。

### 3. 使用指南
*   **输入**：无约束的真实场景图像集合（支持多视角，也可极端至单张图像）以及一张光栅化（2D）的参考平面图。
*   **输出**：图像在平面图中的精确定位（相机位姿），以及对齐到平面图全局坐标系下的3D场景。
*   **硬件要求**：由于管线包含3D场景重建与2D视觉大模型（DINOv3）的推理，需要具备较好算力的GPU设备（实验部分使用了 NVIDIA RTX A6000）。
*   **开源情况**：论文明确表示相关代码与数据集将公开发布。

### 4. 主要创新点
1. **基于3D几何的物理代理（3D-Grounded Proxy）**：打破传统依靠穷举搜索离散位姿空间的局限，将输入图像重建为重力对齐的3D点云，并向下正交投影为“2D密度图”。该密度图作为连接透视视角的自然照片与自顶向下视角的抽象平面图之间的中间代理。
2. **基于大模型的跨模态特征匹配**：针对含噪密度图与干净建筑平面图之间的巨大模态鸿沟，引入并适配了预训练的2D视觉基础模型（DINOv3 + LoRA微调），使模型能够在这两种截然不同的表征之间学习到可靠的共享语义特征空间。
3. **结构一致性的多任务学习目标**：为防止特征点匹配中的空间结构崩塌，不仅提出了亚Patch级别的坐标回归损失，还引入了“拓扑保持（Topology preservation）”和“几何保持（Geometry preservation）”损失函数，将相似变换的物理刚性约束作为先验知识注入到网络训练中。

### 5. 实验效果
*   **真实无约束场景（C3数据集）**：在核心图像级和像素级指标上，相比于现有最强基线（如C3Po），实现了两到三倍的性能提升。例如综合角度-位置召回率提升了 123.24%（73.58 vs 32.96），RMSE误差降低了约 56%。
*   **极端稀疏视角鲁棒性**：即使在仅有单张输入图像（Single-view）的极端条件下，依然能够构建有效的密度图并完成定位，准确率大幅超越基线方法，当视角数量达到10至30张时，其性能就可媲美150张图像的密集重建水平。
*   **标准室内基准（Structured3D数据集）**：在更通用的设定下（无需离散位姿网格和平面图预处理），其单视图定位精度超越了现有的室内定位 SOTA 方法（如 UnLoc，综合召回率提升 37.2%）。
*   **额外应用能力**：通过平面图作为共享几何锚点，成功实现了将缺乏视觉重叠的独立室内、室外3D场景重建成统一的全局坐标系对齐。
