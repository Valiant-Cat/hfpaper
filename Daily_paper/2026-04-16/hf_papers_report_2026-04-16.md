## 📄 OccuBench: Evaluating AI Agents on Real-World Professional Tasks via Language World Models

- **链接**: https://huggingface.co/papers/2604.10866
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - AI Agent（智能体）评测 / 大语言模型（LLM）

### 2. 一句话核心贡献
提出了 OccuBench 评测基准，通过**语言世界模型（LWMs）**低成本模拟专业领域的交互环境，实现了首个针对 AI 智能体在真实跨行业（10大行业、65个专业领域）复杂任务及故障环境下的系统性能力评估。

### 3. 使用指南
*   **输入**：待评测的 AI 智能体（Agent LLM），以及特定领域的环境配置（包含系统提示词、工具 API 模式、初始状态、状态语义描述）。
*   **交互过程**：智能体通过调用工具（Tool Calls）发出动作，语言世界模型（LWM，通常由另一个 LLM 充当）根据当前状态和逻辑规则生成结构化 JSON 响应（即环境观察），双方进行多轮交互。
*   **输出**：智能体执行任务的完整轨迹（Trajectory），最终由基于规则的验证器（Rubric-based verifier）输出任务完成通过率（Completion Rate）。
*   **硬件要求/部署**：无需搭建真实的数据库、企业内网或物理沙箱系统。仅需具备调用常规大语言模型 API（作为 Agent 和模拟器）的算力及网络条件即可，极大降低了环境工程开销。

### 4. 主要创新点
*   **基于语言世界模型（LWMs）的环境模拟**：利用 LLM 预训练中包含的领域操作逻辑作为状态转移函数（$E_{LWM}(s, a | c) \to (s', o)$），将繁重的环境搭建从“工程部署问题”转化为“提示词配置问题”，成功解锁了医疗急诊、核反应堆监控等无公开 API 领域的评测。
*   **多维度的真实职业场景覆盖**：跨越 10 个行业类别（医疗、金融、工业、农业等）和 65 个细分领域，包含 100 种真实的职业角色场景，填补了现有基准仅局限于网页操作、代码编写等少数场景的盲区。
*   **系统级的环境故障注入评估**：不仅测试理想状态（Happy Path），还通过受控参数随机注入**显性故障**（如超时、HTTP 500 报错）和**隐性故障**（如静默的数据截断、字段缺失），真实反映智能体在现实嘈杂系统中的鲁棒性与数据降级检测能力。

### 5. 实验效果
论文在 8 个模型家族的 15 个前沿模型上进行了评测，核心结论如下：
*   **综合表现与能力图谱**：GPT-5.2 综合得分最高（79.6%），但没有单一模型能统治所有行业。每个模型都有独特的“偏科”现象（例如：Gemini 3.1 Pro 擅长教育和科学，Claude Opus 4.6 擅长交通，Qwen 3.5 Plus 擅长医疗和商业）。
*   **抗故障鲁棒性极差**：面对故障环境，所有模型表现大幅下滑。其中**隐性故障（E2，平均通过率 53.4%）比显性故障（E1，62.6%）更致命**，因为模型普遍缺乏主动发现数据内容缺失的判断力，常常基于错误数据得出错误结论。
*   **Scaling Law 依然有效**：模型参数越大、代际越新、分配的推理算力（Reasoning effort）越多，表现越好。例如 GPT-5.2 在最高推理设置下成绩跃升了 27.5 分。
*   **“强智能体 ≠ 强模拟器”现象**：GPT-5.2 作为智能体表现最好，但作为环境模拟器表现最差（会强行虚构规则和状态）；而在使用能力合适的模拟器（如 Gemini-3-Flash 或 Qwen）时，该基准的评估排名一致性可达 85.7%，证明了评测方法的可靠性。

## 📄 From P(y|x) to P(y): Investigating Reinforcement Learning in Pre-train Space

- **链接**: https://huggingface.co/papers/2604.14142
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理（NLP） - 大语言模型微调、强化学习与复杂推理（LLM Reasoning & RL）

### 2. 一句话核心贡献
提出了一种双空间强化学习框架（DSRL），通过在预训练空间（$P(y)$）使用负样本强化学习（NSR-PreRL）剪枝错误推理路径并激发模型内在反思能力，随后切换到后训练空间（$P(y|x)$）进行细粒度优化，从而显著突破了大模型复杂推理能力的上限。

### 3. 使用指南
*   **输入**：复杂推理任务的问题（如数学题或逻辑问题）。
*   **输出**：包含自我纠错与长逻辑链（CoT）的推理过程及最终答案。
*   **训练流程**：采用两阶段训练机制。第一阶段为“热身”（Warmup），计算梯度时屏蔽输入条件 $x$，仅利用奖励小于0的负样本更新边缘分布 $P(y)$（即 NSR-PreRL）；在训练 10-25 步后进入第二阶段，将模型无缝切换至标准在线 RL（如 GRPO）以更新条件分布 $P(y|x)$。
*   **硬件与开源情况**：训练依赖标准的大语言模型强化学习硬件设施（多GPU集群）。作者基于开源 RL 框架 `verl` 实现了该方法，并在论文中明确表示代码已开源。

### 4. 主要创新点
1.  **提出预训练空间强化学习（PreRL）**：打破传统 RLVR（带可验证奖励的强化学习）仅局限于优化条件分布 $P(y|x)$ 的瓶颈，通过在梯度更新中移除输入依赖直接优化边缘分布 $P(y)$，从理论和实验层面证明了两者梯度的高度一致性，使模型在提升推理能力的同时保留了广泛的探索空间。
2.  **揭示负样本强化（NSR）的关键机制**：深入剖析发现预训练空间中的正样本强化会导致策略崩溃，而负样本强化（NSR-PreRL）则极为有效——它能迅速剪枝错误的推理子空间，同时内源性地激发模型的“反思（Reflection）”与“过渡（Transition）”思维（占比分别提升 14.89%），拓宽了概率探索空间。
3.  **设计“策略转世”的双空间框架（DSRL）**：首创结合预训练空间与后训练空间的强化学习范式，利用 NSR-PreRL 进行初始化以消除通用的错误模式并扩展推理视野，随后切换为标准 RL 进行针对性调优，完美协同了“广泛知识内化”与“定向目标对齐”。

### 5. 实验效果
*   **核心数学基准**：在 MATH500、AMC23、AIME24/25 及 OlympiadBench 等高难度基准上全面且稳定地超越强基线算法（如 GRPO）。在 Qwen3-4B 模型上，DSRL 在 AIME24 提升 4.69 分，AIME25 提升 2.50 分；在 Qwen3-8B 上取得 58.47 的平均最高分，具有极高的样本效率和更快的收敛速度。
*   **泛化能力（OOD测试）**：在未见过的非数学领域任务上展现出卓越的正向迁移能力，Qwen3-4B 在 GPQA-Diamond 提升 3.79 分，MMLU-Pro 提升 5.37 分；Qwen3-8B 在 HumanEval 代码任务上提升 2.44 分。
*   **行为演变**：显著加快了模型高级推理行为（如子目标设定、回溯、验证和枚举）的涌现速度与频率，有效降低了“全部回答错误（All-Wrong）”问题的比例。

## 📄 Mobile GUI Agents under Real-world Threats: Are We There Yet?

- **链接**: https://huggingface.co/papers/2507.04227
- **阅读来源**: HTML

### 1. 应用领域
大模型安全与评估 / 移动端GUI智能体 (LLM Agents Security / Mobile GUI Agents)

### 2. 一句话核心贡献
提出了首个针对移动GUI智能体的真实应用内容插桩框架与基准测试（AgentHazard），系统性地揭示并量化了现有大模型智能体在面对未经授权的第三方恶意诱导内容（如伪造的帖子、商品信息等）时存在的严重脆弱性。

### 3. 使用指南
*   **输入**：自然语言的任务指令（如“在笔记应用中添加新内容”）以及实时的Android界面状态（包括屏幕截图和无障碍UI树）。
*   **输出**：智能体的多步操作轨迹（点击、滑动、终止任务等）以及评估指标结果（如任务成功率下降幅度 $\Delta\text{SR}$、被误导率 $\text{MR}$）。
*   **硬件要求**：运行动态环境需要配置Android SDK和Android虚拟设备（AVD），推荐使用GPU加速模型推理（官方动态评估使用 2x24GB RTX 3090，对抗训练使用 4x80GB A100）。
*   **软件/代码**：已开源（包含预编译的劫持App、动态/静态任务数据及评测脚本），项目地址为 `https://agenthazard.github.io`。
*   **使用流程**：配置Python虚拟环境与API Keys -> 启动Android模拟器并保存快照 -> 运行提供的测试脚本（动态注入修改原生UI内容）-> 通过输出的Parquet文件分析评测指标。

### 4. 主要创新点
1.  **无Root权限的动态内容插桩框架**：设计了一种通过拦截系统无障碍事件（Accessibility events）实时劫持和修改Android原生UI元素（同时修改截图和UI树）的工具。无需Root设备或修改App源码，即可生成极其隐蔽且符合原生视觉的对抗性攻击场景。
2.  **双模式（动态+静态）的系统性评测基准**：构建了包含 122 个可复现动态交互任务的端到端执行环境，以及包含基于 12 款真实App提取的 3000+ 个“状态-规则”对的高难度静态数据集，兼顾了评测的真实多步推理与大规模快速迭代。
3.  **基于LLM的自动化对抗内容生成策略**：利用大语言模型模拟人类设计欺骗性文本的认知过程，通过“起草-增强相关性-指定误导行为-精简压缩（平均10个Token）”的自动化Prompt流水线，实现了大规模、高上下文相关性的攻击场景生成。

### 5. 实验效果
*   **整体脆弱性**：在对 6 个主流开源/商业移动智能体和 5 种底层大模型（如GPT-4o, Claude-3.5, DeepSeek等）的测试中，智能体极易被平均长度仅10个Token的诱导内容欺骗。动态环境下的平均被误导率（MR）高达 **42.0%**，静态环境下为 **36.1%**。
*   **多模态的“反直觉”弱点**：引入视觉模态（Vision）虽然提升了智能体在正常环境下的成功率，但在面临攻击时，多模态智能体比纯文本智能体表现出更强的脆弱性（成功率下降更严重，误导率更高）。
*   **模型表现对比**：Claude-4-sonnet 展现出相对最强的鲁棒性（最高抗攻击成功率和最低误导率），而 GPT-4o 系列误导率超过 50%（GPT-5 有显著改善）。
*   **防御效果的局限**：尽管对抗性微调（Adversarial Fine-tuning）能降低部分误导率，但防御后模型的被误导率依然超过 30%，证明仅靠简单的对抗训练无法从根本上解决该安全漏洞，需要对智能体架构进行底层改进。

## 📄 MERRIN: A Benchmark for Multimodal Evidence Retrieval and Reasoning in Noisy Web Environments

- **链接**: https://huggingface.co/papers/2604.13418
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - AI智能体评测 (Agent Evaluation) / 多模态检索与多跳推理 (Multimodal Retrieval and Reasoning)

### 2. 一句话核心贡献
提出了 **MERRIN**，这是一个全新的人工标注基准测试集，专门用于评估搜索增强型 AI 智能体在充满噪音和冲突的真实网络环境中，自主识别模态、检索多模态证据（含视频/音频）并进行多跳推理的能力。

### 3. 使用指南
*   **输入**：无显式模态提示的自然语言问题（例如：“根据最新数据，哪一年 Safari 的市场份额超过了 IE？”，而不是“请看这张图回答...”）。
*   **输出**：简短、唯一且无歧义的最终答案。
*   **评测设置**：支持三种评估模式：无搜索 (No Search)、原生搜索 (Native Search，调用模型自带搜索工具) 和 智能体搜索 (Agentic Search，论文中基于 `smolagents` 框架构建，外接了 Google Search、网页解析和 YouTube 视频处理工具)。
*   **资源需求**：评测闭源模型（如 Gemini 3.1 Pro, GPT-5.4）需相应的 API 访问权限和搜索 API 额度；评测开源模型（如 Qwen3-235B）则需要大规模的 GPU 集群支持。
*   **评估方式**：采用 LLM-as-Judge（以 BrowseComp 提示词为基准）进行自动化准确率评估。

### 4. 主要创新点
1.  **消除显式模态提示 (No Explicit Modality Cues)**：查询完全采用自然语言构建，不包含诸如“在这个视频中”或“查看图表”的提示，迫使智能体必须自主推断需要检索何种模态的信息。
2.  **引入未被充分探索的模态 (Underexplored Modalities)**：打破了以往基准局限于文本和图像的现状，深度整合了视频和音频等多模态网络资源，要求模型具备跨更广泛模态的处理能力。
3.  **高度还原真实的嘈杂网络环境 (Noisy & Conflicting Web Environments)**：问题设计天然会触发搜索引擎返回不完整、相互冲突或具有误导性的干扰项。智能体不仅需要检索，还必须在充满噪音的真实网页结果中进行多源冲突解决和多跳推理。

### 5. 实验效果
*   **整体表现极具挑战性**：在对 10 种前沿大语言模型（闭源与开源）的全面评测中，所有智能体的**平均准确率仅为 22.3%**。
*   **最佳模型**：表现最好的配置是搭载 Agentic 搜索框架的 **Gemini-3.1-Pro**，但其准确率也**仅达到 40.1%**。
*   **人类与 AI 差距显著**：在子集测试中，人类在时间更短、搜索次数和网页访问量大幅减少（资源消耗少约3倍）的情况下，达到了 **71.4%** 的准确率，且人类的信源选择精确度远高于模型（38.1% vs 1.8%）。
*   **核心瓶颈发现**：即使直接为模型提供无噪音的“黄金证据”（消除搜索障碍），准确率提升也仅有 7.6%。这表明，虽然搜索策略仍需优化，但**多模态推理能力不足**才是当前智能体面临的最紧迫瓶颈；此外，能力较强的模型（如 Gemini Deep Research）在嘈杂环境中容易陷入“过度探索（Over-exploration）”的陷阱，导致浪费算力且无法收敛到正确答案。

## 📄 Free Geometry: Refining 3D Reconstruction from Longer Versions of Itself

- **链接**: https://huggingface.co/papers/2604.14048
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 3D重建（多视图3D重建、测试时自适应/TTA）

### 2. 一句话核心贡献
提出了一种名为“Free Geometry”的测试时自适应框架，利用模型“输入视图越多重建效果越好”的内在特性，通过完整视图的特征去监督掩码视图的特征，使前馈3D重建模型能在无3D真值标签的新测试场景中实现自我进化与几何校准。

### 3. 使用指南
*   **输入**：目标测试场景的多视图图像序列（无须3D标注）。
*   **输出**：优化后的高精度3D几何结构（包括深度图、相机位姿和3D点云）。
*   **硬件要求**：极低。单张普通GPU（如 RTX Pro 6000）即可运行，每个数据集的测试时训练耗时不到 2 分钟。
*   **使用方式**：即插即用型框架。冻结原有的3D大模型（如 Depth Anything 3）骨干网络，仅在其多视图Transformer中插入并微调轻量级的 LoRA 模块和相机 Token。
*   **开源状态**：代码已开源（地址：https://github.com/hiteacherIamhumble/Free-Geometry ）。

### 4. 主要创新点
1.  **基于“视图越多越好”的无标签自监督机制**：打破了传统测试时自适应依赖不确定或噪声信号的局限，巧妙利用全视图（如8帧）提取的特征作为高质量的“教师”，指导掩码后的部分视图（如4帧）“学生”提取特征，构建了具有架构保证质量的自监督蒸馏信号。
2.  **特征级别的非对称蒸馏与轻量化更新**：发现测试时重建错误的根源在于编码器特征的一致性不足。因此放弃了传统的解码器输出级对齐，直接在 Transformer 编码器层进行特征级监督，并结合 LoRA 技术（参数更新量极小），大幅提升了训练稳定性和运行效率。
3.  **双层特征一致性目标函数设计**：设计了协同作用的损失函数。包含**帧内一致性损失**（Intra-frame consistency，精准对齐匹配特征的方向与幅值）与**跨帧关系损失**（Cross-frame relational loss，通过在特征空间构建锚点三元组，保留因掩码操作而隐藏的全局拓扑与几何关系），从局部到整体纠正模型的几何表征。

### 5. 实验效果
在 ETH3D、ScanNet++、7Scenes 和 HiRoom 共 4 个具有挑战性的主流3D数据集上进行了广泛评估，显著提升了当前 SOTA 基础模型（Depth Anything 3 和 VGGT）的零样本表现：
*   **精度稳定提升**：相比于未调整的冻结基线模型，相机位姿准确率（AUC指标）平均提升 **3.73%**，点云地图预测质量（F1-score）平均提升 **2.88%**。
*   **极强的视图数量泛化性**：尽管测试时自适应是在固定的视图数量设定下完成的（如8帧指导4帧），但优化后的模型权重可以无缝泛化应用到不同长度的推理视图（4、8、16、32帧）中，均表现出稳定的指标增长。
*   **难点区域显著改善**：可视化结果表明，在遇到遮挡、薄结构、反光表面等前馈模型极易出错的区域，Free Geometry 生成的深度图和3D表面具有更少的离群点和更高的局部连贯性。

## 📄 TREX: Automating LLM Fine-tuning via Agent-Driven Tree-based Exploration

- **链接**: https://huggingface.co/papers/2604.14116
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理-大语言模型微调（NLP-LLM Fine-tuning）、自动化机器学习（AutoML）、AI智能体（AI Agents）。

### 2. 一句话核心贡献
提出了首个基于多智能体协同与蒙特卡洛树搜索（MCTS）的自动化系统 TREX，实现了大语言模型微调全生命周期（方案设计、数据准备、训练执行与评估反馈）的完全自动化，并构建了专属评测基准 FT-Bench。

### 3. 使用指南
* **输入**：开放式的任务目标定义（包含任务描述、评估指标/协议、初始原始数据或数据获取要求、实验环境配置）。
* **输出**：经过多次迭代优化后的最佳模型微调方案（包含具体的数据集处理逻辑、训练超参数配方）以及最终微调完毕的高性能模型。
* **环境与硬件需求**：需要 GPU 集群计算资源。系统内的执行者智能体（Executor）封装了集群任务调度 API，需在具备沙盒环境的集群中运行以保障数据安全和训练执行。
* **代码与实现基础**：底层执行模块基于开源框架 OpenHands 开发，数据处理依赖 HuggingFace 生态，需配置前沿的高级 LLM（如 Gemini 3 Pro 或 Claude 4.5 Sonnet）作为智能体的推理引擎以获得最佳效果。

### 4. 主要创新点
1. **双智能体协同闭环（Dual-Agent Workflow）**：设计了“研究员（Researcher）”和“执行者（Executor）”双层架构。研究员负责文献检索、高层策略制定与细粒度结果诊断；执行者部署在 GPU 集群上，负责将实验计划转化为可执行代码并完成数据处理与模型训练。
2. **基于 MCTS 的树状实验空间探索（Tree-based Exploration）**：将昂贵且复杂的多轮实验迭代建模为蒙特卡洛树搜索（MCTS）。通过 UCT 算法和上下文记忆压缩机制，在有限的算力和时间预算下，高效平衡“利用历史高分方案”与“探索新方案”的收益。
3. **引入 AIDP 工具库与细粒度实验诊断机制**：开发了专为 LLM 训练定制的高性能 AI 数据处理工具库（AIDP），支持智能体自主编排复杂的数据清洗与合成流水线；同时在评估阶段引入 Bad-case（错误样本）对比归因分析，让系统能从失败实验中提取高价值的改进指导信息。

### 5. 实验效果
在涵盖通用能力到垂直领域（如医学、化学、金融）的 10 个真实场景基准测试 **FT-Bench** 上，TREX 使用 Qwen3-1.7B 作为基座模型：
* **稳定提升**：在资源受限的情况下（单次实验不超过5万条数据，最大迭代20轮），TREX 在所有 10 个任务上均成功实现了基座模型性能的显著优化。
* **超越人类专家**：在部分复杂任务上（如 TOMG-Bench 分子生成任务和 OpenFinData 金融任务），TREX 自动生成的微调配方所带来的性能增益，超越了人类专家针对特定任务手工调整的 Llama-3.1/3.2 或 Qwen2.5 微调模型。

## 📄 Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents

- **链接**: https://huggingface.co/papers/2604.14004
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大语言模型智能体 (LLM Agents) / 代码生成与自动化软件工程 (Code Generation & Software Engineering)

### 2. 一句话核心贡献
首次系统性提出了代码智能体领域的“记忆迁移学习”（Memory Transfer Learning），通过构建跨异构任务的统一记忆池，证明了高层抽象的元知识（Meta-knowledge，如验证流程、安全编辑规范）能够突破领域壁垒，显著提升智能体在全新代码任务中的泛化与推理能力。

### 3. 使用指南
*   **输入**：自然语言描述的新编程任务或代码缺陷修复指令（Query）。
*   **处理流程**：
    1.  **离线记忆生成**：预先在多个异构代码任务（源领域）中运行智能体，收集成功与失败的经验，并利用 LLM 将其提炼为高层抽象格式（特别是“Insight/洞察”格式，去除了具体的代码实现细节）。
    2.  **在线检索**：推理时，使用文本嵌入模型（如 text-embedding-3-small）提取当前任务的特征，在跨领域记忆池中通过余弦相似度检索出 Top-K（论文中为3条）最相关的记忆。
    3.  **注入推理**：将检索到的记忆作为元知识（Meta-knowledge）附加到代码智能体的系统提示词（System Prompt）中，指导其规划和执行。
*   **输出**：结合了历史跨领域成功经验和避错指南的代码解决方案或环境操作动作。
*   **硬件与部署**：不需要特殊的底层硬件，但依赖于大模型推理（如 GPT-4o-mini 或 DeepSeek 等）以及代码执行环境（如 Docker 容器中的 Linux bash、Python 环境）。基于论文描述，该方法架构简单，无需对模型进行参数微调（非参数化知识迁移）。

### 4. 主要创新点
1.  **突破单领域局限的统一跨领域记忆池**：打破了现有自进化智能体仅在单一（同构）基准测试中生成和利用记忆的惯例，证明了跨异构领域（如从机器学习代码迁移到仓库级软件工程）的记忆池具有极高的利用价值。
2.  **揭示了“抽象程度决定可迁移性”的内在机制**：设计并对比了四种不同抽象层级的记忆表示（原始轨迹 Trajectory、工作流 Workflow、摘要 Summary、洞察 Insight）。实证发现低层级的“轨迹”容易引发“错误锚定”导致负迁移，而高层级的“洞察”（Insight）因去除了任务特异性噪音，能提供最通用的策略指导（如结构检查、严格测试流程）。
3.  **验证了记忆的规模扩展性与跨模型迁移能力**：证明了记忆迁移学习的收益随记忆池规模和数据源领域数量的增加而稳步上升（Scaling 效应）；同时发现提取的元知识具有高度的模型无关性，能在不同大语言模型（如从闭源 GPT 模型转移到开源的 DeepSeek、Qwen 模型）之间成功迁移并产生增益。

### 5. 实验效果
在 6 个极具挑战性的异构代码基准测试（包含基础代码生成 LiveCodeBench、仓库级开发 SWE-Bench Verified、特定领域生成 ReplicationBench 等）上进行了全面评估：
*   **性能提升**：与 Zero-shot 基线相比，跨领域记忆迁移学习使 6 个测试集的平均性能（Pass@3）提升了 3.7%，其中最高在单一测试集上提升达 8.3%。
*   **对比强基线**：在与现有的强自进化框架（AgentKB 和 ReasoningBank）对比中，本方法仅使用极少量的跨领域记忆（431条），平均性能却分别反超了 1.7% 和 2.9%，展现了极高的效率与有效性。
*   **开源模型泛化**：在开源模型 DeepSeek V3.2 和 Qwen3-Coder 上，该方法同样分别取得了 1.8% 和 2.6% 的平均性能提升。

## 📄 RationalRewards: Reasoning Rewards Scale Visual Generation Both Training and Test Time

- **链接**: https://huggingface.co/papers/2604.11626
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉与多模态大模型：视觉生成（文生图 Text-to-Image、图像编辑 Image Editing）、多模态奖励模型（Reward Modeling）、基于人类反馈的强化学习（RLHF）以及测试时计算缩放（Test-Time Compute Scaling）。

### 2. 一句话核心贡献
提出了一种基于推理的视觉奖励模型 **RationalRewards** 及其数据高效的训练框架 **PARROT**，将传统的黑盒标量奖励转化为多维度结构化评估，从而在训练阶段（通过抗作弊的RL奖励）和测试阶段（通过提示词修正）双重提升视觉生成质量。

### 3. 使用指南
*   **输入**：用户的文本指令、生成的候选图像（对于图像编辑任务还包括源图像）。
*   **输出**：包含四个维度的结构化自然语言评估（文本忠实度、图像忠实度、物理质量、文本渲染）、1-4分的浮点绝对评分；在测试时还可直接输出“针对性修正后的提示词”。
*   **硬件要求**：模型参数量为 8B（基于 Qwen3-VL-Instruct-8B），对硬件要求较低。结合 vLLM（支持前缀缓存和 Paged Attention）部署时，单张图像的评估和提示词修正推理开销仅约 0.4 秒。
*   **代码及开源**：论文指出该模型为开源奖励模型（open-source reward models），并附带了详细的训练超参数和 RL 算法伪代码以支持复现。

### 4. 主要创新点
1.  **数据高效的 PARROT 变分训练框架**：无需昂贵的人工“评估理由”标注。通过变分证据下界（ELBO）推导，设计了“锚定生成 -> 预测一致性过滤 -> 学生模型蒸馏”的三阶段流水线，直接从开源的低成本成对偏好数据中提取高质量的推理数据，数据消耗量比基准模型少 10-20 倍。
2.  **抗作弊的结构化多维奖励（Mitigating Reward Hacking）**：改变了传统奖励模型直接输出单一分数的黑盒模式。模型必须先输出多维度的自然语言评价，再给出分数。这种“先推理后打分”的机制形成了隐式正则化，极大缓解了强化学习优化过程中常见的“奖励欺骗（Reward Hacking）”导致生成质量下降的问题。
3.  **从评估器到优化器的双空间能力（Dual-Space Optimization）**：既能在参数空间（训练时）为强化学习提供细粒度、多维度的可解释奖励梯度；又能在提示词空间（测试时）无需任何参数更新，通过“生成-批评-修正（Generate-Critique-Refine）”循环，将多维度的批评转化为精准的提示词修改，解锁生成器的潜在能力。

### 5. 实验效果
*   **偏好预测（作为评估器）**：在 MMRB2、EditReward-Bench 和 GenAI-Bench 等基准测试中，8B 版本的 RationalRewards 击败了所有开源的标量奖励模型，性能比肩甚至逼近 Gemini-2.5-Pro 和 GPT-5 级别的商业模型。
*   **强化学习（参数空间优化）**：在文生图基准 UniGenBench++ 上，作为 RL 奖励引导 FLUX.1-dev 实现了 9.37 分的大幅提升（从 60.97 提升至 70.34）；在 ImgEdit-Bench 图像编辑上也一致超越了基准标量奖励模型。
*   **测试时提示词优化（无需参数更新）**：仅通过单轮的推理期“批评与修正”循环，其带来的质量提升在多个基准上就持平甚至超越了计算昂贵的 RL 微调效果，并且与 RL 微调具有正向叠加效应（例如将 RL 优化后的 Flux.1 评分从 3.84 进一步提升至 4.01）。

## 📄 Target Policy Optimization

- **链接**: https://huggingface.co/papers/2604.06159
- **阅读来源**: HTML

### 1. 应用领域
强化学习 - 大语言模型对齐与微调（特别适用于存在稀疏奖励或可验证奖励的 RLVR（Reinforcement Learning from Verifiable Reward）推理场景）。

### 2. 一句话核心贡献
提出了一种目标策略优化方法（TPO），通过将“期望的概率重分配”与“优化器更新机制”解耦，在闭式下构建目标分布并使用交叉熵拟合，解决了传统策略梯度方法在稀疏奖励下容易失稳和过度更新的问题。

### 3. 使用指南
*   **输入**：Context（如 Prompt 的前缀 state），当前策略模型采样生成的一组候选结果（Completions），以及这组结果对应的奖励评分。
*   **输出**：优化后的大模型策略参数。
*   **使用流程**：
    1. 对同一 Prompt 采样一组回答；
    2. 对回答的奖励分数进行组内标准化（Z-score）；
    3. 利用旧策略（Rollout Policy）和标准化分数，直接计算出带有指数倾斜（Exponential-tilting）的闭式目标概率分布 $q$；
    4. 对当前策略模型输出使用简单的交叉熵损失（Cross-Entropy）去拟合目标分布 $q$；
    5. 反向传播更新模型。
*   **资源需求**：与 GRPO 相同，属于基于组的强化学习（Group-based RL），无需训练和加载额外的 Critic（价值网络）模型。实验中 1.5B 级别大语言模型训练使用了 A100-80GB GPU。
*   **开源情况**：论文中已明确表明代码开源。

### 4. 主要创新点
1.  **解耦“目标构建”与“策略拟合”**：摒弃了传统策略梯度（如 PG, PPO, GRPO）中的标量加权更新机制。TPO 提取采样的候选集作为闭式目标分布（Target Distribution），将策略更新转化为单纯的分布匹配（交叉熵）问题。
2.  **具备天然的梯度消失固定点（Fixed Point）**：由于采用目标分布拟合机制，一旦当前策略概率匹配了目标分布，TPO 的对数几率梯度（$p^\theta - q$）会精确降至零。这直接避免了 GRPO 等方法在策略收敛后仍存在持续非零梯度所导致的盲目游走与过度更新问题。
3.  **稀疏奖励下的高效信号分配与多轮复用安全性**：在极端稀疏奖励场景下，绝大多数采样批次全盘失败。TPO 能够天然地在第一轮 Epoch 将更新信号集中在包含有效排序信息的极少数组别上；同时，固定的目标分布为优化提供了一个稳定的吸引子，使得同一批次数据可以安全地进行多次梯度迭代（Multi-epoch reuse）而不会发散，且无需引入截断（Clipping）机制。

### 5. 实验效果
*   **简单/密集奖励任务**：在表格化 Bandit、MNIST 上下文 Bandit 和具有密集奖励的 Transformer 序列预测任务中，TPO 的收敛速度优于或比肩现有的主流基线算法（PG, PPO, GRPO, DG）。
*   **稀疏奖励序列任务**：在仅有序列末尾判定奖励的 Token 反转任务中，TPO 成功解决了任务，而 GRPO 和 DG 几乎完全停滞在随机猜测水平。
*   **大语言模型真实推理（LLM RLVR）**：基于 Qwen3-1.7B 和 DeepSeek-R1-Distill-Qwen-1.5B 进行了测试。
    *   在 **GSM8K** 任务上，TPO 早期学习速度更快，最终准确率与 GRPO 相当（~85%-87%）。
    *   在极度困难且稀疏的 **Reasoning Gym 任务（如 Graph Coloring, Knights & Knaves）** 上取得碾压性优势。例如在图着色（Graph Coloring）任务中，Qwen3-1.7B 使用 GRPO 训练 300 步得分几乎为 0（彻底失败），而使用 TPO 训练则成功将平均得分稳定提升至 0.96。

## 📄 Exploration and Exploitation Errors Are Measurable for Language Model Agents

- **链接**: https://huggingface.co/papers/2604.13151
- **阅读来源**: HTML

### 1. 应用领域
人工智能-大语言模型智能体 (LLM Agents) / 开放式决策制定与系统化评估

### 2. 一句话核心贡献
本文提出了一种与策略无关的评估框架和误差度量指标，通过结合部分可观测的2D网格地图和未知任务有向无环图（DAG），仅基于智能体的动作轨迹即可系统性地分离并量化大语言模型的“探索（Exploration）”与“利用（Exploitation）”误差。

### 3. 使用指南
*   **输入**：智能体在交互环境中的多步动作轨迹（Action trajectories）以及每个时间步的地图状态。
*   **输出**：量化的“探索误差（Exploration error）”和“利用误差（Exploitation error）”指标，以及任务总体成功率。
*   **硬件要求**：运行评估环境和误差计算过程对硬件要求极低（普通CPU即可），测试智能体性能需依赖调用相关大模型（如GPT-4, Claude, Gemini）的API。
*   **代码开源**：代码已开源（论文摘要中明确指明“We release our code”）。

### 4. 主要创新点
1. **策略无关（Policy-agnostic）的评估度量**：无需访问智能体内部的价值函数或假设固定的最优参考轨迹。该指标受经典图论启发，通过计算无进展轨迹段中的环路数量（cyclomatic number）和节点/边的冗余重访次数（预算限制），来识别“任何合理策略都不会产生的”结构性冗余行为。
2. **符号化DAG任务与受控环境设计**：构建了2D网格地图与有向无环图（DAG）结合的测试床。通过用随机符号替换任务节点中的语义信息，有效防止了模型利用预训练语义先验“走捷径”，从而纯粹测试其基于环境观察的推理能力。同时，环境的拓扑结构（如走廊宽度、节点密度）可编程调节，以控制所需的探索或利用难度。
3. **细粒度的误差状态归因机制**：打破了传统仅看“成功率”的黑盒评估。系统根据每个时间步的地图状态（是否存在已知且前提满足的待办任务、是否存在未观测的相邻网格），将冗余无效动作精确归类为“探索误差”、“利用误差”或两者兼有。

### 5. 实验效果
作者在多种前沿模型（GPT-4.1, Gemini 3.1 Pro/Flash Lite, Claude Opus 4.6等）上进行了评估，主要发现如下：
*   **成功率与探索误差强相关**：即使是SOTA模型在该任务中也存在挣扎。实验发现，探索误差与任务成功率呈现强烈的**负相关**，而利用误差与成功率相关性较弱（表明模型必须先有效探索环境，才有可能完成目标）。
*   **行为模式差异**：不同模型即使成功率相近（如Claude Opus 4.6 和 Gemini 3.1 Pro 均能达到100%），其底层行为模式截然不同（Claude倾向直接利用已知路径，Gemini倾向在移动中持续探索未知区域）。
*   **工程优化的有效性**：通过简单的提示词工程（Prompting，强调探索或利用）或脚手架工程（Harness Engineering，如为智能体提供明确的结构化记忆摘要），可以显著降低这两种误差并大幅提升成功率。
*   **语义信息的双刃剑效应**：重新引入真实语义（如做意大利面任务）后，发现语义信息会对模型产生不同影响——对某些模型（如GPT-4.1）能引导其更好探索，对其他模型（如Gemini Flash）则可能干扰推理，使其陷入短视的过度利用（myopic exploitation）。

## 📄 TIP: Token Importance in On-Policy Distillation

- **链接**: https://huggingface.co/papers/2604.14084
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理（NLP） - 大语言模型微调与知识蒸馏（特别是在线知识蒸馏 On-Policy Distillation）。

### 2. 一句话核心贡献
提出了一种基于“学生熵”和“师生散度”的双轴 Token 重要性评估法（TIP）及 Soft-OR 评分机制，精准识别并利用了传统方法容易遗漏的“过度自信但错误”的 Token，在大幅降低显存占用的同时提升了在线知识蒸馏的性能。

### 3. 使用指南
*   **输入**：学生模型自身的预测分布（用于计算 Student Entropy）与教师模型在该位置的预测分布（用于计算 Teacher-Student Divergence/KL散度）。
*   **输出**：经过重要性打分筛选后的 Top-K Token 集合，后续仅在这些 Token 上计算并反向传播蒸馏损失。
*   **硬件要求与开销**：降低了显存门槛。无需引入额外的神经网络计算或辅助目标，相关指标（熵和散度）在标准蒸馏计算中即可顺便获取。丢弃不重要的 Token 后，峰值显存可降低 47% - 58%，非常适合在有限 GPU 预算下对大模型进行高效蒸馏。
*   **开源情况**：代码已开源，链接为 `https://github.com/HJSang/OPSD_OnPolicyDistillation`。

### 4. 主要创新点
*   **构建双轴分类法（TIP Taxonomy）**：首次系统性地通过“学生分布熵”（代表模型置信度）和“师生KL散度”（代表师生分歧）两个维度，将 Token 划分为四个象限，揭示了学习信号在 Token 状态空间中的分布规律。
*   **打破“唯熵论”盲区（发掘 Q3 象限）**：理论证明并实验验证了单纯依赖学生“熵”进行筛选存在结构性缺陷——会直接漏掉“低熵且高散度”的 Token（即学生极度自信但预测错误的点）。研究发现这些仅占总数不到 10% 的 Token 蕴含着极高密度的纠错信号。
*   **提出无参的 Soft-OR 评分函数**：设计了一种无需调参的归一化融合分数（$s_t = \hat{h}_t + \hat{\delta}_t - \hat{h}_t \cdot \hat{\delta}_t$）。该分数自动结合了不确定性和分歧，完美覆盖所有高信息量的 Token 区域，弥补了单一指标的不足。

### 5. 实验效果
*   **数学推理任务**：在 MATH-500 和 AIME 2024/2025 数据集上（涵盖 Qwen3, Llama, Qwen2.5 等系列模型），采用 Soft-OR 分数仅保留 **50%** 的 Token 进行训练，其效果就能稳定追平甚至超越使用 100% 全量 Token 的基准表现。
*   **长视野智能体规划（Agentic Planning）**：在 DeepPlanning 基准测试中，效果更加显著。仅针对筛选出的 **20% 包含极密纠错信号的 Q3 象限 Token** 进行训练，其整体规划准确率直接超越了全量 Token 的在线蒸馏（例如 14B 教师模型下，Avg@16 准确率由 11.7% 提升至 12.6%），证明了纠正关键错误对整体任务成功率的决定性作用。

### 3. 使用指南

### 4. 主要创新点
1.  **置信度感知门控机制（Confidence-Aware Gating）**：巧妙融合了多重采样候选框的“空间一致性（Spatial Consensus，即平均 IoU）”与“Token 级生成置信度”，构建了一个可靠的门控指标。仅在模型表现出不确定时才触发放大操作，确保简单样本不丢失全局上下文。
2.  **基于方差分解的自适应裁剪（Adaptive Crop Sizing）**：运用全方差定律（Law of Total Variance），将候选预测框的不确定性分解为“样本间的位置离散度（Inter-sample spread）”和“样本内的预测尺度（Intra-sample box extent）”，从而动态生成贴合当前实例难度的裁剪半径。
3.  **保持视觉上下文的裁剪优化策略**：摒弃了传统的固定比例裁剪，引入了自适应调整策略，包括：去除异常离群点（仅保留最接近中位数的 Top 75% 候选者）、边界向内偏移（防止越界导致的有效特征裁切），以及强制正方形裁剪（防止界面 UI 元素长宽比失真）。

### 5. 实验效果
在三大主流 GUI 定位基准数据集上进行了综合评估：
*   **ScreenSpot-Pro**（高分辨率、密集专业桌面软件）：准确率获得大幅提升，最高增加 **+13.4%**。
*   **UI-Vision**（多平台基准）：准确率最高增加 **+10.3%**。
*   **ScreenSpot-v2**（标准分辨率的移动端/Web端）：准确率最高增加 **+4.2%**。
*   **对比优势**：在相同推理成本下，UI-Zoomer 显著优于朴素的多次采样投票（Naive Sampling）和以往的固定比例裁剪方法（如 RegionFocus），并且证明了对图标类小目标（Icon targets）的提升尤为明显。

*   **长周期可用性**：在数周至数月的真实个人助手部署测试中，基于本地 Markdown 的混合检索机制（Vector+FTS5 降级策略）和 Wiki 知识沉淀有效避免了传统智能体因上下文无限堆叠导致的“推理灾难性遗忘”和幻觉问题。

*   **输入**：多视角或单张RGB图像，以及自然语言形式的空间查询（注：在训练阶段，DGE环境还需要场景底层的3D点云和相机位姿数据用于计算真实值）。
*   **输出**：关于场景空间几何关系的自然语言逐步推理过程及精准答案。
*   **硬件要求**：训练阶段采用GRPO强化学习算法，显存与算力需求较高（论文实验使用了多节点 8×H800 80GB GPU）；推理阶段硬件需求取决于基座模型大小（如Qwen2.5-VL 3B或7B），常规商用或高端消费级显卡即可运行。
*   **开发生态**：模型基于开源的 Qwen2.5-VL 初始化，训练管道依赖 EasyR1（强化学习）和 MS-Swift（微调）框架，环境构建依赖 ScanNet 等开源3D室内场景数据集。

### 4. 主要创新点
1. **确定性几何环境（DGE）**：将16类空间推理任务（如物体距离、深度排序、相机相对位置等）解耦为可执行的原子几何验证规则。直接通过3D点云和相机位姿进行程序化数学计算以获取绝对正确的真实值，将3D场景转化为零噪声的在线交互式裁判，消除了传统自演进方法中伪标签带来的错误强化风险。
2. **问题与解答策略协同进化（Policy Co-Evolution）**：使用单一VLM同时扮演“提问者”和“解答者”。提问者基于全局场景观测生成符合物理逻辑的空间问题；解答者在DGE生成的绝对真实值约束下进行推理。两者的参数共享使得几何验证知识与视觉感知能力形成互相促进的闭环。
3. **内生驱动的自适应任务调度（Adaptive Scheduler）**：根据模型在各个空间任务上的历史准确率，动态调整采样权重。系统会自动将训练资源倾斜到模型当前表现最弱的任务类别上，无需人工干预即可涌现出由易到难的动态课程学习（Curriculum Learning）。

### 5. 实验效果
*   **综合表现**：在涵盖空间推理与通用视觉的 9 个基准测试中，SpatialEvo 在 3B 和 7B 两个参数规模下均取得了最高平均分（3B: 51.1分，7B: 54.7分），全面超越 SpatialLadder、SpaceR 等依赖静态数据集微调或固定奖励函数的基线方法。
*   **核心空间任务**：在强相关的多视角空间基准（VSI-Bench）和具身空间理解基准（EmbSpatial）上取得显著提升（例如 7B 模型在 VSI-Bench 上达到 46.1 分，大幅领先最佳基线的 45.4 分）。
*   **通用能力保持**：在通用视觉理解基准（如 MMStar 和 RealWorldQA）上未出现性能退化，证明该框架在高度专业化空间推理能力的同时，完美保留了模型的通用视觉认知能力。
