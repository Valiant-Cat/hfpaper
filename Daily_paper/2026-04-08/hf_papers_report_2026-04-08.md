## 📄 Paper Circle: An Open-source Multi-agent Research Discovery and Analysis Framework

- **链接**: https://huggingface.co/papers/2604.06170
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理-多智能体系统（Multi-agent Systems） / AI辅助科研（AI for Science） / 文献检索与文档理解（Document AI）。

### 2. 一句话核心贡献
提出并开源了一个基于大模型多智能体协作的学术文献发现与分析框架（Paper Circle），通过整合多源混合检索与动态知识图谱构建技术，将非结构化的论文网络转化为高溯源性的结构化知识，辅助研究人员高效完成文献搜索、评估和深度阅读。

### 3. 使用指南
*   **输入**：自然语言搜索查询（用于文献发现）或指定的论文 PDF 文件/URL（用于文献分析）。
*   **输出**：
    *   *发现模式*：多维度打分排序的论文列表，并在每一步同步生成结构化结果（JSON、CSV、BibTeX、Markdown 格式以及实时更新的 HTML 仪表板）。
    *   *分析模式*：结构化的论文知识图谱（包含概念、方法、实验、图表等节点及其关系图），带有溯源出处的问答（QA）结果，以及类似同行评审的详细分析报告。
*   **硬件与模型要求**：支持主流开源大模型，实验表明量化版模型（如 `qwen3-coder-30b-Q3KM`）即可达到极佳效果且内存占用较小。系统包含 Python/FastAPI 后端和 React 前端。
*   **开源状态**：代码与框架完全开源。

### 4. 主要创新点
*   **确定性多源检索与多维评分管线（Discovery Pipeline）**：结合了离线本地数据库与在线学术API（arXiv, Semantic Scholar, OpenAlex等）。摒弃了单纯的语义匹配，采用定制化的多维评分函数（综合 BM25、语义相似度、时效性、新颖度和引用量），并结合最大边缘相关性（MMR）重排序，确保检索结果的准确性与视角多样性。
*   **构建高溯源性的论文知识图谱（Paper Mind）**：首创将单篇 PDF 动态转化为强类型的知识图谱。通过概念、方法、实验和图表链接四个专门的智能体，提取结构化节点与逻辑边。图谱中的每一个元素均带有严格的元数据溯源（如原始文本块ID、页码、置信度），彻底解决大模型分析文献时的“幻觉”和难以溯源的问题。
*   **高透明度与结构化输出的多智能体工作流**：有别于试图完全替代人类的“黑盒”AI科学家，Paper Circle 被设计为增强人类智能的协作台。由追踪器（Tracker）协调意图解析、搜索、排序、分析等专职智能体，每一步交互都会强制更新共享状态并输出确定性的结构化构件，保证了整个科研探索过程的完全可复现性和可审计性。

### 5. 实验效果
*   **文献检索任务**：在内部构建的评测基准上，基于 `qwen3-coder-30b-Q3KM` 智能体实现了高达 80% 的命中率（Hit Rate）和极佳的排序质量（MRR = 0.627），且单次查询仅需 21-22 秒。若采用“BM25 + 交叉编码器重排”配置，MRR 最高可达 0.8692，但计算成本显著增加。
*   **用户体验**：针对 81 次真实科研检索会话的用户调研显示，系统认知负荷极低（NASA-TLX 综合得分仅 1.2/7），易用性强（学习门槛评分为 1/10）。
*   **自动审稿评估**：在对 ICLR 2024 真实评审数据的预测对比中发现，尽管大模型擅长梳理技术细节并生成连贯的分析报告，但其最终评分与人类评审的相对排序相关性较弱（皮尔逊/斯皮尔曼系数偏低），证明该系统目前更适合作为辅助分析工具，而非可靠的机器评审裁判。

## 📄 Learning to Retrieve from Agent Trajectories

- **链接**: https://huggingface.co/papers/2604.04949
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 智能体搜索（Agentic Search）、信息检索（Information Retrieval）、大模型（LLM）微调。

### 2. 一句话核心贡献
提出了一种名为 LRAT（Learning to Retrieve from Agent Trajectories）的新训练范式，直接利用智能体在多轮检索中的交互轨迹（浏览行为、未浏览拒斥和浏览后推理链）来训练和优化检索模型，解决了传统以人为中心的检索模型与智能体任务需求不匹配的问题。

### 3. 使用指南
*   **输入**：智能体执行复杂搜索任务时产生的多轮交互轨迹数据集合（包含中间查询语句、被浏览的文档片段、未被浏览的候选文档，以及浏览后的推理链文本）。
*   **输出**：一个针对智能体多步推理过程优化过的稠密检索模型（Dense Retriever），能为智能体输出更符合其推理演进的文档排序列表。
*   **硬件与资源依赖**：无需额外的人工标注成本。需要基础大语言模型用于生成轨迹和进行数据过滤验证（LLM-based verifier），并需要常规 GPU 资源以计算对比学习损失（加权 InfoNCE Loss）微调双编码器检索模型。
*   **开源与部署**：方法基于开源的 FlagEmbedding 框架实现，兼容主流开源基座检索模型（如 Qwen3-Embedding、E5-Large）和开源智能体（如 Tongyi-DeepResearch、Llama-3 等），可在现有大模型架构上即插即用，支持构建持续迭代的“数据飞轮”。

### 4. 主要创新点
1. **基于“搜索-浏览”转换的无偏负样本挖掘**：发现智能体的浏览行为是任务成功的必要条件（作为正样本），同时发现智能体浏览受位置偏差（Position Bias）影响极小，因此同批次中“未被浏览”的文档可直接作为高质量、无偏的负样本。
2. **利用推理链进行 LLM 交叉验证去噪**：智能体在浏览后的显式推理链是判断文档真实效用的强指标。LRAT 引入基于 LLM 的验证器，分析浏览后的推理文本是否真正使用了该文档信息，从而过滤掉“被浏览但无用”的假阳性噪声数据。
3. **基于推理长度的关联强度估计（加权优化）**：借鉴人类搜索的“停留时间”概念，创新性地将智能体“浏览后的推理链长度”映射为连续的关联强度权重（采用指数饱和函数），在对比学习中对引发更深层推理、对任务推进贡献更大的文档赋予更高的训练权重。

### 5. 实验效果
在分布内（InfoSeek-Eval）和分布外（BrowseComp-Plus）深度研究基准上，LRAT 均展现出卓越且一致的性能，适用于不同规模（4B 到 358B参数）和架构的智能体：
*   **检索召回率大幅跃升**：在 BrowseComp-Plus 核心数据集上，不同检索基座（Qwen3-Emb, E5-Large）的证据文档召回率实现了 7% 到 37% 以上的相对提升。
*   **端到端性能与效率双增**：显著提高了各智能体的最终任务成功率（Success Rate），同时大幅降低了完成任务所需的平均交互步数（Average Steps），证明优化的检索器让信息获取更精准直接。
*   **成功实现数据飞轮**：验证了即使在包含错误结果的不完美轨迹上训练也能带来增益；在模拟的流式交互环境中，随交互数据增加，检索质量和成功率能保持稳步、持续地提升（Self-improving Data Flywheel）。

## 📄 MegaTrain: Full Precision Training of 100B+ Parameter Large Language Models on a Single GPU

- **链接**: https://huggingface.co/papers/2604.05091
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大语言模型全精度训练与微调 (LLM Full-precision Training / Fine-tuning)

### 2. 一句话核心贡献
提出了一种以内存为中心的训练系统 MegaTrain，通过将模型参数和优化器状态驻留在 CPU 内存中并与 GPU 进行高效的流式交互，突破了显存限制，实现了在单张 GPU 上对千亿（100B+）参数量级大语言模型的高效全精度训练。

### 3. 使用指南
*   **输入**：大语言模型架构（支持分层结构的 Transformer，如 LLaMA、Qwen 等）及训练/微调数据集（如 MetaMathQA）。
*   **输出**：完成全精度训练或微调后的模型参数。
*   **硬件要求**：仅需单张 GPU（兼容高端数据中心卡如 H200、GH200、A100，以及消费级/工作站显卡如 RTX A6000、RTX 3090）。**核心要求是极大的主机内存（CPU RAM）**，例如训练 120B 模型需要约 1.5TB 主机内存。
*   **开源情况**：代码已完全开源（GitHub 地址：https://github.com/DLYuanGod/MegaTrain）。

### 4. 主要创新点
1.  **以内存为中心的训练范式 (Memory-centric Paradigm)**：彻底改变了传统以 GPU 显存为主的策略，将模型参数、梯度和优化器状态全部存储在 CPU 内存中，将 GPU 仅视为“瞬态计算引擎”。这一设计成功将模型规模的限制因素从 GPU 显存容量转移到了成本更低、容量更大的 CPU 内存上。
2.  **流水线双缓冲执行引擎 (Pipelined Double-buffered Execution Engine)**：为了克服 CPU-GPU 之间的数据传输带宽瓶颈，设计了基于多 CUDA 流的并发引擎。利用双缓冲（Ping-Pong Buffer）策略，将下一层参数的预取、当前层的计算以及上一层梯度的卸载高度重叠，完美隐藏了 I/O 延迟，使 GPU 能够保持持续的高负载计算。
3.  **无状态层模板机制 (Stateless Layer Templates)**：摒弃了 PyTorch 中会持续占用大量显存的全局 Autograd 计算图。GPU 显存中仅保留“空白”的计算模板，权重数据以流的形式进入并在计算时动态绑定，计算完毕后立即丢弃。此举彻底消除了持久化图元数据的开销，确保显存占用永远不超过单层网络的峰值需求。

### 5. 实验效果
*   **极致模型规模**：在单张 H200 GPU（配备 1.5TB 内存）上，稳定实现了高达 120B 参数大模型的全精度训练，而现有的卸载系统（如 ZeRO-3、FSDP Offload）在更小的规模下就已发生内存溢出（OOM）。
*   **吞吐量全面领先**：在训练 14B 模型时，MegaTrain 的训练吞吐量是 DeepSpeed ZeRO-3（CPU 卸载）的 **1.84倍**；在 32B 规模下依然能维持超过 250 TFLOPS 的算力，即使在 RTX 3090 (24GB) 消费级显卡上也能高效完成 14B 模型的训练（达到 30.19 TFLOPS）。
*   **超长上下文支持**：得益于逐层计算和块状重计算（Block-wise recomputation）策略对激活显存的严格控制，MegaTrain 能够在单张 GH200 GPU 上支持 7B 模型进行长达 **512K tokens** 的超长上下文训练，且吞吐量随上下文长度增加而提升。精度方面与标准全 GPU 训练完全一致。

## 📄 Action Images: End-to-End Policy Learning via Multiview Video Generation

- **链接**: https://huggingface.co/papers/2604.06168
- **阅读来源**: HTML

### 1. 应用领域
具身智能（Embodied AI） / 机器人控制策略学习（Robot Policy Learning） / 计算机视觉-多视角视频生成（Multiview Video Generation）。

### 2. 一句话核心贡献
将 7 自由度（7-DoF）的机器人动作转化为像素级接地的多视角“动作图像”（Action Images），使单一的视频生成大模型无需外加策略头，即可实现端到端的零样本策略学习与视频-动作联合生成。

### 3. 使用指南
*   **输入**：自然语言指令 + 初始的多视角传感器图像（观察视频帧）。
*   **输出**：生成的未来多视角机器人视频 + 对应的多视角“动作图像”序列（可通过内置的几何解码器转换为可执行的 7-DoF 机器人控制信号）。
*   **硬件要求**：极高。该方法基于 14B 参数的 Wan 2.2 视频生成大模型构建。论文指出训练使用了 32 张 A100 GPU；推理阶段使用了 4 卡 GPU 统一序列并行（Unified Sequence Parallelism）以及多种系统级优化（如 VAE 并行、缓存等）来保证生成速度。
*   **使用方式**：由于视觉和动作在模型中被统一为视频模态，通过改变输入阶段的掩码（Mask），同一个模型可直接用于不同任务：
    1. 输入指令+初始帧 $\rightarrow$ 联合生成未来视频与动作（作为控制策略）。
    2. 输入指令+完整动作序列+初始帧 $\rightarrow$ 生成条件化的未来视频（作为环境模拟器）。
    3. 输入完整观察视频 $\rightarrow$ 生成动作图像（作为数据标注工具）。

### 4. 主要创新点
1. **像素级动作表示（Action Images）**：抛弃了将控制信号编码为低维抽象 Token 的传统做法，将 7-DoF 动作转化为 3 个语义 3D 点（位置、法向、朝向），并使用 2D 高斯投影将其渲染为多视角 RGB 热图视频（红通道为位置，绿通道为法向，蓝通道同时编码朝向与夹爪开合状态），使得动作在多视角空间中完全可视化和空间接地（Pixel-grounded）。
2. **统一的生成式世界模型架构**：通过将视觉视频和动作视频在 3D-VAE 的潜在空间中进行时序拼接，模型将“策略学习”完全转化为“视觉预测追踪”问题。视频主干网络（Video Backbone）本身直接充当零策略生成器，彻底消除了视觉预测与动作执行之间的特征鸿沟，无需额外训练专用的动作模块（Action Module）。
3. **多视角几何解码机制（Multi-view Geometric Decoding）**：设计了一种简单有效的三维动作重构算法。以生成的动作图像为主，在主视角热图中提取 2D 锚点并进行射线投射（Ray casting），结合侧视角热图进行响应匹配以消除深度歧义，从而将生成的 2D 动作图像高精度地还原为现实中可执行的连续三维控制指令。

### 5. 实验效果
*   **零样本控制（Zero-shot Policy）**：在 RLBench 仿真环境（未见过的任务）和真实环境（xArm，未见过的物体与场景）中，取得了最强的零样本任务成功率，显著优于现有的具身控制基线（如 MV-Policy、OpenVLA）和其他世界模型（如 TesserAct、Cosmos-Policy）。
*   **视频与动作联合生成质量**：在生成未来预测视频时，其图像保真度（PSNR、SSIM）、感知真实度（FVD、LPIPS）以及动作轨迹预测误差（2D/3D Trajectory Error）均全面超越了先前的视频空间世界模型。
*   **泛化能力**：面对跨域数据（如从 BridgeV2、DROID 或野外 FR3M 数据集采样的图像），模型能够在即使缺乏准确相机外参监督的情况下，依然生成符合物理规律的动作图像并保持多视角的几何一致性。

## 📄 Demystifying When Pruning Works via Representation Hierarchies

- **链接**: https://huggingface.co/papers/2603.24652
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大语言模型压缩与剪枝（NLP - LLM Compression and Pruning）

### 2. 一句话核心贡献
通过解构大语言模型的表征层级（嵌入、逻辑值、概率空间），本文揭示了网络剪枝在非生成式任务上有效、却在生成式任务上失效的根本原因，指出 Softmax 非线性映射与自回归解码是导致剪枝误差被急剧放大的关键机制。

### 3. 使用指南
*   **输入**：预训练的大语言模型（如 Qwen-2.5-7B、Mistral-7B）、剪枝算法配置（如层丢弃或稀疏化参数）以及任务 Prompt。
*   **输出**：模型在不同空间（Embedding、Logit、Probability）的扰动偏差评估指标（如余弦相似度、KL散度），以及任务维度的性能反馈。
*   **硬件依赖**：无需特殊定制硬件，标准的单卡或多卡 GPU 即可运行评估与分析过程。
*   **应用建议**：该研究提供了剪枝应用指导。对于分类、检索等非生成式任务，可放心使用高压缩率剪枝；对于生成式任务，剪枝会导致累积性崩溃，需结合概率空间的 KL 散度偏移进行严格评估，或依赖剪枝后微调。
*   **代码开源**：已开源，GitHub 地址为 `https://github.com/CASE-Lab-UMD/Pruning-on-Representations`

### 4. 主要创新点
1.  **提出基于表征层级的扰动分析框架**：创新性地将语言模型的推理流水线解耦为三个连续的空间：嵌入空间（Embedding）、逻辑值空间（Logit）和概率空间（Probability），以此精确追踪剪枝引入的误差是如何在网络内部传递和演化的。
2.  **理论证明 Softmax 映射的误差放大效应**：通过引入二阶泰勒展开式进行局部理论分析，推导并证明了：尽管剪枝在嵌入空间和 Logit 空间引发的扰动极小（具有高鲁棒性），但 Softmax 的非线性转换会基于温度系数和 Logit 方差剧烈放大这些微小偏差，直接导致高维概率空间的输出分布产生严重偏移。
3.  **解析自回归误差传播与子空间稳定性**：揭示了生成任务崩溃的动态机制（自注意力机制将前序生成的错误 Token 作为上下文，导致误差跨时间步呈滚雪球式累积）；同时解释了多项选择等非生成式任务之所以对剪枝鲁棒，是因为它们仅依赖特定候选词构成的低维“稳定概率子空间”，而不受全局高频词概率偏移的影响。

### 5. 实验效果
研究在 Qwen-2.5-7B 和 Mistral-7B 模型上测试了块级剪枝（移除 8 层 Attention/MLP）和层内剪枝（如 Wanda, SparseGPT 50% 稀疏度）：
*   **非生成式任务**：在多项选择（如 HellaSwag、MMLU）和向量检索（如 BEIR）任务上，严重剪枝后的模型几乎保持了与原模型相当的性能（准确率无明显下降）。
*   **生成式任务**：在数学推理（GSM8K）、代码生成（HumanEval）和自由文本生成中，剪枝导致性能灾难性崩塌，输出出现严重的逻辑混乱和重复现象。

3.  **基于强化学习的自适应效用控制 (Adaptive Reasoning Control via RL)**：引入基于 GRPO（Group Relative Policy Optimization）的强化学习阶段。将推理前后的“相似度几何增益”转化为连续的效用信号作为奖励（Reward），辅以生成长度惩罚和格式奖励。最终使模型学会“看题下菜碟”：面对复杂或时序特征强的输入时主动触发推理，面对简单直白输入时直接生成向量，彻底解决“过度思考”问题。

### 5. 实验效果
*   **核心数据集表现**：在覆盖 78 个多模态任务的大型评测基准 MMEB-V2 上取得全新 SOTA。

*   **效率大幅提升**：得益于强化学习训练出的“自适应跳过推理”机制，相较于“始终强制推理”的模型（如 UME-R1），MMEmb-R1 在保持乃至提升精度的同时，将推理延迟显著降低了约 1.8 倍至 2.5 倍。

## 📄 QiMeng-PRepair: Precise Code Repair via Edit-Aware Reward Optimization

- **链接**: https://huggingface.co/papers/2604.05963
- **阅读来源**: HTML

### 1. 应用领域
NLP-大语言模型、自动程序修复（Automated Program Repair, APR）、强化学习（RLHF / GRPO）。

### 2. 一句话核心贡献
提出 PRepair 框架及 EA-GRPO 强化学习算法，通过“编辑感知”的动态奖励机制，解决了大语言模型在代码修复中过度修改正确代码（Over-editing）的问题，实现了既准确又最小化的精准代码修复。

### 3. 使用指南
*   **输入**：包含错误逻辑的源代码（Buggy Code）及程序功能描述。
*   **输出**：修复后的正确代码（最大程度复用原代码逻辑，仅对错误部分作最小化修改）。
*   **使用流程**：分为两阶段。首先通过 `Self-Breaking` 让模型对正确代码注入缺陷生成无标签训练数据；然后通过 `Self-Repairing` 阶段，使用 EA-GRPO 算法对模型进行强化学习微调。
*   **硬件要求**：强化学习训练阶段计算开销较大（实验基于 8 张 A100/L40S GPU 和 VeRL 框架）；推理阶段正常部署即可。
*   **部署优化（免额外硬件）**：由于该模型输出的修改量极小，输入代码可直接作为“推测解码（Speculative Decoding / Prompt Lookup Decoding）”的高质量草稿（Draft），在不增加额外模型开销的情况下显著加速推理。

### 4. 主要创新点
1.  **动态编辑感知策略优化 (EA-GRPO)**：在强化学习中设计了一种动态奖励机制。只有当模型生成的代码在组内（Group）达到基础正确率阈值后，才会触发基于“编辑代价（Edit Cost）”的惩罚项。这一设计有效平衡了“修复正确性”和“修改程度”，防止模型为了追求低修改量而牺牲正确率。
2.  **自动化的缺陷数据生成与过滤 (Self-Breaking & Min-Max Sampling)**：设计了无监督的数据生成框架，通过受控的 Prompt 注入逻辑错误，并采用 Min-Max 采样策略筛选出编辑距离最多样化的错误样本，有效解决了高质量、具备局部缺陷的代码修复数据极度匮乏的问题。
3.  **提出全新评估指标 $\text{Fix}@k$**：打破了传统评估仅关注功能是否跑通（如 Pass@k）的局限性，首次设计了联合考量“修复正确性”和“编辑代价（行级修改量）”的新指标 $\text{Fix}@k$，为精准代码修复任务提供了更加符合实际开发场景的度量标准。

### 5. 实验效果
*   **精准度 ($Fix@k$) 大幅提升**：在 Python (HumanEvalFix) 和 Verilog (QiMeng-CodeV) 基准测试上（基于 Qwen2.5-Coder 3B/7B 模型），对比常规 GRPO 算法，PRepair 显著缓解了过度修改现象，核心指标 $\text{Fix}@k$ 实现了最高 **31.41%** 的提升。
*   **正确率 (Pass@k) 稳定且有增益**：显式限制编辑行数并没有损害模型的修复能力，反而促使模型更好地进行错误定位。在 Verilog 上，3B 模型的 Pass@1 提升了 4.74%；同时在跨语言（跨域）评估中展现出极强的泛化稳定性。
*   **推理吞吐量显著提升**：在实际落地应用中，结合基于 Prompt 查找的推测解码技术，由于修复前后的代码差异极小，Draft Token 的接受率大幅提高，使得系统解码吞吐量（Throughput）提升了最高 **15%**。

## 📄 ACES: Who Tests the Tests? Leave-One-Out AUC Consistency for Code Generation

- **链接**: https://huggingface.co/papers/2604.03922
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大模型代码生成 (NLP - LLM Code Generation) / 测试时计算 (Test-Time Computation) 与代码候选排序。

### 2. 一句话核心贡献
提出了一种基于留一法AUC（LOO-AUC）的测试用例一致性打分方法（ACES），仅依赖代码执行的二值通过矩阵，在无需真实标签的情况下打破了“用不可靠测试评估不可靠代码”的循环依赖，实现了高质量的代码候选集排序。

### 3. 使用指南
*   **输入**：二值通过矩阵（Binary Pass Matrix），即大模型生成的 $n$ 个代码候选者在 $m$ 个生成的测试用例上的执行结果（0表示失败，1表示通过）。
*   **输出**：每个代码候选者的加权聚合得分（依此进行降序排列，取 Top-1 即为最终选择的代码）。
*   **硬件需求**：计算开销极低。由于仅涉及基础的矩阵和向量运算，无需 GPU 或引入额外的神经网络推理，普通 CPU 即可在毫秒级完成整个排序过程。
*   **代码/模型调用**：无需额外训练。方法提供两个版本：
    *   **ACES-C**：闭式解（一步计算），适用于大多数平均测试质量尚可的场景，作为即插即用的打分器。
    *   **ACES-O**：基于梯度的迭代优化，适用于测试用例噪声极大、难度较高的场景（需使用基础的自动微分框架）。

### 4. 主要创新点
1.  **破局视角的转变与留一法评估（LOO-AUC）**：指出评估测试用例的价值不在于判定其是否“绝对正确”，而在于衡量其“区分正确与错误代码的能力”。通过引入留一法 AUC，用剩余测试用例对代码的排序来校验被留出测试用例的判别一致性，完全脱离了对真实标签（Ground Truth）的依赖。
2.  **具备理论保证的闭式加权算法（ACES-C）**：在数学上严格证明了每个测试用例的期望 LOO-AUC 与其真实的潜在区分能力成正比。基于此恒等式，推导出一种结合通过率方差校正的闭式权重计算公式，在平均测试质量大于随机的温和假设下，能够在期望上逼近理论最优权重。
3.  **自适应的可微分联合优化策略（ACES-O）**：针对闭式解假设可能失效的困难任务（即误导性测试用例极多），将离散的 LOO-AUC 转化为基于逻辑斯蒂替代函数的可微目标。通过梯度上升迭代优化测试权重，使排名质量和测试权重之间形成正反馈相互增强。

### 5. 实验效果
*   **核心数据集**：在 HumanEval、HumanEval+、MBPP 和 LeetCodeDataset 上进行了广泛测试。
*   **显著超越基线**：在所有仅依赖执行结果（无需额外静态分析或大模型充当裁判）的后处理排序方法中，ACES 实现了最先进的 Pass@1 性能（SOTA）。例如在 HumanEval 上，ACES-O 将 Pass@1 提升至 84.15%，大幅超越传统的多数投票法（Majority Voting）和 CodeT 等启发式方法。
*   **高噪声下表现优异**：在误导性测试更多的严格基准（如 HumanEval+ 和 MBPP）上，ACES-O 展现出极强的鲁棒性，精准抑制了“坏测试”对排序的干扰。

- **链接**: https://huggingface.co/papers/2604.05404
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大模型推理效率评估与智能体（NLP - LLM Inference & Tool-Integrated Reasoning / Agents）

### 2. 一句话核心贡献
提出了一种感知硬件的效率评估指标 PTE（Prefill Token Equivalents），统一了工具集成推理中计算与内存瓶颈的不对称物理成本，并借此揭示了导致大模型工具调用低效的四种具体模式。

### 3. 使用指南
*   **输入**：模型的推理轨迹（包含每轮生成的 token 数、解码前的累积上下文长度）以及目标硬件的参数特征（如计算吞吐量和内存带宽的比值，即硬件操作强度 HOI）。
*   **输出**：PTE 成本得分，即通过系数 $\gamma$ 将受内存限制的解码成本（Decode）转化为等效的受计算限制的预填充（Prefill）Token 数量。
*   **硬件要求**：计算该指标不需要特定的物理硬件，但它可以根据不同 GPU 架构（如 NVIDIA H100、A100、RTX 4090 等）的峰值性能参数进行适配，从而反映特定硬件下的效率表现。

### 4. 主要创新点
1. **提出统一的物理级指标 PTE**：打破了传统“Token 计数”或“工具调用次数”无法反映真实延迟的局限，首次将 Transformer 推理中计算密集的预填充（Prefill）阶段与受限于内存带宽的解码（Decode）阶段统一到同一量纲下进行成本核算。
2. **揭示 TIR 场景下的非对称成本陷阱**：明确指出了在真实的工具调用场景中，工具请求造成的暂停会导致 KV-Cache 驱逐（强制重新计算），且未经过滤的冗长工具响应会迅速膨胀上下文，导致后续解码步骤呈线性变慢。
3. **系统性提取四种低效模式**：通过大规模轨迹分析，提炼出大模型在工具使用中的四种低效行为：验证性工具使用（过度思考）、无效的工具混合、缺乏工具使用先验知识、以及工具格式崩溃，并证明了过高的 PTE 往往意味着较低的推理准确率。

### 5. 实验效果
*   **指标有效性验证**：在模拟工业级高并发场景下（256 并发），PTE 与实际物理耗时（Wall-clock latency）呈现极强的线性相关性（$R^2 = 0.94$），显著优于传统的输出 Token 计数（$R^2 = 0.40$）和商业 API 计费模型。
*   **硬件鲁棒性**：敏感性分析表明，即使在内存计算比差异巨大的不同硬件设备上（如 H200 vs. 消费级 4090），基于 PTE 算出的各模型效率排名仍保持高度一致（Spearman 秩相关系数 $> 0.95$）。
*   **核心基准测试洞察**：在 MATH500、AIME24/25、SimpleQA 等 5 个基准测试中对上千条推理轨迹进行量化发现：准确率相近的 SOTA 模型，其 PTE 成本可能相差几个数量级。同时证实了启用“思考模式（Thinking mode）”的模型在复杂任务中收益明显，但在简单任务中会因过度思考导致计算严重浪费且准确率下降。

## 📄 In-Place Test-Time Training

- **链接**: https://huggingface.co/papers/2604.06169
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大语言模型 (LLM) 动态自适应、长上下文建模与持续学习。

### 2. 一句话核心贡献
提出 In-Place TTT 框架，通过原位复用现有 Transformer 模型中 MLP 层的投影矩阵作为可动态更新的“快速权重”，为预训练大语言模型提供了“即插即用”的测试时训练（Test-Time Training）能力，高效解决了模型部署后无法动态适应长文本及持续输入流的痛点。

### 3. 使用指南
*   **输入**：按块（Chunk）划分的流式输入文本序列（Tokens）。
*   **输出**：经过动态自适应调整权重后，模型生成的预测结果（Next-Token）。
*   **使用方式**：作为预训练 LLM 的“即插即用”增强模块。无需修改原有模型架构或从头进行昂贵的预训练，仅需在推理阶段（或少量的持续训练阶段）对 MLP 的下投影矩阵（`W_down`）应用基于块的梯度下降更新即可。
*   **硬件要求**：无需特殊硬件。该方法天然契合现代并行加速器（如 GPU/TPU），通过块级更新机制（Chunk-wise）和上下文并行（Context Parallelism）算法，可实现极高的计算吞吐量。
*   **代码开源情况**：论文文本中未明确提及开源代码库，但明确指出其算法基于标准 Transformer 算子（如 Conv1D、并行扫描算法）实现，可通过现有的深度学习框架在常规 GPU 环境下复现。

### 4. 主要创新点
1.  **即插即用的原位架构设计 (In-Place Design)**：摒弃了以往 TTT 方法需要引入独立且难以初始化的新 RNN/特殊层的做法。将大模型中无处不在的 MLP 模块的最终投影矩阵（`W_down`）直接作为推理时动态更新的“快速权重（Fast Weights）”，不仅保护了预训练权重的完整性，还实现了与现有百亿参数规模大模型的完美兼容。
2.  **面向语言建模对齐的专属目标函数 (LM-Aligned Objective)**：针对原有 TTT 使用的“通用重构误差”不适合自回归语言模型的问题，提出了一种与“下一个 Token 预测 (NTP)”任务显式对齐的新目标函数。该目标利用 1D 卷积和可学习投影矩阵捕获未来的预测信息，在理论上被证明能显著提升正确 Token 的预测 Logit。
3.  **高效的块级更新与上下文并行机制 (Chunk-Wise & Context Parallelism)**：由于未替换原有的注意力机制，系统摆脱了必须使用小 Chunk 的限制。它采用大尺寸块级更新（Chunk-wise，如 512-1024 长度），并结合并行扫描（Parallel Scan）算法，在保持严格因果语义等价的前提下，打破了传统 TTT 逐 Token 串行更新的性能瓶颈。

### 5. 实验效果
*   **在预训练模型增强方面**：通过低成本的持续训练，In-Place TTT 为开源模型（如 Qwen3-4B/14B、LLaMA-3.1-8B）带来了显著提升。在 RULER 长文本基准测试中，模型在 64k 和 128k 上下文长度下的表现大幅提高，甚至在 256k 长度展现出极强的外推泛化能力。
*   **在从头预训练表现方面**：在 500M、1.5B 和 4B 参数规模的实验中，搭载 In-Place TTT 的模型在长上下文困惑度（Perplexity）表现上持续优于现有的高效注意力或 TTT 基线方法（如 SWA、LaCT），并在多项零样本常识推理任务（如 HellaSwag、PIQA）中实现了性能超越。同时，分析表明该方法在实际推理场景中带来的计算和显存开销极小。

## 📄 Video-MME-v2: Towards the Next Stage in Benchmarks for Comprehensive Video Understanding

- **链接**: https://huggingface.co/papers/2604.05015
- **阅读来源**: HTML

### 1. 应用领域
多模态大语言模型（MLLMs） - 视频理解与模型评估评测（Benchmark）。

### 2. 一句话核心贡献
提出了全面且高难度的视频理解评测基准 Video-MME-v2，通过渐进式的能力分级体系和创新的“基于题组的非线性评分机制”，严格评估多模态大模型在复杂动态视频场景下的感知一致性与推理连贯性。

### 3. 使用指南
*   **输入**：视频数据（可选择输入纯视频帧、带字幕/ASR转录文本、或包含原始音频）以及具有逻辑关联的题组（每题为 8 选项的单选题，包含强干扰项）。
*   **输出**：模型预测的选择题答案（A-H）。
*   **硬件要求**：不需要除推理目标模型之外的特殊硬件，但被评测模型需具备长上下文视频处理能力（支持 64 帧、512 帧或原生视频流提取）。
*   **开源与应用**：该基准提供高质量人工标注的数据集（800 个视频，3200 道 QA）。用户可将模型的预测结果通过该项目特有的“非线性评分算法（Non-Lin Score）”进行计算，以获得比传统单题准确率更真实的性能评估。

### 4. 主要创新点
1.  **渐进式的三级能力评价体系**：摒弃扁平化分类，将视频理解能力解构为三个递进层级：Level 1（多点视觉信息聚合与感知）、Level 2（时间动态与时序建模）、Level 3（复杂的物理、社交及情节多步推理），从而能精准定位模型产生级联错误的瓶颈。
2.  **基于题组的评估策略（Group-Based Evaluation）**：设计了两种题组结构。“一致性题组”通过不同角度和粒度考察特定基础能力的广度；“连贯性题组”通过模拟人类逻辑推导的连续提问（如：定位线索->验证异常->解释目的->得出结论）考察模型多步推理的深度。
3.  **引入非线性评分机制（Non-Linear Scoring）**：彻底打破传统的“单题平均准确率（Avg Acc）”算法。在一致性题组中采用二次函数惩罚孤立的正确猜测；在连贯性题组中采用“链式截断”机制（一旦某一步推理出错，后续正确答案不再计分），严格过滤依靠盲猜获得的高分，确保得分反映的是真实、忠实的推理链条。

### 5. 实验效果
*   **模型与人类存在巨大鸿沟**：在严格的非线性评分下，人类专家得分为 **90.7**，而当前最强商业模型 Gemini-3-Pro 仅为 **49.4**，开源最强模型 Qwen3.5-397B-Think 仅为 **39.1**。
*   **揭示传统指标的“虚假繁荣”**：以 Gemini-3-Pro 为例，其传统的单题准确率高达 66.1%，但在非线性组级评分下暴跌至 49.4，证明当前 SOTA 模型在处理相关的连续提问时极度缺乏一致性与稳定性。
*   **暴露模型对“语言先验”的过度依赖**：实验发现，赋予模型“思考（Thinking）”能力在有字幕时能提升表现，但在移除文本线索的纯视觉环境下，性能会出现明显衰退（如 KimiVL-16B 在 Level 3 下降 12.2 分），表明当前视频推理模型仍严重依赖语言先验，而非真正的视觉深度理解。

推出并开源了具有 4B 参数的 MedGemma 1.5 模型，在单一统一架构内首次集成了高维3D医学影像（CT/MRI）、全切片病理图像（WSI）、多时间点纵向X光分析以及复杂医疗文档（化验单/电子病历）的解析与结构化能力。

### 3. 使用指南
*   **输入**：支持纯文本，以及与单张 2D 图像（X光、皮肤图）、高维 3D 影像序列（预处理为 2D 切片序列）、全切片病理图像（切割为图块序列）或文档截图（PDF转图像）交错排列的多模态提示。
*   **输出**：文本回答（临床问答、诊断意见）、2D 边界框坐标（解剖定位）或结构化 JSON 对象（化验单实体提取）。
*   **硬件与部署**：模型为 4B 小参数规模，对显存和计算资源要求较低。高维影像输入需要遵循特定的预处理流水线（例如 CT 最多输入 85 张切片，WSI 最多 126 个图块以控制 Token 长度）。
*   **开源状态**：模型权重已完全开源（提供于 Google Health AI Developer Foundations），开发者可直接使用其进行下游特定临床任务的微调（注意：基础模型不可直接用于自动化临床决策）。

### 4. 主要创新点
1. **高维医学影像的长上下文适配机制**：通过将 3D CT/MRI 转换为轴向切片序列并限制最大数量（约 21,760 个视觉 Token），实现了模型对 3D 空间的理解；特别是对 CT 图像引入了独特的多通道窗口映射（将 Hounsfield 单位映射至 RGB 的不同通道以凸显骨骼、软组织和脑实质）。
2. **全切片病理图像（WSI）采样与序列化**：开发了定制化的多阶段组织分割算法，通过在特定放大倍率下对组织区域进行随机无放回采样（生成最多 126 个非重叠图块），并严格保留原始空间顺序，使得模型能够高效处理超高分辨率的病理图像。
3. **针对临床任务的蒸馏与强化学习（RL）配方**：在冻结视觉编码器（MedSigLIP）的基础上，不仅引入了海量高质量的医疗多模态预训练数据，还通过引入特定领域的教师模型（专门针对 CT、MRI、病理学和 EHRQA 训练的教师模型）进行知识蒸馏，并结合基于规则的强化学习显著增强了多模态对齐能力。

### 5. 实验效果
模型在多项核心医疗基准测试上实现了显著突破（对比前代 MedGemma 1 4B）：
*   **高维影像分类**：3D MRI 疾病分类准确率绝对提升 11%，3D CT 提升 3%；全切片病理（WSI）图像分类 Macro F1 暴涨 47%。
*   **空间与时序推理**：胸部 X 光解剖定位（边界框）的 IoU 提升 35%；多时间点（纵向）胸部 X 光变化分析准确率提升 4%。
*   **文本与文档理解**：临床医学问答 MedQA 准确率提升 5%，电子病历问答 EHRQA 准确率大幅提升 22%；在 4 个化验单 JSON 信息提取数据集上，Macro F1 平均提升 18%。
*   **横向对比**：在所有专业医疗视觉任务中，其表现均超越了同等参数规模的最先进多模态模型（如 Qwen3 VL 4B）。

## 📄 Experience Transfer for Multimodal LLM Agents in Minecraft Game

- **链接**: https://huggingface.co/papers/2604.05533
- **阅读来源**: HTML

### 1. 应用领域
人工智能 - 具身智能（Embodied Intelligence） / 多模态大模型智能体（Multimodal LLM Agents） / 开放世界游戏决策控制（如 Minecraft）。

### 2. 一句话核心贡献
提出了一种面向迁移的记忆框架 **Echo**，通过将过往经验显式分解为结构、属性、过程、功能和交互五个维度，并结合上下文类比学习（ICAL），解决了多模态大模型智能体在复杂环境中难以高效重用历史经验、实现跨任务泛化的问题。

### 3. 使用指南
*   **输入**：多模态环境信号（视觉图像、文本描述、交互反馈）以及当前待解决的新任务指令。
*   **输出**：解决当前任务的逻辑推理过程及可执行的动作序列（Action Sequences）。
*   **系统需求/硬件**：依赖于经过指令微调的多模态大语言模型（MLLM）作为推理核心，需要具备向量检索能力的数据库来维护长短期记忆库（Memory Bank），运行过程需要高性能 GPU 支持大模型的推理与多模态嵌入计算。
*   **开源情况**：文中未明确提及代码是否已开源，但基于论文描述，使用者需结合 Minecraft 仿真环境接口，并实现文中所述的 CSD（上下文状态描述符）生成与聚类流程方可复现。

### 4. 主要创新点
1.  **提出五维经验迁移分类学（Five-dimensional Transfer Axes）**：打破传统智能体将记忆作为“被动仓库”的局限，将环境与经验知识显式拆解为结构（空间/几何）、属性（视觉/物理特征）、过程（因果/状态转移）、功能（用途）、交互（感知-动作循环）五个可解释的维度，使得智能体能识别不同任务间的深层逻辑共性。
2.  **设计统一的上下文状态描述符（CSD, Contextual State Descriptor）**：将异构的多模态输入（视觉、文本、交互）压缩、对齐为紧凑的语义快照。CSD 不仅包含符号化的内容，还包含用于快速检索的全局向量嵌入，为跨任务比较和类比推理提供了标准化格式。
3.  **构建结构化上下文类比学习闭环（Structured ICAL）**：智能体不再是被动检索 few-shot 示例，而是主动通过多维语义相似度计算，从记忆库中提取 CSD，利用类比推理（Analogy Learning）生成新任务策略，并结合自我一致性校验（Self-verification）和环境执行反馈不断更新记忆库，显著缓解了开放环境下的模型幻觉与控制不稳定问题。

### 5. 实验效果
在 Minecraft 开放世界“从零学习”环境下的核心表现如下：
*   **学习效率与涌现现象**：在物品解锁任务中实现了 **1.3倍** 的速度提升。在积累一定基础知识后（冷启动期后），智能体在中后期展现出**“爆发式”（burst-like）链式解锁现象**，能在短时间内连续解锁多种相似物品。
*   **基线对比领先**：在少样本（Few-shot）规划设定下，仅需 1-shot 即可持平或超越强基线模型（如 JARVIS-1, MP5, MrSteve, Voyager 等）。在 4-shot 和 8-shot 设定下，于工具台制造类任务（Crafting Table）上达到了最高 **92.5%** 的任务成功率。
*   **长程学习稳定性**：在跨世界适应和多步依赖任务（如工艺链、功能等效替换）中表现出极强的可持续适应能力。相比于部分基线模型在 20 个 episode 后性能饱和，Echo 在 10-30 个 episode 阶段仍保持稳定的成功率增长。

## 📄 GBQA: A Game Benchmark for Evaluating LLMs as Quality Assurance Engineers

- **链接**: https://huggingface.co/papers/2604.02648
- **阅读来源**: HTML

### 1. 应用领域
NLP-大模型评测 / 智能体软件工程 (Agentic Software Engineering) / 自动化软件测试与质量保证 (QA)

### 2. 一句话核心贡献
提出了首个专注于评估大语言模型在交互式环境（游戏）中**自主发现软件缺陷（Bug）能力**的基准测试（GBQA），填补了智能体软件自动化开发周期中“自主质量保证（QA）”环节的空白。

### 3. 使用指南
*   **输入**：交互式游戏环境的后端API（代表游戏当前状态与可用动作）。可选输入包含游戏设计文档和源代码（用于QA测试模式）。
*   **输出**：结构化的Bug报告（包含对异常行为的描述以及复现步骤）。
*   **评估机制**：系统内置“Critic Agent”（评判智能体），自动将模型输出的Bug报告与人工验证的 Ground Truth 进行语义匹配，主要计算缺陷发现的**召回率 (Recall)**。
*   **硬件与运行环境**：无需特殊底层算力硬件，通过调用各大模型厂商的API（如Claude、GPT、Qwen等）运行代理代码即可与轻量级Web游戏后端交互。
*   **开源情况**：作为一套Benchmark提出，包含30款游戏、124个人类验证的Bug数据集、多智能体环境构建器以及基线QA智能体代码（论文明确提供了数据集规模与评测协议，通常伴随开源发布）。

### 4. 主要创新点
1.  **开创性的“自主缺陷发现”评估范式**：现有的软件工程基准（如SWE-Bench）多假设Bug已被人类发现并描述，主要测试“代码修复”能力；而GBQA要求智能体在无明确预期的动态环境中，主动进行长时间探索、状态跟踪和异常感知，直击软件开发中最困难的环节之一。
2.  **多智能体协作的可扩展基准构建系统**：为了自动化、低成本地生成测试环境，设计了一个模拟专业游戏工作室层级（制作人、设计、程序、美术代理）的多智能体系统，能够程序化地构建游戏并按难度（Easy/Medium/Hard）注入逻辑与状态Bug，并结合“人类专家在环（Human-in-the-loop）”确保标注的绝对准确性。
3.  **针对长周期QA任务的基线智能体架构**：提出了一种强化的QA代理基线，结合了“带有验证反思的ReAct探索机制”（遇到可疑现象时先进行局部复现验证以降低误报），以及“层级记忆系统”（区分单次Session内的短期状态压缩与跨Session的长期经验积累），有效缓解了LLM在长序列交互中的幻觉和遗忘问题。

### 5. 实验效果
*   **整体任务极具挑战性**：当前最前沿的大模型在自主Bug发现任务上依然捉襟见肘。即使是表现最佳的 **Claude-4.6-Opus (思考模式)** 在提供完整设计文档的QA模式、并给予500步最高交互预算的情况下，也**仅能发现 48.39% 的Bug**。
*   **推理能力优于单纯的参数扩张**：实验表明，带有强化推理模式的中等规模模型（如 Qwen3-32B-Thinking 得分 33.87%）显著优于常规的大规模模型（如 Llama-3.1-70B 得分 14.52%），甚至比肩超大参数模型。
*   **长周期状态Bug是核心瓶颈**：单步感知类的简单Bug能在前300步被有效发现，但需要长期状态追踪和逻辑推理的困难（Hard）Bug发现率极低，证明了当前大模型在系统性测试启发式策略和长线一致性追踪上存在严重不足。

## 📄 How Well Do Agentic Skills Work in the Wild: Benchmarking LLM Skill Usage in Realistic Settings

- **链接**: https://huggingface.co/papers/2604.04323
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大语言模型智能体 (LLM Agents) 与 技能增强/工具调用 (Skill Retrieval & Tool Use)

### 2. 一句话核心贡献
首次在包含3.4万个真实开源技能的开放式设定下系统评估了LLM智能体技能的实际效用，揭示了其性能随真实度提升而急剧下降的现象，并提出了“特定查询的技能优化（Query-specific Refinement）”策略以大幅恢复并提升智能体解决复杂任务的能力。

### 3. 使用指南
*   **输入**：用户的具体任务指令（Task Instruction）或查询。
*   **输出**：智能体结合检索并优化后的技能，执行并输出完成任务所需的具体代码、命令或工作流。
*   **操作流程**：
    1. **技能检索**：系统利用“智能体混合检索（Agentic Hybrid Search）”工具，在包含 3.4 万个真实技能的开源库中进行迭代搜索（综合考虑元数据与全文本的 BM25 关键词匹配及向量语义检索）。
    2. **技能优化（核心推荐）**：在执行最终任务前，使用 Query-specific Refinement 策略。智能体在沙盒中利用检索到的粗糙/通用技能进行试错探索、自我评估（Self-evaluation），随后提取有效片段，合成一份专为当前任务定制的高质量技能。
    3. **任务执行**：智能体加载合成后的专属技能，完成最终任务。
*   **硬件与部署**：依赖强大的 LLM（如 Claude 3.5, Qwen, Kimi）及其对应的智能体框架（如 Claude Code）。检索模块需配置 SQLite FTS5 与 Embedding 模型（如 Qwen3-Embedding-4B）。任务执行和技能探索需在隔离的 Docker 容器中运行以确保安全。论文使用的数据集源自公有 GitHub 仓库（MIT/Apache 2.0 协议）。

### 4. 主要创新点
1. **渐进式的真实场景评估框架**：打破了以往研究中“直接向智能体喂给完美定制技能”的理想化假设。设计了从“强制加载”、“添加干扰项”、“带有定制技能的全库检索”到“纯通用技能的全库检索”等难度递进的评估设定，真实反映了智能体在技能选择、检索和适配上面临的瓶颈。
2. **智能体混合检索机制 (Agentic Hybrid Search)**：不仅依赖传统的固定向量检索，而是赋予智能体搜索工具（涵盖关键词匹配与基于内容的语义检索）。智能体可以通过多次 formulating queries（构建查询）、阅读技能元数据和全文来动态调整检索策略，大幅提升了召回率。
3. **基于特定任务的技能探索与合成技术 (Query-specific Refinement)**：针对通用技能难以直接适配具体任务的痛点，创新性地引入了“测试时（Test-time）探索与合成”机制。智能体通过预演任务，识别出不同通用技能中有价值的部分（如从一个技能中提取张量并行知识，从另一个提取梯度模式），进而将其合成为一个高相关性、无噪音的新技能，有效弥补了通用技能与特定任务之间的鸿沟。

### 5. 实验效果
*   **真实场景下性能急剧退化**：在最理想的“提供定制技能并强制加载”设定下，Claude 任务通过率高达 55.4%；但在最真实的“全库自主检索通用技能”设定下，通过率暴跌至 38.4%（仅比不使用任何技能的基线高 3.0%）。Kimi 和 Qwen 在真实检索下的表现甚至低于“无技能基线”，表明低质量/不相关的检索技能会严重误导模型。
*   **检索策略对比**：智能体混合检索效果卓越，其 Recall@5 相比传统的单一语义检索大幅提升了约 18.7 个百分点。
*   **优化策略(Refinement)大幅挽回性能**：使用特定查询的技能优化（Query-specific refinement）后，Claude 在带有定制技能的检索设定中通过率从 40.1% 恢复至 48.2%（追回了大部分性能损失）。即使在完全没有人类定制技能的通用智能体基准测试中，该策略也使多个模型（Claude, Kimi, Qwen）的通过率稳定提升了 4.1% 到 5.6% 不等，同时极大提高了智能体主动加载并使用技能的意愿。

## 📄 Claw-Eval: Toward Trustworthy Evaluation of Autonomous Agents

- **链接**: https://huggingface.co/papers/2604.06132
- **阅读来源**: HTML

### 1. 应用领域
NLP - 大语言模型自主智能体（LLM-based Autonomous Agents）评测

### 2. 一句话核心贡献
提出了 **Claw-Eval** 端到端评估套件，通过收集全执行轨迹的三重独立证据（审计日志、环境快照、执行轨迹），并在统一架构下衡量智能体的完成度、安全性和鲁棒性，解决了现有基准“仅看最终输出”、“缺乏安全与故障测试”以及“模态覆盖狭窄”的痛点。

### 3. 使用指南
*   **输入内容**：测试任务的声明式配置（支持通用服务编排、多模态视觉文件、多轮对话背景设定）。
*   **输出内容**：针对每次执行的多维度评分（Completion 完成度、Safety 安全性、Robustness 鲁棒性），以及同一任务独立运行 3 次后得出的综合指标：Average Score（总体能力）、Pass@3（峰值能力/最高通过率）和 Pass^3（可靠性下限/全通过率）。
*   **运行环境要求**：框架分为三个时间阶段（设置、执行、评判）和三个空间层（宿主机、Docker隔离环境、模拟服务Mock Services）。评测时需要系统支持 Docker 以建立安全的沙盒执行环境，并将智能体的执行与评判系统进行严格的“时间防火墙”隔离。
*   **开源与可用性**：该基准包含 300 个经过人工验证的任务（涵盖 9 个细分类别），采用混合评分管道（规则校验+LLM裁判）。

### 4. 主要创新点
1. **全轨迹可审计的混合评分系统（Trajectory-aware Grading）**：打破了“只看最终结果”的黑盒评测，通过三个独立且对智能体隐蔽的渠道（执行轨迹、服务端 API 审计日志、执行后的环境快照）提取证据，并使用 2159 个细粒度评分项验证智能体“实际做了什么”而不是“声称做了什么”。
2. **耦合安全性与鲁棒性的多维评估（Multi-dimensional Scoring）**：将安全性测试和鲁棒性测试直接嵌入到高压的正常任务流中。安全性作为一票否决的乘法门控因子；鲁棒性则通过系统内置的受控错误注入机制（如模拟 API 超时、限流错误）来衡量智能体的策略恢复能力。
3. **统一架构下的全模态/多轮交互覆盖（Unified Task Schema）**：在单一框架下统一了三种截然不同的评估范式：1) 多服务环境的通用业务编排；2) 视频/文档/图像的多模态感知与生成；3) 与模拟用户（带有潜在意图和渐进式信息揭示策略）的专业多轮对话。

### 5. 实验效果
该研究在 14 个前沿闭源/开源大模型上进行了全面测试，核心发现如下：
*   **防欺骗能力提升**：如果只看轨迹和输出，普通的 LLM 裁判会漏掉 44% 的安全违规和 13% 的鲁棒性错误，而 Claw-Eval 的多证据混合管道能有效捕捉这些隐蔽问题。
*   **真实可靠性（Pass^3）与峰值能力（Pass@3）脱节**：引入模拟故障（错误注入）后，模型的峰值能力（偶尔能成功）基本保持稳定，但一致性/可靠性（每次都能成功）急剧下降，部分模型 Pass^3 暴跌达 24%。
*   **多模态表现存在严重领域割裂**：当前模型处理多模态任务的成功率远低于纯文本。并且在视频理解、图文文档、代码生图三个细分领域中，**没有一个模型能够全面霸榜**（例如 GPT-5.4 擅长图文但视频极差）。
*   **提问质量决定多轮交互胜负**：在多轮对话任务中，智能体问了“多少个问题”与最终任务成功率几乎无关（解释度不到 1%），而智能体的“提问精确度（问得多好）”解释了 76% 的表现差异。
