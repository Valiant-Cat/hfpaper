## 📄 ArtifactNet: Detecting AI-Generated Music via Forensic Residual Physics

- **链接**: https://huggingface.co/papers/2604.16254
- **阅读来源**: ArXiv Abs

### 1. 应用领域
多媒体取证-AI音频检测 (Audio Processing / AIGC Detection)

### 2. 一句话核心贡献
提出了一种参数极度高效的轻量级框架 ArtifactNet，通过将 AI 音乐检测重新定义为“取证物理学”问题（直接提取神经编解码器遗留的物理伪影），实现了高精度且具有极强泛化能力的 AI 生成音乐检测。

### 3. 使用指南
*   **输入**：音频信号对应的幅度频谱图 (Magnitude spectrograms)。
*   **输出**：分类结果，判定该音频是真实录制还是由 AI 生成的音乐。
*   **硬件需求**：模型总参数量极低（仅 4.0M 参数），计算负荷轻量，无需高端 GPU，非常适合在普通计算设备或边缘侧部署。
*   **其他要求**：支持多种常见音频格式，模型采用了针对 WAV/MP3/AAC/Opus 的编解码器感知训练策略以保证鲁棒性（摘要未明确声明代码开源状态，但发布了评估基准 ArtifactBench）。

### 4. 主要创新点
1.  **全新的检测范式（取证残差物理学）**：放弃传统的表示学习路线，将检测核心转移到直接提取和分析神经音频编解码器不可避免地印刻在生成音频上的“物理伪影（Artifacts）”，提升了检测的泛化性。
2.  **极简高效的网络架构**：设计了包含有界掩码的 ArtifactUNet（3.6M 参数）用于提取编解码器残差，再通过 HPSS（谐波打击乐分离）将其分解为 7 通道取证特征，最后用微型 CNN（0.4M 参数）进行分类，兼顾了高性能与极低的参数量。
3.  **编解码器感知训练（Codec-aware training）**：引入了涵盖 WAV、MP3、AAC 和 Opus 四种格式的数据增强策略，成功将跨编解码器的概率漂移降低了 83%，有效解决了 AI 音频检测中常见的“编解码器不变性”失效问题。

### 5. 实验效果
*   **评估数据集**：在作者提出的 ArtifactBench 多生成器基准数据集（含 6,183 首曲目，覆盖 22 个 AI 生成器和 6 个真实数据源）上进行了零样本 (Zero-shot) 评估。
*   **核心指标**：在未见过的测试集（n=2,263）上，ArtifactNet 的 **F1 分数达到 0.9829**，且**假阳性率 (FPR) 仅为 1.49%**。
*   **对比优势**：在相同评估条件下，显著优于现有主流方法 CLAM (F1=0.7576, FPR=69.26%) 和 SpecTTTra (F1=0.7713, FPR=19.43%)，且参数量比两者分别少了 49 倍和 4.8 倍。

## 📄 TIPSv2: Advancing Vision-Language Pretraining with Enhanced Patch-Text Alignment

- **链接**: https://huggingface.co/papers/2604.12012
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型 / 计算机视觉 - 视觉-语言预训练 (Vision-Language Pretraining) 及下游密集预测任务（如零样本语义分割、图像检索、单目深度估计等）。

### 2. 一句话核心贡献
提出了 TIPSv2 预训练框架，通过引入监督可见图像块的 iBOT++ 目标、多粒度文本采样和轻量化的头部 EMA 机制，显著解决了现有基础视觉编码器在局部图像块（Patch）与文本对齐能力不足的痛点。

### 3. 使用指南
*   **输入数据**：图像以及对应的文本（如类别标签、简短描述或详细描述）。
*   **输出结果**：图像与文本的全局特征嵌入（Global Embeddings，用于分类/检索）以及高分辨率的密集图像块特征（Patch Embeddings，用于零样本分割和深度预测等密集任务）。
*   **使用方式**：可作为开箱即用的通用视觉编码器。在下游任务中，通常将预训练的视觉编码器参数冻结（Frozen），直接提取特征（例如，将图像块特征与各类别的文本特征计算余弦相似度即可完成零样本语义分割）。
*   **硬件需求**：大规模预训练阶段算力开销较大（论文在512张 TPUv5 上训练2天），但因采用 Head-only EMA 机制，比同类方法节省了近一半的模型显存开销；下游推理阶段仅需常规 GPU 即可运行。
*   **开源情况**：代码和预训练模型（包含 ViT-B, ViT-L, ViT-g 及 SO-400m 等多种尺寸）均已开源。

### 4. 主要创新点
1.  **iBOT++ 掩码图像建模目标**：打破了传统掩码图像建模（MIM）仅计算被掩码区域损失的惯例。iBOT++ 强制要求**可见（未被掩码的）图像块**特征也必须与教师模型对应的表征对齐，这一简单的改进极大增强了局部语义特征的空间感知能力，从而实现更精准的 Patch-Text 密集对齐。
2.  **仅限头部的指数移动平均机制 (Head-only EMA)**：在结合自监督学习与图文对比学习的框架中，发现对比损失本身已能防止模型坍塌。因此，TIPSv2 放弃了对庞大的视觉主干网络进行 EMA 复制，仅对轻量级的投影头（Projection Heads）执行 EMA 更新。此举在保持训练稳定和性能的同时，大幅削减了显存占用和训练计算成本。
3.  **多粒度文本标题增强策略**：为了提高图像文本对齐的鲁棒性，TIPSv2 除了使用网络抓取的弱监督文本外，还引入了基于大模型（PaliGemma 和 Gemini）生成的不同细节层级的合成描述。通过在训练中交替采样简单和极度丰富的文本描述，避免了图文对比任务过于困难或退化，从而同时提升了模型的全局与密集表征能力。

### 5. 实验效果
在涵盖多模态和纯视觉领域的 **9 个任务、20 个数据集**上进行了全面评估：
*   **零样本语义分割（核心优势）**：在 ADE150、PASCAL Context 等基准上大幅超越此前领先模型（如 DINOv2、SigLIP 2 以及最新的 DINOv3），取得了全新的 SOTA 成绩，其生成的特征图不仅语义更精准，且在空间分布上更加平滑且边界清晰。
*   **全面且均衡的性能**：作为冻结的特征提取器，在图像-文本检索（Flickr30K）、零样本分类（ImageNet）、单目深度估计（NYUv2）、表面法线估计等任务上，均达到甚至超越了最先进纯视觉编码器或多模态大模型的同期水平，证明了其作为通用基础模型的卓越有效性。

## 📄 Web Retrieval-Aware Chunking (W-RAC) for Efficient and Cost-Effective Retrieval-Augmented Generation Systems

- **链接**: https://huggingface.co/papers/2604.04936
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 检索增强生成（NLP - RAG）、信息检索、大模型文档预处理（文本分块 Chunking）

### 2. 一句话核心贡献
提出了一种面向网页及异构文档的检索感知分块框架（W-RAC），通过将文本提取与大模型语义规划解耦，仅让大模型基于结构化ID进行分块决策，从而在消除文本幻觉、提升检索精度的同时，将LLM分块成本降低了一个数量级。

### 3. 使用指南
*   **输入**：经过确定性解析后的文档结构化元数据（包含各个语义单元如标题、段落的唯一标识符ID、层级结构、顺序和Token数量等），**不输入长篇原始文本**。
*   **处理过程**：调用大语言模型（LLM），LLM仅承担“规划器”角色，根据输入的结构关系生成包含ID序列的分块计划（例如返回 `["heading_1", "heading_2", "text_3", "text_4"]`）。
*   **输出**：在本地系统中，通过LLM返回的ID列表反向映射到原始文本，拼接形成最终用于向量化（Embedding）和索引的文本块（Chunks）。
*   **硬件与部署**：不需要特殊硬件，依赖常规的LLM API服务即可。该方法大幅降低了对LLM输出长度的要求，极大地降低了推理延迟和API调用成本。论文未明确说明代码是否完全开源，但其Prompt结构和流水线逻辑在文中已详细给出，易于工程复现。

### 4. 主要创新点
*   **无文本重新生成（No Text Regeneration）的ID寻址**：摒弃了让LLM直接阅读并输出长文本段落的传统代理分块（Agentic Chunking）方式，转而让LLM处理轻量级的ID标识符。这彻底消除了模型在分块时可能产生的文本幻觉或内容篡改风险。
*   **检索感知策略（Retrieval-Awareness）**：在分块规划中深度结合实际检索需求（如强制保留3级标题结构、禁止将步骤/流程列表拆分、合并零散片段等），使生成的文本块在语义和结构上高度匹配真实用户的查询模式。
*   **高可观测性与系统扩展性（Observability and Determinism）**：分块计划以明确的ID映射关系存在，这意味着整个过程是可解释、可调试、可缓存且易于重新计算的。无需重新处理源文本即可快速迭代检索策略，满足工业级大规模数据注入的需求。

### 5. 实验效果
在模拟真实企业知识库的多格式数据集（RAG-Multi-Corpus）上，W-RAC 展现出卓越的性能：
*   **成本与效率**：平均单文件处理时间下降 **59.10%**（从9.18秒降至3.78秒），LLM输出Token消耗锐减 **84.6%**，整体分块相关的LLM API成本降低了 **51.7%**。
*   **检索表现**：在保持极具竞争力的召回率（Recall）、MRR和NDCG的同时，**显著提升了检索准确率（Precision）**。例如在具体组织数据集上 Precision@3 相对提升了 50%~65%；在处理时间顺序（Temporal）和对比类（Comparative）复杂查询时，精确度相对提升最高可达 84%，大幅提升了RAG系统的回答可靠性。

## 📄 (1D) Ordered Tokens Enable Efficient Test-Time Search

- **链接**: https://huggingface.co/papers/2604.15453
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 图像生成与测试时搜索（Computer Vision - Image Generation and Test-Time Search）

### 2. 一句话核心贡献
揭示了在自回归图像生成中，具有“从粗到细”（Coarse-to-Fine）层级结构的 1D 有序 Token 比传统 2D 网格 Token 更适合进行测试时搜索（Test-Time Search），并证明了利用该结构甚至能通过纯搜索实现免训练的文本到图像生成。

### 3. 使用指南
*   **输入**：文本提示词（Text Prompt），也可选输入参考图像（用于零样本图像引导）。
*   **输出**：与输入条件高度对齐且质量优化的生成图像。
*   **使用流程**：无需重新训练基础模型，在推理（Inference）阶段引入验证器（Verifier，例如 CLIPScore、ImageReward、GroundedSAM 或 DreamSim），使用经典搜索算法（如 Beam Search 或 Lookahead Search）在自回归模型（如 FlexTok）的 1D 有序 Token 空间中逐步评估中间生成状态并剪枝，最终输出得分最高的图像。
*   **硬件需求**：需要现代 GPU（如 H100 / GH200）以支撑多步自回归解码及验证器的并行计算开销。算力消耗与设定的搜索步数、Beam 宽度成正比。
*   **开源状态**：作者在论文中明确表示代码、模型权重以及额外的交互式可视化工具均已开源。

### 4. 主要创新点
1.  **揭示 Token 结构对测试时算力扩展（Test-Time Scaling）的决定性作用**：首次从结构层面证明，1D 有序 Token 因其优先生成全局结构再细化局部细节的特性，使得部分生成的 Token 序列（中间状态）已经具备明确的语义信息。这种特性让验证器在生成早期就能提供可靠的引导信号，使得 Beam Search 等启发式搜索算法的效率大幅超越传统的 2D 网格 Token。
2.  **实现免训练（Training-Free）的条件生成与零样本控制**：证明了得益于 1D 有序 Token 优秀的搜索空间结构，即便在没有自回归模型先验（即在 Token 词表中进行均匀随机采样）的情况下，仅靠图像-文本相似度验证器的引导，通过纯搜索也能生成语义合理的图像；若引入图像-图像验证器，则可直接实现零样本（Zero-shot）的多模态概念保留与控制。
3.  **构建视觉大模型测试时搜索的系统性分析框架**：系统性地评估了三种经典搜索算法（Best-of-N、Beam Search、Lookahead Search）、八种验证器（覆盖图文对齐、美学、图像相似度、规则逻辑等）以及不同强度模型先验（文本条件、无条件、均匀先验）之间的交互机制，为视觉生成模型的推理期算力扩展提供了完整的理论与实践指导。

### 5. 实验效果
*   **基础生成与算力扩展（MS-COCO & GenEval）**：在同等推理计算量（NFE）下，基于 1D 有序 Token 的模型（FlexTok）配合 Beam Search 展现出陡峭的性能增长曲线，显著优于相同架构的 2D Token 基线模型以及开源强基线（Janus-1.3B）。实验表明，一个 530M 参数的小模型通过充足的测试时搜索，其表现可以超越不使用搜索的 3.4B 巨大参数模型，符合推理期算力扩展的幂律分布。
*   **零样本概念保留（DreamBench++）**：在无需微调的前提下，通过结合 DreamSim 验证器进行搜索，FlexTok 在概念保留指标上取得了巨大提升（DINO-I 提升 18.4 分），同时维持了出色的文本跟随能力，而 2D 模型即便使用计算成本更高的 Lookahead Search，提升幅度也十分有限（DINO-I 仅提升 5.9 分）。
*   **验证器表现**：单一验证器中，基于人类偏好的模型（如 ImageReward、HPSv2）展现出最强且最均衡的综合引导能力；而将多种验证器基于排序进行融合（Ensemble）则能有效避免“验证器过度拟合（Verifier Hacking）”，实现最鲁棒的整体生成质量。

## 📄 PRL-Bench: A Comprehensive Benchmark Evaluating LLMs' Capabilities in Frontier Physics Research

- **链接**: https://huggingface.co/papers/2604.15411
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理-大语言模型评估 (NLP-LLM Evaluation) / AI for Science

### 2. 一句话核心贡献
构建了 PRL-Bench，首个基于《物理评论快报》(PRL) 最新前沿论文、旨在系统评估大语言模型执行端到端、长周期、探索性真实物理研究能力的综合基准测试。

### 3. 使用指南
* **输入**：包含科学研究动机和具体目标的物理任务（不提供显式解决路径和公式提示）。
* **输出**：客观可验证的物理结果（如数值、解析公式或离散判断）及其详细推理推导过程。
* **硬件/环境要求**：评估时要求为模型提供**代码解释器（Code Interpreter）**以支持数值计算和程序化验证，同时必须**禁用网络搜索工具**以防原始文献泄露。
* **资源获取**：数据集已开源，可通过 Hugging Face 下载访问（https://huggingface.co/datasets/AdrianMiao/PRL_Bench）。

### 4. 主要创新点
* **探索导向的开放性任务设计**：打破传统基准测试中“路径明确、目标单一”的局限，任务仅保留物理动机与核心目标，要求模型自主规划求解路径并选择适用的物理框架，真实还原科研探索过程。
* **长周期与异构工作流重构**：将复杂的物理论文转化为相对独立但逻辑统一的子任务序列（涵盖理论推导、数值计算等异构形式），模拟真实的物理研究长线工作流，并通过非严格线性依赖设计减缓误差累积。
* **客观可验证的细粒度评估体系**：设计了包含最终答案与中间过程检查点（Rubrics）的双层评价标准，并采用 LLM-as-judge 范式，实现了对长序列科学探索任务的自动化、客观且细粒度的打分与错误归因分析。

### 5. 实验效果
* **整体表现欠佳**：在对 GPT-5.4、Gemini-3.1-Pro、Qwen-3.5-Plus 等 6 款前沿大模型的评测中，最强模型（Gemini-3.1-Pro）的总体得分**仍远低于 50 分（仅为 44.27）**，表明当前大模型在真实物理研究面前存在巨大能力鸿沟。
* **主要瓶颈暴露**：错误分类分析显示，**45%–55% 的失败源于物理概念与公式套用错误**，说明前沿理论物理领域知识仍极为匮乏；同时，推理链条断裂和未完成任务等现象表明，现有大模型在维持长周期逻辑一致性和研究策略规划上存在显著缺陷。

## 📄 QuantCode-Bench: A Benchmark for Evaluating the Ability of Large Language Models to Generate Executable Algorithmic Trading Strategies

- **链接**: https://huggingface.co/papers/2604.15151
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理-大模型代码生成 / 金融人工智能（量化交易与算法策略生成）

### 2. 一句话核心贡献
提出了 **QuantCode-Bench** 基准测试，旨在系统性评估大语言模型将自然语言交易策略描述转化为可执行、且具备真实交易行为及语义对齐的算法交易代码（基于 Backtrader 框架）的能力。

### 3. 使用指南
*   **输入**：一段英文的交易策略自然语言描述（包含技术指标、入场/出场条件、仓位管理等规则）。
*   **输出**：基于 Python `Backtrader` 框架的量化交易策略代码。
*   **环境/硬件需求**：标准 Python 环境，需安装 `Backtrader` 框架。需要相应的历史市场数据集以供回测运行（基准测试自身已提供包含多资产、多时间框架的数据集）。无需特殊底层硬件，能正常运行回测和调用模型即可。

### 1. 应用领域

### 3. 使用指南
* **输入与输出**：输入为待解决的复杂推理问题（Query）；输出为剔除低质量路径后，通过多数投票（Majority Voting）等共识策略聚合得出的最终答案。
* **运行机制**：
  1. 生成多条初步的短推理路径（Prefix，例如前 1024 个 Token）。
  2. 暂停生成并提取这些前缀片段的内部状态（KV Cache）。
  3. 通过预训练的轻量级模块（附加在模型上的 LoRA 适配器）结合特殊占位符 `<verify>` 进行一次前向传播，输出每条路径的质量得分。

* **硬件与资源**：标准 GPU 即可部署（实验基于单张 NVIDIA H100）。由于剪枝模块复用了前缀阶段的 KV Cache 且避免了重新编码全文，推理时的额外延迟几乎为零。
* **开源情况**：作者明确表示将开源构建的训练数据集以及训练好的 Checkpoints 模型权重，从业者可直接跳过高成本的训练数据构建阶段进行开箱即用部署。

### 4. 主要创新点
* **建立统一的路径剪枝分类学（Taxonomy）**：首次从“信号源（内部状态 vs 外部文本）”和“可学习性（可训练 vs 启发式规则）”两个维度对现有的剪枝方法进行分类，并指出“内部且可学习（Learnable Internal）”是目前未被充分探索的最优范式。
* **基于内部特征的柔性监督极简架构（STOP）**：设计了一种不破坏基座大模型生成能力的即插即用评估模块。使用参数高效微调（PEFT/LoRA），通过直接读取高维内部状态（而非低维文本输出）来评估模型的真实置信度。训练时首创使用蒙特卡洛（MC）估计来构建软标签（0到1之间的成功率概率），解决了单一硬标签（对/错）方差过高的问题。
* **提出交互式缩放定律与部署指南（Interaction Scaling Law）**：量化了计算预算、前缀长度和任务复杂度之间的非线性关系，推导出一个清晰的幂律公式。该公式可以自动为不同的实际业务场景预测出最佳的路径保留率（Retention Ratio），免去了昂贵的超参数网格搜索。

### 5. 实验效果
* **评测设置**：在参数量从 1.5B 到 20B 的系列大模型（DeepSeek-R1-Distill-Qwen）上进行了详尽测试。覆盖了极难的数学（AIME 2024/2025）、高端科学（GPQA Diamond）以及通用逻辑谜题（LogicGrid）数据集。

* **通用性与延迟**：相比于使用外部流程奖励模型（PRM）或相似度计算的基线方法，STOP 展现出了极高的吞吐量和最低的系统开销（仅需微秒级额外延迟）。同时，该模块学到的内部寻址模式（聚焦“自我纠错”和“逻辑转折”标志）展现了其在不同推理任务和外部工具调用竞赛（AIMO3）中的强泛化能力。

提出并开放了千亿参数规模的原生全模态大模型 Qwen3.5-Omni，首创了动态图文音同步的流式生成与对齐技术（ARIA），并在无损文本/视觉能力的前提下，实现了极低延迟的超长音视频实时交互以及基于音视频指令直接生成代码的“音视频氛围编程（Audio-Visual Vibe Coding）”能力。

*   **输入格式**：支持异构多模态输入，包括纯文本、图像、音频以及视音频交错数据。得益于 256k 的超大上下文窗口，模型最高支持输入长达 10 小时的音频或 400 秒的 720P 视频（1 FPS）。
*   **输出格式**：可实时生成文本及流式语音响应，支持带情感的类人多语种语音合成，并能基于用户提供的音频样本进行零样本（Zero-shot）声音克隆。
*   **资源与获取**：模型包含 Plus 和 Flash 两个变体版本，目前已通过阿里云 Model Studio 以 API 形式对外开放调用。
*   **部署要求**：由于模型达到百亿至千亿级参数规模，完整本地部署需要极其庞大的显存与算力支持，但其底层架构对高并发服务进行了优化（支持 KV-cache 优化及 Chunked-prefilling 机制）。

### 4. 主要创新点
1. **统一的混合专家双轨架构（Thinker-Talker with Hybrid MoE）**：
   采用“思考者（Thinker，负责文本与高维表示生成）”与“讲述者（Talker，负责流式语音 Token 预测）”的联合架构，且两部分均采用了 Hybrid-Attention MoE 架构。在输入端通过显式注入时间戳格式字符串，极大增强了长视频和音视频的绝对时序感知与同步能力。
2. **自适应速率交错对齐机制（ARIA，Adaptive Rate Interleaved Alignment）**：
   针对文本与语音 Token 编码效率差异导致的流式合成不稳定（吞字、发音错误等）问题，ARIA 将传统的双通道生成统一为单流约束模式，在流式解码中动态对齐文本和语音单元，以极小的延迟代价大幅提升了交互语音的稳定性与自然度。
3. **原生全模态智能体与涌现能力（Native Omni Agent & Audio-Visual Vibe Coding）**：
   超越了被动感知响应范式，具备原生全模态工具调用能力（如自主 WebSearch 与复杂 FunctionCall）。同时展现出一种全新的涌现能力：能够直接理解包含空间、时间和听觉特征的音视频指令，并据此直接编写可执行代码（Audio-Visual Vibe Coding）。

### 5. 实验效果
模型在多达 215 个音频与音视频理解、推理及交互子任务上进行了全面评估：
*   **综合表现**：Qwen3.5-Omni-Plus 取得了当前最佳（SOTA）的整体成绩，在通用音频理解、推理、识别和语音对话任务上 **超越了 Gemini-3.1 Pro**，在综合音视频理解层面与 Gemini-3.1 Pro 持平。
*   **多语种识别与翻译**：在 FLEURS 测试集上表现出压倒性优势，其多语种 ASR 和语音到文本翻译（S2TT）错误率/BLEU得分显著优于竞品，尤其在粤语、泰语、越南语等复杂声调和亚洲语言中优势极大。
*   **无损对齐**：在保持顶尖视听多模态能力的同时，其纯文本及图像视觉能力并未出现“多模态税（退化）”，性能与同尺寸的单模态 Qwen 模型完全持平。
*   **语音生成基准**：在 SEED-TTS 等语音合成和跨语种声音克隆基准上具有极高的保真度与稳定性。

## 📄 NTIRE 2026 Challenge on Video Saliency Prediction: Methods and Results

- **链接**: https://huggingface.co/papers/2604.14816
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 视频显著性预测（Video Saliency Prediction）

### 2. 一句话核心贡献
举办了 NTIRE 2026 视频显著性预测挑战赛，构建了包含 2,000 个视频和超百万帧的大规模高质量众包数据集，并总结评估了当前最先进的视频显著性预测模型与方法。

### 3. 使用指南
*   **输入**：一段视频序列（RGB 帧流，部分前沿方法还支持音频或光流特征作为辅助输入）。
*   **输出**：对应每一帧的连续显著性图（Saliency Map），用于表示视频中每个像素吸引人类视觉注意力的概率分布。
*   **硬件需求**：当前最优方法通常依赖大规模视频基础模型（如参数量达 6B 的 InternVideo2、V-JEPA2 等）作为特征提取器，推理和模型微调需要较高显存的 GPU 硬件支持。
*   **开源情况**：挑战赛使用的含有 2,000 个视频的开源协议数据集，以及最终通过审核的前排参赛队伍的复现代码已在 GitHub 免费公开（https://github.com/msu-video-group/NTIRE26_Saliency_Prediction）。

### 4. 主要创新点
1. **高扩展性的大规模众包数据采集与处理**：为了打破昂贵眼动仪带来的数据规模瓶颈，构建了一套基于鼠标轨迹收集人类视觉刺激的方法。超过 5,000 名评估者参与标注，并结合复杂的过滤与后处理技术，生成了质量逼近眼动仪的超大规模（2,000个视频）基准数据集。
2. **基于大视频模型与双专家解码器的集成架构（冠军方案）**：冠军队伍（iLearn）采用 InternVideo2-Stage2_6B 作为主干并结合 LoRA 微调，创新性地设计了两个互补的解码器：一个专注于中心偏置先验和时间调制，另一个利用多尺度特征融合与深度辅助监督，最终在对数空间对两者进行集成，大幅提升了复杂场景下的预测能力。
3. **引入多样化的前沿模型范式进行时空建模**：各顶尖参赛队伍将多种最新网络架构成功迁移至显著性预测任务中。例如，亚军（CVSP）基于预测编码理论利用自监督模型 V-JEPA2 提取特征；其他队伍则引入了状态空间模型（Mamba）结合扩散网络（Diffusion）、双向多尺度特征融合，以及运用双曲空间进行音视频多模态融合建模。

### 5. 实验效果
挑战赛使用 500 个视频的私有测试集对最终提交的模型进行客观评估，采用了 4 种公认的评价指标：皮尔逊相关系数（CC）、相似度（SIM）、AUC-Judd 曲线下面积和归一化扫描路径显著性（NSS）。
*   **排名表现**：最终 7 支队伍成功通过代码审核。冠军队伍（iLearn）与亚军队伍（CVSP）在综合指标上均取得了全场最佳的平均排名，最终冠军凭借在第一顺位指标（CC）上的微弱优势胜出。
*   **核心结论**：实验结果充分证明，**大规模预训练视频大模型（VLMs/Video Foundation Models）配合复杂的多尺度时空特征解码** 是当前解决视频显著性预测任务的最有效路径。

## 📄 Maximal Brain Damage Without Data or Optimization: Disrupting Neural Networks via Sign-Bit Flips

- **链接**: https://huggingface.co/papers/2502.07408
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉（图像分类、目标检测、实例分割）与 自然语言处理（推理大语言模型、混合专家模型MoE、文本编码器微调）。

### 2. 一句话核心贡献
提出了一种完全无需数据和迭代优化的轻量级攻击方法（DNL及1P-DNL），仅通过翻转极少量（甚至1-2个）关键参数的符号位（Sign-Bit），即可引发多领域深度神经网络性能的灾难性崩溃。

### 3. 使用指南
*   **输入**：受害模型的权重参数文件。（注：对于单次传递变体 1P-DNL，还需要输入一个随机生成的无意义数据，如高斯噪声图像或随机Token串）。
*   **输出**：经过极少量比特篡改后的模型权重（该模型在实际任务中会输出完全错误或乱码的结果）。
*   **硬件与环境要求**：无需高性能GPU进行大量梯度计算，无需访问受害者的训练或验证数据。但攻击者**必须具备对模型存储参数的“写权限”**（在实际场景中，可结合Rowhammer、直接内存访问(DMA)、硬件固件漏洞或Rootkit等系统级漏洞来实现内存比特翻转）。
*   **代码开源情况**：作者在论文中承诺“所有代码将在文章被接收后公开”。

### 4. 主要创新点
*   **零数据与零优化的启发式攻击算法**：打破了传统比特翻转攻击（BFA）依赖数据集和多次梯度优化搜索的瓶颈。其基础版（DNL）基于参数绝对值大小的启发式规则直接定位关键位；增强版（1P-DNL）创新性地融合了一阶和二阶泰勒展开近似，仅需一次随机输入的前向和反向传播即可精确定位最脆弱的参数。
*   **揭示网络早期层与局部特征的极端脆弱性（架构特定启发式）**：研究发现破坏网络早期层（如基础边缘检测滤波器）会引发严重的误差级联放大。此外，针对卷积神经网络（CNN）首次提出了“单核单翻转（One-Flip-Per-Kernel）”约束，证明了在同一个卷积核内翻转多个比特会产生误差抵消，分散在不同核进行翻转能将破坏力最大化。
*   **验证了跨模态攻击的有效性并提出极简防御策略**：证明了该漏洞无视模型规模，广泛存在于CNN、ViT以及MoE大语言模型中（在MoE中仅破坏冷门专家也会毒化全局特征）。同时，利用“定位关键位”的逆向思维提出了高效防御方案：只需对极小比例（如1%）的高危参数符号位进行重点保护（如冗余备份或ECC纠错），即可极低成本地免疫此类攻击。

### 5. 实验效果
*   **图像分类（ImageNet等）**：在ImageNet数据集上，仅需翻转 ResNet-50 中的 **2个符号位**，即可导致模型准确率下降 **99.8%**。
*   **目标检测与分割（COCO 2017）**：仅在骨干网络中翻转 **1到2个符号位**，即可让 Mask R-CNN 和 YOLOv8-seg 的检测与掩码平均精度（AP）出现断崖式崩溃（下降77%以上甚至归零）。
*   **推理大语言模型（MATH-500基准）**：对于百亿参数的MoE模型 Qwen3-30B-A3B-Thinking，只需在两个不同的专家模块下各翻转1个符号位（**共计2个比特**），即可将其数学推理准确率从 **78% 直接降至 0%**（模型输出完全退化为无意义重复的乱码）。
