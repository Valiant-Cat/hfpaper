## 📄 Timer-S1: A Billion-Scale Time Series Foundation Model with Serial Scaling

- **链接**: https://huggingface.co/papers/2603.04791
- **阅读来源**: HTML

### 1. 应用领域
**时间序列预测 (Time Series Forecasting)** - **时间序列基础模型 (Time Series Foundation Models)**，适用于工业传感、金融评估、医疗监测、气象预报等多种场景的通用预测任务。

### 2. 一句话核心贡献
提出了 Timer-S1，一个拥有 83 亿参数的稀疏混合专家（MoE）时间序列基础模型，通过创新的“串行标记预测”（Serial-Token Prediction）机制，有效解决了现有模型在大规模扩展及长期预测中存在的误差累积与计算效率瓶颈问题。

### 3. 使用指南
*   **输入数据**：单变量时间序列数据。模型会对输入进行实例重归一化（Instance Re-normalization）并切分为 Patch Token。虽然主要针对单变量模式训练，但可通过微调处理多变量依赖。
*   **模型推理**：
    *   模型基于 Decoder-only Transformer 架构，包含 TimeMoE 模块和 TimeSTP 模块。
    *   在推理时，利用 TimeSTP 模块在**单次前向传播**中自适应地生成多步预测结果，无需进行昂贵的逐步自回归（Rolling-style）推理。
    *   支持 11.5K 的长上下文窗口输入。
*   **输出结果**：未来时间步的预测值，支持分位数（Quantile）概率预测。
*   **硬件与开源**：模型总参数量为 8.3B（每个 Token 激活参数为 0.75B），需要具备相应显存的 GPU 进行推理或微调。文中明确表示模型将开源以促进研究。

### 4. 主要创新点
1.  **串行标记预测（Serial-Token Prediction, STP）架构**：设计了通用的 TimeSTP 模块，这是一种序列化的 Transformer 块。它既保留了长期预测所需的渐进式串行计算能力（避免并行预测的依赖丢失），又消除了传统自回归模型在推理时的滚动操作，从而大幅降低了误差累积和推理延迟。
2.  **十亿级稀疏混合专家（MoE）扩展**：采用了 TimeMoE 模块，构建了高达 83 亿参数的模型，但在推理时通过稀疏激活机制（每个 Token 仅激活 0.75B 参数），在保证模型能捕捉大规模异构数据模式的同时，维持了较低的推理成本。
3.  **万亿级数据构建与解耦训练流程**：构建了包含超过一万亿个时间点的高质量语料库 **TimeBench**，并设计了独特的“三阶段”训练策略：首先是密集监督预训练，其次是针对短期的加权持续预训练，最后是基于 RoPE 的长上下文扩展后训练，全面提升了模型在不同预测视界下的性能。

### 5. 实验效果
*   **核心数据集**：在大规模基准测试 **GIFT-Eval**（包含 24 个数据集，17.7 亿数据点）上进行了评估。
*   **主要表现**：
    *   Timer-S1 达到了 **SOTA (State-of-the-art)** 的预测性能，取得了最佳的 **MASE (0.693)** 和 **CRPS (0.485)** 分数。
    *   相比于前代模型 Timer-3 (Sundial)，MASE 降低了 7.6%，CRPS 降低了 13.2%。
    *   在中长期预测任务上表现尤为出色，且消融实验证明 Serial-Token Prediction 目标在相同计算预算下显著优于传统的 Next-token 和 Multi-token 预测目标。

## 📄 DreamWorld: Unified World Modeling in Video Generation

- **链接**: https://huggingface.co/papers/2603.00466
- **阅读来源**: HTML

1. **应用领域**：计算机视觉 - 视频生成 (Text-to-Video Generation)、世界模型 (World Models)。

2. **一句话核心贡献**：提出了一种名为 DreamWorld 的统一框架，通过在视频生成器中联合建模多源世界知识（物理常识、3D几何、时序动态），并结合动态约束退火策略，解决了现有模型在视觉逼真度与物理逻辑一致性之间的优化冲突问题。

3. **使用指南**：
    *   **输入**：文本提示词 (Prompt)。
    *   **输出**：符合物理规律、时空一致性强的高质量视频（分辨率480p/720p，81帧）。
    *   **核心流程**：
        *   **预处理**：利用 RAFT 提取光流特征（时序），利用 DINOv2 提取语义和空间几何特征。
        *   **训练**：基于 Wan2.1 (DiT结构) 进行微调，扩展模型的输入/输出投影层，使其能同时预测视频像素潜变量和上述世界知识特征。
    *   **推理**：使用“多源内部引导 (Multi-Source Inner-Guidance)”机制，无需外部辅助模型，直接利用模型自身预测的特征修正生成轨迹。
    *   **硬件与代码**：实验在 8x NVIDIA A100 GPU 上完成；代码将公开。

4. **主要创新点**：
    *   **联合世界知识学习 (Joint World Knowledge Learning)**：不同于以往单一的特征对齐，该方法扩展了 Diffusion Transformer 的架构，使其在一个统一的流匹配 (Flow Matching) 目标下，同时学习视频外观与多模态世界先验（时序动态、空间几何、语义一致性）。
    *   **一致性约束退火 (Consistent Constraint Annealing, CCA)**：提出了一种动态损失权重衰减机制，在训练初期强力注入物理约束，后期逐渐降低权重，有效解决了多目标优化导致的画面闪烁和视觉伪影问题，平衡了物理准确性与视觉生成质量。
    *   **多源内部引导 (Multi-Source Inner-Guidance)**：设计了一种推理时的引导策略，通过贝叶斯框架利用模型自身预测的“世界特征”作为条件（类似 Classifier-Free Guidance），在不引入额外计算开销的情况下，强制生成结果遵循真实世界的物理定律。

5. **实验效果**：
    *   **VBench**：总分达到 **80.97**，比基线模型 Wan2.1 提高了 2.26 分，在“空间关系”和“动态程度”子项上优势明显。
    *   **VBench 2.0**：以 **52.97** 的总分在所有对比模型中位居第一，优于 VideoJAM 和大规模基线模型。
    *   **VideoPhy (物理常识评测)**：实现了 **52.9%** 的语义依从性 (SA) 和 **26.2%** 的物理常识得分 (PC)，显著超越了现有最先进方法 (SOTA)。
    *   **定性表现**：在“倾倒液体”、“物体遮挡”等复杂物理交互场景中，DreamWorld 能生成符合重力和空间逻辑的视频，避免了基线模型中常见的物体穿模和时序形变问题。

## 📄 HiFi-Inpaint: Towards High-Fidelity Reference-Based Inpainting for Generating Detail-Preserving Human-Product Images

- **链接**: https://huggingface.co/papers/2603.02210
- **阅读来源**: HTML

1. **应用领域**：
计算机视觉 - 图像生成与修复（Specifically：基于参考图像的商业/电商广告图像生成）。

2. **一句话核心贡献**：
提出了一种名为 HiFi-Inpaint 的高保真图像修复框架，通过引入高频信息引导和构建大规模合成数据集，解决了现有模型在生成人-商品合成图时难以精确保留商品细粒度细节（如文字、Logo、纹理）的问题。

3. **使用指南**：
*   **输入**：
    1.  简练的文本提示词 (Text Prompt)。
    2.  经过掩码处理的人物背景图像 (Masked Human Image)。
    3.  商品参考图像 (Product Reference Image)。
*   **输出**：
    *   高分辨率、细节保留完整且与背景自然融合的人-商品合成图像。
*   **实现流程**：
    *   模型基于 FLUX.1-Dev 架构。
    *   首先从商品图中提取高频特征图（High-Frequency Map）。
    *   通过双流 DiT 模块融合视觉 Token 和高频 Token 进行生成。
*   **硬件要求**：
    *   论文中训练使用了 8 张 NVIDIA A800 GPU。由于基于 FLUX 架构，推断通常需要大显存的高性能 GPU。

4. **主要创新点**：
*   **HP-Image-40K 数据集**：构建了一个包含 40,000+ 样本的高质量数据集。利用预训练 T2I 模型生成“双联画”格式图像，并通过自动化过滤流程（边缘检测分割、CLIP 相似度筛选、OCR 文本一致性校验）确保数据的多样性和高质量，解决了真实数据收集难的问题。
*   **共享增强注意力机制 (Shared Enhancement Attention, SEA)**：设计了一种改进的 DiT 架构，在双流视觉块中引入了基于商品高频图的 Token 分支。通过参数共享机制，显式地利用高频信息增强模型对商品细粒度特征（形状、图案）的感知与重建。
*   **细节感知损失 (Detail-Aware Loss, DAL)**：引入了一种基于高频信息的像素级监督损失函数。该损失函数专门针对掩码区域的高频分量进行约束，弥补了仅依赖潜在空间 MSE 损失导致的细节模糊问题，强制模型精确恢复复杂的纹理和文字。

5. **实验效果**：
*   **核心数据集表现**：在 HP-Image-40K 测试集（合成数据）和收集的真实世界测试集（Real-world dataset）上均进行了评估。
*   **定量结果**：HiFi-Inpaint 在所有关键指标上均达到 SOTA（State-of-the-Art）水平。特别是在视觉一致性指标（CLIP-I: 0.950, DINO: 0.919）和结构/细节保留指标（SSIM: 0.634, SSIM-HF: 0.429）上大幅领先于 ACE++、Insert Anything 和 FLUX-Kontext 等基线模型。
*   **定性评估**：用户研究和可视化结果显示，该方法生成的图像不仅整体美观度高，而且能极好地保留商品上的微小文字和复杂 Logo，且在大姿态变化或小掩码区域等困难场景下仍表现稳健。

## 📄 RealWonder: Real-Time Physical Action-Conditioned Video Generation

- **链接**: https://huggingface.co/papers/2603.05449
- **阅读来源**: HTML

1. **应用领域**：计算机视觉-视频生成（特别是基于物理动作条件的交互式视频生成、世界模型构建、机器人模拟及AR/VR）。

2. **一句话核心贡献**：提出了 RealWonder 系统，通过将物理模拟作为中间桥梁，首次实现了从单张图像出发，基于连续3D物理动作（如力和机器人操作）的实时（13.2 FPS）流式视频生成，解决了视频模型难以理解复杂3D物理互动的问题。

3. **使用指南**：
    *   **输入**：单张 RGB 图像、文本提示（Prompt）、以及连续的 3D 物理动作序列（包括外力点/力场、机器人末端执行器指令、相机位姿）。
    *   **输出**：符合物理规律的实时光影视频流（分辨率 480x832）。
    *   **流程**：
        1.  系统首先利用 SAM 2 和深度估计模型将2D图像重建为可模拟的3D点云场景并估算材质。
        2.  物理模拟器（Genesis）根据输入动作计算动态演变，输出光流（Optical Flow）和粗糙的 RGB 预览。
        3.  这些中间信号作为条件输入到一个蒸馏后的4步扩散视频生成模型中，合成最终视频。
    *   **资源**：代码和模型权重公开，支持在单张 H200 GPU 上进行实时推理。

4. **主要创新点**：
    1.  **物理模拟作为中间表征桥梁**：利用物理引擎将连续、无界的 3D 物理动作转化为视频模型可理解的视觉信号（光流和 RGB 预览），从而避开了对连续动作进行 Token 化的难题，并解决了缺乏“动作-视频”配对训练数据的瓶颈。
    2.  **无需动作数据的训练策略**：视频生成器的训练仅依赖“光流-视频”配对数据（易于通过现有视频合成），无需昂贵且难以获取的真实物理动作标注数据，极大地提升了系统的可扩展性。
    3.  **实时流式生成架构**：通过分布匹配蒸馏（Distribution Matching Distillation）技术，将视频扩散模型压缩至仅需 4 步推理，结合轻量级点云渲染，实现了端到端的低延迟流式生成（约 75ms/帧）。

5. **实验效果**：
    *   **数据集**：构建了包含30个多样化场景（刚体、布料、流体、颗粒物等）的测试集。
    *   **对比基线**：与 PhysGaussian（基于物理高斯）、CogVideoX-I2V 和 Tora（基于轨迹控制的视频模型）进行了对比。
    *   **性能指标**：在 480x832 分辨率下达到 **13.2 FPS** 的生成速度。
    *   **用户评价**：在 400 人参与的 2AFC（迫选法）用户研究中，RealWonder 在物理响应准确性、运动合理性和视频自然度方面均显著优于所有基线模型。
    *   **鲁棒性**：实验表明系统对3D重建误差和材质估算误差具有较强的鲁棒性，视频模型能自动修复模拟器输出的视觉瑕疵（如补充水波纹细节）。

## 📄 UltraDexGrasp: Learning Universal Dexterous Grasping for Bimanual Robots with Synthetic Data

- **链接**: https://huggingface.co/papers/2603.05312
- **阅读来源**: HTML

1. **应用领域**：
   机器人学 - 双臂灵巧抓取与操作 (Robotics - Bimanual Dexterous Grasping and Manipulation)

2. **一句话核心贡献**：
   提出了一套结合优化合成与运动规划的数据生成流程，构建了首个大规模双臂灵巧抓取合成数据集 UltraDexGrasp-20M，并据此训练出具备强泛化能力和零样本 Sim-to-Real 迁移能力的通用抓取策略。

3. **使用指南**：
   *   **输入**：场景的 3D 点云数据（通常由深度相机如 Azure Kinect 采集）。
   *   **输出**：双臂及灵巧手的控制指令（动作向量）。
   *   **硬件需求**：需要双臂机器人平台（论文中使用两台 UR5e 机械臂）及多指灵巧手（论文中使用两台 XHand），以及用于推理的 GPU 计算资源。
   *   **开源情况**：论文明确表示开源了数据生成流程。
   *   **流程**：首先利用数据生成管线在仿真中生成海量多样化轨迹，训练策略网络，然后直接部署到真实机器人上，无需真实世界微调。

4. **主要创新点**：
   1.  **互补式数据生成流水线**：创新性地集成了“基于优化的抓取合成”（确保抓取姿态的物理合理性与几何契合度）与“基于规划的演示生成”（确保双臂协同运动的运动学可行性与连贯性），解决了高质量双臂灵巧抓取数据生成的难题。
   2.  **多策略自适应数据集 (UltraDexGrasp-20M)**：构建了包含 1000 个物体、2000 万帧的大规模数据集，涵盖了针对不同尺寸物体的多种抓取策略（如双臂环抱大物体、全手抓取中物体、两/三指捏取小物体）。
   3.  **基于单向注意力的通用策略网络**：设计了一种简单高效的策略架构，采用 PointNet++ 编码点云，通过解码器 Transformer 的单向注意力机制聚合场景特征，并使用有界高斯分布预测动作，显著提升了对新颖物体的泛化性。

5. **实验效果**：
   *   **仿真环境**：在包含 600 个不同形状、重量（5g-1000g）和尺寸物体的测试集中（含训练未见物体），该策略达到了 **84.0%** 的平均成功率，超越强基线方法（如 DP3）约 25.2 个百分点。
   *   **真机实验**：模型仅在合成数据上训练，直接在真实世界中部署（Zero-shot Sim-to-Real），对 25 个具有挑战性的真实物体实现了 **81.2%** 的平均抓取成功率，展现了极强的鲁棒性。

## 📄 AgentVista: Evaluating Multimodal Agents in Ultra-Challenging Realistic Visual Scenarios

- **链接**: https://huggingface.co/papers/2602.23166
- **阅读来源**: HTML

1. **应用领域**：多模态大模型评测 (Multimodal Agent Evaluation)、具身智能体 (Embodied AI)、大模型工具调用 (Tool-use)。

2. **一句话核心贡献**：提出了一项名为 AgentVista 的高难度基准测试，通过 209 个基于真实视觉场景的任务，系统评估了多模态智能体在长视程、混合工具调用（如搜索、代码执行）环境下的综合解决问题能力，揭示了当前顶尖模型在视觉定位与复杂推理上的巨大差距。

3. **使用指南**：
    *   **输入**：包含丰富细节的真实世界图像（单图或多图，如设备照片、截屏、图表）以及基于该视觉场景的特定用户查询。
    *   **工具配置**：需为智能体提供四类核心工具接口：Web Search（网页搜索）、Image Search（图像搜索）、Page Navigation（网页浏览）、Code Interpreter（代码解释器，支持图像处理库如 PIL/OpenCV）。
    *   **输出**：智能体需经过多步推理，输出一个确定的、简短的答案（如实体名称、数值或短语）。
    *   **评估**：使用提供的轻量级代理框架执行任务，利用 judge 模型（如 GPT-4o）将智能体的最终输出与人工标注的标准答案（Ground Truth）进行比对计算准确率。

4. **主要创新点**：
    *   **超高难度的真实视觉场景（Ultra-Challenging Realistic Scenarios）**：区别于传统基准中简化的视觉输入，AgentVista 从 30 万张图像中筛选出需极细粒度视觉定位的任务，强制模型必须“看懂”复杂的视觉细节（如模糊标签、微小组件差异）才能进行下一步，而非仅依赖文本。
    *   **强制长视程混合工具交互（Long-Horizon Hybrid Tool Use）**：任务设计要求智能体必须交替使用视觉感知工具（如用代码裁剪/增强图片）和外部知识检索工具。困难样本可能需要超过 25 轮交互，远超现有基准的平均轮数，有效测试了智能体的规划稳定性。
    *   **严格的四阶段构建与验证流程**：采用“模型辅助筛选 -> 专家重写 -> 执行验证 -> 双重人工校验”的流水线。特别是“执行验证”阶段，剔除了无需工具即可回答或工具调用单一的样本，确保每个任务都有唯一可验证的答案且必须依赖多步工具推理。

5. **实验效果**：
    *   **整体表现低迷**：在测评的 14 个前沿模型中，即使是表现最好的 **Claude 3.5 Sonnet，其准确率也仅为 27.3%**，GPT-4o 为 23.9%，说明该基准远未被解决。
    *   **开源与闭源差距大**：代表性开源模型（如 LLaVA-OneVision）的准确率均低于 13%，与闭源顶尖模型存在显著差距。

# 论文报告：On-Policy Self-Distillation for Reasoning Compression

1. **应用领域**

2. **一句话核心贡献**

   *   **输出**：一个能够自动生成简洁且高准确率推理路径的模型。

4. **主要创新点**
   1.  **无监督且在线的自蒸馏框架**：打破了以往压缩方法对真值标签（Ground-Truth）或外部奖励模型的依赖。利用模型具备的指令遵循能力，以“添加简洁指令”后的自身作为教师，解决了传统 RL 方法需要复杂奖励工程的问题。
   2.  **自适应难度的压缩机制**：该方法不需要专门的难度评估器。理论和实验证明，OPSDC 能自动根据问题难度调整压缩率——对简单问题进行大幅压缩（去除冗余），而对复杂问题保留必要的深思熟虑步骤（Weak Compression Signal for Hard Problems）。
   3.  **利用反向 KL 散度避免熵坍缩与提升准确率**：
        *   使用 Reverse KL 使得学生模型仅覆盖教师的高概率模式，避免了传统 Forward KL 导致的训练不稳定性（Saw-tooth pattern）。
        *   证明了推理模型的“冗余”往往包含有害的幻觉或自我怀疑，压缩过程通过减少这些 token，实际上降低了“一步错步步错”的累积误差概率，从而实现了“少思考，高准确”的反直觉效果。

5. **实验效果**
   在 Qwen3-8B 和 Qwen3-14B 模型上进行了广泛测试：
   *   **MATH-500**：实现了 **57%–59%** 的 token 缩减，同时准确率提升了 **9–16 个百分点**（例如 Qwen3-14B 从 70.0% 提升至 86.1%）。

   *   **通用能力**：在 MMLU 测试中，模型的通用知识能力保持稳定，未出现灾难性遗忘。
   *   **对比结论**：相比于定量设置压缩目标（如“减少50%”），定性的简洁指令（如“Be concise”）能带来更好的准确率表现。

## 📄 SageBwd: A Trainable Low-bit Attention

- **链接**: https://huggingface.co/papers/2603.02170
- **阅读来源**: HTML

### 1. 应用领域
**NLP - 大语言模型预训练与高效训练**
(主要针对 Transformer 架构的大语言模型在预训练（Pre-training）和微调阶段的计算加速与显存优化)。

### 2. 一句话核心贡献
本文深入分析了低比特注意力机制（Low-bit Attention）在预训练中性能下降的根本原因（反向传播中的 $\mathbf{dS}$ 误差），并通过引入 QK-Norm 和优化 Tokens-per-step 策略，成功实现了一种能匹配全精度注意力（FPA）预训练效果的可训练 INT8 注意力机制 SageBwd。

### 3. 使用指南
*   **输入**：Query ($\mathbf{Q}$)、Key ($\mathbf{K}$)、Value ($\mathbf{V}$) 张量。
*   **输出**：注意力输出 ($\mathbf{O}$) 及反向传播过程中的梯度。
*   **硬件要求**：支持 INT8 Tensor Core 计算的 GPU（文中测试使用了 NVIDIA B200 和 RTX4090）。
*   **集成方式**：作为标准 FlashAttention 的替代模块集成到 Transformer 模型中。
*   **关键设置**：
    *   **INT8 量化**：在前向和反向传播的 7 个矩阵乘法中量化其中 6 个。
    *   **稳定性配置**：必须开启 **K-smoothing**（Key 平滑）和 **QK-Norm**（Query-Key 归一化），无需开启 Q-smoothing。
    *   **超参建议**：在超大 Batch Size 训练时，QK-Norm 尤为关键；适当降低 Tokens-per-step (TPS) 有助于利用梯度噪声掩盖量化误差。

### 4. 主要创新点
1.  **量化误差源理论定位**：明确指出反向传播中的**注意力分数梯度（$\mathbf{dS}$）**是低比特训练的主要精度瓶颈。由于 $\mathbf{dS}$ 数值极小且位于连乘链中，极易受 INT8 量化噪声影响导致信噪比恶化。
2.  **QK-Norm 稳定机制**：发现并验证了 QK-Norm 在大 Tokens-per-step (TPS) 训练场景下的必要性。它通过约束 Query 和 Key 的动态范围，有效抑制了 Logits 过大导致的 Softmax 梯度数值不稳定，从而提升量化精度。
3.  **平滑策略与 TPS 关系解耦**：
    *   **平滑策略**：通过消融实验证明，**K-smoothing** 对训练稳定性至关重要，而 Q-smoothing 在预训练中收益有限甚至可能引入梯度偏差。
    *   **TPS 效应**：揭示了较小的 TPS 会产生较大的梯度噪声，这种随机性反而能“掩盖”系统性的量化误差，使低比特训练更易收敛至全精度水平。

### 5. 实验效果
在 **325M 参数的 Llama 模型**上，使用 **78B Tokens** 数据进行了预训练对比实验：
*   **低 TPS 场景 (260K Tokens/step)**：SageBwd 的最终验证集 Loss 为 **2.561**，与全精度注意力（FPA）的 **2.563** 在统计上无显著差异，证明了其无损训练能力。
*   **高 TPS 场景 (2.1M Tokens/step)**：在未加 QK-Norm 时，传统低比特方法 Loss 会发散（2.640 vs FPA 2.586）；加入 QK-Norm 后，SageBwd 能够保持训练稳定并显著缩小与全精度的差距。
*   **误差分析**：通过中间张量对比，证实了 SageBwd 能有效控制反向传播中 $\mathbf{dS}$ 等关键梯度的 RMS 误差。

## 📄 Towards Multimodal Lifelong Understanding: A Dataset and Agentic Baseline

- **链接**: https://huggingface.co/papers/2603.05484
- **阅读来源**: HTML

### 1. 应用领域
**多模态大模型 (MLLM) / 长视频理解 / 智能体 (Agentic AI)**
特别关注跨越数天至数月的超长周期视频流（如个人生活记录、长期直播、游戏进程）的记忆、检索与推理。

### 2. 一句话核心贡献
本文提出了首个针对**多模态终身理解（Multimodal Lifelong Understanding）**的数据集 **MM-Lifelong**，并设计了**递归多模态智能体 (ReMA)**，通过动态记忆管理有效解决了现有端到端模型在处理具有高时间稀疏性的超长视频流时面临的上下文饱和与遗忘问题。

### 3. 使用指南
*   **输入**：跨越长周期的原始视频流（包含视觉和音频）以及用户的自然语言查询（Query）。
*   **输出**：针对问题的自然语言回答，以及支撑该推理过程的视频时间片段（Temporal Grounding，即证据所在的具体起止时间）。
*   **核心流程**：
    1.  **离线感知阶段**：使用基础模型（如 Whisper, Qwen-VL）将视频流转化为结构化的文本/向量记忆库（Memory Bank）。

*   **硬件与依赖**：需要大显存 GPU 支持基础视觉模型推理，依赖向量数据库（如 FAISS）存储长期记忆，通常需要调用高性能 LLM（如 GPT-4/5 或 Qwen）作为 Agent 的核心大脑。

### 4. 主要创新点
1.  **定义“终身理解”范式与 MM-Lifelong 数据集**：
    不同于传统的“长上下文”视频任务，本文严格定义了**终身理解 (Lifelong Understanding)**，强调**时间稀疏性（Temporal Sparsity）**和**未被观测的时间间隔（Unobserved Gaps）**。构建了包含 181.1 小时、跨越日/周/月三个时间尺度的数据集，模拟真实生活中的概念漂移和非连续性。
2.  **提出递归多模态智能体架构 (ReMA)**：
    设计了一种非端到端的 Agent 基线，不直接将所有视频帧塞入 Context Window，而是将长视频流视为一个“活跃知识库”。ReMA 利用递归策略管理记忆，通过“回忆-检查-更新”的循环机制，能够在极长的时间跨度中精准定位“大海捞针”般的关键信息。
3.  **揭示端到端模型的“上下文饱和”失效模式**：
    实验发现，随着输入视频长度增加，当前的端到端 MLLM（即使支持无限上下文）会出现性能震荡甚至急剧下降（Context Saturation）。这表明单纯扩大上下文窗口无法解决终身理解中的噪声积累问题，验证了 Agentic 记忆架构的必要性。

### 5. 实验效果
*   **基准对比**：在 MM-Lifelong 数据集的 Day、Week、Month 三个子集上，ReMA 均显著优于现有的 SOTA 端到端模型（如 Gemini 1.5 Pro, GPT-4o）以及其他视频 Agent 基线（如 DeepVideoDiscovery, LVAgent）。
*   **具体表现**：
    *   **回答准确率**：ReMA 在最难的 Month-Scale 子集上实现了最高的准确率，证明了其处理跨越数周甚至数月依赖关系的能力。
    *   **时间定位 (Grounding)**：在 Ref@N 指标上，ReMA 展现了卓越的证据定位能力。相比之下，端到端模型在面对 100+ 小时的视频流时，几乎无法输出有效的证据时间戳（Ref 得分极低），主要依赖语义先验而非视觉证据，而 ReMA 能够有效召回并定位具体的视频片段。

## 📄 Distribution-Conditioned Transport

- **链接**: https://huggingface.co/papers/2603.04736
- **阅读来源**: HTML

### 1. 应用领域
**机器学习-生成模型 / 计算生物学**
具体场景包括：单细胞基因组学中的批次效应校正（Batch Effect Transfer）、细胞药物扰动响应预测、细胞分化动力学推断，以及离散序列数据的演化建模（如T细胞受体序列）。

### 2. 一句话核心贡献
提出了一种名为“分布条件传输”（Distribution-Conditioned Transport, DCT）的通用框架，通过学习源分布和目标分布的嵌入表示来调节传输映射，从而实现了在训练期间未见过的分布对之间的泛化传输，并能有效利用非配对的边缘分布数据进行半监督学习。

### 3. 使用指南
*   **输入**：
    1.  **源样本集合**：来自源分布的一组样本（如未处理的细胞基因表达数据）。
    2.  **目标分布信息**：可以是目标样本集合（用于提取嵌入）或预测的目标分布嵌入向量。
*   **模型流程**：
    1.  **分布编码**：使用置换不变的神经网络（如 DeepSets）将输入的样本集合映射为低维分布嵌入向量（Distribution Embedding）。
    2.  **条件传输**：将源样本、源分布嵌入和目标分布嵌入输入到生成模型（如 Flow Matching、基于 OT 的模型或离散序列生成器），生成符合目标分布的样本。
*   **输出**：映射后的样本集合，使其统计特性符合目标分布。
*   **硬件与代码**：需要 GPU 进行训练（涉及神经网络优化）。论文中提到代码已开源。

### 4. 主要创新点
1.  **全对全（Any-to-Any）的泛化能力**：
    传统的传输模型通常针对固定的源-目标对训练。DCT 通过学习分布的连续嵌入空间，而非使用离散的条件标签，使得模型能够对**训练期间从未见过的**源分布和目标分布进行传输（Zero-shot generalization）。
2.  **利用孤立边缘分布（Orphan Marginals）的半监督学习**：
    在生物学数据中，成对的样本往往稀缺（如同一克隆在不同时间点的观测）。DCT 允许在训练中混合使用成对数据和非配对的“孤立”数据（仅在一个时间点观测到的分布），从而显著提高了模型在稀疏数据下的性能。
3.  **兼容多种传输机制的通用框架**：
    DCT 不依赖于特定的底层生成模型。它可以与**连续流模型**（如 Flow Matching, Stochastic Interpolants）、**散度最小化模型**（如 Sliced Wasserstein, MMD）以及**离散序列模型**（如基于 ESM2 的 Transformer）结合使用，具有极强的扩展性。

### 5. 实验效果
*   **合成高斯数据基准**：
    在分布外（OOD）测试中，DCT 相比于传统的基于离散标签（K-to-K）的模型，能够准确地推断未见过的目标分布，误差显著更低，证明了其通过嵌入空间进行插值和外推的能力。
*   **单细胞药物扰动预测 (Trellis 数据集)**：
    在预测未见过的患者（Held-out patients）对化疗药物的反应时，源-目标条件化（Source-Target-Conditioned）的 DCT 模型优于仅基于源条件的模型（如 Meta Flow Matching），展现了更好的跨患者泛化能力。
*   **T细胞受体 (TCR) 序列演化**：
    在仅有 30% 患者拥有多个时间点数据的稀疏场景下，利用非配对数据进行半监督训练的 DCT 模型（使用 ESM2 主干），相比仅使用成对数据的监督基线，将预测分布的 Energy Distance 降低了一半以上，有效解决了小样本纵向数据的预测难题。

## 📄 KARL: Knowledge Agents via Reinforcement Learning

- **链接**: https://huggingface.co/papers/2603.05218
- **阅读来源**: HTML

1. **应用领域**：
   NLP-大模型智能体 (Agents)、检索增强生成 (RAG)、强化学习 (Reinforcement Learning)、企业级知识搜索与有据推理 (Grounded Reasoning)。

2. **一句话核心贡献**：
   提出了一种结合多任务强化学习与自举式合成数据管道的训练框架 (KARL)，通过一种新的离线 RL 算法 (OAPL)，在低成本和低延迟下实现了超越顶尖闭源模型（如 Claude Opus 4.6）的复杂文档搜索与推理能力。

3. **使用指南**：
   *   **输入**：用户查询以及对应的封闭域文档库（需预先进行向量化索引）。
   *   **流程**：
     1. **数据生成**：使用代理（Agent）利用向量搜索工具在语料库中探索，生成“问题-答案”对及推理轨迹，并通过去重和质量过滤代理进行清洗。
     2. **训练**：采用 **OAPL** (Optimal Advantage-based Policy Optimization with Lagged Inference) 算法，对基座模型（如 GLM 4.5 Air）进行迭代式的大批量离线强化学习微调。
     3. **推理**：模型在推理时仅使用向量搜索工具，支持自动上下文压缩。可通过“并行思考 (Parallel Thinking)”策略（并行生成多个轨迹并聚合）进一步提升效果。
   *   **基础设施**：训练依赖高吞吐量的嵌入式向量数据库以支持大规模 rollout，实验中使用 H200 GPU 节点进行推理和训练。

4. **主要创新点**：
   *   **自举式代理合成数据管道 (Agentic Synthesis Pipeline)**：设计了一套动态流程，让代理主动探索语料库生成数据，而非静态提示。随着模型能力迭代，其合成的数据质量也随之提升（Self-improvement），解决了复杂推理任务训练数据匮乏的问题。
   *   **OAPL 离线强化学习范式**：提出了一种迭代式、大批量的离线策略 RL 方法。相比在线 RL（如 PPO/GRPO），该方法样本效率更高，无需复杂的稳定性技巧（如截断重要性采样），且对训练端与推理端（如 vLLM）的差异具有鲁棒性。
   *   **多任务异构泛化**：证明了在异构搜索行为（如深度搜索 BrowseComp-Plus 和广度搜索 TREC-Biogen）上进行联合训练，能显著提升模型在未见过的分布外任务（Out-of-Distribution）上的泛化能力。

5. **实验效果**：
   *   **综合性能**：在提出的 **KARLBench**（包含表格推理、多文档合成等 6 种搜索模式）上，KARL 模型在成本-质量和延迟-质量的权衡曲线上均达到 Pareto 最优。
   *   **对比顶尖模型**：在不使用额外推理计算时，KARL 性能匹敌 Claude Sonnet 4.5；在使用 10 条并行推理轨迹（Test-time Compute）后，其表现匹配甚至超越了最佳闭源模型 Claude Opus 4.6。
   *   **效率提升**：相比基座模型 GLM 4.5 Air，KARL 将搜索轨迹缩短且准确率大幅提升，能够解决基座模型无法收敛的长视距搜索问题，同时推理成本降低约 33%，延迟降低约 47%。
