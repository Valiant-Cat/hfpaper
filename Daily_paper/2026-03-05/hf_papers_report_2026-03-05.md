## 📄 ArtHOI: Articulated Human-Object Interaction Synthesis by 4D Reconstruction from Video Priors

- **链接**: https://huggingface.co/papers/2603.04338
- **阅读来源**: HTML

1. **应用领域**：计算机视觉 - 4D 场景重建与人机交互（HOI）生成、具身智能数据合成、VR/AR 内容创作。

2. **一句话核心贡献**：ArtHOI 提出了首个零样本关节体人机交互合成框架，通过将合成任务转化为基于单目视频先验的 4D 重建问题，成功在无 3D 监督的情况下实现了具有物理合理性（如开门、开柜子）的动态场景生成。

3. **使用指南**：
    *   **输入**：自然语言提示词（Text Prompt，例如“一个人打开冰箱”）。
    *   **流程**：
        1.  使用视频扩散模型（如 KLing）根据文本生成一段单目 2D 视频。
        2.  **第一阶段**：利用光流和 SAM（Segment Anything）自动分割物体的动态（关节）与静态部分，并通过运动学约束重建 4D 物体几何。
        3.  **第二阶段**：将重建的物体作为固定的时空支架，优化人体运动以满足接触和非穿透等物理约束。
    *   **输出**：几何一致且符合物理规律的 4D 人机交互场景（通常表示为 3D 高斯或 Mesh 序列）。
    *   **硬件与耗时**：实验在 NVIDIA A6000 (48GB) 上进行，单个场景处理流程约需 20 分钟。

4. **主要创新点**：
    1.  **基于 4D 重建的合成范式**：摒弃了传统的端到端生成，将生成式视频模型作为先验，通过“逆渲染”的方式从生成的 2D 视频中重建 4D 场景。这种方法在无 3D 数据监督的情况下，有效引入了显式的几何和物理约束。
    2.  **解耦的两阶段优化策略**：针对单目视频中人与物体运动耦合导致的优化不稳定问题，提出了“先重建物体、后合成人体”的策略。通过先恢复刚性的物体关节运动作为“支架”，再以此为约束求解人体动作，解决了单目深度模糊性问题。
    3.  **无模板的关节结构发现**：提出了一套基于光流的分割与绑定算法。利用光流作为几何线索区分动态与静态区域，结合“准静态绑定（Quasi-static binding）”策略，无需预定义的物体类别模板即可自动识别并建模物体的关节结构（如门轴）。

5. **实验效果**：
    *   **物理合理性大幅提升**：在核心评价指标上，ArtHOI 的接触准确率（Contact%）达到 **96.5%**，显著优于现有最佳方法（如 ZeroHSI 的 83.3%）；同时实现了最低的穿模误差（Penetration 为 0.009）。
    *   **关节动力学精准**：在关节物体动力学估计方面，其旋转误差（Rotation Error）远低于专门的单目估计方法（D3D-HOI），证明了重建的物体运动具有高度的几何保真度。
    *   **用户偏好**：在涉及冰箱、柜子等关节物体的交互生成用户调研中，ArtHOI 在物理真实感和接触准确性上均获得了最高的用户偏好度。

## 📄 CubeComposer: Spatio-Temporal Autoregressive 4K 360° Video Generation from Perspective Video

- **链接**: https://huggingface.co/papers/2603.04291
- **阅读来源**: HTML

# CubeComposer 论文研读报告

**1. 应用领域**
计算机视觉 - AIGC 视频生成、虚拟现实 (VR) 内容创作、360° 全景视频补全 (Perspective-to-360° Video Generation)。

**2. 一句话核心贡献**
提出了一种基于立方体贴图的时空自回归扩散模型，通过覆盖率引导的生成顺序和稀疏注意力机制，克服了传统方法显存限制，首次实现了从普通视角视频到原生 4K 分辨率 360° 全景视频的高质量生成。

**3. 使用指南**
*   **输入**：一段由普通相机拍摄的透视视频（Perspective Video，包含相机轨迹），以及可选的全局或分面文本提示词（Captions）。
*   **输出**：完整的、高分辨率（最高 4K）的 360° 全景视频（Equirectangular 格式）。
*   **流程**：
    1.  将输入的透视视频投影并转换为包含六个面的立方体贴图（Cubemap）。
    2.  模型将视频划分为多个时间窗口，并根据输入视角的空间覆盖率自动规划生成顺序。
    3.  利用自回归方式逐面、逐窗口生成缺失内容，最终拼接为全景视频。
*   **硬件需求**：基于视频基础模型 Wan 2.2 5B 微调，推理过程虽然优化了峰值显存，但仍需高性能 GPU 支持。

**4. 主要创新点**
1.  **覆盖率引导的时空自回归策略**：放弃一次性生成整个全景图的传统做法，将视频分解为立方体贴图（Cubemap），在时间窗口内根据空间覆盖率（Spatial Coverage）降序规划生成顺序。这种策略优先生成已知信息丰富的面，利用已生成内容作为条件，大幅降低了显存峰值并提升了生成稳定性。
2.  **高效的稀疏上下文注意力机制**：设计了包含“历史窗口”、“当前窗口及未生成面的透视条件”和“未来有效片段”的三重上下文管理机制。特别是引入了**稀疏上下文注意力（Sparse Context Attention）**，通过对角带状掩码让上下文仅进行局部注意力计算，将计算复杂度从上下文长度的平方级降为线性，有效支持长序列生成。
3.  **立方体拓扑感知的连续性设计**：针对分面生成可能产生的边界接缝问题，提出了**立方体感知位置编码（Cube-aware RoPE）**以及**立方体感知填充与混合（Padding and Blending）**技术。通过在生成过程中引入相邻面的重叠区域并进行像素级混合，消除了空间边界的非连续性，确保全景视频的无缝拼接。

**5. 实验效果**
*   **数据集**：构建了包含 11,832 个高分辨率视频片段的 **4K360Vid** 数据集用于训练和评估。
*   **定量对比**：在 ODV360 和 4K360Vid 测试集上，CubeComposer 在 **LPIPS、FID、FVD** 以及 **VBench**（美学质量、成像质量、整体一致性）等指标上均优于 Argus、ViewPoint 和 Imagine360 等现有 SOTA 方法。
*   **定性表现**：相比于仅支持 1K 分辨率或依赖外部超分模型（如 VEnhancer）的现有方法，CubeComposer 原生生成的 4K 视频在细节丰富度、清晰度以及时空一致性上表现更佳，且无明显的拼接痕迹。

## 📄 MemSifter: Offloading LLM Memory Retrieval via Outcome-Driven Proxy Reasoning

- **链接**: https://huggingface.co/papers/2603.03379
- **阅读来源**: HTML

1. **应用领域**：NLP - 大语言模型（长长期记忆机制 / 检索增强生成 RAG / 强化学习）。

2. **一句话核心贡献**：提出了一种名为 MemSifter 的框架，通过结果导向的强化学习训练轻量级代理模型来承担记忆检索与推理任务，在降低主模型计算负担的同时显著提升了长周期任务的检索精度和最终效果。

3. **使用指南**：
    *   **输入**：用户的历史交互记录（按会话分段的原始文本）以及当前的对话上下文/查询。
    *   **流程**：
        1.  若历史记录过长（>128k token），先使用嵌入模型进行粗筛。
        2.  将筛选后的历史记录输入到轻量级 MemSifter 代理模型。
        3.  代理模型执行“先思考后排序（Think-and-Rank）”，生成推理依据并输出 Top-K 最相关的会话 ID。
        4.  将这 Top-K 个会话的原始内容拼接，作为上下文输入给主力工作大模型（Working LLM）。
    *   **输出**：主力大模型基于检索到的高精度记忆生成的最终回复。
    *   **资源**：代码、模型权重及训练数据已开源，推理只需较小的显存资源（相对于全量上下文输入）。

4. **主要创新点**：
    *   **解耦的“推理-检索”代理架构**：利用轻量级代理模型（Proxy）专门负责记忆的筛选和推理，替代了传统的向量检索或依赖主力大模型处理全量历史的方法，实现了“推理前置”的高精度检索，大幅降低了推理成本。
    *   **任务结果导向的强化学习（RL）范式**：摒弃了传统检索依赖静态相关性标签（如 Recall/Precision）的训练方式，直接将主力大模型（Working LLM）完成任务的最终表现作为奖励信号，确保检索结果真正对下游推理有用。
    *   **基于边际效用与位置感知的奖励机制**：设计了包含“消融实验效用估计（Utility Estimation）”和“排序敏感衰减（Rank-sensitive Discounting）”的复合奖励函数，不仅奖励能提升任务分数的记忆，还强迫模型将最关键的信息排在首位。

5. **实验效果**：
    *   在 **8 个大模型记忆基准数据集**（涵盖个人记忆 LoCoMo、PersonaMem 及复杂的 Deep Research 任务）上进行了评估。
    *   **性能表现**：MemSifter 在检索准确率和最终任务完成度（F1 Score等）上均全面超越了现有的 SOTA 方法（包括 Dense Retrieval、MemGPT、Graph RAG 以及原生超长上下文模型）。
    *   **效率优势**：相比直接使用长上下文 LLM 处理全量历史，MemSifter 极大地降低了计算开销，同时在检索精度上优于“先检索后重排（Rerank）”的基线模型。

## 📄 T2S-Bench & Structure-of-Thought: Benchmarking and Prompting Comprehensive Text-to-Structure Reasoning

- **链接**: https://huggingface.co/papers/2603.03790
- **阅读来源**: HTML

1. **应用领域**：自然语言处理 (NLP) - 大模型推理、长文本理解与信息抽取 (LLM Reasoning & Information Extraction)。

2. **一句话核心贡献**：提出了一种显式引导模型构建中间文本结构的提示策略“思维结构”(SoT)，并构建了首个评估模型“文本到结构”转换及推理能力的综合基准 T2S-Bench，显著提升了模型在复杂长文本任务中的表现。

3. **使用指南**：
    *   **输入**：复杂的长文本、科学文献或需要多跳推理的文档。
    *   **使用方法**：
        *   **Prompt 模式 (SoT)**：在提示词中要求模型先识别文本中的关键节点（Nodes）和链接（Links）构建结构图，然后再基于该结构回答问题。
        *   **微调模式**：使用论文提供的 T2S-Train-1.2k 数据集对开源模型（如 Qwen, Llama）进行微调，以内化结构化能力。
    *   **输出**：JSON 格式的文本结构图（包含节点实体和关系边）以及基于该结构的最终推理答案。
    *   **资源**：包含自动化数据构建流程、评估脚本及开源数据集。

4. **主要创新点**：
    1.  **思维结构 (Structure-of-Thought, SoT)**：提出了一种通用的提示范式，将传统的端到端生成分解为“结构化-检索-生成”过程，作为一种稳定的中间表示 (Intermediate Representation)，在 8 个任务中表现优于思维链 (CoT)。
    2.  **T2S-Bench 基准数据集**：构建了首个涵盖 6 大科学领域、32 种结构类型的“文本到结构”基准，包含 1.8k 高质量样本，设计了多跳推理 (MR) 和端到端结构提取 (E2E) 两种评估任务。
    3.  **基于学术图表的数据构建流水线**：创新性地利用学术论文中经过同行评审的流程图/结构图作为“黄金标准”，结合自动化提取与专家（PhD级）多轮校验，解决了结构化数据标注难、容易产生幻觉的问题。

5. **实验效果**：
    *   **SoT 增益**：在 Qwen2.5-7B-Instruct 模型上，使用 SoT 策略在 8 个不同的文本处理任务中平均带来 **+5.7%** 的性能提升；在 T2S 数据集上微调后，提升幅度进一步增加至 **+8.6%**。
    *   **模型排名**：在测评的 45 个主流模型中，**Gemini-2.5-Pro** 表现最佳（MR 任务准确率 81.40%），其次是 Claude-sonnet 和 GPT-5.2；开源模型中 DeepSeek 和 Qwen 系列表现具有竞争力。
    *   **瓶颈揭示**：实验发现即使是顶尖模型，在端到端提取任务中的**节点提取**准确率依然较低（仅约 58.1%），表明从非结构化文本中精准定位实体仍是当前大模型的主要短板。

## 📄 SWE-CI: Evaluating Agent Capabilities in Maintaining Codebases via Continuous Integration

- **链接**: https://huggingface.co/papers/2603.03823
- **阅读来源**: HTML

1. **应用领域**：NLP-大模型代码智能（LLM for Code）、软件工程自动化（Automated Software Engineering）、智能体协作（Multi-Agent Systems）。

2. **一句话核心贡献**：提出了首个基于持续集成（CI）循环的仓库级基准测试 SWE-CI，将代码生成能力的评估从静态的单次修复扩展到了动态的长周期代码维护与演进场景。

3. **使用指南**：
    *   **输入**：基准测试包含 100 个任务，源自 68 个真实的 Python 仓库。每个任务由一个“基础提交（Base Commit）”和一个“目标提交（Oracle Commit）”组成，平均跨越 233 天的开发历史。
    *   **流程**：采用“架构师-程序员（Architect-Programmer）”双智能体协议。架构师分析测试失败原因并撰写需求文档，程序员根据文档修改代码。该过程在 Docker 容器中通过 CI 循环（修改代码 -> 运行测试 -> 反馈）反复迭代（最多 20 轮）。
    *   **输出**：智能体旨在消除与目标代码库的功能差距，输出通过测试的维护后代码库。
    *   **环境**：依赖 Docker 进行环境隔离，使用 `pytest` 作为测试框架，基于 `iFlow CLI` 构建智能体工作流。

4. **主要创新点**：
    *   **基于演进的动态评估范式**：区别于 SWE-bench 的“快照式”单次修复，SWE-CI 模拟了真实的长期软件开发周期（平均 71 次连续提交），要求智能体处理需求变更并在此过程中通过持续集成维护代码质量。
    *   **EvoScore 可维护性指标**：提出了一种加权评价指标 `EvoScore`，通过折扣因子 $\gamma$ 平衡短期修正与长期收益。该指标能奖励那些编写易于扩展代码的智能体，惩罚虽然通过当前测试但积累技术债务导致后续维护困难的行为。
    *   **双智能体协同工作流**：设计了符合工业界实践的架构师与程序员分工模式。架构师负责高层诊断与规划（避免过度设计），程序员负责具体实现，二者配合模拟真实团队在 CI 流程中的快速迭代。

5. **实验效果**：
    *   **模型表现**：对来自 8 个提供商的 18 个模型进行了评估。结果显示 Claude Opus 系列模型在长周期维护任务中表现出显著优势，领跑所有模型；GLM-5 亦表现出色。
    *   **策略差异**：不同厂商的模型在维护策略上表现出不同偏好。MiniMax、DeepSeek 和 GPT 系列倾向于优化长期可维护性（EvoScore 在 $\gamma$ 较大时得分高），而 Kimi 和 GLM 更倾向于短期任务通过率。
    *   **核心短板**：当前 LLM 在防止代码回退（Regression）方面能力严重不足。大多数模型的“零回退率”低于 0.25（即在维护过程中频繁破坏已有功能），仅 Claude Opus 系列超过 0.5，表明现有模型难以在长期维护中保障软件稳定性。

## 📄 Helios: Real Real-Time Long Video Generation Model

- **链接**: https://huggingface.co/papers/2603.04379
- **阅读来源**: ArXiv Abs

# Helios 模型研究报告

1. **应用领域**：
   计算机视觉 - 视频生成（AIGC / Video Generation），具体涵盖文生视频（T2V）、图生视频（I2V）及视频转视频（V2V）。

2. **一句话核心贡献**：
   提出了首个 140 亿参数（14B）的自回归扩散模型，能够在单张 NVIDIA H100 GPU 上以 19.5 FPS 的速度实时生成分钟级高质量长视频，同时解决了长视频生成的画面漂移问题。

3. **使用指南**：
   *   **输入**：支持统一的输入表示，用户可提供文本、图像或视频片段作为条件。
   *   **输出**：分钟级时长的连续视频内容。
   *   **硬件需求**：单张 NVIDIA H100 GPU 即可实现实时推理；得益于内存优化，80GB 显存可容纳多达 4 个 14B 模型。
   *   **开源状态**：计划开源代码、基础模型以及蒸馏模型。

4. **主要创新点**：
   1.  **无需启发式策略的抗漂移训练**：摒弃了 Self-forcing 或关键帧采样等传统抗漂移手段，通过在训练过程中显式模拟漂移模式并从源头消除重复动作，本质上解决了长视频生成中的崩溃与漂移问题。
   2.  **极简的高效推理架构**：在不依赖 KV-cache、稀疏注意力或量化等标准加速技术的前提下，通过对历史和噪声上下文进行重度压缩并减少采样步数，实现了 14B 大模型的高速实时生成。
   3.  **低资源消耗的基础设施优化**：实现了无需并行或分片框架的训练流程，支持与图像扩散模型相当的 Batch Size，并将计算成本降低至与 1.3B 小模型相当甚至更低的水平。

5. **实验效果**：
   *   **速度性能**：在单卡 H100 上达到 **19.5 FPS**，实现了真正的实时生成。
   *   **生成质量**：广泛的实验表明，Helios 在短视频和长视频生成任务上的表现均**一致优于**现有方法，且能够保持分钟级生成的连贯性与高质量。

## 📄 Heterogeneous Agent Collaborative Reinforcement Learning

- **链接**: https://huggingface.co/papers/2603.02604
- **阅读来源**: HTML

1. **应用领域**：NLP-大模型强化学习（RLHF/RLVR）、多智能体协同训练、复杂逻辑推理（如数学问题求解）。

2. **一句话核心贡献**：提出了一种异构智能体协同强化学习框架（HACRL）及其算法 HACPO，通过在训练阶段共享不同架构或规模模型的推理轨迹（Rollouts），实现了在推理时独立执行的前提下，以更低的采样成本显著提升异构模型群体的整体推理性能。

3. **使用指南**：
    *   **输入**：一组异构的大语言模型（即参数量、架构或微调状态不同的 Agents，例如 Qwen3-1.7B 和 Qwen3-4B），以及包含可自动验证奖励信号（如数学题答案）的数据集。
    *   **流程**：
        1.  各智能体针对同一 Prompt 生成推理轨迹（Rollouts）。
        2.  将所有智能体生成的经过验证的轨迹放入共享池。
        3.  使用 HACPO 算法计算能力感知的优势（Advantage），并利用特定的重要性采样机制进行策略更新。
    *   **输出**：完成训练的多个独立智能体，它们在推理阶段不需要彼此交互即可表现出更强的能力。
    *   **工具/硬件**：基于 `verl` 强化学习框架实现，通常需要多 GPU 环境来并行运行多个模型。

4. **主要创新点**：
    1.  **独立执行的协同优化范式**：打破了传统多智能体强化学习（MARL）需协同推理或知识蒸馏单向“教师-学生”传递的限制。HACRL 允许模型在训练时双向互利（强模型提供指导，弱模型提供多样化探索），但在推理时完全独立，无需部署复杂的协同系统。
    2.  **智能体能力感知的优势估计（Agent-Capability-Aware Advantage Estimation）**：设计了一种基于模型相对能力比率的加权机制。该机制能自动调整来自不同智能体样本的权重，使模型更积极地从强智能体学习，同时保持优势估计的无偏性（理论已证明）。
    3.  **针对异构分布偏移的校正机制**：为了解决异构模型间策略分布差异过大的问题，引入了**非梯度指数重加权**（抑制大分布偏移的影响）和**非对称逐步截断**（Asymmetric Stepwise Clipping，防止跨智能体样本主导梯度更新）技术，确保了训练的稳定性。

5. **实验效果**：
    *   **数据集**：在 7 个高难度数学推理基准上进行了评估，包括 MATH-500, MATH, GSM8K, AIME2025 等。
    *   **性能提升**：在多种异构设置下（如不同模型状态、不同尺寸 Qwen3-1.7B/4B/8B、不同架构 Qwen vs Llama），HACPO 相比于强基线算法 GSPO 平均提升了 **3.3%** 的性能。
    *   **效率对比**：HACPO 仅使用了**一半的采样（Rollout）成本**，即达到了优于独立训练两倍采样量的效果，证明了极高的数据利用效率。

- **阅读来源**: HTML

# 论文研报：Memex(RL): Scaling Long-Horizon LLM Agents via Indexed Experience Memory

1. **应用领域**
   NLP-大模型智能体 (LLM Agents)、强化学习 (Reinforcement Learning)、长上下文处理 (Long-Context Handling)。

2. **一句话核心贡献**
   针对长程任务中大模型上下文受限的问题，提出了一种“索引体验记忆”机制及配套的强化学习框架，使智能体学会将上下文压缩为紧凑索引并在需要时精准回溯原始证据，从而在有限上下文中实现高效的长程推理。

3. **使用指南**
   *   **输入**：自然语言形式的任务指令、环境观测流（包含工具执行结果、中间推理步骤等）。
   *   **输出**：具体的决策动作，包含两类：
       *   **环境工具**：如文件操作、API 调用等。
       *   **记忆工具**：`CompressExperience`（生成索引摘要并将详细内容归档至外部 KV 库）、`ReadExperience`（根据索引从外部库取回精确内容）。
   *   **模型训练**：需要使用本文提出的 Memex(RL) 框架对 LLM 进行微调。该框架基于 PPO/GRPO 算法，利用特定的奖励函数（任务成功奖励 - 上下文溢出惩罚 - 冗余操作惩罚）来训练模型自主管理记忆。
   *   **基础设施**：需要维护一个外部 Key-Value 数据库作为体验存储（Experience Store）；代码实现基于开源 RL 框架 Slime。

4. **主要创新点**
   *   **索引体验记忆架构 (Indexed Experience Memory, IEM)**：
       不同于传统的有损摘要或模糊的向量检索，IEM 将记忆分为两部分：在**工作上下文**中仅保留包含稳定索引（Index）的结构化摘要；将**全保真原始数据**（如代码片段、日志）完整保存在**外部数据库**中。这种设计允许智能体通过显式解引用（Dereferencing）零损耗地恢复过去的细节。
   *   **针对记忆管理的 RL 训练框架 (Memex(RL))**：
       提出了一套包含特定奖励塑形（Reward Shaping）的强化学习方法，惩罚上下文溢出和冗余工具调用。特别是引入了**压缩自适应轨迹处理 (Compression-adaptive trajectory processing)**，将压缩后的轨迹分段训练，解决了压缩导致的前缀变化带来的信用分配难题。
   *   **理论与实践的双重保证**：
       论文从理论上证明了该机制能够同时满足“保持决策质量”和“保持工作上下文有界”两个性质；并在实践中通过软触发机制（Soft Triggering）将上下文压缩转化为一种可学习的技能，而非基于硬阈值的规则。

5. **实验效果**
   在经过修改的高难度长程 **ALFWorld** 基准测试（移除了导航辅助，强制依赖记忆回溯，任务跨度可达数百步）上表现显著：
   *   **任务成功率飙升**：相比未训练的基座模型，任务成功率从 **24.22%** 提升至 **85.61%**。
   *   **上下文占用大幅降低**：在成功率提升的同时，峰值工作上下文长度减少了约 **43%**（从 16,934 tokens 降至 9,634 tokens），有效控制在 8k token 的惩罚阈值附近。
   *   **行为模式改变**：训练后的智能体显著减少了盲目的信息压缩，大幅增加了主动检索（Retrieve）操作，证明其学会了构建和利用可复用的外部记忆。

## 📄 Proact-VL: A Proactive VideoLLM for Real-Time AI Companions

- **链接**: https://huggingface.co/papers/2603.03447
- **阅读来源**: HTML

### 1. **应用领域**
多模态大模型（Multimodal LLMs）- 实时视频理解与交互（Real-time Video Understanding & Interaction），具体应用于 **AI 游戏解说（电竞评论）** 和 **实时游戏指导（AI 助手/教练）**。

### 2. **一句话核心贡献**
提出了 Proact-VL 框架及大规模 Live Gaming Dataset，通过“分块流式处理”与“主动响应决策机制”的结合，解决了传统视频大模型在实时流中无法兼顾低延迟、自主发言时机（Proactivity）与生成质量的难题。

### 3. **使用指南**
*   **输入数据**：连续的实时视频流（被切分为固定时长的块，如 1 秒/块）、用户查询（Query）以及上下文历史（Context）。
*   **模型运行**：
    1.  模型接收多源 Token（视频、查询、历史）。
    2.  **决策阶段**：利用一个轻量级的响应头（Response Head）基于 `<|answer|>` Token 的隐藏状态计算“是否发言”的概率分数。
    3.  **生成阶段**：如果分数超过阈值，模型生成短文本回复；否则保持静默（输出特殊占位符）。
    4.  **记忆管理**：采用双缓存滑动窗口（Dual-cache Sliding-window）机制与 Reverse-RoPE 技术，在有限显存下支持无限长视频流推理。
*   **输出结果**：实时的文本解说或操作指导，包含精确的时间戳和与游戏进程同步的内容。

### 4. **主要创新点**
1.  **轻量级主动响应机制 (Lightweight Proactive Mechanism)**：不同于传统模型被动等待指令或持续输出，该模型引入了一个基于门控 MLP 的响应头，使 AI 能够根据视觉和上下文线索自主决定“何时说话”，实现了对发言频率和时机的精准控制。
2.  **多层级损失函数设计 (Multi-tier Loss Function)**：提出结合 **交互损失**（Utterance Loss，监督生成内容质量）和 **响应损失**（Response Loss，监督发言时机）的联合训练目标。特别是引入了“转换感知权重”，重点优化“静默-发言”状态切换的边界，解决了训练数据中静默时长占主导导致的不平衡问题。
3.  **构建 Live Gaming Dataset**：收集并清洗了 561 小时的高质量游戏解说视频，涵盖单人解说、多人协同解说和实时指导三个细分场景，通过自动化流程提取了包含精确时间戳、说话人身份及风格化特征（如停顿、笑声）的训练数据。

### 5. **实验效果**
*   **综合表现**：在 **Live Gaming Benchmark** 上，Proact-VL 在文本质量（LiveU, FinalQ）和主动响应时机（F1, TimeDiff）上均显著优于现有的商业闭源模型（GPT-4o, Gemini 2.5 Pro）及开源基线（VideoLLM-online, LiveCC）。
*   **时机控制**：在多人协同解说场景中，Proact-VL 展现了极高的触发准确率，能够有效避免与人类解说员抢话，同时在精彩时刻及时跟进。
*   **泛化能力**：在域外测试集（如《黑神话：悟空》）中，模型展现了强大的零样本泛化能力，能够准确识别未见过的游戏机制并提供高质量的解说与指导。
*   **实时性**：模型支持 10-15 FPS 的视频流稳定处理，满足实时交互需求。

## 📄 MUSE: A Run-Centric Platform for Multimodal Unified Safety Evaluation of Large Language Models

- **链接**: https://huggingface.co/papers/2603.02482
- **阅读来源**: HTML

1. **应用领域**：NLP/多模态 - 大模型安全评估与红队测试 (Multimodal LLM Safety Evaluation & Red-teaming)

2. **一句话核心贡献**：提出了 MUSE，这是一个开源的以“运行”为中心（Run-Centric）的平台，首次将跨模态载荷生成、多轮攻击编排和基于五级分类法的自动化评估统一在一个架构中，揭示了多轮跨模态交互可系统性绕过现有模型的安全对齐。

3. **使用指南**：
    *   **输入**：用户在浏览器前端选择攻击策略（如 Crescendo, PAIR, Violent Durian）或启用 ITMS（模态切换），设定恶意目标（如制造武器、恶意软件等），并选择目标模型（支持 OpenAI, Google, Anthropic, Qwen 等 API）。
    *   **流程**：系统自动将攻击者的文本提示转换为对应的音频（TTS）、图像（文本渲染）或视频载荷，并进行多轮交互管理。
    *   **输出**：完整的交互日志（包含每一轮的提示、回复、模态）、自动化 LLM 裁判给出的五级安全评级（从完全合规到直接拒绝），以及硬性/软性攻击成功率（ASR）统计。
    *   **部署**：代码开源，采用客户端-服务器架构，后端处理流式传输和 API 路由，无需特定硬件训练模型，但需配置各模型供应商的 API Key。

4. **主要创新点**：
    1.  **ITMS（Inter-Turn Modality Switching）机制**：引入了一种在多轮对话中每轮旋转模态（如文本、音频、图像交替）的攻击增强方法。实验证明，这种机制能通过破坏模型早期的防御稳定性，加速攻击收敛（即用更少的轮次实现越狱）。
    2.  **统一的跨模态多轮攻击平台**：解决了以往研究将“多轮文本攻击”与“单轮多模态攻击”割裂的问题，MUSE 是首个能同时协调攻击者 LLM、多模态转换管道（生成音/视/图）、目标模型路由和自动裁判的集成系统。
    3.  **精细化五级评估分类法**：摒弃了简单的二元（通过/失败）标准，采用五级分类（合规、部分合规、间接拒绝、直接拒绝、无响应）。通过 **Hard ASR**（仅包含完全合规）和 **Soft ASR**（包含部分合规）的双重指标，量化了二元指标容易忽略的“部分信息泄露”灰色地带。

5. **实验效果**：
    *   在对 **6 个多模态大模型**（包括 GPT-4o, Gemini 1.5/Pro, Claude 3.5 Sonnet, Qwen2.5/3-Omni）进行的约 **3700 次**红队测试中显示：
    *   尽管所有模型在单轮测试中表现出近乎完美（90-100%）的拒绝率，但在使用 MUSE 的多轮策略（如 Crescendo）攻击下，**攻击成功率（ASR）高达 90%–100%**。
    *   ITMS 机制被证明能显著加速攻击收敛；消融实验发现，非文本模态对安全性的影响具有**模型家族特异性**（例如，音频/图像输入削弱了 Gemini 的防御，但反而增强了 Qwen 的防御），强调了针对不同供应商进行感知模态测试的必要性。
