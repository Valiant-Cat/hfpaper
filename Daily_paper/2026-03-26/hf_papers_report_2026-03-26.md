## 📄 EVA: Efficient Reinforcement Learning for End-to-End Video Agent

- **链接**: https://huggingface.co/papers/2603.22918
- **阅读来源**: HTML

### 1. 应用领域
多模态大语言模型（MLLMs）- 长视频理解 / 强化学习 / 自主智能体（Agent）

### 2. 一句话核心贡献
提出了一种基于“先规划后感知（Planning-before-perception）”范式的端到端视频智能体强化学习框架（EVA），通过三阶段训练管道使模型能够自主决定视频的采样区间、帧数和分辨率，有效解决了长视频理解中的信息丢失与计算冗余问题。

### 3. 使用指南
*   **输入**：文本问题（Query）以及原始长视频文件。
*   **输出**：多轮动态推理过程（包含工具调用计划、反思思考）以及最终针对问题的准确文本答案。
*   **工作流程**：模型在不预先加载全部视频的情况下，首先根据问题进行推理并调用帧提取工具（可自定义时间范围、提取帧数 `nframes` 和图像缩放比例 `resize`），获取视觉反馈后评估信息是否充足，若不足则继续迭代探索，直至得出答案。
*   **硬件需求**：基础模型采用 Qwen2.5-VL-7B-Instruct，训练和推理需要显存充足的 GPU（如 A100/H800），由于其自适应 Token 节省机制，相比密集采样方法显著降低了对显存和计算流的峰值要求。
*   **数据与开源情况**：作者构建了三个高质量数据集（EVA-SFT, EVA-KTO, EVA-RL），旨在支持稳定和可复现的强化学习训练，通常此类模型及其对应的数据集会进行开源。

### 4. 主要创新点
1.  **“先规划后感知”的交互范式**：打破了传统视频模型或感知优先 Agent “先处理全量/均匀采样视频再推理”的固定工作流。EVA 采用“总结-计划-行动-反思”的迭代循环，让智能体在获取视觉输入前先基于文本进行策略规划，实现了细粒度的主动视觉探索（同时控制空间分辨率与时间跨度）。
2.  **简单有效的三阶段训练管道**：设计了逐步递进的训练策略：
    *   **SFT（监督微调冷启动）**：注入工具调用格式和基础的帧选择策略。
    *   **KTO（Kahneman-Tversky优化）**：利用失败与成功的轨迹数据进行单样本偏好对齐，纠正模型过早输出或无效采样的典型失败模式，稳定后续 RL 过程。
    *   **GRPO（广义奖励策略优化）**：引入数据增强的在线强化学习，让模型在探索中自我完善。
3.  **多格式混合奖励驱动机制**：在 GRPO 阶段，混合了选择题与开放式问答（QA）数据进行训练。选择题采用完整性自验证（CSV）奖励，开放式问题采用 ROUGE 分数奖励；同时设计了特定的“格式奖励（Format Reward）”惩罚随机猜测行为，强迫智能体必须在获取足够视觉证据后才能作答。

### 5. 实验效果
*   **极致的计算效率**：在采样困境基准（LSDBench）上，EVA 仅消耗约 6.2K 视觉 Tokens 即实现了 51.8% 的准确率，相较于耗费巨量 Token 的基线模型（如 Qwen2.5-VL 均匀采样）在极低算力开销下实现了 +2.6% 的性能反超。
*   **综合性能突破**：在四个长视频基准（LongVideoBench, MLVU, VideoMME, LVBench）和零样本推理基准（Video-Holmes）上，EVA 仅需每个视频提取平均 20-30 帧，其准确率便比通用 MLLM 基线高出 **6–12%**，比先前的自适应 Agent 方法进一步提升了 **1–3%**。
*   **幻觉削减**：在 ELV-Halluc 评测中，EVA 将语义聚合幻觉（SAH）比例从基线的 8.8% 显著降低至 5%，证明该框架极大增强了模型对视频时间序列与局部细节的精准定位能力。

## 📄 4DGS360: 360° Gaussian Reconstruction of Dynamic Objects from a Single Video

- **链接**: https://huggingface.co/papers/2603.21618
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 三维重建 / 动态场景重建 / 新视角合成 (Novel View Synthesis)

### 2. 一句话核心贡献
提出了一种无扩散模型（diffusion-free）的单目视频360°动态物体高斯重建框架 **4DGS360**，通过引入基于锚点的 3D 原生跟踪器（AnchorTAP3D）解决了遮挡区域的几何模糊问题，并构建了支持极端视角评估的新数据集 **iPhone360**。

### 3. 使用指南
*   **输入**：一段日常拍摄的单目动态物体视频（通常附带相机的内外参及初始深度图，例如使用iPhone捕获的数据）。
*   **输出**：动态场景的 4D 高斯表示（3DGS），支持在任意时间点、包括极端新视角（如物体的背面）下进行高保真的视频级渲染。
*   **硬件要求**：由于底层基于 3DGS 和 Transformer 跟踪模型，需要具备一定显存的 GPU 进行模型训练、初始化预测及可微渲染优化。
*   **代码开源情况**：文中未明确提及开源链接，但按照该领域学术惯例及建立新基准数据集（iPhone360）的属性，推测后续会开源代码与数据集。

### 4. 主要创新点
1.  **感知遮挡的 3D 原生跟踪与初始化 (AnchorTAP3D)**：提出了一种新型 3D 跟踪器 AnchorTAP3D。它将高置信度的 2D 跟踪点作为“锚点（Anchors）”输入给 3D Transformer 跟踪模型。这既利用了 2D 跟踪的鲁棒性抑制误差漂移，又发挥了 3D 模型的空间感知能力，成功推断出被遮挡区域的 3D 坐标，打破了以往过度依赖 2D 先验导致的几何塌陷局限。
2.  **鲁棒的 360° 4D 高斯优化框架**：在获得可靠的 3D 初始化后，通过随机采样生成动态高斯原语，结合分层运动表示（HiMoR）以及尽可能刚性（ARAP）正则化、深度一致性等损失函数进行联合优化，在不依赖计算昂贵的生成式先验（Diffusion priors）的情况下，实现了连贯的 360° 几何重建。
3.  **提出 iPhone360 基准数据集**：针对现有数据集（测试相机与训练相机视角差异小）无法真实评估 360° 重建性能的问题，构建了 iPhone360 数据集。该数据集在真实的单目手持拍摄设定下，将测试相机放置在与训练视角相差 70°~135° 的极端位置，填补了该领域的评估空白。

### 5. 实验效果
*   **核心数据集表现**：在自建的 **iPhone360** 极端视角数据集，以及现有的 **iPhone** 和 **DAVIS** 数据集上，定性与定量评估（LPIPS、CLIP-I、CLIP-T 等感知指标及像素级指标）均达到了当前的 **SOTA（最优）水平**。
*   **极端视角合成**：在常规视图和极端新视角（如完全不可见的背面）下，相比于严重依赖 2D 跟踪和面临深度图模糊的基线方法，4DGS360 展现出显著更强的几何一致性和形状保持能力，大幅降低了遮挡区域的伪影与几何破损。即使在快速运动的无真实深度的野外（In-the-wild）场景中，也能鲁棒地恢复被遮挡部分的结构。

## 📄 CarePilot: A Multi-Agent Framework for Long-Horizon Computer Task Automation in Healthcare

- **链接**: https://huggingface.co/papers/2603.24157
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型 (Multimodal LLMs) / 智能体 (AI Agents) / 医疗健康软件 GUI 自动化

### 2. 一句话核心贡献
提出了 **CarePilot**（一个结合视觉工具接地与双重记忆机制的 Actor-Critic 多智能体框架）以及首个长周期医疗软件交互基准 **CareFlow**，有效解决了视觉语言模型（VLMs）在复杂、高异构性医疗软件中难以完成多步长序列交互任务的问题。

### 3. 使用指南
*   **输入**：自然语言指令（任务目标）以及当前医疗软件的 GUI 界面截图。
*   **输出**：下一步的具体语义操作动作（如点击坐标、输入文本、滚动、标注区域等）。
*   **硬件要求**：训练阶段使用了 NVIDIA A100 (40GB) GPU 并结合 LoRA 进行高效微调；推理阶段由于采用了“推理蒸馏”技术剔除了 Critic 模块，仅需运行单一 Actor 模型（基于 Qwen-VL 等参数级模型），显著降低了运行开销。
*   **开源情况**：论文明确指出，该项目的代码和 CareFlow 评估数据集（包含1100个真实医疗软件交互任务）将开源，仅供非商业性学术研究使用。

### 4. 主要创新点
1.  **双重记忆机制 (Dual-Memory Mechanism)**：设计了短期记忆（Short-term Memory，记录最近一步的上下文、动作和反馈）与长期记忆（Long-term Memory，维护全局轨迹嵌入与总体进度反思），显著提升了长序列（8-24步）任务中的推理稳定性并缓解了错误累积。
2.  **视觉工具接地 (Visual Tool Grounding)**：为应对医疗软件复杂的专业界面，集成了 4 种轻量级感知模块（UI元素检测、文本/图标匹配、OCR识别、局部放大视图），为大模型提供精确的坐标与视觉特征反馈，极大提升了模型对医学图像和界面的感知准确度。
3.  **基于推理蒸馏的 Actor-Critic 架构**：在训练/模拟阶段，通过 Critic 评估 Actor 的动作并生成结构化修正反馈；随后通过“推理蒸馏（Reasoning Distillation）”使用监督微调（SFT）将高质量轨迹注入 Actor 中。在推理时，Actor 可以脱离 Critic 独立进行准确决策，既保留了反思纠错能力，又消除了多智能体交互的运行时延迟。

### 5. 实验效果
在涵盖 4 大核心医疗系统（DICOM 浏览器 Orthanc、图像计算工具 3D Slicer、电子病历 OpenEMR、医院管理 OpenHospital）的自建基准测试 **CareFlow** 上取得 SOTA（当前最佳）表现：
*   **任务完成率 (Task Accuracy, TA)**：基于 Qwen 3 VL 优化的 CarePilot 达到了 **48.76%**，大幅超越最强的闭源基线模型 GPT-5（36.19%）和同级别开源模型。
*   **步骤准确率 (Step-Wise Accuracy, SWA)**：达到了 **92.50%**，比 GPT-5（85.22%）高出 7 个百分点以上。
*   **泛化能力**：在分布外（OOD）的医疗软件测试集中，CarePilot 依然保持了 36.40% 的任务完成率，展现出强大的鲁棒性与跨软件泛化能力。

## 📄 Why Does Self-Distillation (Sometimes) Degrade the Reasoning Capability of LLMs?

- **链接**: https://huggingface.co/papers/2603.24472
- **阅读来源**: HTML

### 1. 应用领域
NLP-大模型微调（LLM Post-training）、强化学习（RLHF / RLVR）、自动数学推理。

### 2. 一句话核心贡献
揭示了自我蒸馏（Self-Distillation）在数学推理任务中导致大模型性能下降的根本原因：过度抑制了模型在推理过程中的“认知不确定性表达”（Epistemic Verbalization，如“Wait”、“Hmm”），从而严重损害了模型在复杂或分布外（OOD）任务上的泛化与纠错能力。

### 3. 使用指南
*   **输入**：数学推理问题或自然语言指令（Prompt）。
*   **输出**：包含逐步思考过程（Step-by-step reasoning / Chain-of-Thought）及最终答案的文本。
*   **应用建议**：在对大语言模型进行后训练（Post-training）时，若目标任务多样性高且需要强泛化能力（如广泛的数学推理），应避免使用包含“正确答案”等过度丰富的信息直接引导教师模型。应允许并保留模型在推理轨迹中的自我怀疑和探索性词汇（不确定性表达），而非盲目追求推理轨迹的简短与过度自信。
*   **硬件与代码**：依托于常规的大模型微调硬件（GPU集群）。论文实验基于开源的 `verl` 强化学习框架和 `math-verify` 评估工具，并使用了 Qwen3、DeepSeek-Distill-Qwen、OLMo-3 等开源模型权重。 

### 4. 主要创新点
1.  **提出“认知不确定性表达”的正面价值**：首次指出在大模型推理中，表达不确定性的标记（如“Wait”、“Actually”）并非冗余废话，而是模型维持替代假设、逐步减少不确定性并进行错误恢复的重要内部信号。
2.  **揭示教师模型信息量与学生模型推理风格的冲突**：通过信息论视角证实，自我蒸馏中教师模型由于获得了额外的高信息量上下文（如正确答案），会生成过于自信、简短且缺乏不确定性表达的推理轨迹；学生模型模仿这种风格会丧失自主探索和试错的能力。
3.  **发现“任务覆盖率”对蒸馏效果的决定性影响**：证明了在任务覆盖率低、题型单一的领域（如特定科学问答），自我蒸馏抑制不确定性可以提高效率并提升性能；但在任务覆盖率广、需强泛化能力的领域（如数学），保留不确定性表达是维持模型性能的关键。

### 5. 实验效果
在跨多个模型架构（DeepSeek-R1-Distill-Qwen-7B、Qwen3-8B 和 Olmo3-7B-Instruct）的实验中，对比了常规的 GRPO 与自我蒸馏（SDPO）策略。在分布外（OOD）的复杂数学基准测试（如 AIME24 和 AMC23）上，SDPO 虽然缩短了生成长度，但导致了**高达 40% 的性能下降**。例如在 AIME24 评测中，随着训练步数增加，SDPO 极大地减少了模型的不确定性 token 数量，其准确率显著低于基础模型和 GRPO 训练出的模型；而 GRPO 则通过增加不确定性表达维持并提升了推理表现。

## 📄 OmniWeaving: Towards Unified Video Generation with Free-form Composition and Reasoning

- **链接**: https://huggingface.co/papers/2603.24458
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 统一视频生成与多模态学习

### 2. 一句话核心贡献
提出了一种全能型统一视频生成模型 OmniWeaving，通过整合多模态大语言模型（MLLM）和扩散模型（MMDiT），解决了现有开源框架在处理自由形式多模态输入（文本、多图、视频）时缺乏复杂组合与抽象推理能力的问题。

### 3. 使用指南
*   **输入**：自由形式的交错多模态数据，包括单/多张图像、源视频以及复杂的文本提示（支持模糊意图、组合指令或抽象描述）。
*   **输出**：符合空间组合逻辑、时间连贯且语义高度对齐的高质量视频（论文主要聚焦 480p 分辨率）。
*   **硬件要求**：由于融合了多模态大模型（如基于 Qwen2.5-VL）和庞大的多模态扩散 Transformer（MMDiT），对算力要求极高。论文训练阶段使用了 400 张 NVIDIA H20 GPU，预计推理阶段也需要配备大显存的高端 GPU（如 A100/H100）。
*   **开源情况**：作者在论文中明确声明，项目代码和模型权重（Codes and model）即将向公众开源。

### 4. 主要创新点
1.  **带有“思考模式（Thinking Mode）”的统一理解与生成架构**：将 MLLM 从被动的特征提取器升级为主动的推理代理。在面对模糊或复杂的输入时，模型会先生成中间推理步骤以推导出精确的增强提示词，从而弥合抽象用户意图与像素级视频生成之间的认知鸿沟。
2.  **DeepStacking 多粒度特征注入机制**：在组合生成中，为了同时捕捉细粒度细节与高层抽象语义，模型从 MLLM 的不同深度层（如浅、中、深层）提取隐藏状态，通过 MLP 映射后直接注入到 MMDiT 的条件层中，显著增强了多主体和复杂场景下的视觉组合能力。
3.  **构建首个智能视频生成基准（IntelligentVBench）与大规模混合数据集**：构建了涵盖基础任务、多模态组合和推理增强三大类任务的大规模预训练数据集；同时推出了 IntelligentVBench，这是首个利用“VLM-as-a-Judge”范式，专门用于严苛评估视频模型多模态组合与抽象推理能力的全面测试基准。

### 5. 实验效果
*   **核心智能基准（IntelligentVBench）**：在涵盖推理与组合的四个高难度任务（隐式图生视频、插值双图生视频、组合多图生视频、图文视频编辑）中，OmniWeaving 均全面超越现有的开源统一模型（如 VACE、VINO、UniVideo），达到最新的 State-of-the-Art (SoTA) 性能。特别是在推理任务中，开启“思考模式”后性能获得了决定性的提升。
*   **传统基础基准**：在 VBench（文生视频基础测试）和 OpenVE-Bench（视频编辑测试）上，OmniWeaving 不仅超越了其他统一架构模型，其表现甚至匹敌或超越了专门针对单一任务优化的专家模型（如 HunyuanVideo 等），证明其在实现全能性的同时并未牺牲基础生成质量。

## 📄 T-MAP: Red-Teaming LLM Agents with Trajectory-aware Evolutionary Search

- **链接**: https://huggingface.co/papers/2603.22341
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大语言模型安全（LLM Safety）/ AI智能体红队测试（LLM Agent Red-Teaming）

### 2. 一句话核心贡献
提出了一种基于轨迹感知的进化搜索方法 T-MAP，通过融合执行轨迹反馈与工具调用图（TCG），自动生成能够绕过安全护栏并在真实的多步工具交互中执行有害操作的对抗提示，揭示了LLM智能体在复杂环境中的操作性安全漏洞。

### 3. 使用指南
*   **输入**：目标智能体的运行环境（如集成的 MCP 服务器及其可用工具列表）、风险类别配置与攻击风格参数。
*   **输出**：成功诱导目标智能体执行真实恶意行为的对抗性提示词（Adversarial Prompts），以及触发的有害多步工具执行轨迹。
*   **特殊硬件/要求**：主要依赖大模型 API 调用（如使用 DeepSeek-V3.2 作为攻击生成与分析器，另一 LLM 作为裁判）。由于涉及真实工具调用（如执行代码、操作文件系统、发送邮件），测试必须在**严格隔离的沙盒环境**中运行，以防对物理设备或真实用户造成不可逆的破坏。
*   **开源情况**：代码已开源（论文中明确声明 Code is available）。

### 4. 主要创新点
*   **轨迹感知的进化红队框架 (T-MAP)**：打破了传统仅依赖“文本回复是否违规”的红队测试局限。将多维度表型精英图（MAP-Elites）应用于智能体领域，利用真实的环境执行反馈（而不仅仅是文本输出）来指导提示词的进化，从而发现真正能落地执行的恶意动作。
*   **跨维度交叉诊断机制 (Cross-Diagnosis)**：引入 LLM 分析器，通过对比分析，从高分“父代”精英轨迹中提取成功经验，同时诊断当前“目标”单元的失败原因。这种双向反馈使得模型能继承有效的欺骗性叙事，并修正导致工具调用失败的缺陷。
*   **动态工具调用图 (Tool Call Graph, TCG) 引导**：在搜索过程中动态构建并维护一个记录工具到工具跳转结果（成功率、失败原因等元数据）的有向图。LLM 变异器可利用该图规避高失败率的调用路径，精准生成符合底层系统逻辑的复合型攻击链路。

### 5. 实验效果
*   **核心指标提升显著**：在五个多样的 MCP 环境（CodeExecutor, Slack, Gmail, Playwright, Filesystem）中，T-MAP 显著超越现有基线（如零样本、多轮精调等），平均**攻击实现率（ARR）达到 57.8%**。
*   **强泛化性与威胁性**：成功攻破了具备高级安全对齐策略的前沿模型（包括 GPT-5.2, Gemini-3-Pro, Qwen3.5, GLM-5），且发现的攻击轨迹具有良好的跨模型迁移能力（Pass@5 测试表现优异）。
*   **多维度攻击多样性**：在动作执行、语义和词汇三个维度上均达到了最高的多样性得分。在更为复杂的跨服务器调用（Multi-MCP chain）实验中，T-MAP 发掘出的跨域攻击轨迹比例高达 46.28%，远超基线的 14%~23%。

## 📄 When Models Judge Themselves: Unsupervised Self-Evolution for Multimodal Reasoning

- **链接**: https://huggingface.co/papers/2603.21289
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型微调 / 强化学习（RL）/ 无监督推理与自我进化

### 2. 一句话核心贡献
提出了一种用于多模态大模型推理的无监督自我进化框架，通过“Actor-Judge”联合建模与组内相对分布奖励（GRPO）机制，在完全不依赖人工标注标签或外部强奖励模型的情况下，有效防止了策略崩溃并显著提升了模型的数学与逻辑推理能力。

### 3. 使用指南
*   **输入**：无标签的多模态数据（包含图像-问题对，无需提供真实答案）。
*   **输出**：经过无监督强化学习更新后，推理能力和泛化能力得到增强的多模态大模型（策略模型）。
*   **使用流程**：给定一个输入，由 Actor 模型采样多条推理轨迹（形成初始自一致性分布），同时由相同结构但参数冻结的 Judge 模型进行打分并实施有界调制，最后通过 GRPO（组相对策略优化）算法计算组内相对优势并更新 Actor 的参数。
*   **硬件要求**：由于涉及大模型的在线多次采样（Rollout）以及强化学习训练，通常需要显存较大的多 GPU 集群计算节点（论文中基于 Qwen2.5-VL-7B 并在 Verl 框架上运行，计算开销约为常规监督 GRPO 的 1.4 倍）。
*   **开源情况**：代码已开源，地址为 https://github.com/OPPO-Mente-Lab/LLM-Self-Judge。

### 4. 主要创新点
1.  **Actor-Judge 联合自评价机制**：放弃了容易导致模型早期陷入局部极值的“多数投票（Majority Voting）”策略，从单一模型中实例化两个角色——Actor 负责生成多条推理轨迹建立自一致性分布，参数冻结的 Judge 负责对轨迹进行打分。
2.  **有界与连续的打分调制（Bounded Modulation）**：Judge 提供的是一个有界且连续可导的校准信号，用来重塑 Actor 的初始自一致性奖励分布，既避免了单纯依赖一致性带来的偏差，又防止了在无监督闭环中过度放大 Judge 的噪声。
3.  **组内分布式奖励建模（Group-wise Distributional Modeling）**：不对候选轨迹使用难以跨样本比较的绝对分数进行优化，而是引入基于能量的归一化（Energy-based scaling）和 Log-sum-exp 基线，将分数转化为“组内相对优势（Group-relative advantage）”，引导模型逐步实现概率质量的重新分配，有效防止了低熵输出和“回复长度坍塌（Response-length collapse）”。

### 5. 实验效果
*   **评估基准**：在 MathVision、MathVerse、LogicVista、GeoQA、Geo3K 等 5 个多模态数学/逻辑推理基准，以及 ChartQA、MMVP 等通用视觉基准上进行了评估。
*   **核心表现**：
    *   在完全无监督（纯无标签数据）设置下，该方法在各个数据集上均实现稳定的性能提升，并在 MathVision 基准上取得了最高 **+5.9** 的绝对准确率提升（从 25.0% 提升至 30.9%）。
    *   在多个基准测试上，该方法的表现稳定优于现有的无监督自我进化方法（如 VisionZero、多数投票自训练等），且媲美甚至在部分场景下超越了依赖强模型蒸馏和人工标注的有监督微调/强化学习方法。
    *   训练动态更加稳定，有效维持了策略模型的输出熵（Entropy），避免了传统多数投票方法容易引发的训练过早饱和问题。

3.  **物理增强的多视角视频生成器 (PE-MVGen)**：基于高容量扩散模型（Wan 2.1 DiT）进行架构适配，支持多视角联合生成。通过在上述异构数据集上使用整流流（Rectified Flows）进行协同训练，模型将轨迹物理修正与视频渲染解耦，使其在面对极端物理交互时也能保持极高的视觉保真度和结构一致性。

*   **消融收益**：异构数据协同训练使得模型在极具挑战的 CARLA 数据集上的 FVD 降低了约 13.4%（从 89.83 降至 77.83），并且在人类偏好评估（Human Preference）中获得了压倒性的胜率。
*   **OOD 泛化压力测试**：在真实世界场景（nuScenes）中人为构造超出分布的碰撞轨迹测试时，该模型依然能大幅减少环境与车辆间的穿透伪影，展现出极强的鲁棒性。

## 📄 UniFunc3D: Unified Active Spatial-Temporal Grounding for 3D Functionality Segmentation

- **链接**: https://huggingface.co/papers/2603.23478
- **阅读来源**: HTML

### 1. 应用领域
3D计算机视觉 - 3D场景理解与功能性分割 (3D Functionality Segmentation) / 具身智能 (Embodied AI)。

### 2. 一句话核心贡献
提出免训练的统一多模态框架 UniFunc3D，将语义、时间与空间推理整合至单一前向传播中，通过主动时空定位解决了基于自然语言隐式指令的3D细粒度可交互部件分割问题，彻底消除了现有割裂式管线导致的“视觉盲区”与级联错误。

### 3. 使用指南
*   **输入**：3D场景的多视角RGB视频帧集合 + 自然语言任务描述（例如：“打开柜子顶部放着化妆品的左侧抽屉”）。
*   **输出**：目标功能性/可交互部件（如把手、开关）的精确 3D 点云掩码（3D Mask）。
*   **依赖与硬件**：基于多模态大语言模型（如 Qwen3-VL 8B/30B、Qwen2.5-VL-72B）与 SAM3，无需针对特定3D任务进行微调训练。由于涉及大模型推理与多视角图像处理，需要大显存 GPU（如论文实验所用的 NVIDIA H100）支持。
*   **开源情况**：作者在论文中明确表示代码将在项目主页开源。

### 4. 主要创新点
1.  **统一的多模态视觉推理架构 (Unified Multimodal Architecture)**：摒弃了此前“纯文本LLM解析任务 + 独立VLM感知”的割裂式管线，直接使用单一 MLLM 联合完成语义识别（是什么目标）、时间推理（出现在哪一帧）和空间定位（具体像素点坐标），防止错误跨模块级联传播。
2.  **主动时空定位机制 (Active Spatial-Temporal Grounding)**：放弃了传统方法中依赖物体检测置信度的被动启发式帧选择，转而让 MLLM 充当“主动观察者”，通过多重偏移时间采样（Multi-sampling）直接基于视觉证据自主寻找包含目标对象的最具信息量的视频帧，解决了空间上下文不一致的问题。
3.  **仿人类的“由粗到细”感知策略 (Human-like Coarse-to-fine Perception)**：
    *   **粗粒度阶段**：在低分辨率下全局巡视视频帧，识别候选功能对象并生成初始定位锚点。
    *   **细粒度阶段**：在选定候选帧附近提取密集时间窗口，并在**原生高分辨率**下处理整个帧。这种机制实现了“无损放大（Zoom-in）”以精确定位微小部件，同时保留了全局场景上下文用于消歧；最后利用 MLLM 对 SAM3 生成的 2D 掩码进行直观的视觉验证（Overlay Verification），极大地减少了过度分割。

### 5. 实验效果
在 3D 功能性分割权威数据集 **SceneFun3D** 上取得了最新的 State-of-the-Art (SOTA) 表现：
*   **精度大幅提升**：作为完全免训练（Training-free）的方法，相比之前的最佳免训练基线（Fun3DU），mIoU 取得了 **59.9%** 的相对提升（绝对提升 9.1 个点），AP@0.25 提升了 53.2%。
*   **超越训练基线**：在模型参数更小（30B vs 72B）且没有使用任何场景真实3D标注进行微调的情况下，大幅击败了专门在此任务上训练（如训练了 1000 epoch 的 AffordBot）的先进方法（mIoU 相对提升 68.5%）。
*   **推理效率极高**：得益于统一前向传播与高效的时间窗口策略，单场景平均推理时间相比 Fun3DU 缩短了约 **3.2 倍**（仅需约26分钟/场景），实现了精度与速度的双重突破。

### 1. 应用领域

### 3. 使用指南
*   **输入**：文本序列（Token 序列 / Prompt）。
*   **输出**：预测生成的下一个 Token 序列（归纳输出），以及反映全局特征的模型内部“演绎输出 (Deductive Outputs)”张量集合（如密度矩阵、度量张量、电势张量、能量-曲率张量）。
*   **硬件需求**：相对较低。论文证明通过小参数模型（110M）即可观察到上述现象。实验环境使用 2 张 RTX 4090 GPU 进行预训练，单张 RTX 4090 GPU 进行推理。
*   **开源情况**：代码和模型已完全开源。提供支持多 GPU 训练和推理的 PyTorch 实现，预训练模型及自定义代码已适配 Huggingface Transformers 库（Github 仓库为 `burcgokden/PLDR-LLM-Self-Organized-Criticality`）。

### 4. 主要创新点
1.  **通过“自组织临界性”与相变理论解释推理涌现**：首次证明了基于幂律解码器表示的大模型 (PLDR-LLM) 在正确的预热步数和最大学习率组合下，会在预训练中达到自组织临界状态。推理能力的产生类似于物理学中的二级相变，模型通过学习等效于标度函数和重整化群的表示，获得了极强的长程相关性和泛化能力。
2.  **定义无需 Benchmark 测试集的“序参量”评估指标**：提出利用推理时不同随机采样生成下，模型全局演绎输出参数的归一化均方根误差 (Normalized RMSE) 作为“序参量”。序参量越接近于 0，表示模型越处于临界稳态，其推理能力越强。这提供了一种纯内源性的 LLM 评估手段。
3.  **发现模型内部注意力的“亚稳态稳态”物理特征**：揭示了具备推理能力的模型，其内部高度非线性的演绎输出张量在推理阶段表现出亚稳态稳态（Metastable steady state），面对未见过的输入仅发生可忽略的微小扰动。这一发现从底层机制解释了为何 Scaling Law（增加数据和参数）以及更深的网络结构能提升大模型表现。

### 5. 实验效果
*   **内在指标一致性**：在近临界态 (Near-criticality) 下训练的模型，其生成的文本在语义和语法上高度准确，且内部的“序参量”趋近于零；而亚临界态 (Sub-criticality) 下的模型则仅生成随机 Token，序参量成数量级增大。
*   **核心数据集表现**：在 ARC、Hellaswag、OpenBookQA、TruthfulQA 等零样本常识推理和问答基准测试上，模型的基准得分与其序参量大小呈现完美反比关系。
*   **越级对比**：参数量仅为 1.1亿 (110M) 并在近临界条件下训练的 `PLDRv51-SOC-110M-5` 模型，在上述 Benchmark 的平均得分超越了具有相似体量的传统基准模型 `GPT-Neo-125M`，证明了该架构与临界训练方法的高效性。

## 📄 UI-Voyager: A Self-Evolving GUI Agent Learning via Failed Experience

- **链接**: https://huggingface.co/papers/2603.24533
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型 - GUI智能体 (Multimodal Large Language Models - GUI Agents) / 强化学习微调

### 2. 一句话核心贡献
提出了一种包含拒绝微调（RFT）和组相对自蒸馏（GRSD）的两阶段自进化训练框架，有效解决了长视野移动端GUI任务中奖励稀疏导致的信用分配模糊，以及失败经验利用率低的问题。

### 3. 使用指南
*   **输入**：自然语言形式的任务指令（如“打开设置并关闭蓝牙”）以及设备当前屏幕的实时视觉截图（含历史操作上下文）。
*   **输出**：具体的GUI交互动作指令（如点击特定坐标、滑动、输入文本、等待等）。
*   **硬件需求**：该方法在仅4B参数的模型（基于Qwen3-VL-4B）上即实现了SOTA性能，相较于百亿/千亿参数的大模型，其训练和推理的显存占用及算力需求大幅降低，常规单卡或少卡GPU服务器即可运行。

### 4. 主要创新点
*   **组相对自蒸馏机制 (GRSD, Group Relative Self-Distillation)**：突破了传统强化学习（如PPO/GRPO）在长序列GUI任务中轨迹级奖励稀疏的瓶颈。该机制通过从自身生成的同组成功轨迹中提取精确的“词元级（token-level）”监督信号，直接教导失败的轨迹，将稀疏奖励转化为密集的步级监督。
*   **无依赖的分叉点检测 (Fork Point Detection)**：提出一种基于图像结构相似度（SSIM）的轻量化帧匹配算法。无需借助外部视觉大模型或人工标注，即可自动对比成功与失败轨迹，精准定位两者视觉状态相同但决策产生分歧的“关键分叉点”，进而实现错误动作的靶向纠正。
*   **零人工干预的完全自进化循环 (Self-Evolving Loop)**：通过种子任务生成器和基于规则的验证器，设计了拒绝微调（RFT）模块。智能体自主探索环境、生成轨迹、过滤失败样本并保留高质量成功样本进行微调，实现了“数据合成”与“模型能力”的自动协同进化。

### 5. 实验效果
在主流移动端设备控制基准 **AndroidWorld**（包含116种不同复杂度的真实App任务）上进行了充分测试：
*   **极致性能**：UI-Voyager (4B模型) 取得了 **81.0%** 的 Pass@1 成功率，不仅全面超越了规模更大的基线模型（如235B的MAI-UI和UI-Tars-2），更**突破了报告中的人类平均操作水平（80.0%）**。
*   **学习效率**：实验证明，相较于直接使用PPO或GRPO等标准强化学习算法（最终性能停滞在76%左右且收敛极慢），引入GRSD后模型能迅速从失败中学习，成功率大幅跃升至81%，在极难任务（低成功率任务）上的提升尤为显著。

## 📄 Unleashing Spatial Reasoning in Multimodal Large Language Models via Textual Representation Guided Reasoning

- **链接**: https://huggingface.co/papers/2603.23404
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型 (Multimodal LLMs) - 视频理解与 3D 空间推理 (Video Understanding & 3D Spatial Reasoning)

### 2. 一句话核心贡献
提出了一种名为 TRACE 的纯提示工程方法，通过引导多模态大模型从第一人称视频中显式生成结构化的 3D 空间文本表征（包含坐标系、相机轨迹和实体信息）作为中间推理步骤，显著提升了模型在复杂场景下的 3D 空间推理和问答能力。

### 3. 使用指南
*   **输入**：一段第一人称室内视角的视频（Egocentric Video），以及一个空间推理问题（如距离估计、相对位置判断等）。
*   **输出**：符合 YAML 格式规范的中间文本表征（包含全局坐标系、相机历史轨迹、详细的实体登记表），以及最终的问答结果（选项或具体数值）。
*   **使用方式**：无需额外训练或微调模型。只需使用论文附录中提供的系统提示词（Prompt Template），采用单次推理（One-stage prompting）将生成文本表征与最终回答结合在一次对话生成中完成。
*   **硬件与适用性**：无需任何额外的 3D 或几何专用硬件/模块；直接适用于现成且具备较强指令遵循能力的多模态大模型（如 Gemini 3 Pro, Qwen2.5-VL-72B, MiMo 等）。
*   **代码/提示词开源**：论文附录已提供完整的系统角色设定和 TRACE 提示词模板。

### 4. 主要创新点
1. **基于全局视角（Allocentric）的文本表征转换**：受人类空间认知理论启发，突破了现有模型过度依赖 2D 像素信号和隐式特征的局限，强制模型从第一人称视角中构建并输出全局视角的 3D 空间文本描述。
2. **结构化的 3D 场景要素设计（TRACE 核心组件）**：定义了一套结构化解析框架，要求模型输出三大要素：(1) 对齐房间物理布局的全局坐标系（Meta Context）；(2) 离散化的相机运动轨迹（包含时间戳、位置估算和朝向）；(3) 实体登记表（包含每个独立物体的外观、估算 3D 坐标和拓扑空间关系）。
3. **空间感知的专用思维链（Spatial CoT）**：将 TRACE 表征作为模型输出最终答案前的一层“空间缓存（Spatial Cache）”，通过显式构建和验证几何约束条件，克服了标准思维链（Standard CoT）和树状思维（ToT）在复杂空间推理任务中容易产生幻觉或性能下降的问题。

### 5. 实验效果
*   **核心数据集表现**：在两大视频空间智能测试基准 **VSI-Bench** 和 **OST-Bench** 上进行了广泛评估。
*   **全面超越基线**：相比于直接回答（Direct）以及标准的 CoT、ToT、LtM 等高级提示策略，TRACE 在不同规模和架构的模型上均实现了稳定的最优（SOTA）表现。具体而言，在 VSI-Bench 上，使 Gemini 3 Pro 准确率提升 **+7.54%**，Qwen2.5-VL 提升 **+3.10%**，MiMo 提升 **+1.63%**。
*   **优于其他文本表征**：对比已有的认知地图（Cognitive Map）文本表征方法，TRACE 在 VSI-Bench 上取得了 **+9.7%** 的显著领先，证明其捕获了更细粒度的 3D 结构信息。
*   **泛化性强**：实验证实该方法没有偏向特定的室内环境分布，在 ARKitScenes、ScanNet 等不同复杂度、不同布局的场景数据子集中均展现出稳健的性能提升。

## 📄 StreamingClaw Technical Report

- **阅读来源**: HTML

### 1. 应用领域
多模态大模型与智能体 (Multimodal LLMs & Agents)、具身智能 (Embodied AI)、实时流视频理解 (Real-time Streaming Video Understanding)、自动驾驶与智能座舱。

### 2. 一句话核心贡献
提出了一种面向实时流视频理解与具身智能的统一多智能体框架 **StreamingClaw**，系统性打通了低延迟在线感知、多模态长时记忆、主动交互以及“感知-决策-行动”闭环控制能力。

### 3. 使用指南
*   **输入格式**：来自多端设备（手持设备、车辆、智能眼镜、具身机器人）的时间戳对齐的多模态连续流数据（视频帧、音频），以及用户的文本或语音查询指令。
*   **输出格式**：低延迟的实时文本/语音问答响应、主动提醒/警告（Proactive alerts），以及结构化的工具和技能调用指令（直接转化为物理世界的执行动作）。

*   **开源与兼容性**：该框架兼容 OpenClaw 生态及主流开源多模态大模型（MLLMs），支持广泛的社区开源工具与技能库集成。

### 4. 主要创新点
1.  **低延迟流式推理与主从协同调度 (StreamingReasoning)**：
    设计了主智能体来处理帧级流式推理。通过复用流式 KV-Cache 并动态淘汰低重要性的视觉 Token，实现了将随时间增长的全量重算转化为线性增量更新；同时具备自我规划能力，可自适应地将任务分发给记忆或主动交互子智能体。
2.  **多模态层级演化记忆机制 (StreamingMemory)**：
    摒弃了传统的纯文本上下文堆叠，提出层级记忆演化（HME）机制。通过在线更新策略，将碎片化的视频流连续抽象演化为“片段 (segments) -> 原子动作 (atomic actions) -> 事件 (events)”。配合指令驱动的高并发、时序显著性检索策略，解决了长时记忆的信息冗余和检索效率低下问题。
3.  **视觉信号驱动的主动交互范式 (StreamingProactivity)**：
    使智能体从被动回答转为主动在线监测。支持“时间感知（如定时提醒）”和“事件锚定（如动作触发）”两类场景。创新性地提出了基于视觉语言信号驱动的训练方案，将视频状态变化映射为词表中的特定“触发 Token (trigger tokens)”，实现高精度的并发事件识别与主动干预（如疲劳驾驶警告、跌倒呼救）。

### 5. 实验效果
*(注：由于提供的截断文本主要侧重于技术架构与方法论，未包含具体基准测试的定量数据表格，以下效果基于文中阐述的技术表现总结)*
*   **计算与延迟**：在长时连续流视频场景下，计算模式实现了从“历史全量重算”到“增量更新”的转变，保持了稳定的高吞吐量和可控的极低延迟，达成“边看边答/边听边答”的实时体验。
