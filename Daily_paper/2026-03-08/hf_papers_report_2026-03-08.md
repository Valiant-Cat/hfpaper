## 📄 RealWonder: Real-Time Physical Action-Conditioned Video Generation

- **链接**: https://huggingface.co/papers/2603.05449
- **阅读来源**: HTML

1. **应用领域**：计算机视觉 - 视频生成 / 物理模拟 (Computer Vision - Video Generation / Physics Simulation)

2. **一句话核心贡献**：提出了首个基于单张图像的实时物理动作条件视频生成系统，通过引入物理模拟作为中间表征桥梁，实现了对力、力场和机器人操作等连续3D物理动作的逼真视频预测与交互。

3. **使用指南**：
    *   **输入**：单张静态 RGB 图像 + 3D 物理动作序列（包括外力、力矩、机器人末端执行器指令或相机位姿）。
    *   **输出**：反映物理交互后果的实时视频流（分辨率 480x832）。
    *   **硬件要求**：推理阶段在单张 NVIDIA H200 GPU 上可实现实时生成（约 13.2 FPS）；训练阶段使用了 A100 GPU 集群。
    *   **开源情况**：论文声明代码和模型权重已公开可用。

4. **主要创新点**：
    1.  **物理模拟作为中间桥梁架构**：利用物理模拟器将连续、无界的 3D 物理动作转化为视频模型易于处理的视觉信号（光流和粗糙 RGB 预览），巧妙避开了动作 Token 化困难和缺乏“动作-视频”配对训练数据的难题。
    2.  **流式光流条件蒸馏技术**：设计了一种蒸馏方案，将基础视频模型转化为仅需 4 步扩散的因果视频生成器；通过 LoRA 注入光流控制并结合 Distribution Matching Distillation (DMD)，大幅降低了计算开销并保证了生成质量。
    3.  **多材质实时交互系统**：系统集成了从单图 3D 重建、支持多种材质（刚体、流体、颗粒、布料等）的物理求解器到视频生成的全流程，实现了亚 100ms 的帧延迟，支持用户实时干预物理场景。

5. **实验效果**：
    *   **生成速度**：在 480x832 分辨率下，系统在 H200 GPU 上达到了 **13.2 FPS** 的生成帧率，实现了真正的实时交互。
    *   **用户偏好**：在包含 30 个不同材质场景（如沙子、水、烟雾、弹性体）的测试集中，通过 400 人参与的 2AFC（二选一强制选择）用户研究，RealWonder 在“动作遵循度”、“运动合理性”和“视觉质量”三个维度上均显著优于现有基线模型（如 CogVideoX-I2V, Tora, PhysGaussian）。
    *   **物理一致性**：实验表明模型能够正确响应风力、碰撞和机器人抓取等复杂物理动作，且生成的视频在保持物理规律的同时具备动态阴影和纹理细节等视觉真实感。

## 📄 KARL: Knowledge Agents via Reinforcement Learning

- **链接**: https://huggingface.co/papers/2603.05218
- **阅读来源**: HTML

1. **应用领域**：
   NLP-智能体（LLM Agents）、检索增强生成（RAG）、大模型强化学习（RL for LLMs）、企业级知识搜索。

2. **一句话核心贡献**：
   提出了一种名为 KARL 的系统，通过基于多任务强化学习（OAPL）和自举式合成数据生成的训练流程，构建了在成本、延迟和质量上达到 Pareto 最优，且具备强大跨域泛化能力的“有据推理”（Grounded Reasoning）知识智能体。

3. **使用指南**：
   *   **输入**：自然语言提问（Query）以及一个封闭的文档语料库（如企业内部文档、技术手册、财务报告等）。
   *   **流程**：
       1.  **数据合成**：使用代理（Agent）利用向量搜索工具自主探索语料库，生成高质量、有依据的问答对（QA pairs）及推理轨迹。
       2.  **训练**：使用 OAPL 算法对基础模型（如 GLM 4.5 Air）进行迭代式离线强化学习训练。
       3.  **推理**：模型配备向量搜索工具和上下文压缩机制，通过“并行思考”（Parallel Thinking）或“价值导向搜索”（Value-Guided Search）生成最终答案。
   *   **输出**：基于检索证据生成的精准答案（包括实体、数值计算结果或综合报告），并附带引用来源。
   *   **硬件需求**：训练和推理依赖高性能 GPU（文中实验使用了 H200 节点进行大规模并行 rollout 生成和训练）。

4. **主要创新点**：
   *   **迭代式大批量离线强化学习（OAPL）**：提出了一种新的后训练范式“OAPL”（Optimal Advantage-based Policy Optimization with Lagged Inference），利用大批量离线数据进行迭代更新。相比在线 RL（如 GRPO），它样本效率更高，无需剪裁重要性采样权重或重播缓冲区等复杂启发式方法，且对训练/推理引擎差异具有鲁棒性。
   *   **自举式代理数据合成管道**：设计了一套全自动的数据生成流程，智能体通过向量搜索主动探索语料库来合成训练数据（包括问题、答案和推理轨迹），并通过“去重代理”和“质量过滤代理”清洗数据。该过程支持自举（Self-improvement），即用更强的模型生成更难的数据。
   *   **多任务 RL 带来的分布外泛化**：证明了在异构搜索行为（如深度搜索 BrowseComp-Plus 和广度搜索 TREC-Biogen）上进行联合训练，不仅能提升任务内表现，还能显著提高模型在未见过的搜索任务（如表格推理、过程推理）上的分布外（OOD）泛化能力，这一点优于传统的监督微调（SFT）蒸馏。

5. **实验效果**：
   *   **综合性能**：在包含 6 个不同搜索领域的 **KARLBench** 评测集上，KARL 在成本-质量和延迟-质量权衡上均优于 Claude 4.6 系列和 GPT 5.2。
   *   **对比 SOTA**：在不使用测试时算力（Test-time compute）的情况下，KARL 达到了与 Claude Sonnet 4.5 相当的质量；在使用 10 个并行 rollout 的情况下，其性能匹配最强闭源模型 Claude Opus 4.6，同时成本降低约 33%，延迟降低约 47%。
   *   **能力提升**：相比基础模型 GLM 4.5 Air，KARL 在 TREC-Biogen（得分从 66.0 提升至 85.0）和 BrowseComp-Plus 上均有显著提升，并且成功解决了基础模型无法通过长视距搜索解决的复杂推理问题。

## 📄 SageBwd: A Trainable Low-bit Attention

- **链接**: https://huggingface.co/papers/2603.02170
- **阅读来源**: HTML

# SageBwd: A Trainable Low-bit Attention 论文研究报告

### 1. 应用领域
**NLP/CV - 大模型高效预训练与微调**（特别是针对Transformer架构的生成式模型，如Llama系列及视频生成模型）。

### 2. 一句话核心贡献
本文深入探究了低比特（INT8）注意力机制在预训练中性能下降的根源，通过理论分析与实验证明了**QK-Norm**的必要性以及**Tokens-per-step (TPS)** 对量化噪声的影响，实现了首个在预训练阶段能达到全精度注意力（FPA）性能的可训练INT8注意力机制。

### 3. 使用指南
*   **输入与输出**：输入为查询（Q）、键（K）、值（V）张量；输出为经过注意力计算的结果及其反向传播梯度。
*   **配置方法**：
    1.  在模型架构中替换标准的FlashAttention或FPA为SageBwd模块。
    2.  **必须开启 K-smoothing**（对K进行平滑处理）。
    3.  **建议开启 QK-Norm**（特别是在大Batch Size/高TPS训练场景下），以压缩动态范围并稳定Softmax梯度。
    4.  Q-smoothing在预训练阶段收益有限，可选择性关闭以节省开销。
*   **硬件要求**：需要支持INT8 Tensor Core计算的GPU（文中实验使用了NVIDIA B200和RTX4090）。
*   **量化策略**：该方法将注意力机制中7个矩阵乘法中的6个进行INT8量化，仅保留关键部分的高精度，以平衡速度与精度。

### 4. 主要创新点
1.  **量化误差根源定位**：理论推导并经验证发现，反向传播中的**分数梯度 $\mathbf{dS}$**（Attention Score Gradient）是量化误差的主要来源。由于其幅值极小且经过链式法则放大，在低精度下信噪比极低，导致训练不稳定。
2.  **QK-Norm 的训练稳定性分析**：揭示了QK-Norm不仅有助于Softmax数值稳定，更关键的是它通过限制输入动态范围，显著降低了INT8量化误差，是在大Batch Size（Large TPS）下实现稳定预训练的必要条件。
3.  **TPS 与量化噪声的交互机制**：发现较小的Tokens-per-step（即较小的Batch Size）产生的梯度噪声能掩盖INT8的量化误差，使得模型更易收敛；而在大TPS下，量化偏差主导优化轨迹，必须依赖QK-Norm进行矫正。

### 5. 实验效果
在 **325M 参数量的 Llama 模型** 上进行了 **78B token** 的预训练实验，主要结果如下：
*   **性能对齐**：在较小的 TPS (260K) 下，SageBwd 的最终 Loss (**2.561**) 与全精度注意力 (**2.563**) 基本持平，实现了无损训练。
*   **大 Batch 稳定性**：在较大的 TPS (2.1M) 下，未加 QK-Norm 的低比特训练 Loss 明显劣化 (2.640 vs FPA 2.586)，而引入 QK-Norm 后成功恢复了训练稳定性。
*   **消融实验**：验证了仅使用 K-smoothing 即可满足训练需求，而 Q-smoothing 在预训练中未带来持续收益，甚至可能因梯度修正引入额外噪声。

## 📄 DARE: Aligning LLM Agents with the R Statistical Ecosystem via Distribution-Aware Retrieval

- **链接**: https://huggingface.co/papers/2603.04743
- **阅读来源**: HTML

### 1. 应用领域
**NLP - 大语言模型智能体 (LLM Agents) / 数据科学 (Data Science)**
具体针对自动化统计分析、R 语言编程辅助以及检索增强生成 (RAG) 在专业科学领域的应用。

### 2. 一句话核心贡献
提出了一种名为 DARE 的分布感知检索嵌入模型，通过将数据分布特征（如稀疏性、维度、分布假设）融入检索过程，解决了通用 LLM 难以检索和正确使用 R 语言生态系统中严谨统计工具的问题。

### 3. 使用指南
*   **输入**：
    1.  用户的自然语言查询（描述分析意图）。
    2.  数据概况元数据（Data Profile，包含数据模态、特征类型、分布假设、缺失值情况等，可由 LLM 从数据中推断）。
*   **输出**：排序后的 R 语言函数列表及其详细文档（包含参数说明和使用示例）。
*   **使用方式**：
    *   作为“插件”集成到 LLM Agent（如论文中的 RCodingAgent）中。
    *   Agent 首先调用 DARE 检索合适的统计工具，然后利用检索到的文档生成可执行的 R 代码。
*   **资源情况**：论文构建了包含 8,191 个高质量 R 函数的知识库 (RPKB) 和相应的 DARE 模型权重，并提供了评估基准。
*   **硬件要求**：模型非常轻量（仅 23M 参数），推理高效（单次查询仅需 3.7ms），普通 GPU 即可支持高并发使用。

### 4. 主要创新点
1.  **分布感知检索嵌入 (DARE)**：设计了一种轻量级的双编码器检索模型，显式地将“数据分布约束”与“函数语义”融合。这使得模型能区分语义相似但适用数据场景不同的函数（例如区分适用于正态分布与非正态分布的检验函数），这是传统语义检索无法做到的。
2.  **构建 R 语言包知识库 (RPKB)**：从 CRAN 的 8000 多个 R 包中清洗、筛选并构建了结构化知识库。利用 LLM 从非结构化文档中提取了关键的统计元数据（如输入数据要求），为工具学习提供了高质量语料。
3.  **RCodingAgent 架构与评估基准**：提出了一个端到端的 R 语言统计分析 Agent 框架，并设计了包含 16 个代表性统计任务（如假设检验、生存分析、混合效应模型等）的评测集，用于系统评估 Agent 在真实数据分析场景下的代码生成与执行能力。

### 5. 实验效果
*   **检索性能**：在 RPKB 测试集上，DARE 实现了 **93.47% 的 NDCG@10** 和 **87.39% 的 Recall@1**。相比于参数量大 15-25 倍的最先进通用 Embedding 模型（如 BGE-Large, Jina），DARE 的检索性能提升高达 **17%**。
*   **下游任务提升**：在 16 个统计分析任务中，集成 DARE 显著提升了各类 LLM 的成功率。例如，**GPT-4o 的任务成功率从 18.75% 飙升至 75.00%**（提升 56.25%），Llama-3-8B 的成功率从 0% 提升至 50.00%。
*   **效率表现**：DARE 具有极高的推理效率，吞吐量达到 **8,512 QPS**，延迟仅为 3.7ms，比通用大模型快 3-4 倍，极适合实时交互系统。

## 📄 Lightweight Visual Reasoning for Socially-Aware Robots

- **链接**: https://huggingface.co/papers/2603.03942
- **阅读来源**: HTML

**1. 应用领域**
机器人学（Robotics）、视觉语言模型（VLMs）、人机交互（HRI）。

**2. 一句话核心贡献**
提出了一种轻量级的语言到视觉反馈模块，通过建立从大语言模型（LLM）到视觉编码器的闭环回路，使机器人能够利用文本上下文重新解释视觉信息，显著提升了在社交场景下的导航、场景理解和人类意图识别能力。

**3. 使用指南**
*   **输入**：RGB 图像序列和自然语言提示（User Query，如导航指令或关于场景的提问）。
*   **输出**：基于上下文的视觉推理结果，包括机器人导航动作（JSON格式）、场景描述或人类意图分类。
*   **使用流程**：
    1.  **第一遍前向传播**：输入图像和文本，模型生成基于文本的视觉推理“提示”（Hidden States）。
    2.  **反馈注入**：通过一个轻量级 MLP 模块将上述推理提示投射回视觉编码器的输入空间，叠加到原始图像嵌入上。
    3.  **第二遍前向传播**：将原始图像、经过推理增强的新图像嵌入以及文本提示共同输入模型，生成最终回答。
*   **硬件要求**：支持单张消费级 GPU（如 NVIDIA RTX 3090，显存占用增加 <3%）。
*   **代码开源**：已开源 (https://github.com/alessioGalatolo/VLM-Reasoning-for-Robotics)。

**4. 主要创新点**
1.  **跨模态反馈闭环机制**：打破了传统 VLM 视觉单向流向语言的架构，设计了从语言模型末端回传至视觉编码器前端的反馈通路，允许模型根据语言理解动态地“重新审视”视觉场景。
2.  **极低参数量的轻量化设计**：引入的视觉推理模块（Gated MLP）和 LoRA 层仅增加了不到 3% 的总参数量，且无需对原本庞大的视觉编码器进行微调，保证了训练的高效性和对不同 VLM 基座（如 Qwen, Gemma, LLaVA）的兼容性。
3.  **针对社交机器人的意图数据集**：构建并标注了一个专门针对人类-机器人交互（HRI）的新数据集，包含不同紧急程度下人类打断机器人对话的真实行为视频，填补了现有数据集缺乏微小社交线索（Social Cues）和意图标注的空白。

**5. 实验效果**
在三个以机器人为核心的任务中对比了 Qwen 2.5 (7B)、Gemma 3 (4B) 和 LLaVA OV 1.5 (4B) 基座模型：
*   **人类意图识别（HRI Dataset）**：在使用 Qwen 7B 作为基座时，完整方法的准确率达到 **36.97%**，显著优于去除反馈模块或改变输入顺序的变体（如 Prompt-first 仅为 25.53%）。
*   **场景描述（Mementos-Robotics）**：在所有测试模型上，开放式场景描述的质量评分均有提升，表明模型对动态环境的理解能力增强。
*   **模拟导航（Habitat）**：在 Qwen 7B 上，导航任务的“最终目标距离”指标明显降低（性能提升）；但在较小的模型（Gemma/LLaVA）上效果混合，主要受限于小模型生成特定 JSON 格式指令的能力。

## 📄 Locality-Attending Vision Transformer

- **链接**: https://huggingface.co/papers/2603.04892
- **阅读来源**: HTML

1. **应用领域**：计算机视觉 - 语义分割与基础模型骨干网络设计 (Computer Vision - Semantic Segmentation & Backbone Architecture)。

2. **一句话核心贡献**：提出了一种名为 LocAt 的轻量级插件，通过引入可学习的高斯核注意力和补丁表示细化机制，解决了 Vision Transformer 在分类训练中丢失局部细节的问题，在不改变训练范式的前提下显著提升了密集预测（如分割）性能。

3. **使用指南**：
    *   **输入**：标准的图像数据（被处理为 Patch Token 序列）。
    *   **输出**：用于密集预测的高质量空间特征图，或图像分类标签。
    *   **集成方法**：该方法作为附加组件集成到现有的 ViT 架构中：
        1.  在自注意力层中加入 **GAug** 模块（修改 Attention Logits）。
        2.  在分类头之前加入 **PRR** 模块（替代全局平均池化 GAP 或仅用 [CLS] Token）。
    *   **硬件需求**：通用 GPU（实验使用了 NVIDIA RTX A6000/V100/A100），计算开销增加极小。
    *   **代码支持**：基于 PyTorch 和 timm 库实现，论文提及使用了公开代码库。

4. **主要创新点**：
    *   **高斯增强注意力 (GAug)**：在自注意力机制中调制了一个以每个 Token 为中心的可学习高斯核。该核的方差由 Query 动态预测，作为一种软性的归纳偏置，促使模型在保持全局感受野的同时，更关注局部邻域信息。
    *   **补丁表示细化 (PRR)**：提出了一种无参数的注意力聚合操作来替代传统的全局平均池化（GAP）。这解决了标准 ViT 训练中空间 Patch Token 缺乏直接梯度监督的问题，确保了空间位置的输出表示对密集预测任务具有实际意义。
    *   **无缝兼容的“面向分割”设计**：LocAt 设计为一种与目标无关的插件，既适用于监督学习也适用于自监督学习（如 DINO），并且兼容多种位置编码（如 RoPE）和架构变体（如 Swin），无需专门的分割预训练即可生成具备强空间感知能力的特征。

5. **实验效果**：

- **阅读来源**: HTML

1. **应用领域**：

3. **使用指南**：
   *   **输入**：需要外部知识辅助的复杂自然语言问题。

       1.  **采样**：使用修改后的 GRPO 算法。在每一步决策时，固定之前的轨迹前缀，仅对当前步骤（推理思考或搜索查询）采样 $k$ 个候选动作（而非采样完整的后续轨迹）。

4. **主要创新点**：
   1.  **截断式步级采样 (Truncated Step-Level Sampling)**：不同于传统的对整条轨迹进行采样，该方法仅探索当前步骤的多种可能性。论文从理论上证明，这种方法在相同奖励结构下，能将优势估计的方差降低至全轨迹采样的 $1/T$（$T$为轨迹长度）。

   3.  **低方差梯度估计与早停激励**：结合截断采样与密集奖励，实现了更低方差的策略梯度，并设计了早停红利（Early-termination bonus），鼓励模型在获取足够信息后尽早回答，避免冗余搜索。

5. **实验效果**：
   *   **核心数据集**：在 NQ, HotpotQA, TriviaQA, MuSiQue, Bamboogle, 2WikiMultiHopQA, ARC-Challenge 等 7 个问答基准上进行了评估。
   *   **主要结论**：
       *   **全面超越**：Slate 在所有基准测试中均优于稀疏奖励的 GRPO 基线和基于过程奖励的 StepSearch 基线。

       *   **小模型与困难任务收益更高**：在 Qwen2.5-3B 模型上，相对 GRPO 提升高达 **30.7%**；在复杂的域外多跳问答任务（如 MuSiQue, Bamboogle）上取得了最大的绝对性能增益。

## 📄 MASQuant: Modality-Aware Smoothing Quantization for Multimodal Large Language Models

- **链接**: https://huggingface.co/papers/2603.04800
- **阅读来源**: HTML

# MASQuant: 多模态大语言模型的模态感知平滑量化报告

1. **应用领域**
   多模态大语言模型（MLLMs）的模型压缩与加速，具体涉及**后训练量化（PTQ）**技术，适用于视觉-语言模型及全模态（视觉、音频、文本）模型的端侧部署。

2. **一句话核心贡献**
   提出了MASQuant框架，通过**模态感知平滑（MAS）**解决不同模态激活值量级差异导致的量化失效问题，并利用**跨模态补偿（CMC）**将模态间差异转化为低秩矩阵，在保持单一量化权重架构的同时实现了高精度的多模态推理。

3. **使用指南**
   *   **输入**：预训练的多模态大语言模型（如Qwen2.5-VL/Omni）、包含少量多模态样本（文本、图像、音频）的校准数据集。
   *   **流程**：
     1. 使用校准数据运行MAS模块，为每个模态（Text, Vision, Audio）独立优化学习平滑因子。
     2. 以文本模态的平滑权重为基准，计算其他模态的权重差异。
     3. 应用SVD白化变换，将这些差异压缩为轻量级的低秩矩阵（L1, L2）。
   *   **输出**：一套统一的量化主权重（通常为W4A8或W8A8格式）和针对非文本模态的少量低秩补偿参数。
   *   **硬件与代码**：方法设计了自定义CUDA内核（基于Nunchaku）以融合投影和量化算子，在GPU（如RTX 4090）上运行；论文提及有源码但需关注后续发布。

4. **主要创新点**
   1.  **定义并解决“平滑错位”问题**：通过系统分析揭示了SmoothQuant等现有方法在MLLMs上的失效原因——不同模态激活值差异巨大（视觉token范围比文本大10-100倍），统一的平滑因子会被主导模态（通常是视觉）控制，导致非主导模态（如音频、文本）信号被过平滑从而丢失精度。
   2.  **模态感知平滑 (MAS)**：摒弃了传统的统一通道级缩放，提出为每个模态独立学习可优化的平滑因子。这种方法在数学上消除了模态主导效应，确保每个模态的激活值都能映射到适合量化的范围内。
   3.  **基于SVD白化的跨模态补偿 (CMC)**：为了解决独立平滑带来的“多套权重”存储问题，作者在理论上证明了经白化处理后的跨模态激活差异具有**低秩特性**。利用这一特性，MASQuant仅需存储一套基座量化权重，通过极小的低秩矩阵在推理时动态补偿模态差异，维持了计算不变性。

5. **实验效果**
   *   **避免模态崩溃**：在Qwen2.5-Omni-3B的W4A8量化测试中，SmoothQuant导致音频模态性能彻底崩溃（Librispeech测试集的WER从FP16的3.9激增至77.4），而MASQuant将其恢复至3.8，几乎无损。
   *   **综合性能领先**：在Qwen2.5-VL（7B/72B）和Qwen2.5-Omni模型上，涵盖OCRBench、ScienceQA、TextVQA等基准测试中，MASQuant在W8A8和W4A8设置下均优于SmoothQuant、AWQ、GPTQ和MBQ等SOTA方法。
   *   **推理效率**：在RTX 4090上，MASQuant实现了相比FP16基线**2.5倍**的推理加速，且相比于其他补偿方法仅增加了5-10%的计算延迟，显存占用极低。

# 论文报告：Towards Multimodal Lifelong Understanding

1. **应用领域**
   多模态视频理解（Multimodal Video Understanding）、具身智能（Embodied AI）、长时记忆与Agent智能体（Long-term Memory & Agents）。

2. **一句话核心贡献**
   论文正式定义了“多模态终身理解”任务，构建了首个跨越数月且包含稀疏时间隙的视频数据集 **MM-Lifelong**，并提出了递归多模态智能体 **ReMA**，有效解决了端到端大模型在超长时序下因上下文饱和导致的推理失效问题。

3. **使用指南**
   *   **输入**：
       1.  **视频流**：跨越长时间（天/周/月）的连续或非连续视频数据（如直播录像、第一人称生活记录）。
       2.  **用户查询**：需要跨越长时间跨度进行推理的自然语言问题。
   *   **流程（ReMA架构）**：
       1.  **被动感知阶段**：将视频流切分为短片段（如2分钟），利用 MLLM（如 Qwen-VL 或 GPT-4V）生成详细的文本/向量摘要，并存入外部向量数据库（Memory Bank）。
       2.  **主动推理阶段**：当接收到查询时，作为控制器的 LLM 启动递归循环，迭代调用“记忆搜索工具”（检索相关片段）和“视觉重检视工具”（回看原始视频帧），动态更新“信念状态（Belief State）”直至得出答案。
   *   **输出**：文本答案以及支撑该答案的精确视频时间区间（Temporal Grounding）。
   *   **技术栈**：代码依赖于多模态大模型进行特征提取，使用向量数据库（如基于 FAISS 的 Mem0）进行记忆管理。

4. **主要创新点**
   1.  **定义“终身理解”范式与 MM-Lifelong 数据集**：
       区别于传统的“长视频”任务，论文强调了真实世界的时间稀疏性（Temporal Sparsity）和未被观测的空白期（Unobserved Gaps）。构建的数据集包含 181.1 小时素材，涵盖游戏、生活记录和直播三个领域，强制模型处理跨越数月的概念漂移和非连续叙事。
   2.  **递归多模态智能体 (ReMA)**：
       提出了一种“系统2”思维的 Agent 架构。不同于试图一次性处理所有上下文的端到端模型，ReMA 将视频流视为动态知识库，通过递归方式（Recursive Reasoning）迭代地检索记忆、验证假设并更新状态，突破了上下文窗口的物理限制。
   3.  **基于线索的严格评估协议**：
       设计了“生命流中的大海捞针（Needle-in-a-Lifestream）”和多跳推理任务。除了传统的问答准确率，引入了 **Ref@N** 指标，要求模型必须精确定位到数月视频流中的具体几分钟证据片段，并采用按时间顺序划分的 Train/Val/Test 分割，防止基于临近上下文的数据泄漏。

5. **实验效果**
   *   **端到端模型局限性**：在 MM-Lifelong 数据集上，随着视频时长增加，State-of-the-art 的端到端 MLLM（如 GPT-4o, Gemini 1.5 Pro）均表现出“上下文饱和”现象，性能不升反降，难以处理超过数小时的连续流。
   *   **ReMA 性能优势**：
       *   **准确率**：ReMA 在日、周、月三个时间尺度上均取得了最高的答案召回精度。
       *   **定位能力**：在最具挑战性的 **Month-Scale（月度尺度）** 测试中，ReMA 的时间定位得分（Ref@300）显著优于现有最佳 Agent 基线（如 DeepVideoDiscovery），证明了其在处理极度稀疏和含噪数据时的鲁棒性。
   *   **消融实验**：证实了较细粒度（如2分钟）的记忆更新周期和使用 MLLM 作为控制器（而非纯文本 LLM）对性能至关重要。

2.  **基于图谱的状态机控制与HITL机制**：将药物发现的四个典型阶段（靶点识别、苗头发现、苗头至先导、先导优化）建模为可组合的技能图谱，并在高不确定性节点（如选定靶点结构时）强制引入人类专家校验（HITL），有效阻断了长程任务中的错误累积。
3.  **MCP 驱动的工具治理体系**：利用 Model Context Protocol (MCP) 构建了标准化的工具总线，实现了对异构生物工具（数据库读取 vs 计算模拟）的分类管理、参数严格校验和全链路溯源，解决了通用Agent在专业领域“胡乱调用工具”的安全隐患。

### 5. 实验效果
*   **基准测试表现**：在自建的 **PharmaBench** 基准（包含88个涵盖全流程的药物发现任务）上，Mozi 在工具编排和长程推理任务（HLE）上的准确率显著优于现有基线模型（如 Biomni 和 Gemini-2.5-Pro）。

    *   **帕金森病案例**：系统筛选了 377,760 个化合物，通过多目标强化学习优化，最终生成的候选分子在计算模拟中展现出 -8.924 kcal/mol 的结合亲和力，且在 hERG 心脏毒性和血脑屏障渗透性等指标上，优于或持平于临床二期基准药物 DNL-201。

## 📄 Distribution-Conditioned Transport

- **链接**: https://huggingface.co/papers/2603.04736
- **阅读来源**: HTML

1. **应用领域**：
   机器学习-生成模型与最优传输（主要应用于计算生物学，如单细胞基因组学数据分析、药物扰动预测、批次效应校正及细胞谱系动力学推断）。

2. **一句话核心贡献**：
   提出了一种名为“分布条件传输”（Distribution-Conditioned Transport, DCT）的通用框架，通过在学习到的连续分布嵌入上调节传输模型，实现了在训练期间未见过的源分布与目标分布之间进行泛化传输，并能有效利用非配对数据进行半监督学习。

3. **使用指南**：

   *   **流程**：
       1.  **分布编码**：使用基于 DeepSets 或 GNN 的编码器将每个样本集合映射为一个低维向量嵌入（Distribution Embedding）。
       2.  **传输学习**：训练一个条件生成模型（如 Flow Matching、MMD 回归或离散序列模型），该模型接收单个源样本，并以源分布嵌入和目标分布嵌入为条件，生成目标样本。
       3.  **推断**：在测试时，利用编码器计算新数据的分布嵌入，驱动传输模型预测样本在目标条件下的状态（即使该目标条件在训练中未出现）。
   *   **硬件要求**：标准 GPU 训练环境。
   *   **代码**：论文提到代码已开源。

4. **主要创新点**：
   1.  **实现 Any-to-Any 的分布泛化**：与传统的基于离散标签（One-hot）或仅基于源条件的模型不同，DCT 通过学习连续的分布嵌入空间，使得模型能够泛化到训练集之外的全新源分布和目标分布（Zero-shot/Few-shot generalization）。
   2.  **支持“孤立边缘分布”的半监督学习**：框架允许利用未配对的单时刻或单条件数据（Orphan Marginals）来丰富分布流形学习。在谱系追踪等生物学场景中，即使缺乏某些群体的成对时间点观测，也能通过无监督的 Any-to-Any 目标提升整体预测性能。
   3.  **底层传输机制无关性（Model Agnostic）**：DCT 是一个元框架，兼容多种传输机制，既支持连续空间的 Flow Matching 和最优传输（OT）模型，也支持离散空间的蛋白质语言模型（如 ProGen, ESM），适用于多模态数据。

5. **实验效果**：
   *   **合成数据**：在多元高斯分布和混合高斯模型基准测试中，DCT 在分布外（OOD）目标上的传输误差显著低于传统的 K-to-K（离散标签）模型，证明了其强大的插值和外推能力。
   *   **药物扰动预测**：在类器官质谱流式数据（Trellis）上，DCT 相比源条件基线（如 Meta Flow Matching），在预测未见过患者的药物反应时表现更优，有效捕捉了患者间的异质性。
   *   **生物序列预测**：在 COVID-19 患者 T 细胞受体（TCR）序列演化任务中，结合 ESM2 的半监督 DCT 模型利用跨患者的非配对数据，将预测的 Energy Distance 降低了一半以上，显著优于仅使用成对数据的监督基线。

## 📄 On-Policy Self-Distillation for Reasoning Compression

- **链接**: https://huggingface.co/papers/2603.05433
- **阅读来源**: HTML

# 论文报告：On-Policy Self-Distillation for Reasoning Compression

### 1. 应用领域
**NLP - 大模型推理优化 / 链路思维（Chain-of-Thought）压缩与增强**

### 2. 一句话核心贡献
提出了一种名为 OPSDC 的在线自蒸馏方法，无需标准答案或奖励模型，仅通过“简洁”指令引导模型自我学习，即可在大幅减少推理 token 数量（压缩率约 57-59%）的同时，通过消除冗余思维噪声显著提升推理准确率。

### 3. 使用指南
*   **输入**：一个具备基础推理能力的大语言模型（如 Qwen3），以及一批无标签的问题（Prompt），无需标准答案。
*   **流程**：
    1.  **构建双策略**：复制模型为“学生”和“教师”。教师模型的输入 Prompt 前附加“请简洁回答（Be concise）”等指令，学生模型保持原始输入。
    2.  **在线采样**：学生模型基于自身策略生成推理路径（Rollouts）。
    3.  **自蒸馏训练**：计算学生生成路径上，学生分布与教师分布之间的**每 token 逆向 KL 散度（Reverse KL）**，并以此作为损失函数进行梯度更新。
    4.  **教师更新**：训练过程中，周期性地（如每 100 步）将学生模型的参数同步给教师模型，以实现渐进式压缩。
*   **硬件与代码**：使用标准 PyTorch 及分布式训练框架（如 FSDP/SGLang）即可实现，不需要复杂的 RLHF 基础设施（如 PPO 的 Value Network）。

### 4. 主要创新点
1.  **无需真值的在线自蒸馏（On-Policy Self-Distillation）**：
    与依赖标准答案的监督微调（SFT）或依赖奖励模型的强化学习（RL）不同，该方法利用模型自身的指令遵循能力作为监督信号。通过优化**逆向 KL 散度**（Reverse KL），迫使学生模型在保持自身探索能力的同时，模仿附加了简洁指令的教师分布，有效避免了离线策略带来的分布偏移和模式坍塌。

2.  **难度自适应压缩机制（Difficulty-Adaptive Compression）**：
    从理论和实验上证明了该方法的压缩强度能随题目难度自动调整。对于简单问题，教师模型生成的路径非常简洁，产生强烈的压缩信号；对于困难问题，教师模型自身也需要详细推理，压缩信号变弱。这种机制防止了传统长度惩罚（Length Penalty）导致的对困难问题推理步骤的过度截断。

3.  **去噪假设与误差传播抑制**：
    论文提出了一个反直觉的观点：**冗余的思维步骤是有害的**。研究表明，过长的推理链容易引入“幻觉”或错误的自我纠正，从而导致误差累积。OPSDC 通过压缩推理过程，实际上是在去除推理中的“噪声”步骤，从而解释了为何减少思考步骤反而能提升准确率（即 Less thinking, better answers）。

### 5. 实验效果
在 Qwen3-8B 和 Qwen3-14B 模型上，使用 DAPO-Math-17k 数据集进行训练，并在多个数学基准上进行了评估：
*   **MATH-500**：实现了 **57%–59%** 的 Token 缩减，同时准确率提升了 **9–16 个百分点**（例如 Qwen3-14B 从 70.0% 提升至 86.1%）。
*   **AIME 2024**（竞赛级难度）：Qwen3-14B 模型在压缩 **41%** 的 Token 的同时，准确率提升了 **10.5%**。
*   **通用能力**：在 MMLU 测试中，模型的通用能力得到完整保留，证明压缩未导致灾难性遗忘。
*   **对比结果**：显著优于基于长度惩罚的 RL 方法、基于真值的 SFT 方法以及仅推理时的 Prompt 优化方法。

1. **应用领域**：
   时间序列预测 - 基础大模型 (Time Series Forecasting - Foundation Models)

2. **一句话核心贡献**：
   提出了“串行扩展”（Serial Scaling）范式，通过引入串行词元预测（STP）目标和大规模稀疏混合专家（MoE）架构，构建了参数量达 83 亿（激活参数 0.75 亿）的时间序列基础模型 Timer-S1，有效解决了现有模型在长短期预测中的可扩展性瓶颈与误差累积问题。

3. **使用指南**：
   *   **输入**：历史时间序列数据（模型采用 Patch 方式处理，针对多变量序列采用单变量拆分输入并进行实例归一化）。
   *   **输出**：未来的时间序列预测值（支持分位数概率预测）。
   *   **硬件需求**：训练和推理需要高性能 GPU 支持，但得益于 MoE 架构，推理时仅需激活部分参数（0.75B），降低了推理成本。
   *   **代码状态**：论文明确表示 Timer-S1 模型将会开源。

4. **主要创新点**：
   *   **串行词元预测 (Serial-Token Prediction, STP) 范式**：设计了 TimeSTP 模块，取代了传统的自回归滚动预测或直接多步预测。该模块通过保留并迭代优化中间表示，在单次前向传播中实现了具备串行计算特性的多步预测，既避免了滚动预测的高延迟，又减少了累积误差。
   *   **十亿级稀疏 MoE 架构**：将稀疏混合专家（MoE）引入时间序列基础模型，构建了 8.3B 参数量的网络（TimeMoE 块），每个 Token 仅激活 0.75B 参数。这种设计有效应对了跨域时间序列数据的显著异质性，平衡了模型容量与推理效率。
   *   **万亿级数据与多阶段后训练策略**：构建了包含超过 1 万亿个时间点的 TimeBench 数据集，并应用了重采样和翻转等数据增强技术。此外，提出了包含“密集监督预训练”、“加权 STP 持续预训练”和“长上下文扩展（RoPE）”的三阶段训练流程，全面提升了模型在不同预测视界（Horizon）下的性能。

5. **实验效果**：
   *   在权威的大规模评测基准 **GIFT-Eval**（包含 24 个数据集、14.4 万条序列）上进行了评估。
   *   Timer-S1 取得了目前的 **SOTA (State-of-the-art)** 性能，在作为预训练模型评估时，达到了最佳的 **MASE (0.693)** 和 **CRPS (0.485)** 分数。
   *   相比前代模型 Timer-3 (Sundial)，其 MASE 降低了 7.6%，CRPS 降低了 13.2%，尤其在中长期预测任务上表现出显著优势。
