## 📄 ShotStream: Streaming Multi-Shot Video Generation for Interactive Storytelling

- **链接**: https://huggingface.co/papers/2603.25746
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 视频生成（具体为：多镜头长视频生成 / 交互式视频生成）

### 2. 一句话核心贡献
提出了一种名为 ShotStream 的因果自回归多镜头视频生成架构，通过首创的双缓存机制和两阶段蒸馏策略，成功解决了长篇叙事视频生成中的高延迟和交互性差问题，在单卡 GPU 上实现了低延迟（16 FPS）且高度一致的流式交互视频生成。

### 3. 使用指南
*   **输入**：流式文本提示（Streaming Prompts）。用户可以在视频生成过程中逐个镜头输入文本描述（包括整体大纲提示和每个镜头的局部提示）。
*   **输出**：在视觉主体、背景和叙事上保持高度一致的多镜头长视频。
*   **硬件需求**：单张显卡即可高效运行（实验基准在单张 NVIDIA H200 GPU 上可达到 16 FPS 的生成速度，实现亚秒级延迟）。
*   **开源情况**：论文明确表示训练代码、推理代码以及预训练模型均开源。

### 4. 主要创新点
1.  **任务重构与高效因果架构**：将传统的“一次性生成所有多镜头”重构为“基于历史上下文的下一个镜头生成（next-shot generation）”任务。通过将多步双向教师模型使用分布匹配蒸馏（DMD）技术压缩为 4 步因果学生模型，实现了支持流式提示输入的实时视频生成。
2.  **双缓存记忆机制与 RoPE 不连续指示器**：为了在自回归生成中保持一致性，设计了双缓存结构：**全局缓存（Global Cache）**存储稀疏的历史条件帧以保证镜头间（inter-shot）一致性；**局部缓存（Local Cache）**保存当前镜头内生成的帧以保证镜头内（intra-shot）连续性。同时引入带有相位偏移的 RoPE（旋转位置编码）不连续指示器，明确消除两种缓存之间的时序歧义。
3.  **两阶段渐进式蒸馏策略（Two-stage Distillation）**：为解决自回归生成中的误差积累（Train-Test Gap）问题，提出两阶段训练法：第一阶段利用真实的视频历史镜头进行**镜头内自强迫（intra-shot self-forcing）**训练；第二阶段过渡到完全利用模型自己生成的历史视频进行**镜头间自强迫（inter-shot self-forcing）**训练，有效提升了长视频生成的稳定性。

### 5. 实验效果
*   **生成效率**：在单卡 H200 GPU 上达到 16 FPS 的高吞吐量，推理速度比传统双向多镜头视频模型提升了 25 倍以上。
*   **客观指标（SOTA）**：在涵盖不同主题的 100 个复杂多镜头视频评估集上，ShotStream 在主体一致性、背景一致性、提示词遵循度（Prompt Adherence）以及镜头切换控制精度等所有核心指标上，均显著超越了现有开源基线模型（如 Mask2DiT、CineTrans、EchoShot 等）。
*   **主观评价**：在 54 人参与的用户盲测中，ShotStream 在视觉一致性、整体画面质量和文本指令遵循度三个维度上获得了绝对压倒性的用户偏好（Preference Rate）。

## 📄 Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models

- **链接**: https://huggingface.co/papers/2603.25716
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 视频生成 / 视频世界模型 (Video World Models)

### 2. 一句话核心贡献
提出了“混合记忆（Hybrid Memory）”新范式及其专用网络架构 HyDRA 与大规模基准数据集 HM-World，有效解决了视频世界模型中动态主体在移出相机视野后重新进入时，容易出现的消失、冻结或严重变形问题，实现了长时时空生成一致性。

### 3. 使用指南
*   **输入**：历史上下文视频帧（Context Frames）以及目标生成的相机位姿/轨迹序列（Camera Trajectories）。
*   **输出**：与历史帧保持高度一致性（包含静态背景一致性与动态主体外观/运动连续性）的未来预测视频帧。
*   **硬件要求**：模型基于全序列视频扩散架构（DiT）和 3D VAE，训练过程使用了 32 张 GPU，对显存和计算资源要求较高。
*   **代码/数据获取**：论文构建了该领域首个大规模专用数据集 HM-World（包含 5.9 万个高保真视频片段），旨在为社区提供评估基准（通常会开源发布，具体需关注作者后续主页发布）。

### 4. 主要创新点
1.  **提出混合记忆新范式与基准数据集 (HM-World)**：首次将视频生成从“静态背景记忆”拓展到“动态主体跟踪+静态背景记忆”的混合范式，并专门构建了包含 17 种场景、49 种主体且精确解耦相机与物体运动轨迹的脱离/重新进入视野（exit-entry events）数据集 HM-World。
2.  **构建时空记忆分词器 (Memory Tokenizer)**：未使用原始的潜变量直接检索，而是通过 3D 卷积将记忆潜变量在时空维度上压缩成富含运动感知和外观线索的记忆 Token，从而大幅过滤了无关的背景噪声。
3.  **设计动态检索注意力机制 (Dynamic Retrieval Attention, HyDRA)**：替代了基础扩散模型（DiT）中标准的 3D 自注意力层。基于生成目标（Query）与记忆 Token 之间的时空相关性（Affinity）计算，通过 Top-K 检索精准召回隐藏主体的动态和外观线索，与局部时间窗口特征融合去噪，大幅降低了计算成本并保证了长序列一致性。

### 5. 实验效果
*   **评估基准**：在 HM-World 验证集上进行了综合评估，包含新建的动态主体一致性指标（$DSC_{GT}$, $DSC_{ctx}$）。
*   **对比模型**：基线模型（Wan2.1-T2V-1.3B）、现有 SOTA 开源方法（DFoT, Context-as-Memory）以及前沿商业闭源模型（WorldPlay 零样本）。
*   **核心表现**：HyDRA 在所有定量指标上均显著超越现有方法。相较于基准模型，重建质量 PSNR 从 18.696 提升至 20.357，SSIM 从 0.517 提升至 0.606；在核心的动态主体一致性评分上分别达到最高的 0.827 ($DSC_{GT}$) 和 0.849 ($DSC_{ctx}$)。定性实验表明，该方法能够完美还原复杂进出视野场景下动态主体的 ID 及自然运动状态。

## 📄 Know3D: Prompting 3D Generation with Knowledge from Vision-Language Models

- **链接**: https://huggingface.co/papers/2603.22782
- **阅读来源**: HTML

**1. 应用领域**
计算机视觉 - 单图到3D生成（Single-Image-to-3D Generation）与多模态内容生成。

**2. 一句话核心贡献**
提出了一种基于视觉语言模型（VLM）知识引导的3D生成框架 Know3D，通过提取并注入多模态扩散模型中间层的隐状态特征，成功解决了单视角3D生成中不可见区域（背面）高度随机、几何不合理的问题，首次实现了对3D物体背面生成的高质量文本语义控制。

**3. 使用指南**
*   **输入**：一张物体的正视图图像（Condition Image）+ 描述该物体背面期望特征的自然语言提示词（Text Prompt）。
*   **输出**：一个完整且高保真的3D资产（3D Mesh/表示），其正面与输入图像保持一致，背面则完全符合输入的文本语义描述，且整体几何结构合理。
*   **硬件要求**：训练成本极高（论文使用了 32 张 NVIDIA A800 GPU 进行微调训练）；基于其底座（Qwen2.5-VL与TRELLIS.2架构），推理阶段预计也需要显存较大的高端 GPU（如 RTX 3090/4090 或 A00 级别）。
*   **开源情况**：论文已建立项目主页（Project page），相关数据集处理方式与微调策略（如LoRA）有明确描述，预计配套代码与模型权重将随项目主页发布以供复现。

**4. 主要创新点**
*   **基于 VLM 扩散模型作为“知识桥梁”**：摒弃了直接用大模型自回归生成3D或直接将抽象文本/图像特征粗暴输入3D模型的方法，而是微调了 Qwen-Image-Edit（结合了Qwen2.5-VL与DiT）作为语义大脑，将多模态预训练数据中的常识推理和空间知识转化为图像空间的结构先验。
*   **中间层隐状态注入机制（Latent Hidden-State Injection）**：深入探讨了不同特征表达对3D生成的引导作用，发现并验证了多模态扩散Transformer（MMDiT）在去噪过程中特定中间层（第20、30、40层）的隐状态相较于VAE潜变量或DINOv3特征，蕴含着最丰富的3D空间结构和语义先验，将其作为条件信号注入 3D 生成骨干网络，大幅提升了生成鲁棒性。
*   **语义感知的正反面视角对齐微调**：针对现有 VLM 在生成“背面”时容易出现的空间朝向错误和姿态改变问题，构建了具备组件级文本描述的正反面配对数据集（基于TexVerse），采用条件流匹配（CFM）目标结合随机提示策略对 VLM 进行了专门的微调，赋予了模型精准的空间感知与背面生成控制能力。

**5. 实验效果**
*   **核心数据集**：在 HY3D-Bench 和 TexVerse 数据集上进行了广泛评估。
*   **定量表现**：在语义一致性（ULIP, Uni3D 评分）上，优于当前开源的 SOTA 单图 3D 生成模型（如 Hunyuan3D-2.1、TRELLIS）；在提供正反双图作为条件的对比中，超越了专门的多视角 3D 生成基线（Hunyuan3D-2mv），获得了更高的图像质量指标（PSNR, SSIM）以及更低的几何与感知误差（Chamfer Distance, LPIPS）。
*   **定性表现**：实现了极强的可控性，能够仅通过改变文本提示词，为同一张正视图生成结构完全不同、且符合物理常识的多种背面（如：给前视角的椅子背面加上不同款式的背包，或改变房屋背面的结构），彻底打破了以往单图生成中背面只能“随机幻觉生成”的局限。

## 📄 Sommelier: Scalable Open Multi-turn Audio Pre-processing for Full-duplex Speech Language Models

- **链接**: https://huggingface.co/papers/2603.25750
- **阅读来源**: ArXiv Abs

### 1. 应用领域
语音自然语言处理（Speech NLP） - 语音大模型（Speech Language Models, SLMs）数据处理与全双工语音交互。

### 2. 一句话核心贡献
提出并开源了一个名为 Sommelier 的高可扩展多轮音频预处理流水线，解决了全双工语音大模型训练中高质量多说话人交互数据稀缺，以及语音重叠、说话人分离错误和 ASR 幻觉等处理难题。

### 3. 使用指南
*   **输入**：原始的、包含自然对话动态（如打断、重叠发音、垫话等）的多说话人/多轮音频数据。
*   **输出**：经过清洗、精准对齐和错误修复的高质量多轮对话数据对（适用于全双工语音大模型的预训练或微调）。
*   **开源情况**：官方明确指出该数据处理流水线是开源的（open-source data processing pipeline）。
*   **硬件需求**：由于数据流水线通常集成了语音识别（ASR）、说话人分离（Diarization）等深度学习基础模型，进行大规模数据处理时强烈建议使用 GPU 硬件加速。

### 4. 主要创新点
1. **复杂自然对话动态解析**：打破了传统单说话人数据的局限，专门针对真实人类对话中的复杂现象（如语音重叠 Overlapping、非正式反馈/垫话 Back-channeling）设计了有效的处理机制。
2. **高鲁棒性的错误缓解架构**：针对现有标准音频处理流水线的痛点，提出了创新方案以显著降低说话人分离（Diarization）错误率，并有效抑制自动语音识别（ASR）过程中的幻觉现象。
3. **面向全双工系统的高可扩展开源设计**：构建了首个专为全双工（Full-duplex）实时语音交互大模型量身定制的可扩展、工业级开源数据处理框架，填补了多轮语音对话数据构建领域的空白。

### 5. 实验效果
*(注：由于摘要内容被截断，以下基于现有内容与通用研究逻辑推断)*
通过 Sommelier 流水线处理的音频数据，在数据纯净度、说话人分离准确率和 ASR 转录保真度上显著优于传统标准流水线。使用该流水线生成的数据训练的全双工语音大模型（SLMs），在处理多说话人实时交互、应对打断和重叠语音时，预期将展现出更低的出错率与更自然、实时的对话表现力。

## 📄 Diffutron: A Masked Diffusion Language Model for Turkish Language

- **链接**: https://huggingface.co/papers/2603.20466
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 (NLP) - 大语言模型微调 / 非自回归文本生成 / 低资源与形态丰富语言（黏着语）处理

### 2. 一句话核心贡献
提出了首个专为土耳其语设计的参数高效型掩码扩散语言模型（MDLM）**Diffutron**，通过 LoRA 持续预训练与渐进式指令微调，以仅 3.07 亿的参数量实现了媲美数十亿参数自回归大模型的文本生成性能。

### 3. 使用指南
* **输入**：土耳其语文本提示词（Prompt）或自然语言指令。
* **输出**：基于离散扩散过程全局迭代并行生成的土耳其语连贯文本响应（非传统的从左到右逐词生成）。
* **硬件需求**：模型极其轻量（仅 307M 参数），推理阶段对显存要求极低。训练阶段同样资源高效（持续预训练仅耗时 ~5.9 小时 @ 单卡 NVIDIA B200；指令微调使用 2 张 A100）。注：目前模型受限于架构，最大上下文窗口为 256-512 个 Token。
* **开源情况**：模型的权重及所使用的土耳其语指令数据集均已在 Hugging Face 平台全面开源。

### 4. 主要创新点
1. **填补形态丰富语言的扩散生成空白**：首次将掩码扩散语言模型（MDLMs）架构成功应用于土耳其语这种形态复杂的黏着语，验证了非自回归并行生成范式在复杂语言结构中的有效性。
2. **基于全线性层高秩 LoRA 的持续预训练（CPT）**：为了在适应土耳其语动态结构的同时防止灾难性遗忘，冻结了多语言基座模型的参数，并对所有线性模块（Attention 的 Q/K/V/O 及 MLP 的输入/输出）应用高秩（$r=256$）的 LoRA 微调，完美平衡了跨语言知识保留与目标语言适配。
3. **渐进式双阶段指令微调策略**：为激活扩散模型的高质量生成能力，设计了两阶段策略：首先使用通用指令集（Bella-Turca）让模型掌握基础指令遵循模式，随后使用复杂专业指令集（InstrucTurca）进行领域深化，有效提升了非自回归模型的逻辑连贯性与响应帮助性。

### 5. 实验效果
* **语言建模内部评估**：经过持续预训练（CPT）后，模型对土耳其语的困惑度（Perplexity）从 3.42 大幅下降至 2.75，表明模型对目标语言分布的不确定性显著降低。
* **下游基准测试表现**：在土耳其语权威评估套件 CETVEL 的代表性子集（涵盖机器阅读理解 Belebele_TR、新闻分类、语义相似度 STS_TR 等）中，Diffutron 以 **3.07亿** 的参数量取得了 **34.68** 的平均分，成功超越了参数规模约大其 7 倍的自回归基线模型（如 Kumru-2B 得分 34.09，TURNA 得分 33.19），展现了极为出色的参数效率。

## 📄 MedOpenClaw: Auditable Medical Imaging Agents Reasoning over Uncurated Full Studies

- **链接**: https://huggingface.co/papers/2603.24649
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型-医疗智能体（Medical Agents） / 计算机视觉-三维医学图像分析

### 2. 一句话核心贡献
提出了一个可审计的运行时环境 **MedOpenClaw** 及相应的全影像基准测试 **MedDxO**，打破了以往模型仅能在预先筛选的静态2D切片上进行评估的局限，首次实现了允许多模态大模型（VLMs）在未经筛选的完整3D临床影像（如多序列MRI、CT/PET）中进行动态交互、工具调用和可解释推理的系统性评估。

### 3. 使用指南
*   **输入**：未经筛选的完整3D医学影像数据包（包含全部体积数据和元数据）以及自然语言格式的临床任务提示（如病例级诊断请求）。
*   **输出**：透明且可重放的完整推理轨迹（包括切片滚动、序列切换、窗宽调整记录、调用的工具及参数、截取的视图证据等），以及最终的结构化诊断报告或答案。
*   **环境与硬件**：该方法作为一个中间API运行层，无需修改底层医疗查看器源码。它部署在基础VLM（如GPT-5.4、Gemini等，需要调用对应大模型API）和标准医学影像平台（如 3D Slicer）之间。执行复杂专家工具（如MONAI分割模块）可能需要GPU算力支持。
*   **代码开源情况**：作为社区驱动的基准测试，平台框架基于开源的3D Slicer与MONAI生态构建，并建立在公开的临床数据集（UCSF-PDGM和NSCLC等）之上，以确保可重复性与后续扩展性（预计基准与评估代码开源）。

### 4. 主要创新点
1.  **提出可审计的医疗智能体运行环境 (MedOpenClaw)**：设计了一个具有严格边界约束的运行时接口，允许大模型直接驱动标准医疗软件（3D Slicer）执行诸如序列选择、切片滚动、视图保存和专业分析等操作。该设计禁止了任意代码执行，所有交互与中间证据均被详细记录，确保了诊断过程的完全透明和临床可审计性。
2.  **构建全研究级别的交互式基准测试 (MedDxO)**：摒弃了传统的“静态单图问答”模式，构建了包含多序列脑部MRI和肺部CT/PET的全流程基准。该基准分为纯视图导航（视觉感知）、工具增强（系统操作）和开放方法三个独立赛道，以最贴近真实放射科医生工作流的方式评估智能体能力。
3.  **揭示了VLM在临床工具使用中的空间对齐（Spatial Grounding）瓶颈**：通过实验首次揭示了一个反直觉的现象：尽管前沿大模型具备较强的图像认知和系统导航能力，但由于缺乏毫米级的精确三维空间定位能力，它们在调用高级医疗分割或测量工具时经常输入错误的坐标，导致获取到错误的中间证据，进而降低了最终的诊断准确率。

### 5. 实验效果
在涵盖脑部MRI和肺部CT/PET的MedDxO核心数据集上：
*   **纯视图导航表现良好**：在不使用高级专家工具的情况下，前沿模型展现了解决基础研究级任务的能力。例如在脑部MRI任务中，Gemini-3.1-pro和GPT-5.4的病例级准确率分别达到了 0.63 和 0.61；在肺部CT/PET的宏观肿瘤定位任务上也分别取得 0.43 和 0.46 的成绩。
*   **高级工具使用导致性能退化**：当允许模型使用高级分割工具包时，由于空间控制精度不足，模型性能反而下降。例如，GPT-5.4在脑部MRI上的准确率从 0.61 降至 0.57，在肺CT/PET上从 0.32 降至 0.27。
*   **细粒度任务面临挑战**：在病理分级（Histopathological Grade）等高度复杂和细粒度的任务上，所有测试的前沿模型表现均未能显著超越随机猜测的基线水平，表明端到端的临床工作流执行仍有巨大提升空间。

## 📄 PackForcing: Short Video Training Suffices for Long Video Sampling and Long Context Inference

- **链接**: https://huggingface.co/papers/2603.25730
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 视频生成（自回归长视频生成 / 视频扩散模型）

### 2. 一句话核心贡献
提出了一种名为 PackForcing 的统一框架，通过创新的“三段式 KV 缓存策略”严格控制了显存线性增长，并解决了误差累积问题，证明了仅用短视频（5秒）训练即可实现高质量、连贯的长视频（长达2分钟）自回归采样和长上下文推理。

### 3. 使用指南
*   **输入**：文本提示词（Text Prompt）。
*   **输出**：高保真、时间连贯的长视频（支持高达 120 秒、16 FPS 的视频流）。
*   **硬件要求**：显存极度友好，单张常规 GPU（如论文验证使用的单张 H200 GPU）即可完成 2 分钟视频的生成，其 KV 缓存被严格限制在约 4.5 GB 以内。
*   **训练与推理**：支持零样本（Zero-shot）直接运行，或仅需使用 5 秒短视频片段进行训练。采用分块自回归（block-by-block）生成策略，并配备流式 VAE 解码器，可实现边生成边输出的低延迟流式显示。
*   *(注：论文文本中未明确提及代码是否开源，但提供了详细的模型架构与推理流程。)*

### 4. 主要创新点
*   **三段式 KV 缓存机制 (Three-partition KV-cache strategy)**：打破传统的全量缓存或简单截断方法，将历史记录分为三部分：1) **Sink**（全分辨率保留极早期帧以锚定全局语义）；2) **Mid**（极度压缩的中间帧以记录大容量历史）；3) **Recent**（全分辨率保留近期帧以确保局部连贯性）。这使得注意力上下文大小被严格绑定为常数（约 8K tokens）。
*   **双分支时空压缩架构 (Dual-branch Spatiotemporal Compression)**：设计了一个融合网络，包含渐进式 3D 卷积（HR 分支，保留细粒度结构）和低分辨率 VAE 重编码（LR 分支，保留全局语义）。该设计对中间态历史（Mid tokens）实现了高达 64 倍的惊人 Token 压缩率。
*   **增量 RoPE 调整与动态上下文选择 (Incremental RoPE adjustment & Dynamic Context Selection)**：引入了纯时间维度的旋转位置编码 (RoPE) 调整算法，能够以极低开销无缝对齐因 Token 压缩/丢弃产生的位置断层；同时采用基于查询-键相关性的 Top-K 动态选择机制，仅将最相关的中间层压缩块提取参与注意力计算，大幅提升了显存利用率和生成稳定性。

### 5. 实验效果
*   **综合指标领先**：在 VBench-Long 基准测试（60秒和120秒测试）上，PackForcing 取得了 State-of-the-Art 的**时间连贯性 (26.07)** 和**动态程度 (56.25)**，动态表现远超最强基线 CausVid（高出 7.82 分）。
*   **超长的时序稳定性**：在连续 60 秒/120 秒的生成中，展现出最稳定的 CLIP 分数（文本-视频对齐度）。对比基线（如 Self-Forcing 在 60 秒时 CLIP 骤降且画面崩坏），PackForcing 的质量衰减极小（CLIP 仅下降 1.14），能自始至终保持明确的动作和主体一致性。
*   **极高的显存效率**：传统方法生成 2 分钟视频需超过 425 GB 显存，而本方法成功将整个 KV 缓存控制在 ~4.5 GB，将显存复杂度从 $O(N)$ 降至 $O(1)$ 常数级。

## 📄 Trace2Skill: Distill Trajectory-Local Lessons into Transferable Agent Skills

- **链接**: https://huggingface.co/papers/2603.25158
- **阅读来源**: HTML

### 1. 应用领域
人工智能 - 大语言模型智能体 (LLM Agents) / 智能体技能学习与经验蒸馏 (Skill Synthesis & Experience Distillation)

### 2. 一句话核心贡献
提出了 Trace2Skill 框架，通过并行分析大量智能体的执行轨迹并进行层级化归纳合并，将零散的经验教训提炼为高度可迁移、无冲突的声明式全局技能（SOP文档），克服了手工编写成本高以及传统在线学习易过拟合局部特征的瓶颈。

### 3. 使用指南
*   **输入**：一个初始技能文档（可以是专家编写的基础版本，也可以是 LLM 仅依赖参数知识生成的无效草稿）以及智能体在特定任务集上的执行轨迹（包含成功与失败案例）。
*   **输出**：一个结构化的、经过进化的技能目录（包含核心 `SKILL.md` 操作标准程序及辅助参考文件），可直接预加载至智能体的系统提示词中。
*   **工作流程**：分为三个阶段：1) 并行生成任务执行轨迹；2) 调度一组分析子智能体（区分成功与错误分析）独立提出技能修改补丁；3) 通过树状层级合并所有补丁并输出最终技能文档。
*   **硬件要求与使用门槛**：完全免训练（Training-free），无需模型参数更新，也无需部署外部经验检索模块。高度可并行化，效率极高（例如使用 122B 模型处理 200 条长轨迹在 A800 上仅需不到 2 个 GPU 小时）。完全兼容开源小规模模型（如 Qwen3.5-35B），无需依赖闭源商业 API。

### 4. 主要创新点
1.  **并行化全局归纳 (Parallel Holistic Consolidation)**：不同于传统让智能体按顺序在线逐条更新技能的模式，该方法一次性并行处理大量轨迹池，通过归纳推理挖掘跨轨迹的普遍性规律，有效防止了对单一轨迹或近期观测的过拟合。
2.  **交互式智能体错误分析 (Agentic Error Analysis)**：引入了具有 ReAct 机制的分析子智能体。在面对失败轨迹时，分析器能够检查完整日志、读取文件、比对正确答案并迭代定位根本原因，相比于传统的单次 LLM 调用（容易产生幻觉或归因错误），生成的补丁更加精准且具有泛化性。
3.  **带程序化冲突预防的层级合并 (Hierarchical Merging with Programmatic Conflict Prevention)**：采用树状层级结构合并成百上千个补丁。合并过程引入了确定性的代码守卫（如拒绝对不存在文件的修改、拦截对同一行代码的冲突编辑），并通过词频权重保留普遍性规律过滤特异性噪音，确保最终生成的长篇技能文档逻辑一致且无冲突。

### 5. 实验效果
*   **惊人的跨模型与跨分布迁移能力**：在电子表格基准测试中，由较小的 Qwen3.5-35B 模型生成的技能，不仅提升了同级模型，甚至能让 Qwen3.5-122B 智能体在 OOD（分布外）的 WikiTableQuestions 任务上取得高达 **57.65 个百分点**的绝对性能提升。
*   **从零创造成效显著**：即使初始技能是毫无用处的 LLM 草稿（Creation 模式），Trace2Skill 提炼出的技能也能带来超过 20% 的性能增长，其质量甚至可以匹配或超越人类专家精心编写的技能。
*   **全面优于现有范式**：消融实验与对比分析证明，该方法的“并行综合归纳单一技能文档”在性能和计算效率上全面优于“顺序在线更新（Sequential Editing）”以及“基于检索的经验库（ReasoningBank）”方法；同时证明了该框架同样适用于数学推理（AIME）和多模态视觉推理（DocVQA）等复杂通用领域。
