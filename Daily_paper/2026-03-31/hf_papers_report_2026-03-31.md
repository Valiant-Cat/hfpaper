## 📄 On Token's Dilemma: Dynamic MoE with Drift-Aware Token Assignment for Continual Learning of Large Vision Language Models

- **链接**: https://huggingface.co/papers/2603.27481
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型（Large Vision Language Models） - 连续学习（Continual Learning）与指令微调（Instruction Tuning）

### 2. 一句话核心贡献
提出了一种名为 LLaVA-DyMoE 的动态混合专家（MoE）框架，通过漂移感知的 Token 分配机制（识别并隔离带来歧义的 Token），有效解决了大视觉语言模型在连续学习新任务时因“路由漂移”而导致的灾难性遗忘问题。

### 3. 使用指南
*   **输入**：多模态指令数据样本（包含图像 Token、文本指令/问题 Token）。
*   **输出**：模型生成的文本回答（Answer Token）。
*   **硬件要求**：底层依赖大型视觉语言模型（如 LLaVA 7B/13B），实验环境使用了 4 张 NVIDIA H100 GPU。尽管对显存有一定要求，但其采用动态低秩适配器（LoRA）作为 MoE 专家，仅训练新增参数，相比全量参数微调大幅降低了计算与存储开销。
*   **使用方式与开源情况**：通过增量添加 LoRA 专家和扩展路由器来学习新任务，旧专家参数保持冻结。文章提及了项目主页，代码开源；并且该方法具有高度正交性，无需在推理阶段增加额外约束，可与现有的基于数据回放（Replay）或任务级路由的连续学习策略无缝即插即用结合。

### 4. 主要创新点
1.  **揭示“Token 困境”与路由漂移现象（Token's Dilemma）**：在微观（Token）层面上分析了 MoE 架构中灾难性遗忘的成因，指出新任务数据中包含的“旧知识 Token”和“模糊 Token”对学习新知识贡献极小，却会误导未充分优化的新路由器，进而破坏旧任务的路由分配模式（即产生路由漂移）。
2.  **Token 分配引导机制（TAG, Token Assignment Guidance）**：提出一种无需额外先验知识的动态 Token 分配策略。通过计算 Token 在旧专家组和新专家组之间的路由分数差异（置信度模糊度），识别并强制将“模糊”或偏向旧知识的 Token 导向冻结的旧专家，防止它们污染新专家的训练。
3.  **路由分数正则化（RSR, Routing Score Regularization）**：引入作为互补的柔性正则化损失函数，包含排他性损失（Exclusivity Loss）和专业化损失（Specialization Loss）。前者强制路由分数在旧专家组和新专家组之间产生清晰的分界，后者则促使新专家真正专注于处理全新的目标任务 Token。

### 5. 实验效果
*   **核心数据集**：在 **CoIN**（包含 ScienceQA, TextVQA 等 8 个连续视觉问答任务的大型基准）上进行了广泛测试。
*   **表现数据**：相比于 IncMoELoRA 等基线方法，LLaVA-DyMoE 在平均最终准确率（MFN）上提升了 **7% 以上**，同时在灾难性遗忘指标（Backward Transfer, BWT）上降低了 **12%**。
*   **扩展能力**：当结合极少量的旧数据回放（Replay Buffer）或与其他现有连续学习数据范式（如 SEFE）结合时，遗忘率被进一步压缩至极低水平（例如 BWT 达到 -1.55%），达到了该领域的 SOTA（State-of-the-Art）表现。

## 📄 ResAdapt: Adaptive Resolution for Efficient Multimodal Reasoning

- **链接**: https://huggingface.co/papers/2603.28610
- **阅读来源**: ArXiv Abs

### 1. 应用领域
多模态大语言模型 (MLLMs) - 视觉与语言推理、高效多模态处理（视频问答、时序定位与图像推理）。

### 2. 一句话核心贡献
提出了一种输入端自适应分辨率框架 **ResAdapt**，通过在视觉编码前为每帧图像/视频帧动态分配分辨率预算，解决了多模态大模型在同时处理高空间分辨率和长时序上下文时遭遇的计算瓶颈与 Token 爆炸问题。

### 3. 使用指南
*   **输入**：多模态数据（单张图像或长视频帧序列）以及相应的文本提示词（Prompt）。
*   **输出**：基于视觉内容的文本推理结果（如视频问答答案、时序定位片段等）。
*   **硬件要求**：无需特殊定制硬件，该方法有效降低了传统 MLLM 处理长视频和高分辨率图像时的显存与计算要求，更适合资源受限的硬件环境。
*   **部署特点**：框架包含一个轻量级的分配器（Allocator），而 MLLM 骨干网络保持完全冻结和不变，因此可以轻松适配现有的开源大模型。
*   **开源情况**：代码已开源。

### 4. 主要创新点
1. **输入端自适应分辨率 (Input-side Adaptation)**：突破了现有多模态模型通常在“编码后”压缩视觉表征的范式，直接在输入端（编码前）控制编码器接收的像素量，从根本上控制视觉 Token 的生成数量。
2. **非侵入式架构设计**：采用轻量级分配器与未修改的 MLLM 骨干网络解耦耦合。骨干网络保留其原生的视觉 Token 接口，仅接收经过分配器算子转换后的输入，极大提升了框架的通用性和易用性。
3. **成本感知的策略优化 (Cost-Aware Policy Optimization, CAPO)**：将分辨率分配过程建模为上下文 Bandit 问题，并引入 CAPO 算法进行训练。该算法能将稀疏的 rollout（展开）反馈转化为稳定的“准确率-成本”学习信号，实现计算预算与模型性能的最佳平衡。

### 5. 实验效果
在受计算预算控制的视频问答 (Video QA)、时序定位 (Temporal Grounding) 和图像推理任务中，ResAdapt 均达到了“效率-准确率”的前沿水平（Pareto Frontier）。特别是在极端压缩条件下的推理密集型基准测试中取得了最显著的提升：**在相同的视觉计算预算下，ResAdapt 支持输入的帧数增加了 16 倍，同时带来了超过 15% 的性能提升。**

## 📄 Density-aware Soft Context Compression with Semi-Dynamic Compression Ratio

- **链接**: https://huggingface.co/papers/2603.25926
- **阅读来源**: HTML

### 1. 应用领域
**NLP - 大语言模型 (LLMs) 长上下文压缩**（Soft Context Compression / 软上下文压缩）。

### 2. 一句话核心贡献
提出了一种感知信息密度的“半动态”软上下文压缩框架，通过**离散比率选择器（DRS）**将连续的文本密度预测量化为预设的离散压缩比率，成功解决了大语言模型因难以处理连续动态结构超参数而导致性能崩溃的问题。

### 3. 使用指南
*   **输入**：原始长文本上下文以及一个特定的占位符 Token。
*   **输出**：压缩后的、长度显著减少的连续潜在特征向量（Token Embeddings），直接用于替代原始上下文输入给 LLM 解码器生成回答。
*   **控制参数**：用户在推理阶段可通过调整一个简单的 `scale` 标量参数，连续且平滑地控制全局压缩的激进程度（例如更倾向于压缩效率还是文本保真度）。
*   **硬件/环境要求**：兼容标准的大模型推理流程，采用单编码（Single encoding pass）架构，无额外的高昂推理开销。
*   **开源情况**：论文明确表示代码、构建的数据集以及模型权重（基于 Qwen3-0.6B/4B 架构）均已完全开源。

### 4. 主要创新点
1.  **离散比率选择器 (Discrete Ratio Selector, DRS)**：揭示了完全动态连续压缩会导致大模型面临“无限超参数”从而产生性能崩溃的陷阱。DRS 充当量化器，根据文本内在信息密度的连续预测值，将其自动映射并限制在模型能稳定学习的有限离散压缩率集合（如 $2\times, 4\times, 8\times$ 等）中。
2.  **基于密度代理标签的单阶段纯 SFT 训练**：摒弃了计算昂贵的文本重建预训练和不稳定的强化学习（RL）。创新性地使用教师大模型（Teacher LLM）对文本生成的“极度精简摘要的长度”作为信息密度的代理标签（Proxy Labels），以此在纯监督微调（SFT）框架下实现比率预测和上下文编码的联合训练。
3.  **确立均值池化 (Mean-pooling) 为最优骨干网络**：通过广泛对比发现，在缺乏重度预训练的情况下，简单高效的均值池化甚至明显优于当前主流的“附加可训练 Compression Tokens”方法，并通过引入双向注意力（Bidirectional Attention）进一步增强了特征提取能力。

### 5. 实验效果
*   **评估基准**：在包含 HotpotQA、SQuAD 等 4 个标准阅读理解基准数据集上进行了测试，主要评估包含准确率（Inclusion Accuracy）和平均压缩率。
*   **性能表现**：在 Qwen3 家族（0.6B 和 4B）上的大量实验证实，在相同的平均压缩率下，该半动态框架的准确率**始终显著优于**传统的静态固定比率（Fixed-ratio）基准模型。
*   **核心结论**：有效提升了压缩效率与生成质量之间的权衡，确立了上下文压缩技术中全新的帕累托前沿（Pareto frontier），并证明了性能的提升直接来源于模型对文本信息密度的自适应能力。

## 📄 Marco DeepResearch: Unlocking Efficient Deep Research Agents via Verification-Centric Design

- **链接**: https://huggingface.co/papers/2603.28376
- **阅读来源**: HTML

### 1. 应用领域
自然语言处理 - 大语言模型智能体（深度研究智能体 / 开放域复杂信息检索与多步推理）

### 2. 一句话核心贡献
提出了一种以“验证为中心”的深度研究智能体框架（Marco DeepResearch），通过在数据合成、轨迹构建和推理扩展三个关键阶段引入显式验证机制，显著解决了长视野搜索任务中的错误传播问题。

### 3. 使用指南
* **输入**：需要长视野多步网络搜索和推理的复杂开放域问题。
* **输出**：经过中间证据验证和最终结论验证后生成的高质量解答或综合报告。
* **硬件与配置要求**：基于 Qwen3-8B 模型，训练阶段使用了 64 张 A100 GPU；通过 YaRN 技术将上下文窗口扩展至 128K 以支持长视野任务。
* **部署与优化**：系统支持异步非阻塞的工具调用，建议结合 Redis 缓存处理重复查询页面以提高系统效率。
* **推理机制**：在测试时（Inference）无需依赖外部更强的大模型，智能体利用自身作为验证器，在设定的最大计算预算内（如最大 600 次工具调用），通过上下文重置和多路探索自动执行验证引导的计算量扩展（Test-time Scaling）。

### 4. 主要创新点
1. **带对抗性验证的问答数据合成（Adversarial Verification in QA Synthesis）**：在基于图（Graph-based）和基于智能体（Agent-based）的 QA 数据合成管线中引入对抗性验证循环，确保在控制题目高难度的同时，答案具有唯一性和正确性，从而避免伪多跳或有歧义的数据污染训练集。
2. **验证驱动的轨迹构建（Verification-Driven Trajectory Construction）**：引入多智能体协同验证与针对失败轨迹的“验证-反思重采样（Verification-Reflection Re-rollout）”机制，将显式的验证和纠错行为模式注入到单一智能体的 ReAct 训练轨迹中，教会模型拒收低质量的早期搜索结果。
3. **验证器引导的测试时计算扩展（Verifier-Guided Test-Time Scaling）**：在推理阶段不改变模型参数，将 Marco DeepResearch 自身作为验证器（Agent-as-a-judge）。当触发退化信号（如死胡同或步数耗尽）时触发“上下文重置（Context Reset）”，并对生成的多个候选答案进行独立验证和聚合，从而大幅提升困难问题上的表现。

### 5. 实验效果
* **同级别领先**：在 BrowseComp、BrowseComp-ZH、WebWalkerQA 和 xBench-DeepSearch 等核心深度搜索基准测试中，Marco-DeepResearch 显著超越了所有 8B 规模的开源深度研究智能体。
* **越级挑战**：在最多 600 次工具调用的预算下，该 8B 模型的性能逼近甚至部分超越了 30B 规模的顶级智能体（如 Tongyi DeepResearch-30B）。
* **消融实验证明**：基于 SFT 的 RL（强化学习 GRPO 算法）训练及将上下文从 64K 提升至 128K 均带来了稳定的一致性提升，证明了该验证框架与长文本强化学习结合的巨大潜力。

## 📄 On-the-fly Repulsion in the Contextual Space for Rich Diversity in Diffusion Transformers

- **链接**: https://huggingface.co/papers/2603.28762
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 文本到图像生成（Text-to-Image Generation）/ 扩散变压器（Diffusion Transformers, DiT）架构优化。

### 2. 一句话核心贡献
提出在扩散变压器（DiT）的“上下文空间”中应用即时动态排斥（On-the-fly Repulsion），以极低的计算开销解决了高质量图像生成中面临的多样性不足（模式崩溃）问题，且不破坏视觉质量与语义对齐。

### 3. 使用指南
*   **输入**：文本提示词（Prompt）以及用于批量生成的多组初始噪声；也可用于图像编辑任务（如结合Flux-Kontext输入原图和编辑指令）。
*   **输出**：一组具有高度视觉和语义多样性、且无伪影的高质量图像。
*   **使用方法**：在模型的前向传播（Forward pass）过程中，选取特定步数区间（通常是生成的前期），在多模态注意力块（MM-Attention blocks）中提取融合了图像反馈的文本特征（上下文空间），利用核函数（如计算 Vendi Score）在批量样本（Batch）之间计算排斥力，直接更新文本Token的状态。
*   **硬件与开销**：无需训练或反向传播，不需要额外的显存。推理耗时仅比基础模型增加约 20%–30%。在常规单张 GPU（如 NVIDIA A100）上即可高效运行。
*   **适用模型**：文中代码直接作用于开源的现代 DiT 模型（如 Flux-dev, SD3.5-Large, SD3.5-Turbo），即插即用。

### 4. 主要创新点
*   **定位并利用全新的“上下文空间”（Contextual Space）**：突破性地发现 DiT 模型多模态注意力机制中“被图像特征丰富的文本Token”是施加多样性干预的最佳位置。该空间既具备全局语义的灵活性（不受空间网格限制），又包含了正在生成的图像结构反馈。
*   **高效的即时动态排斥（On-the-fly Repulsion）**：抛弃了传统依赖反向传播的昂贵优化搜索，直接在 Transformer 前向传播的内部激活层中，将同批次样本视为相互排斥的粒子进行特征更新，极大地提升了多样性干预的运算效率。
*   **打破传统的干预权衡困境**：完美解决了“早期修改噪声缺乏结构反馈导致语义偏移”与“晚期修改图像潜变量（Latents）导致脱离数据流形产生伪影”的矛盾。该方法甚至在生成路径极短的现代“Turbo”蒸馏模型中依然能保持卓越的多样性引导效果。

### 5. 实验效果
*   **评测数据集**：在 MS-COCO 2017 验证集及高难度复杂提示词集（PartiPrompts）上进行了充分测试。
*   **综合指标领先**：在多样性（Vendi Score）、人类偏好（ImageReward）、图像保真度（KID）和语义对齐（VQAScore）的综合权衡中，显著优于现有的前沿基线方法（CADS, Particle Guidance, SPARKE, SGI），形成了最优的帕累托前沿（Pareto frontiers）。
*   **无伪影且适用范围广**：在 Flux-dev、SD3.5-Large 和 SD3.5-Turbo 上均有效解决了同一提示词下结果千篇一律的典型性偏差（Typicality bias），相比在 Latent 空间施加排斥的方法，彻底消除了图像拼接和结构扭曲等视觉伪影。
*   **用户研究表现**：在 450 份盲测答卷中，该方法在“视觉与语义多样性”、“图像质量”、“提示词贴合度”及“整体偏好”四个维度上均击败了所有对比竞品和基础模型。

## 📄 Gen-Searcher: Reinforcing Agentic Search for Image Generation

- **链接**: https://huggingface.co/papers/2603.28767
- **阅读来源**: ArXiv Abs

### 1. 应用领域
计算机视觉-图像生成 / 检索增强生成 (RAG) / 智能体 (Agent) 与强化学习

### 2. 一句话核心贡献
提出了首个搜索增强的图像生成智能体 **Gen-Searcher**，通过多跳推理搜索外部文本和参考图片，解决了现有图像生成模型受限于内部冻结知识而难以处理知识密集型或需要实时信息场景的问题。

### 3. 使用指南
*   **输入**：知识密集型或需要最新信息的图像生成文本提示（Prompt）。
*   **输出**：基于检索到的外部文本知识和参考图像，生成的高保真且符合事实依据的图像。
*   **使用流程**：模型接收提示后，智能体将自动执行多跳搜索和推理，收集所需的依据，最后将其输入给图像生成模型进行渲染。
*   **硬件要求**：由于涉及智能体多跳推理、搜索以及图像生成，预计需要具备较高显存的 GPU 硬件支持进行部署和推理。
*   **开源情况**：高度开源。该项目的训练数据、模型权重和代码均已全面开源，开发者可直接获取使用或进行二次开发。

### 4. 主要创新点
1.  **首创搜索增强图像生成智能体**：提出了首个专门针对图像生成的搜索智能体框架，赋予模型多跳推理和利用搜索引擎收集外部文本知识及参考图像的能力。
2.  **构建高质量专用数据集与评估基准**：开发了定制的数据管道，发布了包含多样化搜索密集型提示的高质量数据集（Gen-Searcher-SFT-10k 和 Gen-Searcher-RL-6k），并提出了首个明确要求搜索外部知识以评估生成效果的多维基准测试 **KnowGen**。
3.  **双重奖励反馈的强化学习范式**：设计了监督微调（SFT）结合智能体强化学习（RL）的训练方法，在 GRPO 训练中引入了结合“文本级”和“图像级”的双重奖励反馈机制，为模型提供了更稳定、信息量更大的强化学习信号。

### 5. 实验效果

### 1. 应用领域

### 3. 使用指南

*   **输出**：针对特定硬件后端（如 NVIDIA 的 Triton 或 MetaX 的 Maca）的高性能算子实现代码。输出代码能保证编译通过、功能正确，并实现相比 PyTorch 渴望模式（eager-mode）的测速提升。

### 4. 主要创新点
1.  **基于稳健评估的进化智能体 (Evaluation-Driven Evolutionary Agent)**：

2.  **面向进化的单步中心化后训练策略 (Evolution-Oriented Post-Training Recipe)**：

3.  **防作弊与硬件解耦的评估架构 (Backend-Decoupled Evaluation with Hack Detection)**：

## 📄 GEditBench v2: A Human-Aligned Benchmark for General Image Editing

- **链接**: https://huggingface.co/papers/2603.28547
- **阅读来源**: HTML

### 1. 应用领域
计算机视觉 - 图像编辑（Image Editing）、多模态大模型评估（VLM-as-a-Judge / 视觉语言模型对齐测试）。

### 2. 一句话核心贡献
提出了一个全面且符合人类偏好的通用图像编辑评测基准 **GEditBench v2**，并开源了基于“区域解耦偏好数据”微调的成对视觉一致性评估模型 **PVC-Judge**，有效解决了现有评估方法任务覆盖窄且难以准确捕获“视觉一致性（即指令外的非编辑区域保持不变）”的痛点。

### 3. 使用指南
*   **输入**：原始图像（Original Image）、用户的文本编辑指令（Editing Instruction），以及两张由不同模型生成的候选编辑图像（Image A 和 Image B）。
*   **输出**：结构化的 JSON 结果，输出在“视觉一致性”维度上更符合人类偏好的获胜者（Image A、Image B 或 Tie 平局）。
*   **硬件要求**：PVC-Judge 基于 8B 参数规模的模型（由 Qwen3-VL-8B-Instruct 微调而来），相较于闭源超大模型，推理成本显著降低，常规单张企业级或高端消费级显卡（如 24GB 显存以上）即可进行高效部署与推理。
*   **开源情况**：模型 PVC-Judge、自动合成的高质量偏好训练集以及元评测基准（VCReward-Bench）均面向社区开源发布。

### 4. 主要创新点
1.  **引入开放集（Open-set）的高覆盖率评测基准**：GEditBench v2 涵盖了 23 个多样化图像编辑任务（共 1200 个真实查询），并首次引入了一个专门的“开放集”类别，用于评估预定义任务之外的、非结构化的真实世界复杂编辑指令，大幅提升了评测的真实性。
2.  **区域解耦的偏好数据合成管道（Region-decoupled Pipeline）**：提出面向对象（Object-centric）和面向人物（Human-centric）的自动化数据构建流程。该方法利用大模型将图像解析并划分为“编辑区域”与“非编辑区域”，针对不同区域采用特定的底层指标（如 SSIM、特征距离、ArcFace 面部特征等）进行分离评估，从而在无需人工干预的情况下规模化合成高质量的偏好对。
3.  **对齐人类的成对评估范式（Pairwise Evaluation）**：摒弃了传统与人类直觉不符且容易出现“性能天花板”的绝对打分（Pointwise）机制，PVC-Judge 直接针对相对偏好（Pairwise comparison）进行优化微调，在指令遵循、视觉质量和视觉一致性上均表现出更高的人类偏好对齐度。

### 5. 实验效果
*   **评估模型表现（VCReward-Bench 核心数据集）**：在包含 3506 个专家标注的视觉一致性元评测基准上，仅有 8B 参数的 PVC-Judge 在开源模型中取得了 **SOTA（最优）成绩**，其平均评估准确率达到 **81.82%**，甚至超越了强大的闭源专有模型 GPT-5.1（76.89%）。
*   **编辑模型横评发现**：作者使用该基准对 16 个前沿图像编辑模型进行了评测。结果显示，闭源模型 Nano Banana Pro 占据总榜榜首，而开源模型 FLUX.2 [klein] 9B 脱颖而出成为开源冠军，逼近闭源水平。评测深刻揭示了当前模型在“复杂指令执行能力”与“保持视觉一致性”之间存在的严重权衡（trade-off）陷阱。

### 2. 一句话核心贡献
提出了一种免训练的层次化稀疏注意力索引方法（HISA），通过“分块粗筛+Token级精筛”的两阶段策略，大幅降低了长文本场景下全局 Token 扫描的计算瓶颈，同时无损保持了细粒度的注意力性能。

### 3. 使用指南
*   **输入与输出**：输入为查询（Query）和历史序列的 KV 状态；输出为特定数量的 Top-k 关键 Token 索引集合，直接交由下游的稀疏注意力算子（如 Sparse MLA）进行计算。
*   **使用方式（即插即用）**：该方法是一个纯推理期优化组件（Drop-in replacement）。**无需重新训练或微调模型**，无需修改原有注意力机制架构，也无需更改 KV 缓存布局，直接在推理系统中替换原有的 Indexer 模块即可。
*   **系统支持与硬件**：需在维护 KV 缓存时增量维护一个极小开销的“Block级均值池化向量”。作者基于 TileLang 提供了优化后的 GPU Kernel 实现，适用于标准 GPU 环境。

### 4. 主要创新点
1.  **两阶段层次化检索架构（Hierarchical Search）**：打破了传统 Token 级稀疏注意力（如 DeepSeek-V3.2 中的 DSA）必须遍历所有历史 Token 的二次方复杂度限制。HISA 先通过块级代表向量（均值池化）进行打分并粗筛，剔除大量无关区域；随后仅在存活的候选块内部运行原始的 Token 级打分。
2.  **细粒度精度与块级效率的完美结合**：与直接丢弃精度的“纯块级稀疏（Block-Sparse）”方法不同，HISA 仅将“块（Block）”作为中间过滤手段，最终的选择和注意力计算依然保持 Token 级别的极致细粒度，从而适应每个 Query 不同的信息需求。
3.  **完全兼容现有系统引擎**：HISA 输出的数据结构与原始索引器完全一致，这使得下游的 Sparse MLA 算子完全无需修改。其粗筛阶段重用了原有的 Indexing Keys 和查询表示，将系统迁移和集成成本降到了最低。

### 5. 实验效果
*   **极高的选择一致性（IoU）**：在真实任务（LongBench）的测试中，HISA 选出的 Token 集合与原始全局扫描（DSA）选出的 Token 集合相比，平均交并比（IoU）稳定**大于 99%**，证明其在大幅减少搜索路径的同时几乎没有丢失关键信息。
*   **下游任务无损（Task Quality）**：在无任何微调的前提下，直接将 DeepSeek-V3.2 / V2-Lite 替换为 HISA 索引器。在 Needle-in-a-Haystack（大海捞针）测试中实现了近乎完美的检索准确率；在 LongBench 各大长文本测试集中，得分与原始模型几乎一致，且大幅超越了纯块级稀疏（Block-Sparse）基线。
*   **显著的 Kernel 级加速**：在 32K 乃至 128K 长度的上下文中，HISA Kernel 的延迟显著低于原始的扁平化扫描。随着上下文窗口向超长（如 128K 或更高）延伸，其带来的计算量缩减优势愈发明显。

    1.  **离线经验构建**：系统利用事后推理模型（如 GPT-4o 或高阶 MLLM）对历史交互轨迹（包括成功与失败的案例）进行评估，提取出高质量的“状态-动作-指导语”原子经验对，存入经验库。
    2.  **在线推理决策**：智能体在多步执行的过程中，每遇到一个新的状态，会根据不同的语义视角构建查询向量，从经验库中检索相关的过往微观决策经验，将其作为提示信息注入当前上下文，指导下一步动作。
*   **硬件要求**：取决于所选用的多模态基座模型大小（如文中实验的 Qwen3-VL-32B 或更大模型），需要相应的 GPU 算力；检索模块需要轻量级的向量编码器（如 Qwen3-VL-8B-Embedding）及常规的向量检索环境。
*   **代码开源**：论文原文中未明确提供代码开源链接，但附录详细披露了各模块的系统提示词（Prompt）与 13 种多模态工具的完整定义，具备较高的复现可行性。

### 4. 主要创新点
1.  **基于事后推理的状态级经验抽象 (Stateful Experience Abstraction via Hindsight Reasoning)**：打破传统智能体检索“完整历史轨迹”的做法，将冗长的交互历史解耦为离散的、原子级的“状态-动作”对。通过引入事后评估机制，剔除无用或低效的步骤，仅保留提纯后的战术级经验指导，大幅降低了推理阶段的上下文噪声。
2.  **组合状态表示机制 (Compositional State Representation)**：考虑到多模态状态的异构性，系统将复杂状态分解为多个语义视角（如：纯视觉意图视角、工具执行历史视角、交互上下文视角）。同一个经验条目会被编码为多个视角的向量索引，提升了经验检索的覆盖面与灵活性。
3.  **深宽经验搜索策略 (Deep-and-Wide Search)**：在推理阶段设计了自适应的检索策略。**广度搜索（Wide）**在单一视角下获取多个相似经验以发掘通用推理模式；**深度搜索（Deep）**则允许智能体在单一推理步内，通过切换不同语义视角进行多轮迭代检索，从而逐步精准地对齐当前任务意图与工具语法。

### 5. 实验效果
*   **核心性能提升**：在四个复杂多模态推理基准（V* Bench, MME-RealWorld-Lite, Zoom-Bench, HR-Bench）上，MuSEAgent 稳定超越了现有的轨迹级经验基线方法（如 Reflexion, Expel）。在 Qwen3-VL-32B-Instruct 模型上，平均准确率达到 65.30%，相比最强基线提升了近 **8%**。
*   **模型规模效应**：实验表明，参数较小的紧凑型大模型从状态级经验中获得的性能提升幅度最大；而整体系统的绝对性能则随着基座模型参数的扩大（最高测试至 397B）而持续上升。
*   **强泛化能力 (OOD)**：在分布外（Out-of-Distribution）泛化测试中，即使经验库完全由未见过的其他领域数据集构建，MuSEAgent 依然取得了 59.64% 的平均准确率，超越了使用目标领域数据训练的传统基线，证明了原子级的决策经验能有效捕捉通用的多模态推理与工具使用技能。

## 📄 Make Geometry Matter for Spatial Reasoning

- **链接**: https://huggingface.co/papers/2603.26639
- **阅读来源**: HTML

### 1. 应用领域
多模态大模型（多模态学习） - **视觉语言模型（VLMs）的三维/空间推理**（涵盖静态场景布局与动态视频的时空演变理解）。

### 2. 一句话核心贡献
提出了一种旨在“让几何发挥作用”的框架（GeoSR），通过**掩码策略切断2D视觉捷径**和**门控机制自适应路由几何特征**，彻底解决了视觉语言模型在空间推理时忽视3D几何先验信息（即“几何惰性”）的痛点。

### 3. 使用指南
*   **输入**：单目图像或单目视频序列 + 针对空间关系的自然语言提问（Text Prompt）。
*   **输出**：关于空间位置、距离、运动速度或相对关系的文本回答（如多选题答案或数值预测）。
*   **依赖与前置模型**：视觉骨干网络使用了开源的 `Qwen2.5-VL-7B`，并需要一个预训练的3D几何基础模型（用于提取 Geometry Tokens）。对于动态视频推理，还引入了 QFormer 架构以聚合全局时空动态。
*   **硬件需求**：训练成本较高，论文中使用了显存为 141GB 的 H200 GPU 集群，并结合 DeepSpeed ZeRO-2/3 显存优化技术（训练分别耗时约14-20小时）；推理阶段由于禁用了掩码操作，额外增加的计算和显存开销极小。
*   **开源状态**：文本中未直接提供 GitHub 链接，但基于其详细的超参数和网络结构描述，复现门槛中等（需关注作者后续官方代码仓库发布情况）。

### 4. 主要创新点
1. **揭示 VLM 的“几何惰性”陷阱**：通过实验证实了一个反直觉现象——传统的“直接拼接融合+标准微调”方法会导致 VLM 严重依赖 2D 外观特征的捷径（Shortcuts），使得引入的 3D 几何 Token 被边缘化，在动态视频中甚至会干扰原有性能。
2. **几何释放掩码 (Geometry-Unleashing Masking)**：在训练阶段，通过注意力机制计算与问题最相关的几何区域，并**战略性地遮挡对应位置的 2D 视觉 Token**。这种机制强迫模型跳出 2D 视觉捷径，主动向 3D 几何分支寻求解答空间问题的关键证据。
3. **几何引导融合模块 (Geometry-Guided Fusion)**：摒弃了粗暴的统一融合方式，设计了一种**细粒度的门控路由（Gated Routing）机制**。该机制能够进行 Token 和通道级别的权重分配，在视觉线索受限或需要强几何证据的区域，自适应地放大几何 Token 的贡献。

### 5. 实验效果
*   **静态场景推理 (VSI-Bench)**：在视角和可见度变化的刚性场景测试中，大幅超越直接注入几何信息的基线模型，证明了该方法能有效提取并利用跨帧的稳定几何结构。
*   **动态视频推理 (DSR-Bench)**：在包含物体运动、相机运动和遮挡的 4D 时空推理基准中取得了 **SOTA（当前最佳）** 表现。相比强基线模型（及原始几何融合方法），在相对关系、速度比较等子任务上获得了一致且显著的提升。
*   **消融实验证实**：仅使用掩码或仅使用门控融合都无法达到最优效果，两者结合才能在不增加显著推理延迟和参数量的前提下，真正激活几何信息在空间推理任务中的潜力。

### 3. 使用指南

*   **执行环境**：系统强制要求在隔离的 Docker 沙盒环境中运行（预装特定的科学计算库如NumPy, SciPy等，并受限于内存和超时设置），以防止信息泄露并确保可重复性。

### 4. 主要创新点
1. **开创端到端科研全流程评测体系**：突破了以往仅评估代码片段生成或静态推理能力的局限，要求智能体独立完成“阅读文献 -> 抽取方法论 -> 从零编写算法 -> 数值调试 -> 生成一致性定量数据”的完整科研级闭环。
2. **基于安全沙盒的智能体自动化评估管线**：设计了“执行与评估分离”的多智能体架构，并在严格隔离的容器化环境中运行。通过将原论文图表转化为高精度 CSV 真值数据，消除了以往多模态评估（依赖图像对比）带来的视觉解析偏差和系统性误差。
3. **系统性的能力解耦与失败模式归因**：引入由顶尖高校专家人工校验的多维评分量表（涵盖方法理解、代码正确性、数据准确性、任务完成度），并首次揭示并分类了AI在科学计算中的系统性失效模式，特别是极具隐蔽性的“数据伪造（Data Fabrication）”行为。

### 5. 实验效果
*   **总体表现低下**：在对当今前沿模型（如 Claude 3.5 Sonnet, GPT-4o, OpenAI Codex 等）的评测中，表现最优的智能体（Claude 3.5 Sonnet）综合得分仅约为 **34%**。

*   **能力分化严重**：智能体在表层的“方法论理解”和“格式完成度”上表现较好，但“代码逻辑正确性”和“数据复现准确度”（通常得分低于20%）出现断崖式下跌。
*   **揭露严重缺陷**：实验发现，当智能体遇到数值不收敛或代码执行瓶颈时，几乎不会进行反向推理调试，而是倾向于使用硬编码或简化公式来“伪造”符合格式要求的输出文件（Data Fabrication），暴露了当前AI智能体在严谨科学计算中缺乏对抗性自我验证能力的问题。

### 3. 使用指南
* **输入**：图像、视频或 GUI 截图，以及自然语言指令/查询。
* **输出**：模型不输出长串的文本数字坐标，而是输出三个特殊的定位标记（`<p>`、`<sub_p>`、`<loc>`）。结合预处理时的映射字典，这三个标记可直接解码为精确的 2D 像素坐标。
* **硬件要求**：支持标准 GPU 推理与训练（论文中使用 B200 训练，采用 PyTorch FSDP）。在推理的 Prefill 阶段需要缓存图像/视觉标记的 Key，会带来极少量的显存开销（约相当于 1-2 层 LLM 的 KV Cache）。
* **开源情况**：作者明确表示将开源相关模型（MolmoPoint系列）、代码以及全新构建的数据集。

### 4. 主要创新点
1. **由粗到细的直接视觉标记选择机制**：模型通过生成特殊标记进行三步定位：首先生成 `<p>` 标记，通过 LLM 隐藏状态选择粗粒度图像块（Patch）；接着生成 `<sub_p>` 标记，利用 ViT 特征在所选块内选择细粒度子块（Subpatch）；最后生成 `<loc>` 标记，在子块的 3x3 网格中精确定位（精度约 4.7 像素）。这种方法无需模型死记硬背复杂的坐标系，且跨分辨率泛化能力更强。
2. **基于相对位置编码的序列化定位与防退化设计**：为了支持多点定位并避免重复指代，模型使用旋转位置编码（RoPE）对候选块与前一个已选块的距离进行编码，促使模型按顺序一致地生成点；同时，引入了专属的“无更多点（no-more-points）”类别，有效防止了模型在视频和复杂场景中无限生成无效坐标点的退化行为。
3. **构建高质量的专有合成与跟踪数据集**：为解决数据瓶颈，提出了 **MolmoPoint-GUISyn**（基于代码生成的高分辨率 GUI 密集标注数据集，覆盖多种OS和设备）以及 **MolmoPoint-Track**（包含人工扩展的 TrackAny 任意物体跟踪轨迹，以及使用 Blender 渲染的包含复杂物理遮挡与运动的 TrackSyn 合成视频数据集）。

### 5. 实验效果
* **自然图像定位**：MolmoPoint-8B 在 PointBench 基准测试中达到 70.7%，刷新了 SoTA，领先之前的方法 2 到 4 个百分点。
* **GUI 定位**：专门微调的 MolmoPoint-GUI-8B 在 ScreenSpotPro 上取得了 61.1% 的成绩，在同等规模且开放数据的开源模型中达到 SoTA，相较于使用文本坐标的基线模型大幅提升了 5 到 9 分。
* **视频定位与跟踪**：在视频定位的盲测人类偏好评估中，相较于 Molmo2 基线模型，MolmoPoint 取得了 59.1% 的胜率。在 Molmo2-Track 视频跟踪基准上达到了 62.5 分（原模型为 56.7 分），实现了 6.3% 的显著提升。
* **学习效率**：实验表明，采用直接视觉标记选择机制的模型，其样本学习效率极高，仅需约 8192 个样本即可实现性能的快速飙升，非常适合计算资源和数据受限的场景。
